При работе с типами данных в датафреймах следует обратить внимание на то, что по умолчанию значениям датафрейма присваиваются типы данных pandas. 
Типы данных Pandas совместимы с Numpy, что позволяет эффективно совмещать их друг с другом.

|**pandas**|**Python**|**назначение в pandas**|
|---|---|---|
|object|str|Используется для представления текстовой или смешанной текстовой и числовой информации. Например, `'@nickname_35'`. В этом случае тип object аналогичен типу str Python.  <br>Также этот тип данных может использоваться для хранения любых объектов Python, например, списка `['@nickname_35', '@nickname_68']`.|
|int64|int|Целочисленный тип данных. Используется для хранения целых чисел. Например, `5, 11, -39`|
|float64|float|Вещественный тип данных. Используется для хранения чисел с плавающей запятой. Например, `15.3, 17.0, -0.57`|
|bool|bool|Логический тип данных, содержащий всего два значения — `False` и `True`.|
|datetime64|datetime|Тип данных даты и времени, используется для хранения значений даты и времени. Например, `2024-08-12 10:30:00`|
|timedelta64|-|Тип данных для хранения интервала времени между двумя событиями с высокой точностью (до наносекунд). Например, `5 days 01:45:30`|
|category|-|Категориальный тип данных, представляет ограниченный список значений (категорий). Например, `'отлично', 'хорошо', 'удовлетворительно'`|

> [!important]
> Большая часть типов данных использует 64-битное представление данных. Это позволяет хранить данные с высокой точностью. При необходимости можно понизить разрядность числовых типов данных, например, использовать такие типы, как `int8`, `int16`, `int32` вместо `int64`, тогда как в Python тип int имеет неограниченную разрядность, а число с плавающей точкой имеет 64 битную разрядность.

Фиксация типа данных позволяет для числовых и временных типов данных в pandas совершать векторные вычисления — т.е. проводить вычисления сразу с использованием всех данных одного столбца DataFrame, что сокращает время обработки данных и оптимизирует вычисления.

## Изменение типа данных
Наиболее распространённый — метод `.astype()`. Этот метод можно применить к столбцу или всему датафрейму.
```python
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
df = pd.read_csv('/datasets/vehicles_us.csv')

# Заменим пропуски:
df['paint_color'] = df['paint_color'].fillna('unknown')
df['is_4wd'] = df['is_4wd'].fillna(0)
df['odometer'] = df['odometer'].fillna(df.groupby(['model_year','model','condition'])['odometer'].transform('median'))
df['cylinders'] = df['cylinders'].fillna(df.groupby(['type'])['cylinders'].transform(lambda x: x.mode()[0]))

# Остальные пропуски удалим:
df = df.dropna()

# Преобразуем тип данных в столбце model_year:
df['model_year'] = df['model_year'].astype('int64')

# Проверим результат:
df['model_year'].dtypes

dtype('int64')
```

Метод `.astype()` можно применить сразу к нескольким столбцам, если им необходимо назначить один тип данных.
```python
# Преобразуем тип данных в нескольких столбцах:
df[['cylinders','odometer','is_4wd']] = df[['cylinders',
                                            'odometer','is_4wd']].astype('int64')

У метода `.astype()` есть полезный аргумент `errors`, который позволяет указать действия метода при обнаружении в данных ошибки при изменении типа данных. Такие ошибки возникают, когда данные нельзя перевести в другой тип, например, строковые данные, содержащие символы, отличные от цифр, нельзя перевести в тип данных `int` или `float`. Аргумент `errors` определяет, как поступить при обнаружении ошибки. По умолчанию аргумент имеет значение `raise`. В этом случае при обнаружении ошибки в данных при запуске кода будет ошибка, в которой в конце будет указана причина
Также аргумент `errors` может иметь значение `ignore`. В результате столбец не будет преобразован, то есть его просто проигнорируют

метод `.to_numeric()` позволяет переводить данные в численный тип (`int` или `float`).
Метод `.to_numeric()` переведёт данные в численный тип — `int64` или `float64`. При этом метод `.to_numeric()` имеет аргумент `errors`, который так же, как и в методе `.astype()` определяет действия, при обнаружении ошибки, но в отличии от метода `.astype()` имеет значение `coerce`, которое будет заменять ошибочное значение на пропуск и изменять тип данных на `float64`.
```python

                   'is_ordered': [1, 1, 0, '1', 0]})

# Изменим тип данных на численный:
df_['is_ordered_bool'] = pd.to_numeric(df_['is_ordered'], errors='coerce')

# Выведем информацию о датафрейме:
df_.dtypes

id                 object
is_ordered         object
is_ordered_bool     int64
dtype: object
```

## Форматы записи даты и времени
время и даты могут быть записаны в разных форматах:
- дата может не содержать время, или наоборот данные содержат только время, но не содержат дату, например, вместо столбца с датой и временем записи действия пользователя `2024-02-15 12:35:41` может быть два столбца: один с датой — `2024-02-15`, второй с временем — `12:35:41`;
- при написании даты и времени может меняться порядок отображения года / месяца / дня или часов / минут / секунд, или в начале идти время, а затем дата;
- даты и время могут содержать метку часового пояса, `2024-02-15 12:35:41+03:00`;
- дата и время может быть прописаны в виде целочисленного значения — в формате `timestamp`, например, дата и время `2024-02-15 12:35:41` в формате `timestamp` будет отображаться как `1707989741` или после импорта данных из Excel даты могут быть прописаны как целые числа, например, вместо `2024-02-15` будет указано `45337`.

Методу `.to_datetime()` нельзя передать несколько столбцов для преобразования, как это можно сделать в методе `.astype()`. Поэтому для преобразования нескольких столбцов одновременно можно использовать метод `.to_datetime()` в связке с методом `.apply()`, применяя его к необходимому срезу данных:
```python
# Изменим тип данных с помощью метода .to_datetime():
df[['start_dt','end_dt']] = df[['start_dt','end_dt']].apply(pd.to_datetime)

Наиболее широко используемые обозначения при создании шаблона записи даты:
- **`%d`**: день месяца в виде двузначного числа (например, 01, 02, ..., 31);
- **`%m`**: месяц в виде двузначного числа (например, 01, 02, ..., 12);
- **`%y`**: год без столетия в виде двузначного числа (например, 00, 01, ..., 99);
- **`%Y`**: год с веком в виде четырёхзначного числа (например, 0001, 0002, ..., 2013, 2014, ..., 9998, 9999);

и времени:

- **`%H`**: часы в 24-часовом формате в виде двузначного числа (например, 00, 01, ..., 23);
- **`%I`**: часы в 12-часовом формате в виде двузначного числа (например, 01, 02, ..., 12);
- **`%M`**: минуты в виде двузначного числа (например, 00, 01, ..., 59);
- **`%S`**: секунды в виде двузначного числа (например, 00, 01, ..., 59);
- **`%f`**: микросекунды в виде числа с шестью знаками (например, 000000, 000001, ..., 999999);
- **`%p`**: локализованный эквивалент AM или PM.

Более подробно смотрите [в документации на английском](https://pandas.pydata.org/docs/reference/api/pandas.Period.strftime.html).

дополнительные аргументы
- `dayfirst` используется, если первое значение в дате — день, например: дата `10/11/12` будет интерпретирована как `2012-11-10` — 10 ноября 2012 года. По умолчанию указан параметр `False`.
- `yearfirst` — аналогично предыдущему параметру, за исключением того, что первое значение определяется как год, например, дата `10/11/12` будет интерпретирована как `2010-11-12` — 12 ноября 2010 года. По умолчанию указан параметр `False`.

## Формат данных даты и времени `timestamp`
Дата и время в формате `timestamp` представлены в виде целого числа, которое обычно составляет количество секунд или миллисекунд, прошедших с определённой точки отсчёта. За точку отсчёта берут различные даты. Например, в Unix-системах `timestamp` представляет собой количество секунд, прошедших с начала эпохи Unix, которая начинается с 1 января 1970 года 00:00:00 (UTC).

как перевести дату и время из формата `timestamp` в формат `datetime64`. Для этого можно использовать метод `pd.to_datetime()`, в котором нужно указать атрибут `unit='s'`, если время в формате `timestamp` округлено до секунды. Если время задано в миллисекундах, то стоит использовать `'ms'` в аргументе `unit`. По умолчанию этот параметр принимает время в наносекундах — `ns`.
```python
# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],
    'start_ts': [1585382400, 1585396200, 1585398900, 1585410300, 1585416600],     
    'end_ts': [1585384200, 1585398000, 1585399500, 1585412400, 1585422000]})

# Изменим тип данных с помощью метода .to_datetime():
df['start_dt'] = pd.to_datetime(df['start_ts'], unit='s')
df['end_dt'] = pd.to_datetime(df['end_ts'], unit='s')

# Выведем датафрейм на экран:
print(df)

# Проверим типы данных:
print('Проверим типы данных датафрейма df:')

## Дополнительная работа с данными типа `datetime64`
Тип данных `datetime64` позволяет получить из даты и времени много информации, например, выделить год, месяц, день, номер недели или название дня недели.
Чтобы извлечь из данных `datetime64` дополнительную информацию, к столбцу с данными применяется акцессор `.dt`, который позволяет обращаться к каждой строке как к объекту `datetime64`, и специальный атрибут, выполняющий ту или иную функцию.

|Атрибут datetime64|Действие|
|---|---|
|year|Извлечение года|
|quarter|Извлечение квартала|
|month|Извлечение номера месяца|
|week|Извлечение номера недели|
|day|Извлечение дня месяца|
|day_of_year|Извлечение порядкового номера дня в году. Аналогичный атрибут - dayofyear|
|day_of_week|Извлечение дня недели (где 0 — понедельник). Аналогичные атрибуты - dayofweek и weekday|
|day_name()|Извлечение названия дня|

```python
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
df = pd.read_csv('/datasets/vehicles_us.csv')

# Изменим тип данных столбца date_posted:
df['date_posted'] = pd.to_datetime(df['date_posted'])

# Выделим год публикации объявления:
df['year'] = df['date_posted'].dt.year

# Выделим квартал публикации объявления:
df['quarter'] = df['date_posted'].dt.quarter

# Выделим месяц публикации объявления:
df['month'] = df['date_posted'].dt.month

# Выделим номер недели публикации объявления:
df['week'] = df['date_posted'].dt.week

# Выделим день (месяца) публикации объявления:
df['day'] = df['date_posted'].dt.day

# Выделим порядковый номер дня публикации объявления:
df['dayofyear'] = df['date_posted'].dt.dayofyear

# Выделим день недели публикации объявления:
df['dayofweek'] = df['date_posted'].dt.dayofweek

# Выделим название дня недели публикации объявления:
df['day_name'] = df['date_posted'].dt.day_name()

# Проверим результат:
print(df[['date_posted','year','quarter','week','day',
          'dayofyear','dayofweek','day_name']].head())

Поэтому для выделения номера недели лучше использовать метод `isocalendar()` — он преобразует дату в датафрейм — с перечислением года, номера недели и дня

## **Часовой пояс и `.to_datetime()`**
Иногда данные могут быть представлены вместе с информацией о часовом поясе. Обычно он добавляется в виде `+03:00`, где первый знак это плюс `+` или минус `-`, которые указывают на разницу относительно UTC, далее идут часы и минуты. Чтобы убрать часовой пояс и привести время к UTC, метод `.to_datetime()` имеет параметр `utc=True`:
```python
# Создадим датафрейм с данными. 
# Дата и время `2024-03-28 00:00:00+03:00` указывают, что событие было совершено в 0 часов 28 марта 2024 года с учётом часового пояса `+3` (`Europe/Moscow`).
df = pd.DataFrame({'dt': ['2024-03-28 00:00:00+03:00']})

# Преобразование столбца dt в формат datetime64 с учетом часового пояса:
df['dt'] = pd.to_datetime(df['dt'], utc=True)

print(df)

                         dt
0 2024-03-27 21:00:00+00:00
# событие в часовом поясе состоялось в полночь, а относительно UTC — в 21 вечера прошлого дня
```
Чтобы убрать информацию о часовом поясе, можно использовать метод `.tz_localize()` с параметром `None`:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({'dt': ['2024-03-28 03:00:00+03:00']})

# Преобразование столбца dt в формат datetime64 с учетом часового пояса:
df['dt'] = pd.to_datetime(df['dt']).dt.tz_localize(None)

print(df) 
```

```
                   dt
0 2024-03-28 03:00:00 
```

> [!important]
> `tz_localize(None)` **убирает информацию о часовом поясе, но не меняет локальное время**. То есть было «03:00 по местному времени (UTC+3)», стало просто «03:00 без пояса».
> 
> `utc=True` говорит: **сделай время в UTC**. `2024-03-28 00:00:00+03:00` — это «полночь по UTC+3». В UTC это на **3 часа раньше**, то есть `2024-03-27 21:00:00+00:00`

## Временные интервалы `timedelta64` и метод `.to_timedelta()`
При вычитании одних даты и времени из других, например, при подсчёте длительности загрузки приложения по формуле «время окончания загрузки - время старта загрузки» образуется разница — интервал времени, который может быть выражен в секундах, минутах, часах, днях. В pandas результат будет представлен в виде данных типа `timedelta64`:
```python
# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],
    'start_dt': ['2024-03-28 08:00:00', '2024-03-28 12:30:00', '2024-03-28 13:15:00', '2024-03-28 17:45:00', '2024-03-28 19:30:00'],
    'end_dt': ['08:30:00 28.03.2024', '2024-03-28 13:13:00', '12:45:00 28.03.2024', '19:00:13 28.03.2024', '21:00:00 28.03.2024']})

df[['start_dt','end_dt']] = df[['start_dt','end_dt']].apply(pd.to_datetime)

# Рассчитаем время сессии:
df['session_leght'] = df['end_dt'] - df['start_dt']

# Выведем результат расчёта длительности сессии на экран:
print(df['session_leght'])

0     0 days 00:30:00
1     0 days 00:43:00
2   -1 days +23:30:00
3     0 days 01:15:13
4     0 days 01:30:00
Name: session_leght, dtype: timedelta64[ns]
```
`1 days +23:30:00` это то же самое, что `-0:30`, то есть сессия началась на 30 минут позже, чем окончилась.

В качестве единиц измерения в типе данных `timedelta64` могут использоваться: дни (`D` или `days`), часы (`H` или `hours`), минуты (`T` или `m`), секунды (`S` или `s`), миллисекунды (`L` или `ms`), микросекунды (`U` или `us`) и наносекунды (`nanoseconds` или `ns`).

Объекты типа `timedelta64` поддерживают арифметические операции, такие как сложение, вычитание и умножение на число.
```python
# Увеличим интервал в 10 раз:
interval = pd.to_timedelta('5 days 13:10:00') * 10

print(interval)

55 days 11:40:00
```
Интервал `timedelta64` можно использовать для изменения значений даты и времени. Для этого необходимо преобразовать `days_listed` в тип `timedelta64` и добавить к дате.
```python

# Преобразуем date_posted в тип datetime64:
df['date_posted'] = pd.to_datetime(df['date_posted'])

# Преобразуем days_listed в тип timedelta64:
df['days_listed'] = pd.to_timedelta(df['days_listed'], unit='D')

# Проверим результат:
print(df[['date_posted','days_listed']].head())
print(df[['date_posted','days_listed']].dtypes)

Для преобразования `days_listed` в тип `timedelta64` мы передали методу `.to_timedelta()` столбец датасета `days_listed` и использовали параметр `unit='D'`, который указал, что данные в столбце `days_listed` представлены в виде количества дней.
