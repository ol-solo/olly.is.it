# Как проводить статистические тесты. Практика

## Шпаргалка

В этом уроке мы будем практиковать методы и функции Python для проведения одновыборочного t-теста:

- `scipy.stats.ttest_1samp` — функция для проведения одновыборочного t-теста с проверкой того, как среднее выборки отличается от предполагаемого значения генеральной совокупности и насколько это статистически значимо;
    - `popmean` — предполагаемое среднее значение генеральной совокупности, с которым сравнивается выборочное среднее;
- `print()` — функция для вывода результатов на экран.
- Статистические понятия:
    - `t-статистика` — показывает, насколько далеко среднее выборочных данных находится от предполагаемого среднего в единицах стандартного отклонения;
    - `p-value` — вероятность получить такие или более крайние значения t-статистики, если нулевая гипотеза верна. Малые значения p-value (например, меньше 0.05) указывают на то, что различия статистически значимы.


## Задание 1.

Представим, что вы продуктовый аналитик в команде, которая недавно запустила рекламную кампанию для увеличения пользовательской активности на новом лендинге.

Вам нужно оценить, достигла ли кампания своей цели по повышению вовлечённости пользователей. Один из ключевых показателей успеха — увеличение среднего времени, которое пользователи проводят на сайте.

Среднее время, которое люди проводили на сайте раньше, равно пяти минутам. Определите, увеличилось ли время, проведённое пользователями на лендинге, после запуска маркетинговой кампании.

```python
from scipy.stats import ttest_1samp
  
time_spent = [6, 7, 5, 7, 6, 8, 5, 6, 7, 5, 4, 5, 6, 4, 5]
popmean = 5
  
t_statistic, p_value = ttest_1samp(time_spent, popmean)
print(f"t-статистика: {t_statistic:.2f}, p-value: {p_value:.4f}")
```

## Задание 2.

Мы получили значение p-value. Теперь определите, стоит ли отвергать нулевую гипотезу на основе результатов t-теста из Задания 1. Для этого используйте стандартный уровень значимости (0.05) для большинства подобных исследований (`alpha`).

```python
from scipy.stats import ttest_1samp
  
time_spent = [6, 7, 5, 7, 6, 8, 5, 6, 7, 5, 4, 5, 6, 4, 5]

t_statistic, p_value = ttest_1samp(time_spent, popmean)

    print("Не отвергаем нулевую гипотезу: статистически значимых различий нет.")
```

# Сравнение средних. Практика

`scipy.stats.ttest_ind(a, b, equal_var=False)` — функция для проведения t-теста для двух независимых выборок, где `a` и `b` — списки данных для каждой из групп. Параметр `equal_var` указывает, равны ли дисперсии данных выборок (по умолчанию предполагается, что дисперсии равны).

`sum(list)` / `len(list)` — один из способов расчёта среднего значения списка в Python.

# Асимметричные проверки. Практика

Для проведения односторонних t-тестов в Python методам `ttest_1samp` и `ttest_ind` нужно передать аргумент `alternative`.

Для одной выборки — то есть метода `ttest_1samp` — `alternative` задаёт направление сравнения истинного среднего (μμ) совокупности, из которой взята выборка, с заранее выбранным числом aa.

`alternative='less'` задаёт альтернативную гипотезу H1:μ<aH1​:μ<a.

## Задание 1.

Учитель игры на гитаре знает, что в среднем его ученики тратят 11 часов в неделю на практику. Учитель дал группе своих учеников код доступа к новому приложению, помогающему практиковаться, и попросил каждого из них прислать, сколько часов — с точностью до часа — этот ученик потратил на практику за неделю после получения доступа к новому приложению. Проверьте гипотезу учителя, что среднее количество часов практики увеличилось.

```python
from scipy import stats as st
  
# результаты группы учеников:
practice_hours_with_app = [11, 13, 11, 10, 9, 12, 13, 13, 14, 10, 14, 10, 12, 13]
  
hours_before_app = 11
  
alpha = 0.05
  
results = st.ttest_1samp(practice_hours_with_app, hours_before_app, alternative='greater')
  
# выведите на экран полученное p-значение
```

```python
print('p-значение:', p_value)
  
# сравните p-значение с заданным уровнем значимости alpha
if p_value < alpha:
    print('Отвергаем нулевую гипотезу')
else:
    print('Нет оснований отвергнуть нулевую гипотезу')
```

# Робастное сравнение выборок. Практика

## **Шпаргалка:**

- `mannwhitneyu(a, b, alternative='two-sided')` — функция для проведения U-теста Манна-Уитни для двух независимых выборок, где `a` и `b` — списки данных для каждой из групп. Параметр `alternative` указывает тип альтернативной гипотезы (двусторонняя, только больше, только меньше).
- `alternative='two-sided'` — проверяет, есть ли различие между двумя группами в любом направлении.
- `alternative='less'` и `alternative='greater'` — проверяют, имеют ли данные первой выборки тенденцию быть меньше или больше данных второй выборки.

## Задание 1.

Компания «Web Innovate» провела редизайн своего сайта. Аналитический отдел компании собрал данные об удовлетворённости клиентов до изменений (`satisfaction_before`) и сразу после них (`satisfaction_after`). Предполагается, что удовлетворённость до редизайна, скорее всего, была меньше, чем после него. Медианные значения совпали.

Ваша задача — проанализировать эти данные и определить, привёл ли редизайн к значительному улучшению клиентского опыта, исходя из уровня значимости `0.05`.

```python
from scipy.stats import mannwhitneyu
  
satisfaction_before = [ 5, 2, 4, 4, 3, 4, 5, 5, 4, 4,
      4, 4, 5, 5, 2, 4, 6, 3, 4, 5, 4, 5, 4, 5, 4, 4,
      3, 4, 4, 3, 4, 4, 3, 3, 4, 3, 7, 4, 2, 2, 6, 5,
      3, 4, 5, 4, 3, 4, 3, 5 ]
satisfaction_after =[2, 5, 4, 6, 4, 3, 4, 4, 5, 5, 4,
      5, 6, 5, 4, 5, 3, 5, 6, 4, 4, 1, 2, 5, 4, 1, 5,
      3, 4, 4, 5, 6, 4, 6, 5, 5, 4, 2, 4, 4, 5, 4, 5,
      5, 3, 6, 5, 5, 5, 4]
  
stat, p_value = mannwhitneyu(satisfaction_before, satisfaction_after, alternative='less')
print(f"Статистика U-теста: {stat}, p_value: {p_value}")
```

## Задание 2.

```python
print(f"Статистика U-теста между ДО и МЕСЯЦ ПОСЛЕ: {stat_before_month}, p_value: {p_value_before_month}")
  
# Сравнение удовлетворённости сразу после и через месяц после редизайна
```

## Задание 3.

В предыдущей задаче мы получили неоднозначный результат.

Уровень удовлетворённости, повышенный сразу после внедрённых изменений, сохраняется и спустя месяц (между «после» и «месяц после» нет разницы), подтверждая устойчивость положительного эффекта.

При этом уровни удовлетворённости до редизайна и месяц после также не различаются достаточно значимым образом, что говорит скорее об отсутствии эффекта от редизайна.

Теперь команда «Web Innovate» направила усилия на оценку совокупного влияния редизайна. Для этого сотрудники объединили данные об удовлетворённости клиентов сразу после редизайна (`satisfaction_after`) и данные, собранные через месяц после редизайна (`satisfaction_month_after`), в одну группу. Этот анализ поможет сравнить уровень удовлетворённости до внесения изменений (`satisfaction_before`) с удовлетворённостью после редизайна, учитывая как непосредственные, так и долгосрочные эффекты изменений.

```python
from scipy.stats import mannwhitneyu
  
satisfaction_before = [5, 2, 4, 4, 3, 4, 5, 5, 4, 4,
                       4, 4, 5, 5, 2, 4, 6, 3, 4, 5,
                       4, 5, 4, 5, 4, 4, 3, 4, 4, 3,

                      4, 5, 6, 5, 4, 5, 3, 5, 6, 4,
                      4, 1, 2, 5, 4, 1, 5, 3, 4, 4,
                      5, 6, 4, 6, 5, 5, 4, 2, 4, 4,
                      5, 4, 5, 5, 3, 6, 5, 5, 5, 4]
  
satisfaction_month_after = [4, 6, 2, 5, 4, 5, 6, 3, 5,
                            5, 3, 6, 3, 5, 5, 3, 3, 7,
                            5, 5, 4, 7, 4, 4, 3, 5, 5,

                            4, 3, 7, 5, 3]
  
# Объединение данных сразу после редизайна и через месяц после редизайна
satisfaction_after_combined = satisfaction_after + satisfaction_month_after
  
# Сравнение данных до редизайна и объединённых данных после редизайна
stat, p_value = mannwhitneyu(satisfaction_before, satisfaction_after_combined, alternative='two-sided')
print(f"Статистика U-теста между данными до редизайна и после (объединённые): {stat}, p_value: {p_value}")
```
