# Определение типов данных DataFrame и Series

## Данные, но ещё не совсем. Словари
Структуры данных позволяют хранить данные так, чтобы человеку было удобно с ними работать. С одной такой структурой — списками — вы уже познакомились. Списки помогают собрать несколько объектов вместе.

Теперь вы готовы познакомиться с ещё одной, не менее полезной структурой данных в Python — словарями.

### **В этом уроке вы узнаете:**

- Что такое словари в Python и зачем они нужны;
- Какие данные можно хранить в словаре и как с ними работать;
- Когда может пригодиться список со словарями.

### Что такое словарь?

Словари в Python очень похожи на словари и энциклопедии в обычной жизни.

- В энциклопедии, обращаясь к термину, мы получаем его объяснение.
- В англо-русском словаре, обращаясь к слову на английском языке, мы получаем его перевод на русский.

Так выглядит любая словарная статья: это пара значений, где первое помогает найти второе. А сам словарь — это просто набор таких пар: слово — перевод.

Словари в Python устроены так же. Каждый элемент словаря — пара из ключа и значения:

- **ключ** — то, что мы ищем в словаре. Ключом может быть строка или число;
- **значение** — данные, к которым ведёт этот ключ. В отличие от ключа, значением может быть объект любого типа: не только строка или число, но и, например, список.

Для словарей в Python есть свой тип объекта — `dict`.

![](https://pictures.s3.yandex.net/resources/image_1707913127.png)

### Зачем нужен словарь, когда есть списки?

Часто удобнее искать значения не по его индексу, а по ключу, который имеет собственный смысл.

Вот почему мы бы использовали словарь, а не список:

- Данные о пользователе сайта проще найти, используя его имя.
- Рецепт блюда проще найти по его названию.

### Создание словаря

Есть несколько правил синтаксиса для создания словаря:

- словари записываются в фигурных скобках `{ }`;
- элементы словаря перечисляются через запятую;
- ключ и значение разделяют двоеточием `:`.

Создадим словарь с авиарейсами. Ключом станет номер рейса, а значением — дата и время вылета:

```
schedule = {                    # элементы словаря заключают в фигурные скобки
    'SU2222': '12.06.23 12:30', # слева от двоеточия — ключ, справа — значение
    'SU1111': '12.06.23 14:05', # каждый элемент отделяется запятой
    'SU0777': '12.06.23 17:00',
    'SU5555': '15.06.23 11:25'
} 
```

Каждую пару ключ-значение мы записали в новой строке. В правилах синтаксиса мы ничего не сказали о переносах. Если использовать переносы, код становится проще читать. Но ничего не мешает записать словарь и в плоском виде.

```
schedule = {'SU2222': '12.06.23 12:30', 'SU1111': '12.06.23 14:05', 'SU0777': '12.06.23 17:00', 'SU5555': '15.06.23 11:25'} 
```

Оба варианта верны, но первый вариант выглядит удобнее.

Выведем содержимое словаря с помощью `print()`:

```
print(schedule) 
```

```
{'SU2222': '12.06.23 12:30', 'SU1111': '12.06.23 14:05', 'SU0777': '12.06.23 17:00', 'SU5555': '15.06.23 11:25'} 
```

Выведем тип полученного объекта:

```
print(type(schedule)) 
```

```
<class 'dict'> 
```

Мы разобрались, как создавать словари. Теперь давайте посмотрим, как с ними работать.

### Находим значение в словаре по ключу

Обратиться к значению в словаре можно при помощи квадратных скобок. Как со списками — только указываем не индекс, а ключ.

```
schedule = {                    
    'SU2222': '12.06.23 12:30', 
    'SU1111': '12.06.23 14:05', 
    'SU0777': '12.06.23 17:00',
    'SU5555': '15.06.23 11:25'
}

print(schedule['SU2222'])
print(schedule['SU1111']) 
```

```
12.06.23 12:30
12.06.23 14:05 
```

Если обратиться к словарю по ключу, которого нет, то вернётся ошибка, и программа остановится. Нужно быть уверенным в том, что ключ есть в словаре.

```
schedule = {                    
    'SU2222': '12.06.23 12:30', 
    'SU1111': '12.06.23 14:05', 
    'SU0777': '12.06.23 17:00',
    'SU5555': '15.06.23 11:25'
}

print(schedule['DP6831']) 
```

```
KeyError: 'DP6831' 
```

Если мы заранее не знаем, есть ли такой ключ, то нужно использовать метод `get()`:

```
schedule = {                    
    'SU2222': '12.06.23 12:30', 
    'SU1111': '12.06.23 14:05', 
    'SU0777': '12.06.23 17:00',
    'SU5555': '15.06.23 11:25'
}

print(schedule.get('DP6831')) 
```

```
None 
```

Этот способ вернёт значение `None`, если такого ключа не нашлось. При этом ошибки не будет, и программа не остановится.

Что будет, если в словаре для одного слова будет два разных определения, то есть ключ будет встречаться несколько раз?

```
print({
    0: 'ноль',
    1: 'один',
    2: 'два',
    1: 'единица'
}) 
```

```
{0: 'ноль', 1: 'единица', 2: 'два'} 
```

Одно значение пропало! Это потому, что ключи словаря должны быть уникальными. Ниже мы разберёмся, как сохранить в словаре несколько значений по одному ключу.

### Какие данные может хранить словарь?

Значениями словаря могут быть числа и строки, но не только. Списки тоже можно хранить. Создадим ещё один словарь с рейсами, но для каждого сохраним расписание сразу на несколько лет вперёд. Для этого используем списки.

```
schedule = {                   
    'SU2222': ['12.06.23 12:30', '12.06.24 18:30', '12.06.25 22:30'], 
    'SU1111': ['12.06.23 14:05', '12.06.24 20:05', '12.06.25 23:05'], 
    'SU0777': ['12.06.23 17:00', '12.06.24 21:00', '12.06.25 21:00'],
    'SU5555': ['15.06.23 11:25', '12.06.24 17:25', '12.06.25 22:25'], 
}

print(schedule['SU2222']) 
```

```
['12.06.23 12:30', '12.06.24 18:30', '12.06.25 22:30'] 
```

А ещё значениями словаря могут быть другие словари. Их называют вложенными. Создадим словарь с рейсами, но ключом станет не название рейса, а название компании-перевозчика. Значением будет словарь со всеми рейсами и их расписанием.

```
schedule = {
    'Аэрофлот': {
        'SU2222': '12.06.23 12:30',
        'SU1111': '12.06.23 14:05',
        'SU0777': '12.06.23 17:00',
        'SU5555': '15.06.23 11:25'},
    'Победа': {
        'DP6831': '12.06.23 09:30', 
        'DP6800': '12.06.23 11:05'},
    'Nordwind Airlines': {
        'N40001': '12.06.23 08:15', 
        'N48321': '12.06.23 19:15'}
    } 
```

Получим все рейсы компании `Аэрофлот`:

```
aeroflot_flights = schedule['Аэрофлот']

print(aeroflot_flights) 
```

```
{'SU2222': '12.06.23 12:30',
 'SU1111': '12.06.23 14:05',
 'SU0777': '12.06.23 17:00',
 'SU5555': '15.06.23 11:25'} 
```

А теперь получим дату вылета рейса `SU2222`:

```
current_flight = aeroflot_flights['SU2222']

print(current_flight) 
```

```
12.06.23 12:30 
```

То же самое мы могли бы сделать и одной строчкой кода, последовательно обращаясь сначала к верхнему словарю, а затем ко вложенному, используя несколько раз квадратные скобки:

```
current_flight = schedule['Аэрофлот']['SU2222'] 
```

Ключами словаря не могут стать изменяемые объекты, например списки или словари. Обычно ключами в словаре являются строки или числа, будем придерживаться этого.

### Поиск в словаре

Создадим словарь со странами и их столицами.

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
} 
```

Как вы помните, словарь — это структура, которая состоит из ключей и значений. Ключи и значения связаны, но с ними можно работать по отдельности.

Иногда нам может потребоваться получить набор из всех ключей, которые есть в словаре. Например, найдём и выведем все страны (страна — это ключ), название которых состоит из трёх символов. Для этого воспользуемся методом `keys()`.

```
all_countries = country_capitals.keys()

for country in all_countries:
    if len(country) == 3:
        print(country) 
```

```
США 
```

Точно так же мы можем сразу найти и все значения, которые есть в словаре. Найдём все столицы (столица — это значение), название которых состоит из трёх символов. Для этого воспользуемся методом `values()`.

```
all_capitals = country_capitals.values()

for capital in all_capitals:
    if len(capital) == 3:
        print(capital) 
```

```
Рим 
```

Словарь отлично подходит для поиска **значения** по его **ключу**. Но что, если, наоборот, необходимо найти ключ по его значению? Для решения такой задачи придётся перебрать в цикле каждую пару ключ-значение, которые содержатся в словаре, пока не найдётся пара с нужным значением.

Решим похожую задачку на примере данных о городах и их столицах. Найдём страну, в которой столица — Рим.

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}
rome_country = ''

for country in country_capitals.keys(): # keys() вернёт список всех стран
    capital = country_capitals[country] # сохраним столицу текущей страны
    if capital == 'Рим':       # если столица Рим
        rome_country = country # то сохраним страну в переменной rome_country
        break                  # и завершим цикл
                               # если столица не Рим, то продолжаем цикл

print(rome_country) 
```

```
Италия 
```

Сейчас разберёмся, что тут происходит:

1. Первое, что мы сделаем, — создадим цикл, в котором будем перебирать все страны в словаре. Метод `keys()` возвращает набор всех ключей, а ключ — это и есть страна.

```
for country in country_capitals.keys():
    pass 
```

2. Найдём столицу для текущей страны, для этого используем наш словарь `country_capitals`. Теперь в переменной `capital` хранится столица, а в переменной `country` — страна.

```
capital = country_capitals[country] 
```

3. Проверим, является ли текущая столица Римом. Если является, то мы нашли нужную страну. Сохраним результат в переменной `rome_country` и завершим выполнение цикла командой `break`.

```
if capital == 'Рим':
    rome_country = country
    break  
```

4. Если столица не Рим, то продолжаем цикл и повторяем то же самое на следующей итерации.

Наше решение строилось на том, чтобы в цикле перебрать все пары ключ-значение. Для этого при создании цикла мы получили набор ключей методом `keys()`, а уже внутри цикла для каждого нового ключа находили его значения.

А что, если сразу получать эту пару — ключ-значение? Да, так тоже можно! Для этого воспользуемся методом `items()`:

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}

for country, capital in country_capitals.items():
    print(country, capital) 
```

```
США Вашингтон
Италия Рим
Англия Лондон 
```

Решим ту же самую задачу — найдём страну, столица которой — Рим. Но теперь воспользуемся методом `items()`:

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}
rome_country = ''

# запустим цикл: items() вернёт пары — страна и столица
for country, capital in country_capitals.items(): 
    if capital == 'Рим':       # если столица Рим
        # то сохраним страну в переменной rome_country
        rome_country = country 
        # и прервём цикл
        break                  
    # если столица не Рим, то продолжаем цикл

print(rome_country) 
```

```
Италия 
```

Наш код стал на строчку короче, а решает задачу ничуть не хуже!

### Действия со словарями

Работа со словарями очень похожа на работу со списками. В словарь можно добавлять новые элементы, изменять их и удалять.

Для того чтобы добавить в словарь новый ключ, просто присвоим ему новое значение:

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}
country_capitals['Россия'] = 'Москва'

print(country_capitals) 
```

```
{'США': 'Вашингтон', 'Италия': 'Рим', 'Англия': 'Лондон', 'Россия': 'Москва'} 
```

Если такой ключ уже есть в словаре, то его значение обновится.

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}

country_capitals['США'] = 'Washington'
country_capitals['Италия'] = 'Roma'
country_capitals['Англия'] = 'London'

print(country_capitals) 
```

```
{'США': 'Washington', 'Италия': 'Roma', 'Англия': 'London'} 
```

Таким способом, мы можем изменить значение в словаре по его ключу. Однако мы не можем изменить сам ключ. Если по какой-то причине нам нужно изменить его, то необходимо удалить элемент с этим ключом и создать новый с правильным ключом.

Для удаления элемента из словаря можно использовать метод `pop()`. Он работает так же, как и со списками — удаляет элемент и возвращает значение, которое было удалено.

```
country_capitals = {
    'США': 'Вашингтон',
    'Италия': 'Рим',
    'Англия': 'Лондон'
}

deleted_capital = country_capitals.pop('Англия')

print(country_capitals)
print(deleted_capital) 
```

```
{'США': 'Вашингтон', 'Италия': 'Рим'}
Лондон 
```

### Список со словарями

Представим, что у нас есть информация о пользователе: ФИО, дата регистрации, дата последнего заказа и количество заказов. Для хранения такой информации хорошо подходит словарь.

```
user_info = {
    'ФИО': 'Иван Иваныч',
    'Дата регистрации': '01.01.2021',
    'Дата последней покупки': '05.07.2023',
    'Количество заказов': 123
} 
```

А что делать, если таких покупателей будет несколько?

```
user_info_1 = {
    'ФИО': 'Иван Иваныч',
    'Дата регистрации': '01.01.2021',
    'Дата последней покупки': '05.07.2023',
    'Количество заказов': 123
}

user_info_2 = {
    'ФИО': 'Пётр Васильич',
    'Дата регистрации': '03.06.2020',
    'Дата последней покупки': '20.07.2023',
    'Количество заказов': 20
}

user_info_3 = {
    'ФИО': 'Марья Петровна',
    'Дата регистрации': '05.02.2018',
    'Дата последней покупки': '20.05.2021',
    'Количество заказов': 5
} 
```

Нам поможет список! Вы же помните, что любой тип объекта может стать элементом списка? Словарь не исключение.

```
users = [user_info_1, user_info_2, user_info_3] #список словарей 
```

Такой способ хранения данных напоминает привычные таблицы. Попробуем найти общее количество заказов от всех пользователей.

```
orders = 0
for user in users:
        orders += user['Количество заказов']

print(orders) 
```

```
148 
```

### Итоги

Вы познакомились с новой структурой данных — словарём. Словари помогают хранить информацию, как справочники. Повторим самое главное:

- Словарь состоит из ключа и значения. Значением может быть объект любого типа, а ключом, как правило, строка или число.
- Обращение к элементу словаря происходит по его ключу.
- В словарь можно добавлять новые элементы и удалять старые.
- Для того чтобы перебрать все элементы словаря в цикле, можно использовать метод `items()`.

Начиная работать с данными, аналитику необходимо посмотреть на них своими глазами. Иногда одного такого взгляда достаточно, чтобы сделать важные выводы о данных!

После этого стоит подключить и другие инструменты. Работа аналитика всегда связана с данными, и чем больше этих данных, тем сложнее сам анализ.

### Ограничения или свобода

На первых этапах нам помогают специальные программы, например Excel. Но у таких программ всегда есть ограничения. Ограничение на количество строк и колонок и другие. Например, в случае Excel мы точно знаем, что не можем работать с таблицами, в которых больше чем 1 048 576 строк и 16 384 колонки. Это будет проблемой, если нас попросят проанализировать действия пользователей на каком-нибудь популярном сайте с десятками миллионов посещений в месяц. Также для сложных вычислений придётся писать длинные, плохо читаемые формулы, которые добавляют много ячеек для промежуточных вычислений.

С этого вы и начали изучать Python — открыв для себя способы избавиться от этих ограничений. Вы научились основам Python, познакомились с переменными и объектами. Научились объединять объекты и сохранять их в удобные структуры — в списки и словари. Но хватит ли этого для решения любых задач аналитика?

Не совсем. Задачи анализа данных порой требуют серьёзных вычислительных мощностей, тем более когда данных становится много или очень много.

### А много данных — это сколько?

Представим, что мы работаем с данными о покупателях. О каждом у нас есть некая информация. В случае Excel мы использовали таблицу с колонками. В Python для этого можно использовать словарь. Сохраним данные по каждому покупателю в словарь:

```
user_info = {
    'ФИО': 'Иван Иваныч',
    'Дата регистрации': '01.01.2021',
    'Дата последней покупки': '05.07.2023',
    'Количество заказов': 2,
    'Сумма выкупа': 123456,
    'Адрес доставки': {
        'Улица': 'Лубянка', 
        'Дом': '1'}
    } 
```

В Excel это будет выглядеть примерно так:

Много данных или мало — это относительный вопрос, и на него нет чёткого ответа. Все зависит от того, что именно мы будем делать с этими данными, какие методы для работы с ними использовать, в каком виде эти данные хранить и какую именно задачу решать.

Если мы только начинаем исследование, и нам не нужен результат здесь и сейчас, то подождать 30 секунд, пока операция не завершится, вовсе некритично. Мы не можем сказать, что данных много.

Однако если таких «долгих» операций будет много, а результаты нужны прямо сейчас, то задержка выполнения каждой операции даже в секунду будет ощутимой. В таком случае мы можем сказать, что данных много и необходимо оптимизировать вычисления.

Но ведь работать со списками будет уже не так удобно!

### В поисках прибыли

Представьте, что вы работаете аналитиком в интернет-магазине. Покупатель оплачивает заказ — это доход, а сотрудники магазина собирают заказ и отправляют его — это расходы. Чтобы заказ приносил прибыль, нужно чтобы доход был больше расходов на его выполнение. Разницу между доходом и расходами и называют прибылью. Найдём же её!

Итак, чтобы определить прибыль с заказа, необходимо вычесть из дохода расходы. Если бы заказ был всего один, то мы бы посчитали это так:

```
income = 1000 # доходы
expenses = 900 # расходы

profit = income - expenses 
```

А что, если таких заказов много? Допустим, у нас больше 10 миллионов заказов. А мы хотим оптимизировать наши бизнес-процессы. Для этого вычислим, какие именно заказы оказались неприбыльными.

Удобно представить наши данные в виде двух колонок в таблице, где каждому заказу сопоставлен доход от него и расходы на его выполнение. Вы бы вычли из доходов расходы и оставили бы только отрицательные результаты.

![](https://pictures.s3.yandex.net/resources/image_1708075880.png)

В Python мы не можем просто так создать таблицу. Но мы можем попробовать сделать что-то похожее на неё. Например, представим, что каждая колонка в нашей таблице — это отдельный список.

```
incomes = [100, 250, 340, 2000]
expenses = [90, 260, 100, 1800] 
```

По сути, нам нужно вычесть из списка `incomes` список `expenses`. Попробуем сделать что-то подобное с помощью цикла `for`:

```
incomes = [100, 250, 340, 2000] # Создадим список с доходами
expenses = [90, 260, 100, 1800] # Список с расходами
profits = []

for i in range(len(incomes)): # Пройдёмся в цикле по каждому заказу
    # Для каждого заказа вычтем расходы из доходов
    profits.append(incomes[i] - expenses[i]) 

print(profits) 
```

А теперь оставим только те заказы, которые не прибыльны.

```
negative_profit = [] # Список убыточных заказов

for i in range(len(profits)):
    if profits[i] <= 0: # Если заказ неприбыльный
    # То добавим убыток в список
        negative_profit.append(profits[i])  

print(negative_profit) 
```

У каждого заказа должен быть свой номер, а мы его не учитывали. У нас появится новый список, теперь их три. Сразу сохраним неприбыльные заказы.

```
orders = ['001', '002', '003', '004'] # Список с номерами заказов
incomes = [100, 250, 340, 2000]
expenses = [90, 260, 100, 1800]

negative_profit_orders = []

for i in range(len(incomes)):
    # Если для заказа отрицательная прибыль
    if incomes[i] - expenses[i] < 0: 
        # То сохраним номер заказа
        negative_profit_orders.append(orders[i]) 

print(negative_profit_orders) 
```

Это именно то, что мы и хотели, но код выглядит нагромождённым, приходится по порядку перебирать все элементы. В нашем примере всего четыре заказа, но если их будет несколько миллионов, то нам стоит использовать другой подход.

### Здесь начинаются векторные вычисления

Было бы гораздо удобнее просто вычесть `incomes - expenses`, как если бы вы работали с колонками в таблице Excel. Такие вычисления называют векторными. В векторных вычислениях мы проводим операции сразу над целыми группами данных, а не отдельными элементами. Такое название пришло из информатики, где вектором называют упорядоченный набор элементов, например кортеж или список.

Вычесть expenses из incomes — векторная операция. Результат у неё будет тот же, как если бы мы поэлементно вычли расходы из доходов и сохранили в отдельный список. Но при значительном количестве данных она выполнится быстрее.

![](https://pictures.s3.yandex.net/resources/image_1708075911.png)

Результатом выполнения векторного вычисления необязательно будет вектор (список чисел). Вы уже знакомы с векторной функцией `sum()`, которая принимает список значений и возвращает сумму значений, или функцией `len()`, которая возвращает длину списка.

### Датафрейм

Работать с таблицами в Pandas помогает специальная структура — датафрейм (англ. DataFrame).

Датафрейм состоит из колонок и строк — всё, как в таблице. У колонок есть названия, а у строки — индекс (неформально говоря, номер).

Например, используем датафрейм для данных по странам и их столицам. В колонках сохраним названия стран и их столиц, а каждую новую запись — в новой строке.

![](https://pictures.s3.yandex.net/resources/image_1708097222.png)

С датафреймом можно работать как с обычной таблицей. Его можно изменять, дополнять и удалять. Например, можно добавить новую колонку или новую строку.

![](https://pictures.s3.yandex.net/resources/image_1708097243.png)

Датафрейм — это двухмерная структура, у неё есть колонки и строки. В работе мы часто обращаемся к конкретным колонкам или строкам. Каждая колонка или строка — это объект особого типа — серия (англ. Series).

### Серия

Объект датафрейм состоит из объектов типа серия. Если мы обратимся к конкретной колонке или строке датафрейма, то будем работать уже с объектом типа серия.

![](https://pictures.s3.yandex.net/resources/image_1708097268.png)

У серий есть несколько атрибутов, основные — два: название `name` и тип объектов в ячейках `dtype` (англ. data type — тип данных).

![](https://pictures.s3.yandex.net/resources/image_1708097291.png)

Тип объекта расскажет о том, какие данные хранит серия. Основные типы:

- `int`, `float` — числа;
- `datetime` — даты;
- `object` — объекты разных типов. Как правило, это обычные строки. Для хранения строк нет специального типа данных.

Если мы обращаемся к колонке, то `name` — это название колонки.

В таком случае `name` — это название индекса для строки. Получается, что в датафрейме строки и колонки мало чем различаются. Сейчас разберёмся, почему так вышло.

Привычный взгляд на таблицу — сверху вниз. Вверху названия колонок, а слева — индексы.

А теперь перевернём таблицу!

Видно, что индексы строк — это их названия.

Поскольку строки — это тоже серии, у них есть те же параметры, что и у колонок: название и тип данных.

## Заполнение датафрейма на основе списка
В прошлом уроке вы научились создавать датафреймы и наполнять их данными. Вы использовали объекты Series и добавляли их как новые колонки в пустой датафрейм. Но иногда и это не самый удобный способ.

a_list = [[1, 'a'], [2, 'b'], [3, 'c']] # двумерный список

Этот список состоит из похожих друг на друга списков. Но пока не понятно, в чём идея. Попробуем написать всё то же самое, но немного по-другому:

a_list = [[1, 'a'],

### Наполним датафрейм данными

Как всегда, начнём с импорта pandas:

```
import pandas as pd 
```

Создадим вложенный список со странами и их столицами:

```
data_countries = [['Россия', 'Москва'],

Представьте, что вы работаете со списком, где каждый элемент — это строка в таблице. Каждый вложенный список является очередной строкой в датафрейме, а значения в нём разделены по колонкам. Значения могут быть разных типов: строки, числа, даты. Мы используем строки.

Создадим датафрейм, но в этот раз используем параметр `data`. Именно этот параметр отвечает за то, какие данные будут храниться в датафрейме.

df = pd.DataFrame(data=data_countries)

А можно и так:

```
df = pd.DataFrame(data_countries)

import pandas as pd

import pandas as pd

## Заполнение датафрейма на основе словаря
В прошлом уроке вы использовали параметр `data` для наполнения датафрейма. Этот параметр относительно универсальный: он может принимать данные разных типов и форматов. Какой способ удобнее — зависит от задачи. Это могут быть словари, списки или даже другие датафреймы.

![](https://pictures.s3.yandex.net/resources/image_1708262095.png)

Хотя параметр `data` и универсальный, но всё же ограничения есть. Необходимо, чтобы данные в `data` соответствовали определённым правилам, иначе магия превращения этих данных в датафрейм не сработает.

```
# some_data хранит содержимое датафрейма в неком виде
df = pd.DataFrame(data=some_data) 
```

Вы уже научились создавать данные на основе списка. Теперь попробуем словари — при их использовании у нас появится чуть больше возможностей.

### Как сохранить таблицу в словаре

Объясним, как хранить данные в словаре так, чтобы Pandas смог сделать из них датафрейм. Работать будем всё с теми же данными о странах и их столицах.

### **Ключ — колонка**

Разберём способ «ключ — колонка». В нём ключи словаря — это названия колонок, а значения — набор значений в колонке. Вы уже знаете, что Series подходит для хранения значений в колонке. Создадим словарь `data_countries`:

```
data_countries = {
    # По ключу храним набор значений в колонке
        'cтрана': pd.Series(['Россия', 'США', 'Великобритания', 'Италия']),
        'столица': pd.Series(['Москва', 'Вашингтон', 'Лондон', 'Рим'])
    } 
```

Передадим этот словарь как параметр `data`:

```
# Создали датафрейм на основе данных data_countries
pd.DataFrame(data=data_countries) 
```

|**страна**|**столица**|
|---|---|
|Россия|Москва|
|США|Вашингтон|
|Великобритания|Лондон|
|Италия|Рим|

Готово! Ваш первый датафрейм на основе словаря. Мы использовали Series для хранения значений. Но если мы передадим списки, то Pandas всё равно поймет нас, иногда это даже удобнее:

```
data_countries = {
        'cтрана': ['Россия', 'США', 'Великобритания', 'Италия'],

data_countries = {

df = pd.DataFrame(data=data_countries)
display(df)

## Выгрузка датафрейма из файла
Часто аналитик получает данные для работы в виде файлов. Это могут быть Excel-файлы c расширением XLS (или XLSX), а ещё могут быть CSV или TSV файлы. Бывают и другие типы, но чаще всего используют CSV-формат. Этот формат хранения табличных данных — один из самых простых и универсальных, он наиболее распространён.

Используя pandas, аналитик может превратить полученный файл в датафрейм. Этот процесс называют выгрузкой данных.

### Что такое CSV файл

CSV расшифровывается как Comma-Separated Values или значения, разделённые запятой.

Посмотрим на знакомые данные о городах и их столицах.

В формате CSV это выглядело бы так:

```
страна,столица
Россия,Москва
США,Вашингтон
Великобритания,Лондон
Италия,Рим 
```

Данные записываются по строчкам, а все значения по колонкам разделены запятыми. Первой строкой принято указывать названия для колонок.

CSV-файл предполагает, что значения разделены запятой, но так бывает не всегда. Данные может разделять любой другой символ, например точка с запятой или пробел. Эти символы называют разделителями.

Довольно часто разделителем оказывается знак табуляции `\t` — такие файлы иногда сохраняют в специальном формате TSV (tab-seperated values), но не всегда, табуляция может встретиться в CSV-файле.

Итак, вам как аналитику передали файл с данными. Для анализа необходимо выгрузить эти данные в датафрейм. Как же это сделать?

# Заполнение и обработка данных в DataFrame

## Зачем нужна фильтрация данных
В предыдущих уроках вы познакомились с библиотекой **pandas** в **Python** и разобрались, что такое **DataFrame**, из чего он состоит, как в нём хранятся данные и как эти данные получить. Однако для проведения анализа данных и решения конкретных задач часто нужны не все данные датафрейма, а только их часть, которая соответствует определённым критериям. Для её получения аналитики используют фильтрацию данных.

В этом уроке мы разберём, что такое фильтрация данных и зачем она применяется, а в последующих уроках более детально изучим основные инструменты, которые можно использовать для фильтрации данных **DataFrame**.

### Фильтрация данных: наглядно

В повседневной жизни мы часто имеем дело с фильтрацией информации. Например, почтовый сервер фильтрует входящие письма и помещает нежелательные сообщения или рекламу в папку «Спам». Алгоритмы социальных сетей могут исключать из выгрузки публикации, которые нам не интересны. Фильтрация товаров в интернет-магазинах или на маркетплейсах позволяет нам быстрее найти товар необходимого типа или бренда. Таблички на подъездах домов указывают номера только тех квартир, которые находятся в нужном нам подъезде. На остановках общественного транспорта перечислены только местные маршруты — представьте, что было бы, если бы на остановках была информация обо всех маршрутах города.

![](https://pictures.s3.yandex.net/resources/image_1711881108.png)

Таких примеров можно привести множество. В любом из них фильтрация данных позволяет нам получить только необходимую информацию, сократить время на её поиск, избежать ложных или неверных значений. То есть оптимизировать процесс получения данных, будь то ответ на вопрос в интернете или выбор напитка из меню ресторана.

### Фильтрация данных при их анализе

В своей работе аналитик часто использует фильтрацию данных. Она позволяет не только провести необходимый анализ по определённым критериям, но и проверить данные на возможные ошибки и улучшить их качество. Условно фильтрацию данных можно разделить на несколько групп по её назначению:

1. **Фильтрация по определённым значениям или их диапазону:** используется для получения выборки данных, соответствующей определённым значениям каких-либо параметров или метрик. Например:
    
    a. найти пользователей, которые совершили больше 10 заказов; b. провести сегментацию пользователей по их возрасту; c. отобрать данные по тем каналам привлечения, у которых конверсия в покупку выше ожидаемого значения.
    
    Также такой тип фильтрации часто используется для фильтрации аномальных или ошибочных данных. Например: 1. найти заказы с отрицательной стоимостью; 2. при сегментации пользователей проверить аномально высокие значения возраста, например, 200 лет; 3. найти пользовательские сессии за день, длительность которых превысила 24 часа.
    
2. **Фильтрация данных по временному диапазону:** помогает создавать аналитические отчёты по дням, неделям, месяцам или иным временным периодам; сравнивать исторические данные, то есть данные, полученные за предыдущие периоды, с текущими значениями; анализировать изменения метрик во времени и проводить когортный анализ. Например:
    
    a. фильтрация данных по дням поможет оценить изменение количества новых пользователей в интернет-магазине, а изучение продаж в различные месяцы — оценить фактор сезонности; b. изучение временной динамики метрик монетизации и вовлечённости позволяет оценить эффективность вложений в рекламу со временем.
    
3. **Фильтрация данных по категориям и сегментам:** часто используется для создания выборки (или среза) данных на основе принадлежности пользователей или товаров к определённым категориям, группам или сегментам. Например:
    
    a. фильтрация товаров интернет-магазина по их категории; b. фильтрация определённых сегментов пользователей по различным признакам — региону, типу устройства, полу, возрасту, интересам и так далее.
    
4. **Фильтрация данных по отдельным столбцам:** иногда необходимо упростить данные и оставить только часть информации из всего объёма. Например, из всех данных по доставке еды из ресторанов оставить только данные по времени доставки и месту назначения.
    

Приведённые выше подходы к фильтрации данных являются одними из наиболее распространённых. На практике для решения различных задач могут быть использованы более сложные алгоритмы фильтрации. Например, фильтрация строк для поиска и исключения дублирующихся или пропущенных значений в данных.

С этими алгоритмами мы будем постепенно знакомиться по ходу обучения.

### Практическое значение фильтрации данных

Верно проведённая фильтрация позволяет использовать только необходимые для анализа данные. При этом исключается не интересующая аналитика информация, а также некорректные или ошибочные данные. Улучшается качество данных, снижается их объём, оптимизируются время и ресурсы для проведения дальнейшего анализа.

Однако к фильтрации данных следует подходить аккуратно и учитывать её возможное влияние на полноту данных. Некорректная настройка критериев фильтрации может привести к частичной потере данных или искажению результатов, а также к выявлению ложных закономерностей. Например, при фильтрации строк с пропущенными значениями только столбца, который не влияет на основной анализ данных, можно удалить существенную часть информации. Предположим, удаление купленных товаров без комментариев пользователей может исказить результаты изучения среднего количества купленных товаров по различным категориям, поскольку наличие комментария не влияет на факт покупки товара.

Дополнительно критерии фильтрации должны быть чувствительны к возможным изменениям со временем: критерии могут меняться в зависимости от сезонности или изменения внешних факторов. Поэтому при использовании сложных логических выражений важно следить за их корректностью и проверять результат фильтрации данных.

Если вы работали с электронными таблицами в **Excel** или аналогичных редакторах, то с большой вероятностью взаимодействовали с различными встроенными фильтрами данных по значениям в столбцах или скрывали нерабочие интервалы строк. Фильтрация данных **DataFrame** проводится аналогичным образом. В следующих уроках мы рассмотрим, какие операторы и методы используются для фильтрации **DataFrame** на примере данных сервиса Яндекс Музыка.

## Фильтрация по столбцам
Иногда для решения поставленных аналитических задач достаточно только части данных, которые находятся в одном или нескольких столбцах. Например, для изучения пользовательской активности интернет-магазина необходимо определить дату первой покупки каждого покупателя. Для этого достаточно иметь информацию об идентификаторе пользователя и дате совершения первой покупки. Любые другие сведения, например, количество товаров в заказе и их стоимость, адрес доставки или название устройства входа будут избыточными.

![часто для анализа необходима только часть данных.](https://pictures.s3.yandex.net/resources/image_1708616800.png)

Как правило, аналитик располагает полными данными, например, всеми записями действий пользователей (логами активности, если рассматривать пример выше). Поэтому часто стоит задача провести фильтрацию полных данных, оставив только необходимую информацию. Для создания выборки данных, включающей только часть столбцов исходного **DataFrame**, применяют фильтрацию по столбцам.

В этом уроке мы разберём, как провести фильтрацию по столбцам в **pandas:**

- рассмотрим возможности оператора **DataFrame** `[]` при фильтрации столбцов;
- сохраним выборку данных в виде нового набора данных;
- познакомимся с тем, как удалить лишние для анализа данных столбцы **DataFrame** с помощью метода `.drop()`.

Для разбора примеров будем использовать базу данных **Яндекс Музыки**. Загрузим данные и сохраним в переменной `music_df`:

```
import pandas as pd # импортируем библиотеку pandas

# сохраним данные Яндекс Музыки в переменной music_df:
music_df = pd.read_csv('music_log.csv')  
```

### Фильтрация по столбцам: оператор DataFrame `[]`

Из предыдущих уроков вы знаете, что **DataFrame** в **pandas** состоят из отдельных серий данных — **Series**, которые можно представить в виде столбца обычной таблицы. Для того чтобы вывести на экран содержимое одного столбца, достаточно указать его название (метку) в специальном операторе **DataFrame** — квадратных скобках `[]`. В этом случае синтаксис будет следующий:

```
# выборка одного столбца из DataFrame df:
df['column_А'] 
```

Для создания выборки, состоящей из нескольких столбцов, необходимо указать их метки через запятую и передать в оператор `[]` в виде списка. Количество столбцов может быть неограниченным, а их последовательность может не совпадать с последовательностью столбцов в исходном датафрейме:

```
# выборка нескольких столбцов из DataFrame df:
df[['column_А', 'column_B', ... , 'column_N']] 
```

![На схеме изображен основной алгоритм работы оператора [] при создании выборки, состоящей из одного или нескольких столбцов исходного DataFrame.](https://pictures.s3.yandex.net/resources/image_1708616842.png)

Обратите внимание, что когда мы создаем срез данных (то есть выборку значений для анализа), состоящий из одного столбца, то результатом будут данные **Series**. Когда мы выбираем несколько столбцов, то результат будет представлен в виде **DataFrame**, поскольку больше одного **Series** — это **DataFrame**. Если мы хотим вывести на экран значения одного столбца в виде **DataFrame**, то его название необходимо передать в виде списка с одним элементом:

```
# выборка одного столбца из DataFrame df в виде данных DataFrame:
df[['column_А']] 
```

Такой подход может быть полезным, если определенные методы **pandas** применяются только к объектам типа **DataFrame.**

Перейдём к примерам: выделим из базы данных Яндекс Музыки только столбцы, содержащие название артиста (`artist`) и жанр музыки (`genre`). Для этого создадим список с названиями нужных столбцов и передадим его в оператор `[]` датафрейма `music_df`. В качестве результата выведем на экран первые пять строк отфильтрованного датафрейма с помощью метода `.head()`, который по умолчанию выводит именно это количество первых строк:

```
# выборка нескольких столбцов из DataFrame df и вывод первых пяти строк:
print(music_df[['artist','genre']].head()) 
```

```
                                   artist    genre
0                              Marina Rei      pop
1                            Stive Morgan  ambient
2                            Stive Morgan  ambient
3                                  Rixton      pop
4  Henry Hall & His Gleneagles Hotel Band     jazz 
```

В результате были выведены первые пять строк отфильтрованного датафрейма `music_df`, который содержит только два столбца: `artist` и `genre`.

### Сохранение результата фильтрации данных

В примерах выше проведённая фильтрация не была сохранена, а сами исходные данные не претерпели изменений. Иногда при анализе данных появляется необходимость сохранить выборку в новой переменной, чтобы использовать её для дальнейшего анализа. Например, сохраним срез данных датафрейма `music_df`, который будет содержать уникальные значения имени исполнителя `artist` и жанра `genre`, в переменную `artists`:

```
# сохраним срез данных music_df в переменную artists:
artists = music_df[['artist','genre']] 
```

Теперь результат фильтрации сохранён в переменную `artists`, и его можно использовать для дальнейшего анализа данных.

Обратите внимание, что при фильтрации мы сохраняем только необходимые нам данные. Поэтому, если сохранить результат фильтрации данных в переменной датафрейма, как мы это делаем ниже, то часть исходных данных удалится:

```
music_df = music_df[['artist','genre']] 
```

В примере выше при сохранении результата фильтрации данных в переменной `music_df` от исходного датафрейма останется только два столбца: `artist` и `genre`. Остальные данные будут удалены. Поэтому результаты фильтрации данных лучше сохранять в отдельную переменную, чтобы не потерять или не изменить исходные данные.

### Удаление столбцов. Метод `.drop()`

Бывают такие ситуации, когда легче отбросить один или несколько столбцов, чем перечислять все столбцы, которые стоит оставить. Например, когда количество столбцов большое, а лишних данных — один или два столбца.

В этом случае можно использовать специальный метод **pandas**, который позволяет удалить лишние столбцы — `.drop()`.

Для удаления столбцов следует использовать параметр `columns` метода `.drop()`, в котором нужно передать список со столбцами для удаления. Например, удалим из датафрейма `music_df` столбцы `user_id` и `total_play` и выведем первые пять строк на экран:

```
# удалим из датафрейма music_df столбцы user_id и total_play:
print(music_df.drop(columns=['user_id','total_play']).head()) 
```

```
                                   artist    genre                   track
0                              Marina Rei      pop                  Musica
1                            Stive Morgan  ambient             Love Planet
2                            Stive Morgan  ambient             Love Planet
3                                  Rixton      pop  Me And My Broken Heart
4  Henry Hall & His Gleneagles Hotel Band     jazz                    Home 
```

Тут мы один за другим применили к нашим данным два метода, при этом последующий метод применяется к результатам предыдущего. Такой вызов методов называется цепочным.

Заметьте, что в коде выше мы не сохранили результат фильтрации. Поэтому, когда есть необходимость сохранить исходные данные, результат фильтрации следует сохранить в отдельной переменной, например:

```
# удалим из датафрейма music_df столбцы user_id и total_play:
filtered_df = music_df.drop(columns=['user_id','total_play']) 
```

Если необходимо сохранить результат фильтрации в исходном датафрейме, то метод `.drop()` позволяет это сделать, если указать параметр `inplace=True`:

```
# удалим из датафрейма music_df столбцы user_id и total_play:
music_df.drop(columns=['user_id','total_play'], inplace=True)
print(music_df.head()) 
```

```
                                   artist    genre                   track
0                              Marina Rei      pop                  Musica
1                            Stive Morgan  ambient             Love Planet
2                            Stive Morgan  ambient             Love Planet
3                                  Rixton      pop  Me And My Broken Heart
4  Henry Hall & His Gleneagles Hotel Band     jazz                    Home 
```

Однако при использовании параметра `inplace=True` нельзя использовать цепочку методов: в этом случае метод `.drop()` не будет возвращать никакого результата, и дальнейший цепочный вызов `.head()` приведёт к ошибке:

```
# удалим из датафрейма music_df столбцы user_id и total_play:
print(music_df.drop(columns=['user_id','total_play'], inplace=True).head()) 
```

```
AttributeError: 'NoneType' object has no attribute 'head' 
```
Бывают случаи, когда при решении аналитических задач полезно вынести список с названиями столбцов для фильтрации в отдельную переменную и использовать ее в других частях кода. Например, сохраним данные двух столбцов в отдельном датафрейме и удалим их из исходного:

```
# создадим список column_list с метками столбцов:
column_list = ['artist','genre','track'] 

# создадим новый срез данных, содержащий столбцы из column_list:
track_info = music_df[column_list]

# удалим столбцы column_list из исходного датафрейма:
music_df.drop(columns=column_list, inplace=True) 
```

В примере выше мы использовали переменную `column_list`, в которой задали список с метками необходимых столбцов и передали его оператору `[]` датафрейма `music_df` для создания нового среза данных `track_info`. Затем, используя этот же список, удалили эти данные из исходного датафрейма с помощью метода .`drop()` и параметра `inplace=True`.

Такой подход упрощает чтение кода и делает его более универсальным: если необходимо изменить названия столбцов для фильтрации, то для этого потребуется изменить только одну строку кода.

В этом уроке мы рассмотрели один из способов фильтрации данных по столбцам датафрейма с использованием оператора `[]`. При работе с данными в **pandas** существуют другие подходы к фильтрации данных — при помощи операторов `.loc[]` и `.iloc[]`, о которых мы поговорим более детально в следующих уроках. А сейчас перейдём к практике.

## Фильтрация по строкам при помощи индексации
В предыдущих уроках вы узнали, как провести фильтрацию датафрейма по отдельным столбцам, научились отображать данные одного или нескольких столбцов и сохранять выборку данных в отдельную переменную.

Однако часто аналитики используют фильтрацию не только по столбцам, но и по строкам с использованием различных логических условий. Это помогает выбрать необходимые данные для анализа. Например, использовать данные только для определённой группы товара или сегмента пользователей, или получить информацию о покупках, совершённых в определённый день, или найти контент видеохостинга с высоким количеством просмотров.

Для этого в **pandas** используется индексация данных **DataFrame** по индексу строки и столбца. В этом уроке мы рассмотрим:

- что такое индексация данных;
- какие операторы индексации существуют;
- как отобразить выбранные строки датафрейма, задав их индексы вручную.

### Индексация

Любая таблица, будь это таблица в ежедневнике или объёмный датафрейм, например, лог событий пользователей онлайн-кинотеатра, обычно состоит из строк и столбцов. Для того чтобы обратиться к информации в определённой ячейке или группе ячеек, необходимо знать номер строк и название столбцов, в которых они расположены — то есть их координаты в таблице.

Такой способ запрашивать данные называют **индексацией**.

Для индексации в **pandas** существует несколько операторов. Наиболее распространёнными из них являются:

- оператор `[]` — позволяет проводить простую индексацию по целочисленному значению индекса строк или по меткам столбцов;
- оператор `.loc[]` — позволяет проводить индексацию по меткам столбцов и индексов;
- оператор `.iloc[]` — позволяет проводить индексацию по целочисленному индексу столбцов и строк.

В предыдущих уроках вы уже познакомились с оператором индексации `[]`, когда выводили данные одного или нескольких столбцов. Это один из распространённых операторов, используемых для фильтрации данных, который позволяет обращаться к столбцу датафрейма по его названию и к целочисленному значению индекса строки (то есть позиции строки от начала).

Далее мы познакомимся с особенностями использования оператора `[]` для индексации данных, а операторы индексации `.loc[]` и `.iloc[]` рассмотрим в следующем уроке.

### Фильтрация по строкам: оператор DataFrame `[]`

Если в квадратных скобках указать не названия столбцов, как мы делали в предыдущих уроках, а интервал значений целочисленного индекса, то будут выведены только определённые интервалы строк. В этом случае синтаксис оператора будет следующий:

```
# синтаксис оператора [] при выборе среза строк DataFrame:
df[начальное_значение_интервала:конечное_значение_интервала] 
```

Срезы строк в таблицах похожи на срезы в списках: в интервале вы указываете начало и конец среза и получаете все значения между ними. При этом первое значение войдёт в срез данных, а последнее — нет.

Например, выведем первые три строки датафрейма `music_df`:

```
# выведем первые три строки датафрейма music_df:
print(music_df[0:3]) 
```

```
      user_id  total_play        artist    genre        track
0  BF6EA5AF   92.851388    Marina Rei      pop       Musica
1  FB1E568E  282.981000  Stive Morgan  ambient  Love Planet
2  FB1E568E  282.981000  Stive Morgan  ambient  Love Planet 
```

Выше на экран были выведены строки с индексами от 0, 1 и 2. При этом последнее значение интервала — 3 — не учитывается при индексации. Обратите внимание, что строки с индексами 1 и 2 полностью дублируют другу друга по значению. Такие строки называют **дубликатами**, и это может быть ошибкой в данных. В конце урока мы разберём, как можно обработать дублирующиеся значения и отфильтровать полные дубликаты данных, оставив только первое уникальное значение.

Если необходимо вывести определённые строки, начиная с первой, то первое значение в интервале можно опустить. Например, индексацию `music_df[0:3]` можно переписать как:

```
# выведем первые три строки датафрейма music_df:
print(music_df[:3]) 
```

```
# Выведем последние пять строк датафрейма music_df:
print(music_df[-5:]) 
```

print(music_df[11:])

Аналогичным образом можно вывести любой другой произвольный интервал строк. Например, выведем на экран строки с индексом от 7 до 11 включительно:

print(music_df[7:12])

Поэтому для того, чтобы вывести одну строку, необходим такой срез, который будет включать только её. Например, выведем только строку с индексом `9`:

В примерах выше метка индекса строки совпадает с её порядковым номером, но бывают случаи, когда метки индексов могут располагаться не по порядку или вообще быть представлены не числами, а датой или текстом. В этом случае оператор `[]` всё равно будет обращаться к строкам, исходя из целочисленного значения индекса.

# проведём сортировку данных по значению столбца user_id и выведем первые 5 строк:

print(sorted_df.head())

Для сортировки значений мы использовали метод `.sort_values()` с параметром `by='user_id'`, передающим метки столбцов, по которым проводится сортировка данных. По умолчанию метод проводит сортировку от меньшего значения к большему, поэтому, если есть необходимость провести сортировку в обратном порядке, используется параметр `ascending=False`.

print(sorted_df[:5])

### Индексация по строкам и столбцам

Примеры этого и предыдущего урока показывают, что оператор `[]` можно использовать как для фильтрации отдельных столбцов, когда передаётся метка столбца или их список (индекс), так и для фильтрации данных по строкам, когда передаётся срез целочисленных значений индекса строк, разделённых символом `:`.

При фильтрации данных с использованием оператора `[]` можно совмещать индексацию по строкам и столбцам. Это позволит обратиться к определённому диапазону ячеек. Например, выведем на экран значения столбца `genre` в интервале строк с целочисленными индексами 9 и 10:

```
# выведем значения столбца genre в интервале строк с индексами 9 и 10:
print(music_df[9:11]['genre']) 
```

```
9     hiphop
10       pop 
```

Для этого используется несколько операторов `[]` подряд. При этом последовательность операторов `[]` может быть любой, хотя обычно в начале указывают срез по строкам, а затем срез по столбцам. Например, выведем значения нескольких столбцов:

```
print(music_df[9:11][['artist','genre']]) 
```

```

```

Таким образом, оператор `[]` помогает аналитику проводить фильтрацию данных по строкам и столбцам, оставляя только необходимые для анализа данные. При этом достаточно передать один аргумент, и `[]` поймёт, что делать:

- если передать срез целочисленных значений индексов, произойдёт фильтрация по строкам;
- если передать метку столбца или их список, произойдёт фильтрация по столбцам.

Чтобы систематизировать результаты урока, соберём возможные варианты использования оператора `[]` для индексации датафреймов в **pandas** в одну таблицу, которую мы будем дополнять в следующих уроках:

|**Выборка (срез) данных**|**Оператор `[]`**|
|---|---|
|Один столбец (Series)|`df['column']`|
|Один столбец (DataFrame)|`df[['column']]`|
|Несколько столбцов|`df[['column_A','column_B']]`|
|Все строки, начиная с заданной|`df[0:]`|
|Все строки до заданной|`df[:5]` (не включая индекс 5)|
|Срез строк|`df[2:5]` (не включая индекс 5)|
|Одна строка|`df[4:5]`|
|Одна ячейка|`df[4:5]['column']`|

Однако не всегда фильтрация данных ограничивается конкретным интервалом строк или столбцов. Часто возникает ситуация, когда необходимо провести фильтрацию в зависимости от содержимого датафрейма. Для этого используется логическая индексация, с которой вы познакомитесь в следующем уроке.

## Индексация и условные выражения
Способ фильтрации данных по срезам строк, который вы разобрали в предыдущем уроке, удобен, когда мы точно знаем количество строк, которые нужно вывести в отдельную выборку и их положение. Обычно такой способ используется при сортировке значений в одном из столбцов. Например, для информации о первых десяти покупателях, или о пяти самых дорогих заказах.

Однако не всегда при анализе данных аналитик точно знает, какие диапазоны строк необходимо использовать в работе. В этом случае строки выбираются в соответствии с определёнными условиями. Например, если нужно проанализировать продажи товаров в категории «Компьютеры», то необходимо оставить только те строки, в которых категория товаров соответствующая. Или провести анализ среднего количества покупок, совершённых пользователями в мобильном приложении. В этом случае необходима фильтрация данных по названию используемого устройства.

Для этого на помощь приходят логические выражения. Они позволяют провести фильтрацию строк по заданным условиям. В этом уроке вы:

- узнаете, как использовать условные выражения для фильтрации данных;
- изучите, что такое логическая маска;
- выясните, как при помощи логической индексации найти дубликаты в данных.

### Логическая или булевая маска

Обратимся к данным Яндекс Музыки и создадим срез данных с информацией о треках в стиле `metal`. Представьте, что вы музыкальный продюсер предстоящей тематической вечеринки и составляете список музыкальных композиций для вечера, вычёркивая из таблицы неподходящие по стилю записи:

![логическая индексация позволяет отобрать данные, которые соответствуют определенным условиям - например, составить плейлист композиций, написанных в стиле Metal](https://pictures.s3.yandex.net/resources/image_1711880971.png)

Для этого в операторе `[]` пропишем условие, которое будет проверять содержимое столбца `'genre'` и сравнивать с искомым значением — `metal`. Выведем на экран первые пять строк нового среза данных:

```
# Используем логическую маску для фильтрации данных по значению столбца genre:
print(music_df[music_df['genre'] == 'metal'].head())

Для создания логических масок можно использовать логические выражения с любыми логическими операторами: `==`, `!=`, `>`, `<`, `>=`, `<=`, а также содержащие функции-предикаты.

Таким образом, так же как карнавальная маска скрывает часть лица, так и логическая маска — скрыла часть данных, оставив только необходимые для анализа данных значения.

![логическая маска позволяет скрыть часть данных, оставив только необходимые для анализа значения.](https://pictures.s3.yandex.net/resources/image_1711881000.png)

Используя логические маски, можно применять не только логические операторы сравнения, но и логические операторы отрицания. Например, отберем все строки, названия жанров которых не равны значению `pop`:

```
# Использование логических операторов отрицания в логической маске:
print(music_df[music_df['genre'] != 'pop']) 
```

Обращаться к операторам отрицания можно не только в самой логической маске, но и для всей логической маски в целом. В примере ниже битовый оператор отрицания `~` используется для всей логической маски, при этом в логической маске нет операторов неравенства:

```
# Использование битового оператора отрицания в логической маске:
print(music_df[~(music_df['genre'] == 'pop')]) 
```

Результаты двух инструкций выше будут одинаковыми.

В качестве логической маски можно использовать сложные логические выражения, включающие несколько условий. Например, оставим строки исходного датафрейма, в которых жанр имеет значения `metal` или `rock`:

```
# Использование сложного логического выражения в качестве логической маски:
music_df[(music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')] 
```

В примере выше каждое логическое выражение заключено в круглые скобки: `(music_df['genre'] == 'metal')` и `(music_df['genre'] == 'rock')`, а соединяются они битовыми операторами сравнения, в примере выше — оператором `|` — логическое `ИЛИ`.

Вспомните правила умножения в математике — в начале считаем то, что в скобках. Так и здесь — важно использовать скобки для выделения каждого логического условия. Если убрать их, то общее сложное логическое выражение будет таким:

```
music_df['genre'] == 'metal' | music_df['genre'] == 'rock' 
```

По приоритету операторов первым будет отработан битовый оператор `|`. Логическое выражение в этом случае будет следующим:

```
'metal' | music_df['genre'] 
```

Поскольку битовый оператор ожидает, что сравниваться будут два целых числа, или значения `True` и `False`, а в примере выше сравниваются строка и объект `Series`, то в результате выполнения такого кода мы получим ошибку. Чтобы избежать этого, следует убедиться, что в логическом выражении с битовым оператором используются логические значения `True` или `False`.

Если для фильтрации используется сложное логическое выражение, или фильтрацию предполагается использовать несколько раз в разных частях кода, то лучше инициализировать логическую маску в отдельной переменной, например, с названием `mask`:

```
# Сохраним сложное логическое выражение в переменной mask:
mask = (music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')

# Проведём логическую индексацию данных с помощью логической маски mask:
filtered_df = music_df[mask] 
```

Такой подход делает код более читаемым, воспроизводимым и упростит работу, если понадобится изменить условия логической маски.

Также использование логической маски можно комбинировать с другими операторами `[]` для сложной фильтрации данных. Например, выведем первые семь строк данных столбцов `'artist'`, `'genre'` и `'track'`, в которых `'genre'` имеет значения `metal` или `rock`:

```
# Сохраним сложное логическое выражение в переменной mask:
mask = (music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')

# Выведем на экран первые 7 строк данных в столбцах artist, genre и track:
print(music_df[mask][:7][['artist','genre','track']]) 
```

При использовании сложных логических последовательностей не всегда удобно записывать их в одной переменной или логической маске — обычно это делает код длинным и сложным для чтения. Поэтому, если необходимо использовать несколько последовательных условий фильтрации данных, то оптимально в начале применить первое условие и сохранить результат в отдельной переменной. А затем провести логическую индексацию. Например, выберем строки в которых жанр — джаз и время воспроизведения находится в диапазоне от 80 до 130:

```
# Первым действием создадим новый срез данных по первому условию:
filtered_df = music_df[music_df['total_play'] >= 80]
filtered_df = filtered_df[filtered_df['total_play'] <= 130]
filtered_df = filtered_df[filtered_df['genre'] == 'jazz'] 
```

Обратите внимание, что для сохранения результата фильтрации мы задали новую переменную `filtered_df`. При этом первая фильтрация данных — по значению `['total_play'] >= 80` проведена по данным исходного датафрейма `music_df`, а последующие фильтрации — с использованием среза данных `filtered_df`.

## Фильтрация при помощи loc и iloc
В предыдущем уроке вы узнали, как отфильтровать данные по строкам и столбцам при помощи оператора `[]`, а также создать срезы данных, используя логические маски (логическую индексацию).

Однако используя оператор `[]`, провести фильтрацию можно или только по целочисленному значению индекса строк, или только по названию столбцов. Если необходимо проводить фильтрацию как по строкам, так и по столбцам, то придётся использовать несколько операторов `[]` подряд, например:

```
# Выведем на экран первые три строки столбцов 'artist' 'track':
print(music_df[:3][['artist','track']]) 
```

Из-за этого оператор `[]` обычно используется для простой фильтрации или индексации данных: когда нужно выбрать один или несколько столбцов, или только срез строк. Такую опцию применяют, если нужно сделать выборку по названию товара и его стоимости, или вывести десять первых покупателей.

Для более сложных алгоритмов, когда необходимо провести фильтрацию по строкам и столбцам, применяя сложные условия в логической маске, часто используют операторы `.loc[]` и `.iloc[]`. Они позволяют индексировать датафрейм по строкам и столбцам в одно действие. В этом уроке мы рассмотрим:

- особенности работы операторов `.loc[]` и `.iloc[]` и их различия;
- основные подходы к индексации данных по строкам и столбцам;
- сходства и различия операторов `.loc[]` и `.iloc[]` с оператором `[]`.

### Фильтрация строк с помощью операторов `.loc[]` и `.iloc[]`

Синтаксис операторов `.loc[]` и `.iloc[]` схожий: в начале задаётся индексация по строкам, затем, через запятую, индексация по столбцам:

```
df.loc[строки, столбцы]
df.iloc[индексы строк, индексы столбцов] 
```

Основное различие операторов `.loc[]` и `.iloc[]` в том, что:

- `.loc[]` выбирает строки и столбцы на основе их метки (названия), списка или интервала меток, а также с использованием логической индексации.
- `.iloc[]` выбирает строки и столбцы на основе их целочисленного индекса (порядкового номера), списка и интервала этих значений.

Приступим к выбору строк с использованием `.loc[]`. Для этого нужно передать разделённый двоеточием диапазон необходимых значений индексов строк — `[1:3]`. При этом в отличие от оператора `[]`, при использовании диапазона индексов будут выведены все строки включая последнее значение:

```
# Вывод среза строк с метками индексов 1, 2, 3 с помощью оператора .loc[]:
print(music_df.loc[1:3]) 
```

```

1  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
2  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
3  82F52E69  193.776327          Rixton      pop  Me And My Broken Heart   
```

В результате необходимые строки были выведены на экран.

На этом же примере разберём основную особенность оператора `.iloc[]`. В логике этого оператора лежит обращение к целочисленному индексу строк или столбцов, то есть к их порядковому номеру. Его использование будет аналогично использованию оператора `[]` — последнее значение в интервале строк не будет включено в итоговый срез. Поэтому для вывода необходимых строк в оператор `.iloc[]` следует передать такой интервал индексов — `[1:4]`:

```
# Вывод среза строк с индексами 1, 2, 3 с помощью оператора .iloc[]:
print(music_df.iloc[1:4]) 
```

```

2  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
3  82F52E69  193.776327          Rixton      pop  Me And My Broken Heart

Для вывода среза первых или последних строк используется аналогичная логика, как в случае с оператором `[]`: если нужно вывести все строки, начиная с первой до заданной строки, то первое значение среза можно оставить пустым. Если необходимо вывести все значения после определённой строки — то последнее значение среза можно оставить пустым.
В примере выше метки индекса строк совпадали с целочисленным значением номера строки. Но так бывает не всегда. Если данные уже подвергались фильтрации или сортировке, то последовательность меток индексов может нарушиться.

## Фильтрация столбцов с помощью операторов loc и iloc
В этом уроке вы узнаете, как с помощью операторов `.loc[]` и `.iloc[]` проводить фильтрацию столбцов, указав метки или их диапазон. Для этого нужно в начале передать вывод всех строк — для этого достаточно указать двоеточие `:`, а затем передать список столбцов, которые необходимо вывести.

Например, выведем на экран столбцы `'artist'` и `'genre'`. При использовании `.loc[]` необходимо передать список меток этих столбцов:

# Фильтрация столбцов с помощью оператора .loc[]:
print(music_df.loc[:, ['artist','genre']].head())

```
# Фильтрация столбцов с помощью оператора .iloc[]:
print(music_df.iloc[:, [2,3]].head())

### Индексация ячейки данных

Операторы `.loc[]` и `.iloc[]` позволяют провести индексацию ячейки или интервала ячеек с использованием одного выражения. Например, сделаем срез содержащий название трека исполнителя `Rixton` с индексом строки `3`. При использовании `.loc[]` в начале передадим метку индекса строки `3`, а затем название колонки `'track'`:

```
# Индексация ячейки с помощью оператора .loc[]:
print(music_df.loc[3, 'track']) 
```

```
Me And My Broken Heart 
```

Результат получен! В отличие от простой индексации, когда нам пришлось бы использовать два оператора `[]` — `music_df[3:4]['track']`, все инструкции уместились в одном операторе `.loc[]`.

При использовании оператора `.iloc[]` нужно выполнить аналогичные действия, только вместо меток передать целочисленные индексы строки и столбца.

|---|---|---|---|
|Один столбец|`df['column']`|`.loc[:, 'column']`|`.iloc[:, index]`|
|Несколько столбцов|`df[['column_A','column_B']]`|`.loc[:, ['column_A','column_B']]`|`.iloc[:, [index_1, index_2]]`|
|Срез столбцов|-|`.loc[:, 'column_A':'column_B']`|`.iloc[:, index_1: index_2]`|
|Одна строка|`df[4:5]`|`.loc[4]`|`.iloc[4]`|
|Все строки, начиная с заданной|`df[0:]`|`.loc[0:]`|`.iloc[0:]`|
|Все строки до заданной|`df[:5]` (не включая 5)|`.loc[:5]` (включая 5)|`.iloc[:5]` (не включая 5)|

Операторы `.loc[]` и `.iloc[]` предоставляют гибкие возможности для выбора и фильтрации строк и столбцов в датафреймах в зависимости от меток или целочисленных индексов. Такой подход упрощает работу с данными, особенно при реализации сложных алгоритмов фильтрации по столбцам и строкам.

## Фильтрация при помощи loc: логические выражения
В прошлых уроках вы рассмотрели, как проводить фильтрацию данных по координатам ячеек с помощью операторов `[]`, `.loc[]` и `.iloc[]`, а также познакомились с логической индексацией — методом фильтрации с использованием логических условий. Логическая индексация позволяет выбирать подмножество данных, соответствующих заданным критериям. Ключевой инструмент логической индексации — логические маски.

Как правило, для создания простых срезов (среза строк или выбора отдельных столбцов) или условий в логических масках используется оператор `[]`, а при более комплексных фильтрациях данных, когда необходимо выделить строки и столбцы согласно различным условиям, используется оператор `.loc[]`. Он позволяет в одном действии обрабатывать метки строк и столбцов, использовать логические маски. Также оператор `.loc[]` позволяет выполнить все действия, связанные с фильтрацией данных, и широко используется при обработке данных или их замене, что мы рассмотрим в следующих уроках.

Оператор `.iloc[]` используется реже и полезен, когда нужно вывести точно определённое количество строк и столбцов вне зависимости от их названия.

В этом уроке мы разберём, как использовать логическую индексацию в `.loc[]`, а также сравним `[]`, `.loc[]` и `.iloc[]` между собой.

### Логическая индексация в `.loc[]`

Логическую индексацию можно использовать только с оператором `.loc[]`. В качестве логической маски можно использовать логические выражения с простым оператором индексации `[]` или с оператором `.loc[]`. Например, напишем условие, которое выведет все записи автора `Aura`:

```
# Создадим логическую маску, фильтрующую данные по значению столбца artist:
mask = music_df['artist'] == 'Aura' 
```

Переменная `mask` будет содержать объект `pandas.Series` с булевыми значениями. В нашей маске `True` будет указано для тех строк, в которых значение столбца `artist` равно `'Aura'`, и `False` в противном случае.

Используя логическую маску, выведем данные в столбцах `'artist'`, `'genre'` и `'track'`:

```
# Проведём фильтрацию данных с использованием логической маски:
print(music_df.loc[mask, 'artist':'track']) 
```

```
      artist       genre                track
5254    Aura       latin         Amor Gigante
7953    Aura         pop    Everything's Cool
35211   Aura        rock                Shine
38877   Aura  electronic  Destination Skyline
40490   Aura       dance        Time to Shine 
```

В примере выше логическая маска сформирована при помощи оператора `[]`, но в ней можно также использовать оператор `.loc[]` и сочетать несколько логических выражений. Например, выведем все композиции автора `Aura`, написанные в стиле `dance`:

```
# Создадим логическую маску, фильтрующую данные согласно нескольким условиям:
mask = (music_df.loc[:,'artist'] == 'Aura') & (music_df.loc[:,'genre'] == 'dance')
print(music_df.loc[mask, 'artist':'track']) 
```

```
      artist       genre                track
40490   Aura       dance        Time to Shine 
```

В примере выше мы использовали сочетание двух логических выражений:

1. Первое фильтрует данные по имени автора композиции — `(music_df.loc[:,'artist'] == 'Aura').`
2. Второе — по названию жанра: `(music_df.loc[:,'genre'] == 'dance')`.

Два условия соединены между собой битовым оператором `&`, который подразумевает логическое `И`, то есть сочетание истинного результата двух условий.

### Сравнение различных операторов индексации

Ранее в примерах мы рассмотрели основные подходы к фильтрации данных с использованием различных операторов индексации — `[]`, `.loc[]` и `.iloc[]`. В таблице ниже — основной синтаксис различных операторов:

|**Выборка (срез) данных**|**оператор `[]`**|**оператор `.loc[]`**|**оператор `.iloc[]`**|
|---|---|---|---|
|Один столбец|`df['column'`]|`.loc[:, 'column'`]|`.iloc[:, index]`|
|Несколько столбцов|`df[['column_A','column_B']]`|`.loc[:, ['column_A','column_B']]`|`.iloc[:, [index_1, index_2]]`|
|Срез столбцов|-|`.loc[:, 'column_A':'column_B'`]|`.iloc[:, index_1: index_2]`|
|Одна строка|`df[4:5]`|`.loc[4]`|`.iloc[4]`|
|Все строки, начиная с заданной|`df[0:]`|`.loc[0:]`|`.iloc[0:]`|
|Все строки до заданной|`df[:5]` (не включая 5)|`.loc[:5]` (включая 5)|`.iloc[:5]` (не включая 5)|
|Срез строк|`df[2:5]` (не включая 5)|`.loc[2:5]` (включая 5)|`.iloc[2:5]` (не включая 5)|
|Несколько строк (не последовательный срез)|-|`.loc[[2,4,6]]`|`.iloc[[2,4,6]]`|
|Одна ячейка|`df[4:5]['column']`|`.loc[4, 'column']`|`.iloc[4, index]`|
|Логическая индексация (строки)|`df[mask]`|`.loc[mask]`|-|
|Логическая индексация (строки + фильтрация по столбцам)|`df[mask][['column_A','column_B']]`|`.loc[mask, ['column_A','column_B']]`|-|

Как видно из таблицы выше, круг возможных действий с применением `[]`, `.loc[]` и `.iloc[]` широк и покрывает большинство необходимых аналитику задач по фильтрации данных.

## Фильтрация при помощи query
В прошлых уроках вы рассматривали логическую индексацию с помощью операторов `[]` и `.loc[]`. В pandas есть ещё один способ фильтрации данных с помощью условий — метод `.query()` — от английского query — «запрос».

Метод `.query()` позволяет проводить фильтрацию данных по заданным условиям в виде одного запроса. Более простой синтаксис написания и совмещения логических условий, чем у операторов `[]`, `.iloc[]` и `loc[]`, упрощает написание и чтение кода. Благодаря повышенной производительности метод подходит для работы с объёмными данными, состоящими из более десятка тысяч строк.

В этом уроке разберём особенности использования запросов при фильтрации данных, а именно:

- как задать условие в `.query()`;
- правила комбинирования нескольких условий в одном запросе;
- особенности фильтрации данных с методом `.query()`;
- а также сравним метод `.query()` и оператор `.loc[]`

### Метод `.query()` — задаём условие

Синтаксис метода `.query()` выглядит так: `.query('запрос')`. Запрос может содержать одно или несколько логических условий, по которым проводится фильтрация данных. При написании запросов используются только метки столбцов или индексов строк. Например, создадим выборку данных Яндекс Музыки, содержащую строки с жанром `dance`:

```
# Проведём фильтрацию данных с помощью метода query:
dance_df = music_df.query('genre == "dance"')

print(dance_df.head()) 
```

```

Запрос `genre == "dance"` аналогичен по смыслу и результату условию `music_df['genre'] == 'dance'`. Только он выглядит компактнее, поскольку метод `.query()` уже применяется к определённому датафрейму и при указании названия меток нет необходимости указывать название этого датафрейма.

Сравните написание двух инструкций, использующих оператор `[]` или `.loc[]` и метод `.query()`:

```
# Логическая индексация с помощью []:
dance_df = music_df[music_df['genre'] == 'dance']

dance_df = music_df.loc[music_df['genre'] == 'dance']

# Запрос .query():
dance_df = music_df.query('genre == "dance"') 
```

Вместо `music_df['genre']` достаточно указать только название столбца — `genre`.

Обратите внимание, что запрос `query()` заключен в кавычки: `'genre == "dance"'`, обычно это одиночные кавычки `''`. Если в запросе используются подстроки или любые другие элементы, которые обычно также заключаются в кавычки, то для них используют двойные кавычки `""` — в примере выше это `"dance"`. Иными словами, если запрос `query()` помещён в одиночные кавычки, то все другие кавычки, которые используются в запросе должны быть двойными.

### Комбинирование нескольких условий в одном запросе

Метод `.query()` позволяет использовать запросы, содержащие два и более условных выражения. Например, отфильтруем данные жанра `pop` и с значением `total_play` более `200`:

```

Использование меток столбцов упрощает работу с запросами в `.query()`, однако, не все названия можно использовать в запросах. Если название столбца содержит:

- пробелы ``;
- операторы сравнения: `<`, `<=`, `>`, `>=`, `!=`, `==`;
- математические операторы: `+`, `-`, `/`, `*`;
- символы: `.`, `,`, `$`, `&`, `!`, `^`;
- скобки: `()` и `[]`;
- ключевые слова Python;

```
# Примеры запросов, где названия столбцов указаны в обратных кавычках:
df.query('`artist-name` == "Finntroll"')

### Сравнение метода `.query()` и оператора `.loc[]`

Методы `.query()` и `.loc` позволяют проводить основные действия по фильтрации данных. Однако `.query()` предоставляет удобный способ написания условий фильтрации в одну строку, что упрощает код, особенно при сложных и длинных условиях.

Когда количество строк в данных превышает несколько десятков тысяч, фильтрация методом `.query()` может оптимизировать процесс обработки, поскольку проводится быстрее, чем логическая индексация с помощью оператора `.loc[]`.

Однако метод `.query()` в большей степени ориентирован на логические условия при использовании значений столбцов. При необходимости проводить индексацию строк по их индексу предпочтительнее использовать операторы индексации `[]`, `.loc[]` или `.iloc[]`. Они позволяют более точно указать диапазон меток индексов строк или использовать их целочисленное значение.

Также `.query()` позволяет проводить фильтрацию данных и создавать новые выборки данных, а индексация при помощи метода `.loc[]` может использоваться для изменения данных в индексируемых ячейках или группах ячеек.

С заменой данных мы разберёмся чуть позже, а в следующем уроке потренируемся в использовании метода `query()`.

# Изменение данных в соответствии с условиями задачи

## Когда нужна замена данных
В повседневной жизни нам часто нужно внести коррективы в документы или другие данные, дополнить или исправить в них ошибки.

Например, при заключении брака один из супругов может изменить фамилию, что потребует изменения многих документов. Также данные заменяют из-за ошибок в них, например, при неверном написании фамилии, или недостоверных данных. Замена данных бывает и в более глобальном масштабе, связанным с общими изменениями некоторых правил. Например, в 2005 году изменились цифры в телефонных кодах регионов России — тогда «0» заменили на «4».

Как и в повседневной жизни, при анализе данных некоторые значения нужно заменять. В этом уроке мы рассмотрим ситуации, когда требуется изменение данных при их анализе, а в следующих уроках разберем, как это можно сделать в pandas.

### Замена данных при подготовке к анализу

Получаемые аналитиком данные не всегда качественные. Они могут содержать пропущенные значения, дубликаты, опечатки, некорректные или невозможные значения. Человеческий фактор, ошибки системы или процесса выгрузки могут «испортить» данные и сделать их непригодными для анализа. Предварительно изучая и готовя данные к анализу, аналитик ищет возможные ошибки в них и по возможности вносит необходимые изменения. Потребность заменить значения в данных — не единственное, с чем может столкнуться аналитик.

Данные могут содержать неполные сведения — пропущенные значения (пропуски). Такое происходит в случае отсутствия информации или из-за ошибки в сборе данных. Иногда пропущенные значения можно восстановить находя дополнительную информацию или проводя аналогии с полными данными.

Например, если при анализе перелётов для некоторых аэропортов отсутствуют международные коды, то их можно восстановить из открытых источников. Или если в базе объявлений о купле-продажи недвижимости для определённого дома отсутствует год постройки, то его можно восстановить из объявлений о продаже недвижимости в этом же доме, где есть нужная информация. Также бывают случаи, когда все пропущенные значения, если их невозможно восстановить, заменяются на какое-то конкретное, заведомо известное значение. Но об этом вы узнаете позже.

Нереалистичные или некорректные значения — также распространённая ошибка в данных. Например, отрицательные значения в возрасте клиента, или стоимости товара, опечатки в данных. В таком случае достаточно заменить некорректные значения на верные.

На стадии подготовки данных к анализу заменять можно не только сами значения в данных, но и названия столбцов или изменять тип данных, если он не соответствует содержимому. Например, дата или целочисленные значения могут быть сохранены в виде строк. Довольно часто у столбцов в исходных таблицах — неудобные для работы метки, например, пробелы. Тогда следует привести их к единому стилю или сократить. Если сбор данных не отлажен, то их подготовка может отнимать большую часть времени аналитика, это часто встречающаяся задача, требующая разнообразных подходов.

### Замена данных при их анализе

Замена данных может потребоваться не только на стадии их подготовки к анализу, но и при его проведении. Например, изменение формата значений — перевод температуры из одной системы измерения в другую, или перевод метров в километры, или изменение категорий товаров или сегментов пользователей.

Иногда данные заменяют, чтобы «перевести» категориальные данные, например, перевод результативности студентов из формата «отлично», «хорошо», «удовлетворительно», «неудовлетворительно» в целочисленные значения для расчёта среднего балла.
Замена значений в данных может быть локальной. Тогда значения меняются в определённой ячейке или в группе ячеек, которые соответствуют заданным условиям, например, изменение телефонного кода определённого города. Также замену можно проводить по всем данным, например, исправить опечатки в названии товара, или по отдельным столбцам, например, заменить названия категорий пользователей.

Для каждого случая существуют свои алгоритмы замены данных, которые мы разберём в следующих уроках.

## Действия над названиями столбцов
При работе с данными в pandas аналитик часто использует названия или, как их ещё называют, метки столбцов. Это удобно при фильтрации данных, сортировке и группировке значений, а также при подсчёте различных метрик и других действий. Метки столбцов следует оформлять в одной нотации и на одном языке. Названия столбцов не должны дублировать названия встроенных функций Python — в противном случае при работе с некоторыми методами могут возникнуть сложности, например, при составлении запросов для фильтрации данных в методе `.query()`.

В этом уроке вы узнаете, какая основная нотация используется для присваивания названий (меток) столбцам, а также освоите основные подходы к изменению названий столбцов:

- познакомитесь с атрибутом датафрейма `columns`, в котором хранятся названия столбцов;
- изучите переименование столбцов с помощью метода `.rename()`;
- освоите изменение регистра названий столбцов на нижний и замену пробела на символ нижнего подчеркивания, методы `lower()` и `replace()` — как в цикле, так и с помощью атрибута `str`;
- увидите плюсы и минусы разных методов и разберётесь на примерах в алгоритмах изменения названий столбцов датафрейма.

### Стиль названия столбцов и переменных

В программировании в зависимости от языка, используются различные стили названия переменных, функций и других элементов. Наиболее распространены следующие нотации:

|**Нотация**|**Пример**|**Описание**|
|---|---|---|
|Snake Case (змеиный_регистр)|`column_name`|Все слова пишутся в нижнем регистре и разделяются не пробелом, а символом подчёркивания _|
|Camel Case (верблюжийРегистр):|`columnName`|Каждое слово, кроме первого, начинается с заглавной буквы, слова не разделяются символами и пишутся слитно|
|Pascal Case (ПаскальРегистр)|`ColumnName`|Каждое слово, включая первое, начинается с заглавной буквы, слова не разделяются символами и пишутся слитно|
|Upper Case (ВЕРХНИЙ_РЕГИСТР)|`COLUMN_NAME`|Все буквы — заглавные, а слова разделяются символом подчёркивания|
|Kebab Case (кебаб-регистр)|`column-name`|Все слова пишутся в нижнем регистре и разделяются не пробелом, а дефисом -|

Обычно, для названия переменных и меток столбцов при работе с датафреймами в pandas используют Snake Case. Этому же стилю следуют стандартные и распространённые библиотеки Python. Это легко запомнить, так как питон — это змея. Но в специальных случаях при написании кода в Python используются и другие стили — Pascal Case для названия классов (например, `CarModel`) и Upper Case для названия констант (например, `TIME`).
Далее разберём, как привести любое название столбца к Snake Case.

### Атрибут датафрейма `columns`

У датафреймов в pandas есть специальные атрибуты, которые представляют собой список названий всех столбцов или строк в виде объекта Index. Например, это атрибут `columns` — от английского «столбцы». Обратимся к атрибуту `columns` датафрейма `music_df`:

```
# Используем атрибут columns для вывода названия столбцов music_df:
print(music_df.columns) 
```

```
Index(['user_id', 'total_play', 'artist', 'genre', 'track'], dtype='object') 
```

В результате получаем список всех столбцов объекта Index. Этот объект похож на список: используя индексацию, можно вывести определённое значение, например, название второго столбца:

```
# Выведем название второго по счёту столбца:
print(music_df.columns[1]) 
```

```
total_play 
```

Однако в отличие от обычного списка, в объекте Index нельзя изменить какое-то определённое значение, поскольку Index представляет собой неизменяемую структуру данных, аналогичную массиву или множеству. Можно изменить только весь Index сразу.

Для примера изменения Index создадим простой датафрейм с несколькими строками и столбцами:

```
# Зададим данные для датафрейма: названия столбцов и содержание строк:
columns = ['Product ID', 'Category', 'Description', 'Price']
data = [[10001, 'Телефоны','Смартфон', 15999],
       [10002, 'Компьютеры', 'Монитор', 17200],
       [10003, 'Кухонная техника', 'Микроволновка', 6350]]

# Создадим новый датафрейм df:
df = pd.DataFrame(data=data, columns=columns)
print(df) 
```

```
   Product ID          Category    Description  Price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

Приведём названия столбцов в стиль Snake Case. Для этого создадим список `new_columns`, в котором пропишем новые названия столбцов в последовательности, представленной в датафрейме. Затем зададим новое значение индекса столбцов датафрейму `df`, используя атрибут `columns`:

```
# Зададим список с новыми названиями столбцов:
new_columns = ['product_id','category','description','price']

# Используя атрибут columns, зададим новый индекс столбцов df:
df.columns = new_columns
print(df) 
```

```
   product_id          category    description  price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

В результате названия столбцов изменились. При этом список `new_columns` должен содержать такое же количество значений, как количество столбцов. Такой подход позволяет изменить все названия столбцов в одно действие. В этом случае мы должны быть уверены как в последовательности столбцов, так и в их количестве. Если вдруг изменится последовательность столбцов, например, при обновлении данных, то столбцы могут получить неверное название, что повлияет на работоспособность кода.
Изменение всех значений индекса столбцов не всегда удобно, например, при необходимости изменить названия не всех столбцов, а только некоторой их части. В этом случае придётся указывать названия всех столбцов датафрейма.

Поскольку Index это неизменяемый массив данных, то для изменения одного из его значений Index можно перевести в список. Затем изменить необходимое значение, обратившись к нему по его индексу, и используя атрибут `columns` завершить изменения.

Рассмотрим на примере и изменим название столбца `'description'` на `'info'`:

```
# Создадим список на основе индекса столбцов датафрейма df:
columns = df.columns.to_list()

# Изменим третий по счёту элемент списка columns:
columns[2] = 'info'

# Используя атрибут columns зададим новый индекс столбцов df:
df.columns = columns
print(df) 
```

```
   product_id          category           info  price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

В примере выше мы создали новый список `columns` с названиями столбцов. Чтобы перевести Index в список значений, использовали метод pandas `to_list()` — от английского «привести к списку». Обычно это метод используется для перевода значений Series в список, но он также подходит для перевода других массивов, например, Index в список.

### Метод `.rename()`

В примерах выше мы изменяли названия всех столбцов или название одного из них, обращаясь к атрибуту. В pandas существует ещё один подход к изменению названия столбцов — метод `.rename()` — от английского «переименовать».

Для изменения названия столбцов необходимо использовать аргумент `columns` метода `.rename()`, в котором передаётся словарь, содержащий в качестве ключа исходное название столбца, а в качестве значения — новое:

```
# Зададим данные для датафрейма: названия столбцов и содержание строк:
columns = ['Product ID', 'Category', 'Description', 'Price']
data = [[10001, 'Телефоны','Смартфон', 15999],
       [10002, 'Компьютеры', 'Монитор', 17200],
       [10003, 'Кухонная техника', 'Микроволновка', 6350]]

# Создадим новый датафрейм df:
df = pd.DataFrame(data=data, columns=columns)

# Используя метод rename() изменим названия столбцов df:
df = df.rename(columns={'Product ID': 'product_id',
                        'Category': 'category',
                        'Description': 'description',
                        'Price': 'price'})

print(df) 
```

```
   product_id          category    description  price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

Используя метод `.rename()`, мы можем изменить названия как всех столбцов, так и только их часть. При этом нет необходимости соблюдать последовательность столбцов, как это было в случае использования атрибута `columns`. Изменим название столбца `'description'` на `'info'`:

```
# Изменим название столбца 'description' на 'info':
# Используя атрибут inplace=True, сохраним изменения в текущем датафрейме:
df.rename(columns={'description': 'info'}, inplace=True)

print(df) 
```

```
   product_id          category           info  price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

Обратите внимание, что в методе `.rename()` мы используем атрибут `inplace=True`. Этот параметр, как и в некоторых других методах pandas, позволяет внести изменения в исходный `DataFrame` без дополнительной перезаписи переменной датафрейма.

Метод `.rename()`, в сравнении с ручным присвоением новых значений Index, используя `columns`, помогает избежать возможных ошибок, если вдруг по какой-то причине изменится последовательность столбцов. Также этот метод не требует перечисления названия столбцов, которые не используются при изменении названий, что оптимизирует код и упрощает его чтение. А если какого-то столбца, который есть в словаре, не окажется в датафрейме, то метод `.rename()` пропустит этот столбец, и при этом работа кода не вызовет ошибку.

### Перевод названия столбцов в Snake Case

В Snake Case каждое слово пишется в нижнем регистре, а вместо пробелов используется нижнее подчёркивание `_`. Если нет необходимости изменять сами названия столбцов, то для их перевода в нотацию Snake Case достаточно изменить регистр всех слов на нижний, а пробел заменить на нижнее подчёркивание.

В предыдущих уроках вы уже встречались с методом Python `.lower()`. Этот метод позволяет перевести каждый символ строки в нижний регистр:

```
# Зададим исходную строку для перевода в нижний регистр:
name = 'Column Name'

# Используем метод .lower() для перевода всех символов в нижний регистр:
lower_name = name.lower()
print(lower_name) 
```

```
column name 
```

Теперь остаётся заменить пробел на нижнее подчёркивание. Для замены одних символов на другие можно использовать метод `.replace()` — от английского «заменять». Например, заменим пробел на нижнее подчёркивание:

```
# Используем метод .replace() для замены пробела на нижнее подчёркивание:
lower_name = lower_name.replace(' ', '_')
print(lower_name) 
```

```
column_name 
```

Результат получен! Разберём синтаксис метода: в начале мы указали значение, которое необходимо заменить — строку с пробелом `' '`, а затем через запятую, значение на которое нужно провести замену — строку, содержащую нижнее подчёркивание `'_'`.

Таким образом, мы можем совместить два метода `.lower()` и `.replace()` в одну цепочку действий, что позволит изменить нотацию названия на Snake Case:

```
# Зададим исходную строку для перевода в нижний регистр:
name = 'Column Name'

# Используем метод .lower() для перевода всех символов в нижний регистр:
snake_case_name = name.lower().replace(' ', '_')
print(snake_case_name) 
```

```
column_name 
```

Такую цепочку методов можно использовать для приведения названия столбцов к нотации Snake Case. Однако эти методы работают только со строками и их нельзя применить к массивам. А названия столбцов представлены в виде Index.

Эту задачу можно решить, если применять методы `.lower()` и `.replace()` для каждого значения массива индексов столбцов. С этим отлично справится цикл в сочетании с атрибутом датафрейма `columns` и методом `.rename()`:

```
# Зададим данные для датафрейма: названия столбцов и содержание строк:
columns = ['Product ID', 'Category', 'Description', 'Price']
data = [[10001, 'Телефоны','Смартфон', 15999],
       [10002, 'Компьютеры', 'Монитор', 17200],
       [10003, 'Кухонная техника', 'Микроволновка', 6350]]

# Создадим новый датафрейм df:
df = pd.DataFrame(data=data, columns=columns)

# Создадим пустой словарь для записи переименований
rename_dict = {}

# Заполним словарь в цикле
for column in df.columns:
    rename_dict[column] = column.lower().replace(' ', '_')

# Произведём переименование
df.rename(columns=rename_dict, inplace=True)

    
print(df) 
```

```
     product_id          category    description  price
0       10001          Телефоны       Смартфон  15999
1       10002        Компьютеры        Монитор  17200
2       10003  Кухонная техника  Микроволновка   6350 
```

В примере выше был реализован следующий алгоритм:

![](https://pictures.s3.yandex.net/resources/image_1709177590.png)

1. В цикле `for column in df.columns:` произвели перебор всех значений индекса столбцов — их названий.
2. В теле цикла для каждого столбца мы использовали код, который преобразовывал старое название столбца, заданное переменной итератором `column` в нотацию Snake Case — `column.lower().replace(' ', '_')`.
3. Затем мы использовали метод `.rename()`, который переименовал все колонки по указанному словарю.
4. Атрибут метода `.rename()` `inplace=True` позволил провести преобразования названий столбцов непосредственно в исходном датафрейме `df`.

### Изменение названия столбцов с помощью аксессора `str`

Перевести названия столбцов к стилю Snake Case можно ещё одним способом. В цикле из примера выше мы перебирали каждое значение **Index** и изменяли названия столбцов пошагово. При работе с данными pandas можно использовать для объектов Series и Index аксессор .str — от английского string - «строка».

`.str` позволяет использовать различные векторизованные строковые методы, в том числе `.lower()`, `.replace()`, описанные выше, с текстовыми данными внутри объектов Series и Index библиотеки pandas, которые имеют тип данных `object`.

Посмотрим на Index столбцов ещё раз:

```
# Зададим данные для датафрейма: названия столбцов и содержание строк:
columns = ['Product ID', 'Category', 'Description', 'Price']
data = [[10001, 'Телефоны','Смартфон', 15999],
       [10002, 'Компьютеры', 'Монитор', 17200],
       [10003, 'Кухонная техника', 'Микроволновка', 6350]]

# Создадим новый датафрейм df:
df = pd.DataFrame(data=data, columns=columns)
    
print(df.columns) 
```

```
Index(['Product ID', 'Category', 'Description', 'Price'], dtype='object') 
```

В выводе видим, что данные в Index представлены в текстовом виде, поэтому мы можем использовать строковые функции через атрибут `str` для изменения всех объектов внутри Index. Например, изменим регистр слов:

```
# Измененим все объекты внутри Index при помощи метода lower():
print(df.columns.str.lower()) 
```

```
Index(['product id', 'category', 'description', 'price'], dtype='object') 
```

Атрибут `str` сделал своё дело! Он позволил нам обратиться к объектам Index и изменить их с помощью метода `.lower()`. Поскольку этот метод векторный, то изменения произошли для всего массива Index сразу. Перебирать каждое значение, как в случае с использованием цикла, не понадобилось.

Обратите внимание, что в примере выше мы не сохраняли результат изменения названия столбцов, а вывели результат на экран — важно отметить, что после использования `.str.lower()` тип объекта `Index` не изменился.

Теперь мы можем привести к Snake Case, используя цепочку методов `.lower().replace(' ', '_')`. Однако, поскольку `str` предоставляет доступ к данным только для одного строкового метода, то использовать этот атрибут в цепочке методов следует перед каждым методом:

```
# Зададим данные для датафрейма: названия столбцов и содержание строк:
columns = ['Product ID', 'Category', 'Description', 'Price']
data = [[10001, 'Телефоны','Смартфон', 15999],
       [10002, 'Компьютеры', 'Монитор', 17200],
       [10003, 'Кухонная техника', 'Микроволновка', 6350]]

# Приведем названия столбцов к Snake Case:
df.columns = df.columns.str.lower().str.replace(' ', '_')

print(df.columns) 
```

```
Index(['product_id', 'category', 'description', 'price'], dtype='object') 
```
Результат — названия всех столбцов переведены в нотацию Snake Case, при этом преобразование осуществлено в одной инструкции, которая содержала цепочку атрибутов объектов pandas и строковых методов.

### Изменение названия столбцов: общий подход

Таким образом, изменить название столбцов или привести их к нотации Snake Case можно используя различные методы и атрибуты. Ниже мы собрали основные способы, которые обсуждали выше, с указанием особенностей их использования:

- Атрибут DataFrame `columns`

```
# Синтаксис:
df.columns = [элементы через запятую] 
```

```
# Пример:
df.columns = ['column_a','columns_b'] 
```

Замена названий всех столбцов на заранее подготовленный список. Оптимально использовать, когда последовательность и количество столбцов известно. Например, при изменении названий столбцов после создания сводных таблиц.

- Метод pandas `.rename()`

```
# Синтаксис:
df.rename(columns={пары значений}, inplace = True) 
```

```
# Пример:
df.rename(columns={'A': 'column_a', 'B': 'column_b'}, inplace = True) 
```

Замена названий всех или только некоторых столбцов. Используется словарь, содержащий старое и новое названия. Не зависит от последовательности столбцов и их количества. Часто используется при изучении динамических данных и при замене названия единичных столбцов. Делает код более читаемым и минимизирует ошибки при обновлении данных. Параметр `inplace = True` указывает на замену значений индекса в датафрейме сразу при выполнении инструкции.

- Атрибут Index `str` и строковые методы `.lower()` и `.replace()`

```
# Синтаксис:
df.columns = df.columns.str.lower().str.replace() 
```

```
# Пример:
df.columns = df.columns.str.lower().str.replace(' ', '_') 
```

Позволяет быстро перевести названия всех столбцов в нотацию Snake Case. При этом, будут заменяться все названия столбцов, даже неизвестные на момент написания кода. Использовать атрибут `str` необходимо перед каждым строковым методом — `.lower()` и `.replace()`. Используются векторные преобразования, без перебора значений Index.

### Замена данных с помощью метода `.replace()`

В прошлом уроке вы познакомились с методом `.replace()`, который использовали для замены названия столбцов. Рассмотрим метод и его возможности детальнее. Начнём с синтаксиса:

```
DataFrame.replace(to_replace=None, value=None, inplace=False, limit=None, regex=False) 
```

Метод имеет несколько атрибутов:

- `to_replace` — значение или список значений, которые необходимо заменить.
- `value` — значение или их список, на которые будет проводиться замена.
- `inplace` — логический атрибут, который указывает, изменять датафрейм на месте или нет. По умолчанию стоит значение `False` — не изменять на месте.
- `limit` — устанавливает максимальное количество значений, которые необходимо заменить в датафрейме. По умолчанию ограничений нет.
- `regex` — логический атрибут, который указывает, являются ли `to_replace` и `value` регулярным выражением. В будущих уроках вы познакомитесь с регулярными выражениями подробнее.

С помощью метода `.replace()` можно заменить одни данные на другие как во всём датафрейме, так и в отдельных столбцах или строках.

Данные во всём датафрейме обычно меняют, когда необходимо изменить название какой-то группы или категории значений, а также изменить возможные ошибки в данных, например, опечатки.

Разберём на примере с данными маркетплейса. Маркетплейс готовится к выходу на международный рынок, поэтому нужно привести все категории в англоязычный формат. Для этого заменим название категории `'Электроника'` на англоязычное название `'electronics'`:

```
# Заменим название категории 'Электроника' на англоязычное название 'electronics':
df.replace('Электроника', 'electronics', inplace=True)

print(df) 
```

Изменение прошло успешно! Изменённые значения выделены символом `<` в конце строки. В примере метод `.replace()` изучил значение в каждой ячейке и, если среди них была подстрока, содержащая `'Электроника'`, заменил её на `'electronics'`.

Метод `.replace()` чувствителен к регистру значений. Например, в инструкции `df.replace('электроника', 'electronics', inplace=True)` заменяемое значение не учитывает регистр, поэтому её выполнение ничего не изменит.

Это удобно, когда точно известно, в каких столбцах проводить замену. Можно не перебирать все столбцы, а изменить данные точечно в нужных местах.

Например, если посмотреть на названия категорий, то можно заметить неточность во второй строчке — категория называется `Смартфоны`, хотя должна быть `electronics`. Проведём замену:

# Заменим название категории 'Смартфоны' на название 'electronics':

В методе `.replace()` для замены значений можно использовать списки. Например, заменим остальные значения категорий на их англоязычные названия. Для этого создадим два списка: один со старым названием `old_category`, другой — с новым `new_category`:

# Создадим два списка – один со старым названием категорий, другой – с новым:
old_category = ['Бытовая техника', 'Компьютерная техника']
new_category = ['appliances', 'computers']

# Заменим название категории согласно двум спискам:
df['category'].replace(old_category, new_category, inplace=True)

print(df) 
```

Изменённые строки выделены символом `<`. Метод сравнил значения в каждой ячейке столбца `'category'` и, если оно было в списке `old_category`, то заменил на соответствующее значение из списка `new_category`.

В подобном случае можно использовать словарь, где ключ представлен старым названием, а значение — новым:

```
# Создадим словарь для замены значений:
replace_dict = {'Бытовая техника':'appliances',
                'Компьютерная техника':'computers'}

# Заменим название категории согласно словарю:
df['category'].replace(replace_dict, inplace=True)

print(df) 
```

Результат будем таким же, как и в первом способе.

Подход со словарём позволяет лучше ориентироваться в коде, поскольку в словаре каждому старому значению соответствует новое и наглядно видно, как происходит замена, что может быть затруднительно в случае с большими списками.
Обратите внимание, что в примерах выше метод `.replace()` использовался с атрибутом `inplace=True`. Такой подход изменяет исходный датафрейм.

На промежуточной стадии анализа использовать `inplace=True` стоит осторожно — чтобы вернуть изменения, придётся перезагрузить все исходные данные и запустить скрипт с начала. В этом случае удобнее сохранять изменение значений в виде нового объекта — датафрейма или серии. Для этого не прописывают в методе атрибут `inplace` — по умолчанию возможность перезаписи исходных данных отключена (`inplace=False`). Это удобно, когда стоит задача сохранить исходные данные.

Например, объединим группы `Смартфоны`, `Умные часы`, `'Мобильный телефон'` и `Планшет` в одну укрупнённую группу — `gadgets`. Создадим для неё новый столбец — `new_group`:

# Создадим список с названием групп товаров для замены названия:
group_list = ['Смартфоны','Умные часы','Планшет','Мобильный телефон']

# Заменим все значения списка group_list на gadgets:
df['new_group'] = df['group'].replace(group_list, 'gadgets')

print(df) 
```

2  electronics  2338051   Мобильный телефон    817    37.0     88        gadgets  <

### Замена данных с помощью индексации

В примерах выше вы меняли данные по всему датафрейму или по отдельным столбцам. Часто в анализе данных бывает нужно изменить какое-то определённое значение — например, изменить ошибочные данные или скорректировать стоимость товара определённой категории.

Посмотрим на датафрейм маркетплейса электроники ещё раз:

```
          category       id               group  price  rating  votes      new_group
0  electronics  1826254           Смартфоны  14297     4.8   1536        gadgets   
1  electronics  1832281           Смартфоны  32690     4.9    186        gadgets    
2  electronics  2338051   Мобильный телефон    817    37.0     88        gadgets **** < 
3  electronics  6757637          Умные часы   1287     4.5    121        gadgets    
4   appliances  1777135  Микроволновая печь   5206     4.9     25  Микроволно...   
5    computers  1774907             Ноутбук  91200     4.8     98        Ноутбук 
6    computers  1879698             Планшет  30290     4.9    749        gadgets 
```

В столбце `rating` есть высокое значение `37`. Это может быть ошибка в данных, поскольку рейтинг представлен по 5-балльной системе. Скорее всего, вместо `37` рейтинг должен быть `3.7`. Заменим это значение.
Для замены обратимся к ячейке с помощью индексации `loc[]` по `id` товара:

```
# Используя индексацию, сделаем срез данных по id товара и поделим значение на 10:
df.loc[df['id'] == 2338051, 'rating'] = df['rating'] / 10

print(df) 
```

Результат:

```
          category       id               group  price  rating  votes      new_group
0  electronics  1826254           Смартфоны  14297     4.8   1536        gadgets   
1  electronics  1832281           Смартфоны  32690     4.9    186        gadgets    
2  electronics  2338051   Мобильный телефон    817     3.7     88        gadgets    
3  electronics  6757637          Умные часы   1287     4.5    121        gadgets    
4   appliances  1777135  Микроволновая печь   5206     4.9     25  Микроволно...   
5    computers  1774907             Ноутбук  91200     4.8     98        Ноутбук 
6    computers  1879698             Планшет  30290     4.9    749        gadgets 
```

В начале метод обратился к необходимой ячейке с помощью индексацию `loc[]` — `df.loc[df['id'] == 2338051, 'rating']`. Затем этой ячейке присвоено новое значение — ошибочный рейтинг разделён на 10: `df['rating'] / 10`.

В ходе корректировки значений новое значение не было прописано вручную, как `df.loc[df['id'] == 2338051, 'rating'] = 3.7`, а использовалось изменение исходного значения. Такой подход корректнее ручной замены, поскольку помогает избежать ошибок при ручном заполнении большого объёма данных.

Обратите внимание, что в примере выше мы присваиваем ячейкам, которые отобрали по условию, вектор изменённых значений — `df['rating'] / 10`. Когда мы делаем одни и те же математические действия с исходными данными, операторы вида `/=` помогают изменять исходные значения:

```
# Используя индексацию, сделаем срез данных по id товара и поделим значение на 10:
df.loc[df['id'] == 2338051, 'rating'] /= 10 
```

### Замена данных по условию

При анализе данных бывает нужно изменить исходные значения при наступлении заданных условий. Посмотрим на примере, как это происходит.

В датафрейме `df` в примерах выше мы создали новую укрупнённую группу товаров — `gadgets`.

```

# Используя логическую индексацию, проведём замену значений:
df.loc[df['new_group'] != 'gadgets', 'new_group'] = 'other'

Далее в операторе `.loc[]` указали столбец — `'new_group'`, именно в нём нужно заменить значения. При замене значений можно использовать более сложные логические выражения, которые включают в себя несколько условий. Например, создадим новый столбец `'is_popular'`, в котором будет информация о том, является ли товар популярным или нет. Будем считать товар популярным, если количество голосов больше 500 и рейтинг выше 4.7 — тогда значение в столбце `'is_popular'` будет `True`. Если хотя бы одно условие не выполняется, то `False`.

Создадим новый столбец `'is_popular'` со значением по умолчанию `False`. Далее с помощью логической индексации изменим значение на `True` для популярных товаров:

```
# Создадим новый столбец 'is_popular' со значением по умолчанию False:
df['is_popular'] = False

# Используя логическую индексацию, найдём популярные товары:
mask = (df['votes'] > 500) & (df['rating'] > 4.7)
df.loc[mask, 'is_popular'] = True

# Выведем на экран часть данных:
print(df.iloc[:, [1,4,5,6,7]]) 
```

Популярные товары найдены, а значения столбца `is_popular` для них изменено.

Такие данные с булевым значением, которое отражает принадлежность строки к определённому сегменту или условию, часто называют **флагом**, или **индикатором**, признака. В нашем случае — флагом популярности товара.

С помощью такого подхода можно фиксировать, является ли посетитель покупателем, есть ли товар в наличии, имеет ли клиент задолженность.

Обратите внимание, что в примере выше все действия описаны последовательно: в начале создали столбец, заполнили его по умолчанию, затем задали логическую маску и на её основе изменили значения. Такой последовательный подход к написанию кода позволяет зафиксировать все действия в очевидном виде.

Код выше можно оптимизировать, если при создании прописать логическое условие:

```
# Создадим новый столбец 'is_popular' с помощью логической маски:
df['is_popular'] = (df['votes'] > 500) & (df['rating'] > 4.7) 
```

Такой подход хорошо использовать при создании новых столбцов. При изменении ранее созданных данных лучше использовать непосредственно замену данных.

Вместе с логической индексацией можно использовать метод `.replace()`. Например, код выше можно переписать так:

```
# Создадим новый столбец 'is_popular' со значением по умолчанию False:
df['is_popular'] = False

# Используя логическую индексацию, найдём популярные товары:
mask = (df['votes'] > 500) & (df['rating'] > 4.7)
df.loc[mask, 'is_popular'] = df['is_popular'].replace(False, True) 
```

Логическая индексация помогла выделить диапазон ячеек для изменения, а метод `.replace(False, True)`, применённый к столбцу `df['is_popular']`, заменил значения.

### Использование `query()` в условии при замене данных

В примерах выше при замене данных по условию использовались логические маски с использованием оператора `[]`. Для замены данных также можно использовать запросы `query()` в сочетании с оператором `.loc[]`.

Перепишем пример выше с использованием `query()`. Теперь популярными будем считать товары, у которых рейтинг выше `4.5`, а количество оценок — выше `1500`:

```
# Используя запрос query, найдём необходимые товары:
mask = df.query('votes > 1500 and rating > 4.5') 
```

Если передать такой результат оператору `.loc[]` для фильтрации данных, как в примере выше, с помощью `df.loc[mask, 'is_popular']`, то интерпретатор выдаст ошибку. Оператор `.loc[]` ожидает значения индексов строк, а не срез данных. Поэтому передадим в операторе `.loc[]` только индексы необходимых строк, добавив к отфильтрованному датафрейму атрибут `.index`:

```
# Создадим новый столбец 'is_popular' со значением по умолчанию False:
df['is_popular'] = False

# Используя запрос query, найдём необходимые товары, результат сохраним в mask:
mask = df.query('votes > 1500 and rating > 4.5').index
df.loc[mask, 'is_popular'] = True

# Выведем на экран часть данных:
print(df.iloc[:, [1,4,5,6,7]]) 
```

Получилось:

```
        id  rating  votes new_group  is_popular
0  1826254     4.8   1536   gadgets        True
1  1832281     4.9    186   gadgets       False
2  2338051     3.7     88   gadgets       False
3  6757637     4.5    121   gadgets       False
4  1777135     4.9     25     other       False
5  1774907     4.8     98     other       False
6  1879698     4.9    749   gadgets        False 
```

Всё верно! Таким образом, совмещение сложных запросов `query` с индексацией данных помогает заменять определённые значения.

# Типы данных в pandas

## Очень важные типы
В следующих уроках вы рассмотрите, что основа эффективного исследовательского анализа данных — это корректно подготовленные данные. В прошлом уроке вы узнали, как ознакомиться с данными, обработать пропущенные значения и дубликаты в них. Теперь пришло время разобраться с типами данных.

Датафреймы pandas могут содержать различные данные — числовые и текстовые значения, дату и время, логические значения или категории. Pandas позволяет аналитику эффективно работать с любыми данными в различных форматах.

Однако не всегда при загрузке в датафрейм данные получают необходимый тип. Например, если в данных есть пропуски, то целочисленные значения будут отображаться в формате чисел с плавающей точкой — float. Или если числовые значения содержат ошибку, например, одно текстовое значение, то тип всего столбца будет не численным `int` или `float`, а примет значение `object`, что не позволит применять к этому столбцу векторные функции и, например, агрегировать данные. Также pandas обычно рассматривает как строки данные с датами и временем, полученные при открытии csv.

Поэтому при анализе данных важно убедиться, что тип данных верный и подходящий для необходимых расчётов. Иначе можно получить неверные результаты или ошибку при запуске кода. Например, если числовые данные представлены в виде строк, то не получится корректно провести математические операции с ними. Или, если даты сохранены как строки, то при сортировке они будут упорядочены алфавитном порядке, а не в хронологическом.

В этом уроке вы узнаете, какие типы данных бывают в датафреймах pandas и какие методы или функции можно использовать для их корректировки.

### Типы данных в pandas

При работе с типами данных в датафреймах следует обратить внимание на то, что по умолчанию значениям датафрейма присваиваются типы данных pandas. В pandas существуют следующие типы данных, некоторые из которых совпадают с типами данных Python:

|**pandas**|**Python**|**назначение в pandas**|
|---|---|---|
|object|str|Используется для представления текстовой или смешанной текстовой и числовой информации. Например, `'@nickname_35'`. В этом случае тип object аналогичен типу str Python.  <br>Также этот тип данных может использоваться для хранения любых объектов Python, например, списка `['@nickname_35', '@nickname_68']`.|
|int64|int|Целочисленный тип данных. Используется для хранения целых чисел. Например, `5, 11, -39`|
|float64|float|Вещественный тип данных. Используется для хранения чисел с плавающей запятой. Например, `15.3, 17.0, -0.57`|
|bool|bool|Логический тип данных, содержащий всего два значения — `False` и `True`.|
|datetime64|datetime|Тип данных даты и времени, используется для хранения значений даты и времени. Например, `2024-08-12 10:30:00`|
|timedelta64|-|Тип данных для хранения интервала времени между двумя событиями с высокой точностью (до наносекунд). Например, `5 days 01:45:30`|
|category|-|Категориальный тип данных, представляет ограниченный список значений (категорий). Например, `'отлично', 'хорошо', 'удовлетворительно'`|

Большая часть наиболее распространенных типов данных pandas имеет аналоги в Python и сходны по хранимой информации, за исключением типов `timedelta64` и category. Также типы данных Pandas совместимы с Numpy, что позволяет эффективно совмещать их друг с другом.

Обратите внимание, что большая часть типов данных использует 64-битное представление данных. Это позволяет хранить данные с высокой точностью. При необходимости можно понизить разрядность числовых типов данных, например, использовать такие типы, как `int8`, `int16`, `int32` вместо `int64`, тогда как в Python тип int имеет неограниченную разрядность, а число с плавающей точкой имеет 64 битную разрядность.

Фиксация типа данных позволяет для числовых и временных типов данных в pandas совершать векторные вычисления — т.е. проводить вычисления сразу с использованием всех данных одного столбца DataFrame, что сокращает время обработки данных и оптимизирует вычисления.

### Фильтрация данных по их типу

После того как информация о типах данных получена, следует оценить, а действительно ли наши типы данных подойдут для дальнейшего анализа. Например, посмотреть данные типа `float64`, затем `object`.

Для фильтрации данных по их типу можно использовать атрибут `.dtypes`, поскольку он возвращает серию значений. В примере выше видно, что у данных с датой и временем некорректный тип — `object`, и если понадобится использовать эти данные, то их лучше перевести в тип `datetime64`. Формат записи даты год-месяц-число позволяет сортировать данные по дате, но не позволяет проводить другие операции.

Метод `.select_dtypes()` имеет несколько параметров: `include=None`, `exclude=None`, которые позволяют показать только включенные типы данных (`include`), или наоборот, показать все типы данных, кроме определённых (`exclude`). В примере выше в базовом синтаксисе мы передавали список типов данных, которые необходимо включить в выборку, поэтому использовали первый параметр `include`.

Фильтрация данных по их типу может быть полезной не только для ознакомления с данными, но и при проведении однотипных действий с различными данными на стадии их изучения. Например, можно подготовить цикл, который будет выводить полезную информацию о данных и, если данные будут численного типа, то показать результат метода `.describe()`, а если типа `object` — вывести уникальное значение данных и изучить распределение значений по ним с помощью метода `.value_counts()`:

```
# Напишем цикл, который будет выводить полезную информацию для каждого столбца:
for column in df.columns:
    print(f'Информация о столбце {column}')
    
    # Если данные представлены в виде числа:     
    if df[column].dtypes in ['int64','float64']:
        print(df[column].describe())
        
    # Если данные имеют объектный тип данных:        
    elif df[column].dtypes == 'object':
        print('Количество уникальных значений', df[column].nunique())
        print(df[column].value_counts()) 
```

### Изменение типа данных

После того как информация о типах данных получена, проведён анализ соответствия типа данных значениям, которые содержит каждый столбец, необходимо скорректировать типы данных.

Обратите внимание, что корректировку типов данных, как правило, проводят в тех случаях, когда это необходимо при работе с данными и их анализе. Например, если необходимо проводить арифметические операции, а числа представлены в виде строк, или необходимо выделить временной интервал, а данные не имеют тип даты и времени. Или когда необходимо перевести данные с плавающей точкой в целочисленный формат, рассматривая их как дискретные значения — о них мы расскажем в следующих уроках. Также ещё одной причиной смены типа данных может быть сокращение их объёма для оптимизации ресурсов, затрачиваемых на обработку данных.

Для изменения типа данных можно использовать несколько подходов. Наиболее распространённый — метод `.astype()`. Этот метод можно применить к столбцу или всему датафрейму. В первом случае в базовом синтаксисе достаточно передать методу новое название типа данных.

Вернёмся к данным объявлений о продаже автомобилей. В следующих уроках мы проверим работу по замене пропущенных значений. Теперь займёмся типами данных. Преобразуем данные в столбце `model_year` в тип данных `int64`:

```
# Импортируем библиотеку pandas и загрузим данные:

# Заменим пропуски:

df['is_4wd'] = df['is_4wd'].fillna(0)

# Остальные пропуски удалим:
df = df.dropna()

# Преобразуем тип данных в столбце model_year:
df['model_year'] = df['model_year'].astype('int64')

# Проверим результат:
df['model_year'].dtypes

### Форматы записи даты и времени

При работе с различными данными вы столкнётесь с тем, что время и даты могут быть записаны в разных форматах:

- дата может не содержать время, или наоборот данные содержат только время, но не содержат дату, например, вместо столбца с датой и временем записи действия пользователя `2024-02-15 12:35:41` может быть два столбца: один с датой — `2024-02-15`, второй с временем — `12:35:41`;
- при написании даты и времени может меняться порядок отображения года / месяца / дня или часов / минут / секунд, или в начале идти время, а затем дата;
- даты и время могут содержать метку часового пояса, `2024-02-15 12:35:41+03:00`;
- дата и время может быть прописаны в виде целочисленного значения — в формате `timestamp`, например, дата и время `2024-02-15 12:35:41` в формате `timestamp` будет отображаться как `1707989741` или после импорта данных из Excel даты могут быть прописаны как целые числа, например, вместо `2024-02-15` будет указано `45337`.

При анализе данных следует быть готовыми к тому, что каждый разный формат записи даты и времени нужно привести к единому виду pandas `datetime64` и иногда это потребует дополнительного внимания. Разберём, как это можно сделать.

### Метод pandas `.to_datetime()`

Метод pandas `.to_datetime()` _**_для перевода данных в формат даты и времени в базовом синтаксисе устроен так: в качестве аргумента передаются данные, которые нужно преобразовать. Обычно, это серия значения — столбец датафрейма. Для примера создадим небольшой датафрейм:

```

# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],

                 '2024-03-28 19:30:00'],

               '12:45:00 28.03.2024', '19:00:00 28.03.2024', '21:00:00 28.03.2024']
})

# Выведем датафрейм на экран:
print(df)

# Проверим типы данных:
print('Проверим типы данных датафрейма df:')
print(df.dtypes) 
```

Обратите внимание, что значение в столбце `start_dt` представлены в стандартном виде — в начале дата, потом время, а вот значения в столбце `end_dt` — наоборот, в начале время, затем дата, при этом дата указана как день, месяц и год.

# Изменим тип данных с помощью метода .to_datetime():
df['start_dt'] = pd.to_datetime(df['start_dt'])
df['end_dt'] = pd.to_datetime(df['end_dt'])

print(df.dtypes)

Обратите внимание, что в коде выше каждый столбец был преобразован отдельно. Методу `.to_datetime()` нельзя передать несколько столбцов для преобразования, как это можно сделать в методе `.astype()`. Поэтому для преобразования нескольких столбцов одновременно можно использовать метод `.to_datetime()` в связке с методом `.apply()`, применяя его к необходимому срезу данных:

```

```

В примере выше данные были представлены в виде день, месяц, год. Однако метод `.to_datetime()` обработал их в иной последовательности: месяц, день, год. В результате данные исказились, а это может привести к непредсказуемым результатам.

Чтобы избежать такой ошибки, в методе `.to_datetime()` есть параметр `format`, в котором можно указать, какой формат времени и даты используется. Сравните результат с предыдущим:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],
    'reg_dt': ['01.03.2024', '02.03.2024', '01.03.2024', 
               '10.03.2024', '11.03.2024']
})

# Выведем датафрейм на экран:
print(df)

# Преобразуем типы данных с датой:
df['reg_dt'] = pd.to_datetime(df['reg_dt'], format='%d.%m.%Y')

# Выведем результ на экран:
print(df) 
```

```

0  001244  01.03.2024
1  001037  02.03.2024
2  001655  01.03.2024
3  000942  10.03.2024
4  001244  11.03.2024
  user_id     reg_dt
0  001244 2024-03-01
1  001037 2024-03-02
2  001655 2024-03-01
3  000942 2024-03-10
4  001244 2024-03-11 
```

Преобразование типа данных прошло корректно — день и месяц на своих местах. В аргументе `format` был передан шаблон `'%d.%m.%Y'`, по которому ориентировался метод `.to_datetime()` при корректировке типа данных. В результате месяц и дни были на своих местах.

Наиболее широко используемые обозначения при создании шаблона записи даты:

- **`%d`**: день месяца в виде двузначного числа (например, 01, 02, ..., 31);
- **`%m`**: месяц в виде двузначного числа (например, 01, 02, ..., 12);
- **`%y`**: год без столетия в виде двузначного числа (например, 00, 01, ..., 99);
- **`%Y`**: год с веком в виде четырёхзначного числа (например, 0001, 0002, ..., 2013, 2014, ..., 9998, 9999);

и времени:

- **`%H`**: часы в 24-часовом формате в виде двузначного числа (например, 00, 01, ..., 23);
- **`%I`**: часы в 12-часовом формате в виде двузначного числа (например, 01, 02, ..., 12);
- **`%M`**: минуты в виде двузначного числа (например, 00, 01, ..., 59);
- **`%S`**: секунды в виде двузначного числа (например, 00, 01, ..., 59);
- **`%f`**: микросекунды в виде числа с шестью знаками (например, 000000, 000001, ..., 999999);
- **`%p`**: локализованный эквивалент AM или PM.

Например, запись `2024-03-28 21:00:00` будет иметь такой формат `%Y-%m-%d %H:%M:%S`. Более подробно смотрите [в документации на английском](https://pandas.pydata.org/docs/reference/api/pandas.Period.strftime.html).

В методе `.to_datetime()` также есть некоторые дополнительные аргументы, кроме аргумента `format`, которые позволяют корректно указать формат даты, особенно, если год представлен в виде двузначного числа:

- `dayfirst` используется, если первое значение в дате — день, например: дата `10/11/12` будет интерпретирована как `2012-11-10` — 10 ноября 2012 года. По умолчанию указан параметр `False`.
- `yearfirst` — аналогично предыдущему параметру, за исключением того, что первое значение определяется как год, например, дата `10/11/12` будет интерпретирована как `2010-11-12` — 12 ноября 2010 года. По умолчанию указан параметр `False`.

### Формат данных даты и времени `timestamp`

Дата и время в формате `timestamp` представлены в виде целого числа, которое обычно составляет количество секунд или миллисекунд, прошедших с определённой точки отсчёта. За точку отсчёта берут различные даты. Например, в Unix-системах `timestamp` представляет собой количество секунд, прошедших с начала эпохи Unix, которая начинается с 1 января 1970 года 00:00:00 (UTC).

Например, дата и время `2024-02-15 12:35:41` в формате `timestamp` будут записаны как `1707989741`, поскольку именно такое количество секунд прошло с 1 января 1970 года 00:00:00 (UTC). Этот формат данных широко используется для хранения и обработки даты и времени, так как позволяет легко выполнять различные операции и вычисления с временными данными. Например, подсчитать разницу в секундах между двумя значениями.

Посмотрим, как перевести дату и время из формата `timestamp` в формат `datetime64`. Для этого можно использовать метод `pd.to_datetime()`, в котором нужно указать атрибут `unit='s'`, если время в формате `timestamp` округлено до секунды:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],
    'start_ts': [1585382400, 1585396200, 1585398900, 1585410300, 1585416600],     
    'end_ts': [1585384200, 1585398000, 1585399500, 1585412400, 1585422000]})

# Изменим тип данных с помощью метода .to_datetime():
df['start_dt'] = pd.to_datetime(df['start_ts'], unit='s')
df['end_dt'] = pd.to_datetime(df['end_ts'], unit='s')

# Выведем датафрейм на экран:
print(df)

# Проверим типы данных:
print('Проверим типы данных датафрейма df:')
print(df.dtypes) 
```

```

В результате все преобразования типов прошли корректно — были образованы два новых столбца с датой и временем в формате `datetime64`. Если время задано в миллисекундах, то стоит использовать `'ms'` в аргументе `unit`. По умолчанию этот параметр принимает время в наносекундах — `ns`.

### Дополнительная работа с данными типа `datetime64`

Тип данных `datetime64` позволяет получить из даты и времени много информации, например, выделить год, месяц, день, номер недели или название дня недели. Выделение этих составляющих даты и времени позволяет аналитикам проводить сравнительный анализ данных, например, сопоставлять данные за один месяц с другим, просмотреть изменения метрик по дням или создать ежеквартальные сводные отчёты. Возможных примеров гораздо больше, поэтому разберём наиболее часто встречающиеся ситуации.

Чтобы извлечь из данных `datetime64` дополнительную информацию, к столбцу с данными применяется акцессор `.dt`, который позволяет обращаться к каждой строке как к объекту `datetime64`, и специальный атрибут, выполняющий ту или иную функцию. Рассмотрим это на примере данных по продаже автомобилей и выделим год публикации объявления в отдельный столбец:

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
df = pd.read_csv('/datasets/vehicles_us.csv')

# Изменим тип данных столбца date_posted:
df['date_posted'] = pd.to_datetime(df['date_posted'])

# Выделим год публикации объявления:
df['year'] = df['date_posted'].dt.year

# Проверим результат:
print(df[['date_posted','year']].head()) 
```

```

0  2018-06-23  2018
1  2018-10-19  2018
2  2019-02-07  2019
3  2019-03-22  2019
4  2019-04-02  2019 
```

Выше для выделения года мы использовали атрибут `datetime64` `year`, который с помощью акцессора `.dt` преобразовал каждую дату в год. Есть множество атрибутов, которые позволяют проводить различные операции с данными `datetime64`.

Рассмотрим основные из них:

|Атрибут datetime64|Действие|
|---|---|
|year|Извлечение года|
|quarter|Извлечение квартала|
|month|Извлечение номера месяца|
|week|Извлечение номера недели|
|day|Извлечение дня месяца|
|day_of_year|Извлечение порядкового номера дня в году. Аналогичный атрибут - dayofyear|
|day_of_week|Извлечение дня недели (где 0 — понедельник). Аналогичные атрибуты - dayofweek и weekday|
|day_name()|Извлечение названия дня|

Например, выделим год, квартал, номер месяца, номер недели, день:

```

# Выделим квартал публикации объявления:

# Выделим месяц публикации объявления:

# Выделим номер недели публикации объявления:
df['week'] = df['date_posted'].dt.week

# Выделим день (месяца) публикации объявления:
df['day'] = df['date_posted'].dt.day

# Выделим порядковый номер дня публикации объявления:
df['dayofyear'] = df['date_posted'].dt.dayofyear

# Выделим день недели публикации объявления:
df['dayofweek'] = df['date_posted'].dt.dayofweek

# Выделим название дня недели публикации объявления:
df['day_name'] = df['date_posted'].dt.day_name()

# Проверим результат:
print(df[['date_posted','year','quarter','week','day',
          'dayofyear','dayofweek','day_name']].head()) 
```

Поэтому для выделения номера недели лучше использовать метод `isocalendar()` — он преобразует дату в датафрейм — с перечислением года, номера недели и дня:

Аналогичным образом можно проводить различные действия с временем, например, выделить час, день или секунды:

# Выделим минуты:

Также существуют атрибуты, выводящие только дату или только время, или задающие сложный формат по шаблону:

|Атрибут datetime64|Действие|

# Создадим датафрейм с данными:

# Выделим только время:
df['time'] = pd.to_datetime(df['dt']).dt.time

# Выделим дату как объект datetime64:
df['date_norm'] = pd.to_datetime(df['dt']).dt.normalize()

# Проверим результат:
print(df)
print(df.dtypes) 
```

### **Часовой пояс и `.to_datetime()`**

Иногда данные могут быть представлены вместе с информацией о часовом поясе. Обычно он добавляется в виде `+03:00`, где первый знак это плюс `+` или минус `-`, которые указывают на разницу относительно UTC, далее идут часы и минуты.

Например, дата и время `2024-03-28 00:00:00+03:00` указывают, что событие было совершено в 0 часов 28 марта 2024 года с учётом часового пояса `+3` (`Europe/Moscow`). Чтобы убрать часовой пояс и привести время к UTC, метод `.to_datetime()` имеет параметр `utc=True`:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({'dt': ['2024-03-28 00:00:00+03:00']})

# Преобразование столбца dt в формат datetime64 с учетом часового пояса:
df['dt'] = pd.to_datetime(df['dt'], utc=True)

print(df) 
```

```
                         dt
0 2024-03-27 21:00:00+00:00 
```

Как видно из результата, событие в часовом поясе состоялось в полночь, а относительно UTC — в 21 вечера прошлого дня. Такие особенности следует учитывать при анализе данных, которые фиксируются в различных часовых поясах.

Чтобы убрать информацию о часовом поясе, можно использовать метод `.tz_localize()` с параметром `None`:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({'dt': ['2024-03-28 03:00:00+03:00']})

# Преобразование столбца dt в формат datetime64 с учетом часового пояса:
df['dt'] = pd.to_datetime(df['dt']).dt.tz_localize(None)

print(df) 
```

```
                   dt
0 2024-03-28 03:00:00 
```

### Временные интервалы `timedelta64` и метод `.to_timedelta()`

При работе с датами и временем аналитик часто проводит расчёты временного интервала, например, изучает время регистрации в приложении, длительность пользовательской сессии, количество покупок пользователя за последние пять дней или рассчитывает лайфтайм событий с момента регистрации в когортном анализе.

При вычитании одних даты и времени из других, например, при подсчёте длительности загрузки приложения по формуле «время окончания загрузки - время старта загрузки» образуется разница — интервал времени, который может быть выражен в секундах, минутах, часах, днях. В pandas результат будет представлен в виде данных типа `timedelta64`:

```
# Создадим датафрейм с данными:
df = pd.DataFrame({
    'user_id': ['001244', '001037', '001655', '000942', '001244'],
    'start_dt': ['2024-03-28 08:00:00', '2024-03-28 12:30:00', '2024-03-28 13:15:00', '2024-03-28 17:45:00', '2024-03-28 19:30:00'],
    'end_dt': ['08:30:00 28.03.2024', '2024-03-28 13:13:00', '12:45:00 28.03.2024', '19:00:13 28.03.2024', '21:00:00 28.03.2024']})

df[['start_dt','end_dt']] = df[['start_dt','end_dt']].apply(pd.to_datetime)

# Рассчитаем время сессии:
df['session_leght'] = df['end_dt'] - df['start_dt']

# Выведем результат расчёта длительности сессии на экран:
print(df['session_leght']) 
```

```
0     0 days 00:30:00
1     0 days 00:43:00
2   -1 days +23:30:00
3     0 days 01:15:13
4     0 days 01:30:00
Name: session_leght, dtype: timedelta64[ns] 
```

Из результатов видно, что некоторые сессии длились 30 минут и больше, а какие-то имеют отрицательные значения. Видимо, закралась ошибка в записи результатов: `1 days +23:30:00` это то же самое, что `-0:30`, то есть сессия началась на 30 минут позже, чем окончилась. Таким образом интервал данных `timedelta64` может быть как положительным, там и отрицательным.

В качестве единиц измерения в типе данных `timedelta64` могут использоваться: дни (`D` или `days`), часы (`H` или `hours`), минуты (`T` или `m`), секунды (`S` или `s`), миллисекунды (`L` или `ms`), микросекунды (`U` или `us`) и наносекунды (`nanoseconds` или `ns`). Например, создадим интервал в 5 дней, 13 часов и 10 минут:

```
# Создадим интервал в 5 дней, 13 часов и 10 минут:
interval = pd.to_timedelta('5 days 13:10:00')

print(interval) 
```

```
5 days 13:10:00 
```

Объекты типа `timedelta64` поддерживают арифметические операции, такие как сложение, вычитание и умножение на число. Это позволяет выполнять вычисления с временными интервалами:

```
# Увеличим интервал в 10 раз:
interval = pd.to_timedelta('5 days 13:10:00') * 10

print(interval) 
```

```
55 days 11:40:00 
```

Интервал `timedelta64` можно использовать для изменения значений даты и времени. Например, в датафрейме с информацией о продаже автомобилей есть дата публикации объявления `date_posted` и количество дней, которое это объявление было активным `days_listed`. Если добавить к дате публикации количество дней объявления в активном статусе, то можно определить дату снятия объявления. Например, из-за продажи автомобиля.

Для этого необходимо преобразовать `days_listed` в тип `timedelta64` и добавить к дате публикации объявления. Используем для этого метод pandas `.to_timedelta()`:

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
df = pd.read_csv('/datasets/vehicles_us.csv')

# Преобразуем date_posted в тип datetime64:
df['date_posted'] = pd.to_datetime(df['date_posted'])

# Преобразуем days_listed в тип timedelta64:
df['days_listed'] = pd.to_timedelta(df['days_listed'], unit='D')

# Проверим результат:
print(df[['date_posted','days_listed']].head())
print(df[['date_posted','days_listed']].dtypes) 
```

```
  date_posted days_listed
0  2018-06-23     19 days
1  2018-10-19     50 days
2  2019-02-07     79 days
3  2019-03-22      9 days
4  2019-04-02     28 days
date_posted     datetime64[ns]
days_listed    timedelta64[ns]
dtype: object 
```

Для преобразования `days_listed` в тип `timedelta64` мы передали методу `.to_timedelta()` столбец датасета `days_listed` и использовали параметр `unit='D'`, который указал, что данные в столбце `days_listed` представлены в виде количества дней.

Теперь прибавим к дате публикации объявления количество дней:

```
# Найдем дату продажи:
df['date_sale'] = df['date_posted'] + df['days_listed']

# Проверим результат:
print(df[['date_posted','days_listed','date_sale']].head())
print(df[['date_posted','days_listed','date_sale']].dtypes) 
```

```
  date_posted days_listed  date_sale
0  2018-06-23     19 days 2018-07-12
1  2018-10-19     50 days 2018-12-08
2  2019-02-07     79 days 2019-04-27
3  2019-03-22      9 days 2019-03-31
4  2019-04-02     28 days 2019-04-30
date_posted     datetime64[ns]
days_listed    timedelta64[ns]
date_sale       datetime64[ns]
dtype: object 
```

В результате получили новый столбец, содержащий дату продажи автомобиля. При этом тип новых данных — `datetime64`.
