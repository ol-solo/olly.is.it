### Задание 5.

В результате стандартизации данных в одном из столбцов количество явных дубликатов увеличилось. Трудно сказать, с чем могут быть связаны дубликаты. Это может быть ошибка при сборе данных: например, если данные собирались из разных баз, где фигурировали одни и те же клиенты, но в базах были приняты разные стили написания уровня образования. Или дубликаты — это клиенты с полностью одинаковыми данными.

Поскольку в данных нет номера клиента или другой персональной информации, то этот вопрос решить сложно. Однако количество таких дубликатов невысокое (менее 1%), поэтому их можно отфильтровать.

Отфильтруйте явные дубликаты в данных и сохраните результат в исходном датасете.

```python
# Импортируем библиотеку pandas и загрузим данные в переменную df:
import pandas as pd
df = pd.read_csv('/datasets/data.csv')
  
# Изменим регистр значений в столбце education на нижний:
df['education'] = df['education'].str.lower()
  
# Удалим явные дубликаты:
df = df.drop_duplicates()
  
# Подсчитаем общее количество дубликатов:
total_dupl = df.duplicated().sum()
  
# Выведем результат:
print(f'Количество дубликатов в данных: {total_dupl}.')
```

В датасете с информацией о продаже квартир не так много категориальных данных. Одно из них — название населённого пункта, к которому квартира относится. Проверьте возможные аномалии в этом столбце. Для этого:

# Корреляции. Практика

### Шпаргалка

Использовать функцию библиотеки pandas `corr()` для расчёта корреляции можно так:

```
df['колонка переменной №1'].corr(df['колонка переменной №2']) 
```

Для множественных сравнений можно построить матрицу корреляции. Тогда результат будет в формате таблицы `DataFrame`:

```
df[['колонка №1', 'колонка №2', 'колонка №3', 'колонка №4']].corr() 
```

Функция `pd.get_dummies()` библиотеки pandas позволяет преобразовать категориальные данные в числовые, создавая новые столбцы для каждой категории.

Вызов метода `pd.get_dummies()` выглядит следующим образом:

```
pd.get_dummies(df, columns=['имя столбца'], prefix='префикс', dtype='тип данных') 
```

Аргументы:

- **`df`** — датафрейм с категориальными данными;
- **`columns`** — список столбцов, которые необходимо преобразовать;
- **`prefix`** — префикс для новых столбцов;
- **`dtype`** — тип данных новых столбцов (по умолчанию True\False).

---

Интерпретация значений корреляции:

- **Значение 0** означает отсутствие корреляции.
- **Значение 1** означает полную положительную корреляцию — переменные увеличиваются вместе.
- **Значение -1** означает полную отрицательную корреляцию — переменные меняются в противоположных направлениях.

**Важно:**

- **Сила корреляции не говорит о причинно-следственной связи.**
- **Корреляция может быть искажена выбросами.**
- **При интерпретации корреляции важно учитывать контекст.**

```python
# Импортируем библиотеки
import pandas as pd
  
data = pd.read_csv('/datasets/real_estate_data_n_missing.csv')
  
# Функция формирование категории этажности
def get_floor_category(floor, floor_total):
    if floor_total <= 3:
        return 'low_floor'
    elif floor <= floor_total * 0.25:
        return 'low_floor'
    elif floor <= floor_total * 0.75:
        return 'middle_floor'
    else:
        return 'high_floor'
# Формирование колонки с категориями      
data['floor_category'] = data.apply(lambda x: get_floor_category(x['floor'], x['floors_total']), axis=1)
  
# Преобразование категориальных данных
dummies = pd.get_dummies(
    data['floor_category'],
    prefix='category_',
    dtype=int
)
data = pd.concat([data, dummies], axis=1)
  
  
# Расчет корреляции
cols = ['last_price', 'category__high_floor', 'category__low_floor']
data_floor_corr = data[cols].corr().round(2)
  
print(data_floor_corr)
```

# Матрица распределений. Практика

### **Шпаргалка**

Функция **`pairplot()`** библиотеки `seaborn` позволяет построить матрицу распределений для набора данных. Это удобный инструмент визуализации для оценки взаимосвязи между несколькими переменными в наборе данных.

Вызов функции **`pairplot()`** выглядит следующим образом:

```
sns.pairplot(data='датафрейм', 
             kind='тип графика', 
             plot_kws='словарь с параметрами', 
             diag_kws='словарь с параметрами') 
```

Основные аргументы функции:

- **`data`** — датафрейм, содержащий данные для построения матрицы распределений;
- **`kind`** — тип графика, который будет использован для отображения данных. Допустимые значения: `'scatter'` (график рассеивания), `'reg'` (график рассеивания с линией регрессии), `'hist'` (гистограмма), `'kde'` (плотность распределения), `'hex'` (шестиугольная биновая сетка);
- **`plot_kws`** — словарь с параметрами визуализации для графиков рассеивания (например, цвет точек, размер точек, стиль линии и так далее);
- **`diag_kws`** — словарь с параметрами визуализации для гистограмм (например, цвет гистограмм, ширина столбцов, прозрачность).

Аргументы **`plot_kws`** и **`diag_kws`** позволяют настроить параметры визуализации для графиков рассеивания и гистограмм соответственно. Вы можете изменить цвет точек, размер точек, стиль линии, ширину столбцов, прозрачность:

```
# Настройка параметров визуализации для графиков рассеивания:
plot_kws = {'color': 'blue', 'marker': '*', 'linewidth': 0.5}

# Настройка параметров визуализации для гистограмм:
diag_kws = {'color': 'red', 'bins': 10, 'linewidth': 1.5, 'alpha': 0.7} 
```
