## Как объявить и назвать функцию
В Python функцию определяют с помощью ключевого слова `def`.
```
def имя_функции(аргументы):
    тело_функции 
```
![[Pasted image 20260305151106.png]]

**Создаём пустую функцию**
```
def my_first_function():
    #наполнение функции допишем позднее
    #пока используем пустую инструкцию
    pass # когда нужно пропустить блок кода и не выполнять его, можно использовать ключевое слово `pass`
```
Обратите внимание на имя функции — оно создаётся по тем же правилам, что и название переменной. 
Название функции должно быть:
- **Синтаксически корректным.** Имя функции начинается с буквы или знака нижнего подчёркивания `_` и содержит только буквы, цифры и знак нижнего подчёркивания. Нельзя добавлять другие символы, например, `!`, и разделять слова пробелом.
- **Уникальным.** В программе не должно быть двух функций с одинаковыми наименованиями. Если нарушать это правило, код может работать совсем не так, как предполагалось: та функция, которая была определена первой, станет недоступна, и всё будет обрабатывать вторая.
- **Понятным.** Здесь несколько моментов:
    - Обычно используют англоязычные названия функций. Это помогает понимать код сотрудникам из разных стран и просто хороший тон в разработке.
    - Название функции должно описывать то, что она делает — это облегчит чтение кода. Например, если функция находит количество пользователей сайта за день, то хорошим названием будет `get_users_count_day()`.
    - В питоне принята «змеиная» запись, когда слова разделены нижним подчёркиванием и состоят из строчных букв. Без разделителей понять название функции может быть трудно.
- **В меру кратким.** Длинные названия выглядят громоздко, код становится плохо читаемым. Короткие названия облегчают код и упрощают его восприятие.

Напишем функцию, которая будет выводить на экран текст о завершении работы программы.

```
def print_tech_info(): 
	print('Работа программы завершена успешно') 
```

Вызовем функцию и посмотрим, как она работает. Для этого пропишем название функции и скобки. Скобки будут пустыми, так как у нашей функции нет аргументов:

```
#вызов функции
print_tech_info() 

Работа программы завершена успешно 
```

Такую функцию можно использовать, например, в конце долго выполняющейся программы. Кроме указанного текста, можно добавить и другую информацию: длительность выполнения, время окончания работы, место, куда сохранён файл с информацией, и т. д.

## Аргументы

**Аргументы** функции — изменяемые вводные данные, которые функция принимает и использует в своей работе.

Например, функция считает сумму двух чисел — эти числа подаются ей как аргументы.

Аргументами функции могут быть число, строка, список, даже другая функция. Существуют функции, которые не принимают аргументы вообще, — именно про такой пример мы писали в прошлом уроке. Бывают функции, принимающие один или любое другое число аргументов.

Рассмотрим примеры.

Функция `triple_num()` принимает один аргумент и выводит его утроенное значение:

```
def triple_num(num) : 
    print(3*num) 
```

Вызовем функцию с аргументом `2024`:

```
triple_num(num=2024) 
```

Получим:

```
6072 
```

Можно использовать короткую запись:

```
triple_num(2024) 
```

Получим тот же результат:

```
6072 
```

Функция сама не может придумать значение аргумента — его нужно ей передать. Если не указать значение аргумента, то получим ошибку:

```
triple_num()# аргумент не задан
> TypeError: triple_num() missing 1 required positional argument: 'num' 
```

Функция `get_sum()` принимает два аргумента и выводит на экран их сумму:

```
def get_sum(first_num, second_num=12):
    print(first_num + second_num) 
```

Внутри скобок указано `second_num=12`. Здесь `12` — это **значение аргумента по умолчанию**. Это значит, что при вызове функции аргументу `second_num` необязательно присваивать значение. Если присвоим, то функция возьмёт его, а если нет — будет использовать `12`.

Вот как это работает:

```
get_sum(first_num=1) 
```

Вывод:

```
# в этом случае first_num = 1 (мы задали), second_num = 12 (значение по умолчанию)
13 
```

Вот так тоже можно:

```
get_sum(first_num=1, second_num=15) 
```

Вывод:

```
# в этом случае first_num = 1 (мы задали), second_num = 15 (мы задали)
16 
```

Для `first_num` значение по умолчанию не указано, значит, его нужно передать функции. Если не передать этой функции ни одного обязательного параметра, получим ошибку:

```
# first_num не задан, second_num = 12(значение по умолчанию)
get_sum() 
```

Ошибка:

```
TypeError: get_sum() missing 1 required positional argument: 'first_num' 
```

Можно вызывать функцию ещё вот так:

```
get_sum(7) 
```

Получим:

```
# в этом случае first_num = 7 (мы задали), second_num = 12 (значение по умолчанию)
19 
```

Так тоже можно:

```
get_sum(5, 15) 
```

Получим:

```
# в этом случае first_num = 5 (мы задали), second_num = 15 (мы задали)
20 
```

Когда мы не указываем названия аргументов, значения присваиваются аргументам по их порядку. Таким образом, `get_sum(5, 15)` эквивалентно вызову `get_sum(first_num=5, second_num=15)`.

Создадим функцию `calculate_sum(num1, num2, num3)`, выводящую на экран сумму трёх аргументов:

```
def calculate_sum(num1, num2, num3):
    print(num1+num2+num3) 
```

Передадим параметры уже знакомыми способами. Укажем явно названия всех параметров:

```
calculate_sum(num1=2, num2=2, num3=2) 
```

Вывод:

```
# здесь num1=2, num2=2 , num3=2
6 
```

Передадим аргументы коротким способом:

```

calculate_sum(2, 2, 2) 
```

Вывод:

```
# здесь num1=2, num2=2, num3=2
6 
```

А теперь попробуем передать параметры иначе — укажем явно названия некоторых из них:

```
calculate_sum(2, num2=2, num3=2) 
```

Вывод:

```
# и здесь num1=2, num2=2, num3=2
6 
```

А теперь названия других:

```
calculate_sum(2, num2=2, 2) 
```

Получим:

```
SyntaxError: positional argument follows keyword argument 
```

Всё шло хорошо до последнего случая. И это как раз тот нюанс — если у какого-то аргумента указать название явно, то у всех последующих аргументов названия тоже нужно указывать явно.

Подведём итог по аргументам:

- Аргументы функции — изменяемые вводные данные, которые функция принимает и использует в своей работе.
- Аргументы указываются в круглых скобках после имени функции.
- Можно задавать значения по умолчанию.
- Синтаксически корректные записи:

```
get_sum(first_num=1, second_num=15)
get_sum(1, 15) # значения аргументов по порядку
get_sum(first_num=1) # значение second_num задано по умолчанию
get_sum(1) # значение второго аргумента задано по умолчанию 
```

## Тело функции

Тело функции — часть, содержащая блок кода с основным алгоритмом.

После круглых скобок в объявлении функции следует поставить **`:`** и перейти на новую строку — здесь начинается тело функции. На новой строке появляется отступ — это важная часть синтаксиса функции в Python, его не нужно удалять.

Вы уже многое знаете о составляющих функции — давайте решим задачу с помощью собственной функции.

**Задача.** Создать подпись в электронном письме для сотрудников компании ООО «Баг и Фича». Подпись должна выводиться в формате:

С уважением, `Имя и Фамилия сотрудника` — — — — — — — — — `должность`, ООО «Баг и Фича»

**Решение.**

1. Определим функцию:

```
def get_signature(name, position): 
```

Здесь:

`def` — ключевое слово для объявления функции;

`get_signature` _—_ название функции (англ. signature — подпись_);_

`(name, position):` — аргументы функции. Параметры, которые изменяются в задаче, — это имя и должность. Именно эту информацию мы и будем передавать в скобках.

2. Результат работы функции — вывод подписи. В теле функции предстоит использовать встроенную функцию `print()`. Оформим вывод в нужном формате. В первой строке — фиксированная часть: `С уважением,`. После неё идёт изменяемая составляющая — имя и фамилия сотрудника, их нужно взять из аргумента `name`, который передаётся функции. Помним про отступ и прописываем этот шаг в тело функции:

```
print('С уважением,', name) 
```

Вторая строка — набор символов `— — — — — — — — —`. Здесь нет изменяемых параметров, так что следующий шаг в теле функции прост:

```
print('— — — — — — — — —') 
```

Следующая строка, как и первая, содержит фиксированную и изменяемую части. Оформим её по тому же принципу, что и первую:

```
print(position,', ООО «Баг и Фича»', sep = '') 
```

Фрагмент `sep = ''` помогает избежать лишнего пробела.

3. Соберём всё вместе:

```
# объявление функции get_signature с аргументами name, position **
def get_signature(name, position): 
# вывод по образцу
# name - параметр из аргументов функции
        print('С уважением,', name) 
        print('— — — — — — — — —')
# position- параметр из аргументов функции
        print(position,', ООО «Баг и Фича»', sep = '') 
```

Готово! Теперь можно вызвать `get_signature()` и передать ей значения для аргументов `name` и `position`.

4. Рассмотрим примеры и сформируем подписи для двух сотрудников:
5. Павел Александров, должность — ведущий аналитик;
6. Александра Павлова, должность — инженер данных.

Передаём аргументам соответствующие значения:

```
get_signature(name = 'Павел Александров', position = 'Ведущий аналитик') 
```

```
С уважением, Павел Александров
— — — — — — — — —
Ведущий аналитик, ООО «Баг и Фича» 
```

```

get_signature(name = 'Александра Павлова', position = 'Инженер данных') 
```

```
С уважением, Александра Павлова
— — — — — — — — —
Инженер данных, ООО «Баг и Фича» 
```

Обратите внимание: для разных передаваемых параметров заданный формат вывода подписи сохраняется, меняются только имя и должность, которые как раз передавались аргументами функции.

Задача решена, и теперь рабочая переписка станет комфортнее!

# Вывод результатов. return

Итак, в предыдущих ~~сериях~~ уроках вы разобрались с синтаксисом объявления функции, правилами её названия, аргументами и телом функции. В этом уроке познакомитесь с последней составляющей — оператором `return`.

После оператора `return` указывают то, что функция выдаст после окончания работы. Это позволяет использовать результат функции в других частях программы.

Оператор `return` используют, когда нужно сохранить результат работы функции — в виде переменной, списка или др. Например:

```
def multiply(num1, num2):
    # перемножаем два числа
    result = num1 * num2 
    # return возвращает значение result
    return result   
    
# res_1 присваиваем результат работы multiply()
res_1 = multiply(2,5) 
# то же для res_2
res_2 = multiply(4,2) 

# выведем res_1, чтобы посмотреть, что хранится в переменной
print(res_1) 
# то же для res_2
print(res_2) 
```

Получим:

```
# res_1
10 
# res_2
8  
```

Можно провести аналогию работы функции и блендера. На вход функции подают аргументы — как в блендер складывают ингредиенты: бананы, молоко, ягоды. Функция-блендер обрабатывает всё внутри. В `return` определим результат работы блендера — коктейль.
![[Pasted image 20260305152333.png]]
Оператор`return` завершает работу алгоритма, и всё, что написано после, Python не отрабатывает. Если коктейль из блендера перелили в стакан, то у блендера задач не осталось — логично!

Зависит от того, что делает функция и нужно ли сохранить результаты её работы. Возможные варианты:

- У функции вообще нет `return`. Например, такова функция `print()` — она просто выводит на экран информацию, не надо сохранять или использовать в коде результаты её работы.
- Один `return` — частый случай. Функция что-то посчитала и вернула результат. Например, функция `sqrt()` из библиотеки NumPy извлекает квадратный корень из числа и возвращает, что получилось — результат можно использовать дальше в программе.
- Несколько return — тоже «штатная» ситуация. Рассмотрим её на примере задачи.

**Задача.** В сервисе доставки еды запускают акцию. Условия такие: если сумма в чеке с учётом доставки превышает 1000 денег, то клиенту дают скидку 10%. Создадим функцию, которая:

- принимает сумму из ресторана и стоимость доставки;
- возвращает итоговую стоимость заказа с учётом условий акции.

1. **Разберём условие.** Аргументами функций будут параметры, которые меняются от заказа к заказу — это сумма чека из ресторана и стоимость доставки. По условию акции нужно посчитать сумму этих аргументов и сравнить её с 1000. Дальше возможны два варианта:
    - Если полная стоимость больше 1000, нужно применить скидку 10% к полной стоимости — то есть вернуть 90% от неё.  
        Для этого нужно сначала разделить сумму на 100 — так мы узнаем, сколько составляет 1 процент. А затем умножить полученное значение на 90.  
        Запишем формулу и упростим её:  
          
        стоимость100⋅90=стоимость⋅0,9.100стоимость​⋅90=стоимость⋅0,9.
    - Если полная стоимость меньше или равна 1000, скидка не начисляется. В этом случае нужно вернуть просто общую стоимость.
        
        Работать с несколькими вариантами поможет условная конструкция `if`-`else`, вернуть значение — оператор `return`.
        
2. Определим `amount_estimator`:

```
def amount_estimator(sum_cafe, sum_delivery): 
```

Здесь:

- `def` — ключевое слово для объявления функции;
- `amount_estimator` — название функции;
- `(sum_cafe, sum_delivery):` — аргументы функции: стоимость заказа в ресторане и стоимость доставки соответственно.

3. Напишем тело функции. Будем действовать так, как определили в шаге 1. Сначала найдём полную стоимость:

```
total_sum = sum_cafe + sum_delivery 
```

Теперь полученное значение нужно сравнить с 1000 и реализовать два исхода.

Используем оператор `if` для первого случая: если `total_sum` больше 1000. `if total_sum>1000:` Пропишем, что делать, если условие выполняется. В этом случае надо применить скидку. Формулу мы получили выше, переведём её в код: `total_sum*0.9`. Это значение вернём как результат функции, для этого прописываем оператор `return`: `return total_sum*0.9`

```
total_sum = sum_cafe + sum_delivery
if total_sum>1000:
    return total_sum*0.9 
```

Для работы со вторым случаем (`total_sum <= 1000`) достаточно использовать оператор `else`, так как других возможных случаев не может быть. Здесь просто вернём это значение: `return total_sum`.

Получаем такой фрагмент кода с условной конструкцией:

```
if total_sum>1000:
  return total_sum*0.9
else:
  return total_sum
 
```

4. Посмотрим на весь код целиком:

```
# объявление функции amount_estimator 
# с аргументами sum1, sum2
def amount_estimator(sum1, sum2):
    # вычисление полной стоимости заказа
    total_sum = sum1 + sum2
    # сравнение с 1000
    # выбор случая, когда сумма больше 1000 
    if total_sum > 1000:        
        # возвращение значения согласно условию 
        # акции: с начисленной скидкой
        return total_sum*0.9       
        # сравнение с 1000, выбор
        # всех оставшихся случаев
    else:       
        # возвращение значения согласно условию акции
        return total_sum            
```

Функция готова! Протестируем её на заказах трёх клиентов:

```

amount_estimator(sum1=620, sum2=100) 
```

```
720 
#полная сумма заказа 720 -  скидки нет, так как по условию 
# она предоставляется на заказы больше 1000 
```

```
# так как по условию она предоставляется на заказы больше 1000 
```

Несколько операторов `return` в одной функции позволили реализовать ветвление в алгоритме. В этом коде две ветки — поэтому и операторов `return` — два. Вызов одной и той же функции всегда возвращает одно корректное значение.

# Создание функции с несколькими аргументами и встроенными методами

Как вы уже знаете из предыдущих уроков, в Python функции — это способ организации и повторного использования кода. Напомним, встроенные методы — это функции, уже включенные в Python, например, `len()` или `print()`. Пользовательские функции — это функции, которые создаются самим пользователем для выполнения разных задач.

Функции в программировании и аналитике можно сравнить с рецептом приготовления блюда. Если вам нужно несколько раз приготовить одно и то же блюдо, вы не изучаете рецепт каждый раз заново. Вместо этого вы создаёте рецепт, и следуете ему каждый раз, когда готовите. Функции работают так же: они помогают нам избежать повторения кода и делают его более читабельным и организованным.

А чтобы улучшить код, можно комбинировать встроенные методы с пользовательскими функциями. Такое комбинирование в Python может значительно повысить гибкость и эффективность вашего кода подобно тому, как в повседневной жизни мы используем сочетание инструментов и индивидуальных навыков для решения задач. Как например, если вы используете готовое тесто (встроенный метод) и добавляете свои ингредиенты (пользовательская функция) для создания уникальной пиццы.

В этом уроке вы узнайте, как комбинировать встроенные методы и пользовательские функции.

Для этого представим, что вы планируете организовать мероприятие, будь то праздник, вечеринка или корпоратив.
Для начала напишем простую функцию `create_invitation` с одним аргументом, которую мы используем для создания индивидуальных приглашений. Эта функция принимает имя гостя и возвращает строку с приглашением на мероприятие, содержащую это имя.

Для создания такой функции можно использовать f-строку, которая позволяет вставлять переменные и выражения непосредственно в текст. Чтобы её применить, нужно поставить букву f перед началом кавычек, а название аргумента обрамить фигурными скобками с каждой стороны.

```
def create_invitation(name):
    return f"Уважаемый(ая) {name}, приглашаем Вас на наше мероприятие!"

# Примеры вызова функции
print(create_invitation("Анна"))
print(create_invitation("Иван"))
print(create_invitation("Ольга")) 
```

Важные составляющие любой пользовательской функции — аргументы и возвращаемые значения. Напомним, аргумент — это значение, которое передаётся функции при её вызове. Эти значения перечисляются в круглых скобках и разделяются запятыми. Сравнивая пользовательскую функцию с калькулятором можно сказать что, аргументы — это числа, которые вы вводите (например, 2 и 3), а возвращаемое значение — это результат операции (например, сумма этих чисел).

Пользовательские функции в Python могут принимать разное количество аргументов и возвращать множество значений, перечисленных через запятую. Вместе они формируются в кортеж.

Для успешного проведения мероприятия нам необходимо подобрать разные продукты. Вдруг у кого-то аллергия? Начнём с выбора фруктов. Вам понадобятся яблоки, бананы и апельсины. Напишем такую функцию `fruit_count`, которая принимает количество яблок, бананов и апельсинов, а затем возвращает общее количество фруктов.

```
def fruit_count(apples, bananas, oranges):
    return apples + bananas + oranges 
```

Используем нашу функцию. Каждое число — это количество соответствующих фруктов. Возьмем 12 яблок, 13 бананов и 9 апельсинов.

```
print(fruit_count(12, 13, 9)) 
```

Фрукты могут различаться по количеству и виду (а не только яблоки, бананы и апельсины). В связи с этим правильнее записать их в виде списка list.

```
fruits = [12, 13, 9] 
```

Напомним, что если вы хотите просуммировать элементы внутри списка, вы можете использовать метод `sum(fruits)`. В свою очередь, если вы захотите узнать, каких фруктов у вас больше всего, вы можете воспользоваться методом `max(fruits)`. В вашем случае такая функция выдаст 13. И наоборот, метод `min(fruits)` покажет, каких фруктов у вас меньше всего. Тогда функция выдаст 9.

Вам нужно учесть количество продуктов, чтобы обеспечить достаточное и разнообразное угощение для гостей. После покупки фруктов, вычислим, сколько каждого вида останется, если каждый участник съест по одному фрукту. Предположим, что у нас будет восемь гостей:

```
def remaining_fruits(participants):
    # Количество каждого вида фруктов на начало мероприятия
    apples = 12
    bananas = 13
    oranges = 9

    # Вычисляем количество оставшихся фруктов
    remaining_apples = apples - participants
    remaining_bananas = bananas - participants
    remaining_oranges = oranges - participants

    # Печатаем результаты с использованием f-строк
    print(f"Оставшиеся яблоки: {remaining_apples}")
    print(f"Оставшиеся бананы: {remaining_bananas}")
    print(f"Оставшиеся апельсины: {remaining_oranges}")

# Вывод функции
remaining_fruits(8) 
```

# Создание функций с циклами и условиями

Часто возникает ситуация, когда в рамках одной пользовательской функции нужно многократно выполнять одно или несколько действий. Например:

- помесячно рассчитать финансовый показатель, чтобы построить график его динамики;
- распределить клиентов по сегментам — для каждого клиента определить, в какой сегмент он попадает по заранее определённым условиям;
- перебрать все маркетинговые кампании, чтобы определить более и менее эффективные;
- проверить каждый склад и определить, достаточно ли на в нём запасов для территории, которую он обслуживает.

Такая обработка данных нетипична, и конкретной функции, которая сразу решит такую задачу для всего массива данных, часто нет. В таких случаях помогают циклы. Напомним: циклы — это конструкции, которые используются для повторения одного или нескольких действий множество раз. Ранее в курсе вы познакомились с двумя основными типами циклов: `for` и `while`. Освежить в памяти их синтасис можно в теме «Циклы и их организация».
Ещё в пользовательских функциях можно использовать условия — когда нужно выполнить разные блоки кода в зависимости от того, истинно ли заданное условие.

Более того, циклы и условия можно комбинировать! Например, функция для подсчёта среднего значения списка чисел может:

- с помощью условия проверять перед расчётом, не пуст ли список;
- а затем с помощью цикла перебирать все числа.
Обратимся к списку стоимостей покупок из магазина электронной техники.

```
purchases = [7000, 14000, 9000, 70000, 3000] 
```

Рассмотрим функцию `price_range()` , которая принимает список цен, минимальную и максимальную границы и возвращает список цен, которые попадают в заданный диапазон.

```
def price_range(prices, min_price, max_price):
    # Создаём пустой список для хранения результатов
    result = []  
    # Итерируем по каждому элементу в списке prices
    for price in prices:  
        # Проверяем, попадает ли цена в заданный диапазон
        if min_price <= price <= max_price:  
            # Если да, добавляем цену в список результатов
            result.append(price)  
    return result 
```

Используем данную функцию на нашем списке `purchases`. Зададим нижнюю границу равной 5 000, а верхнюю — 15 000.

```
purchases = [7000, 14000, 9000, 70000, 3000]

def price_range(prices, min_price, max_price):
    # Создаём пустой список для хранения результатов
    result = []  
    # Итерируем по каждому элементу в списке prices
    for price in prices:  
        # Проверяем, попадает ли цена в заданный диапазон
        if min_price <= price <= max_price:  
            # Если да, добавляем цену в список результатов
            result.append(price)  
    return result

print(price_range(purchases, 5000, 15000)) 
```

## Списковые включения

Более элегантно решать подобные задачи и создавать новые списки на основе существующих помогают **списковые включения** (list comprehensions). Они позволяют комбинировать цикл и условие — генерировать новый список, применяя выражение к каждому элементу итерируемого объекта (например, списка), и при необходимости фильтровать элементы с помощью условного выражения.

**Списковое включение** — цикл, который включён внутрь определения списка.

Сначала возьмём обычный цикл и создадим с его помощью список квадратов чисел:

```
numbers = [1, 2, 3, 4, 5]
squared_numbers = []

for x in numbers:
    squared_numbers.append(x**2)
print(squared_numbers) 
```

Вывод:

```
[1, 4, 9, 16, 25] 
```

Задача простая, а код немного громоздкий. Списковое включение поможет с этим справиться.

Простой пример спискового включения выглядит так:

```
numbers = [2, 3, 4, 5, 6]
squared_numbers = [x**2 for x in numbers]
print(squared_numbers) 
```

Вывод:

```
[4, 9, 16, 25, 36] 
```

В этом примере `squared_numbers` содержит квадраты чисел из списка `numbers`. Здесь `[x**2 for x in numbers]` — это списковое включение. Оно итерирует по каждому элементу `x` из списка `numbers` и генерирует новый список, содержащий квадраты этих чисел. Списковое включение может итерироваться также по генераторам `range` и другим итерируемым объектам.

Чтобы дополнительно фильтровать результат, нужно добавить в конце спискового включения `if` и после него логическое выражение, которое будет проверяться для каждого `х`.

Например, получим квадраты только чётных чисел от 1 до 10:

```
print([x**2 for x in range(1, 11) if x % 2 == 0]) 
```

```
[4, 16, 36, 64, 100] 
```

Коротко и элегантно!

Рассмотрим пример посложнее и запишем функцию `price_range()`, о которой мы говорили выше, с помощью списковых включений.

С помощью списка включений создадим новый список `prices`, который содержит только те цены, которые попадают в диапазон между `min_price` и `max_price`.

```
purchases = [7000, 14000, 9000, 70000, 3000]

def price_range(prices, min_price, max_price):
    return [price for price in prices if min_price <= price <= max_price]
print(price_range(purchases, 5000, 15000)) 
```

Вывод:

```
[7000, 14000, 9000] 
```

Что происходит в коде:

1. Сначала функция перебирает все элементы списка `prices`.
2. Затем для каждой цены (`price`) из списка `prices` проверяется, удовлетворяет ли она условию `min_price <= price <= max_price`. Это условие гарантирует, что в результат попадут только те цены, которые находятся в заданном диапазоне.
3. Если цена удовлетворяет условию, она добавляется в новый список.

Таким образом, результатом работы функции является список, содержащий только те цены, которые находятся в заданном диапазоне.

Результатом будет тот же список `[7000, 14000, 9000]`.
