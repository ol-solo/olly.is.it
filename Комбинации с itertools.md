## Объект итератор и цикл `for`
При создании цикла мы использовали переменные итераторы, поочерёдно перебирающие данные в списке, строке или в других коллекциях, в том числе созданных различными функциями, например `range()`, `zip()` или `enumerate()`. Немного подробнее остановимся на том, как **Python** перебирает объекты в цикле.

В **Python** существуют такие объекты-**итераторы**, помогающие циклу последовательно перебирать все элементы. Именно из итератора берётся необходимое значение из перебираемого списка:
```python
# Создадим цикл, перебирающий все значения в списке trees:
trees = ['Ольха','Берёза','Ясень','Кипарис']

for tree in trees:
    print(tree) 

Ольха
Берёза
Ясень
Кипарис 
```

Результат работы цикла простой: на каждой новой строке отображён элемент списка `trees`. В этом случае цикл `for` взаимодействует не напрямую со списком `trees`, а с объектом-итератором, созданным на основании списка. Именно объект-итератор на каждом шаге цикла выбирает необходимое для перебора значение из списка `trees` и передаёт его в переменную-итератор `tree`. Так происходит, пока все элементы не будут перебраны.

## Что у итератора под капотом
Ошибка `StopIteration` помогает определить циклу `for`, что количество элементов для перебора закончилось и следует завершить свою работу.
```python
# Продолжим перебирать значения списка trees:
print(next(trees_iter)) 
```

Если значений в списке больше нет, то при следующем обращении функция `next()` выведет ошибку `StopIteration`:
```python
**StopIteration** 
```

В коде выше мы четыре раза подряд обратились к объекту-итератору, используя функцию `next()`, и вывели на экран каждое последующее значение списка `trees`. Особенность работы объекта-итератора в том, что он не загружает все объекты исходной коллекции в память, а выводит их последовательно по одному, по мере обращения. Объект-итератор всегда знает, какое значение должно быть выведено при следующем обращении к нему. Эту информацию он хранит в памяти.
```
Ольха
Берёза
Ясень
Кипарис 
```

```
# Создадим список элементов:
trees = ['Ольха','Берёза','Ясень','Кипарис']

# Создадим объект-итератор:
trees_iter = iter(trees)

# Последовательно выведем на экран первые четыре элемента:
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter)) 
```

Чтобы последовательно перебирать элементы, цикл `for` на каждом шаге итерации использует ещё одну функцию **Python** — `next()`. Эта функция при каждом новом обращении к объекту-итератору возвращает следующий итерируемый элемент.

Например:

Функция `iter()` создала на основе списка `trees` объект-итератор, а именно `list_iterator`, то есть итератор списка. Этот объект хранит в себе правила перебора: поскольку это список, то последовательно будет перебираться каждый элемент списка от начала и до конца. И когда все элементы закончатся — перебор остановится. Это мы могли наблюдать в самом первом примере цикла `for`.

```

```

```
iter(trees) 
```

В примере выше цикл `for` не получил объект-итератор и прекратил работу ошибкой, указывающую на то, что объект типа `int`не может быть итерирован и разбит на элементы.

Например, если попробуем перебрать объект типа `int`, то получим ошибку:

```

```

```

for num in 12:
    print(num) 
```

При инициировании цикла `for`, цикл обращается к перебираемому объекту (например, списку `trees`) и запрашивает объект-итератор, который поможет циклу осуществить перебор всех значений исходного списка. Если цикл `for` получает объект-итератор, то происходит его дальнейшая работа. Но если цикл не получит объект-итератор, то код завершается ошибкой.

## Какие ещё функции возвращают итератор?

Объект-итератор возвращает не только функция `iter()`. Таких функций в среде программирования **Python** много. Они поддерживают использование функций `iter()` и `next()`, в них прописаны условия итерации объектов. Однако эти вопросы затрагивают особенности программирования на **Python** и выходят за рамки тех основных знаний, которые необходимы аналитику для работы. Поэтому разберём пример работы функций, возвращающих объект-итератор на примере уже знакомых функций.

В прошлых уроках вы работали в цикле с функциями `enumerate()` и `zip()`. Они также возвращают объекты, которые могут быть итерированы. Например, используем `enumerate()` для создания объекта-итератора и применим функцию `next()`:

```
# Создадим объект-итератор с помощью `enumerate()`:
trees_iter = enumerate(trees)

# Последовательно выведем на экран 4 первых элемента:
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter)) 
```

```
(0, 'Ольха')
(1, 'Берёза')
(2, 'Ясень')
(3, 'Кипарис') 
```

Функция `next()` при каждом новом обращении к объекту-итератору выводила значения сформированного `enumerate()` кортежа. Если продолжить дальше использование функции `next(trees_iter)`, то итератор вернёт ошибку `StopIteration`.

Помимо функций `iter()`, `enumerate()` и `zip()`, в **Python** есть другие функции и модули, создающие объект-итератор из исходного множества значений, одну из которых мы рассмотрим далее — модуль `itertools`. Ещё язык **Python** позволяет создавать собственные объекты-итераторы с необходимыми правилами итерации объектов, но изучение этой темы выходит за рамки курса, поскольку аналитик данных работает уже с готовыми решениями встроенных функций **Python**.

## Модуль `itertools`
Модуль `itertools` (от английского «инструменты для создания итератора») представляет собой встроенный модуль **Python**, содержащий функции для создания различных объектов итераторов. Используя `itertools`, можно создавать такие типы итераторов:

- конечные;
- бесконечные;
- комбинаторные.

Конечные итераторы перебирают все объекты исходного множества и завершают свою работу, когда количество элементов множества закончилось. Например, это итераторы списка, или функций `enumerate()` и `zip()`. В модуле `itertools` конечные итераторы позволяют по-разному объединять несколько множеств для итерации или использовать условия для выбора итерируемых объектов из списка.

В прошлом уроке мы рассмотрели один из таких итераторов — `zip_longest`, на остальных конечных итераторах в рамках урока мы не будем останавливаться, так как по своей логике они дублируют действия со списками и другими коллекциями. Остановимся детальнее на бесконечных и комбинаторных итераторах.

### Бесконечные итераторы

В предыдущих примерах все итераторы, используемые в цикле `for`, имели определённое количество элементов и были конечными. Выше мы рассматривали, что если итерируемых значений больше нет, то итератор выведет ошибку исключение `StopIteration`.

Бесконечный итератор будет генерировать элементы для итерации до тех пор, пока цикл не остановится в явном виде, например, при помощи условного выражения и команды `break`, пока не будет достигнуто определённое условие итерации.

Бесконечный итератор может быть полезным в работе аналитика при создании цикла, когда нет чёткого представления об итоговом количестве элементов в перебираемом множестве значений или существует необходимость обработки поступающих новых данных в режиме реального времени. В этом бесконечные итераторы по своему смыслу похожи на цикл `while`.

Задачей, которая может требовать такой бесконечной генерации, может быть генерация уникальных идентификаторов, присваиваемых новым пользователям или заказам. Также бесконечный итератор может пригодиться при распределении задач между группами или сегментами — например, распределение нового заказа в службе доставки между активными курьерами.

Предположим, в службе доставки есть три курьера и нужно каждый новый заказ последовательно назначать каждому из этих заранее известных курьеров по очереди. В этом случае новые объекты — заказы — поступают из коллекции неизвестного размера, а категории — курьеры — присваиваются им циклично, по заранее заданному правилу, реализованному в виде итератора.

Ещё пример — перебор отображаемых рекламных баннеров в приложении, когда они последовательно сменяются через определённый промежуток времени. Такую задачу также можно осуществить при помощи бесконечных итераторов. Бесконечные итераторы также используются для непрерывной генерации данных при тестировании моделей машинного обучения или проверке работы баз данных, программного обеспечения.

В основном бесконечные итераторы используются больше в программировании, чем в анализе данных. Однако аналитику данных на **Python** полезно понимать общий принцип работы таких итераторов и быть знакомым с инструментами, помогающими их создать.

В модуле `itertools` существуют следующие функции, генерирующие бесконечные итераторы:

- `count()`
- `cycle()`
- `repeat()`

**Функция `count()`** создаёт объект-итератор с бесконечной последовательностью чисел, с определённой последовательностью и началом. Синтаксис функции следующий `itertools.count(start, step)`, где `start` начальное значение, а `step` — интервал до следующего значения.

```
# Создадим итератор, генерирующий бесконечную 
# последовательность чисел от 0 с шагом 5:

# Импорт библиотеки itertools:
import itertools

# Создадим цикл, использующий бесконечный итератор count:
for i in itertools.count(0, 5):
    if i > 50:
        break
    else:
        print(i) 
```

```
0
5
10
15
20
25
30
35
40
45
50 
```

В примере выше функция `itertools.count()` создала бесконечный итератор, начинающийся со значения `0` и шагом `5`. В результате цикл `for` получился бесконечным и, чтобы его остановить, было использовано условие, останавливающее цикл через команду `break`, если перебираемое значение выше `50`. В дальнейших примерах будет использован аналогичный подход для выхода из цикла.

В качестве шага можно использовать не только целые значения, но и вещественные числа, например, сделаем шаг не `5`, а `0.5`:

```
# Создадим цикл, использующий бесконечный итератор count:
for i in itertools.count(0, .5):
    if i > 2:
        break
    else:
        print(i) 
```

```
0
0.5
1.0
1.5
2.0 
```

**Функция `cycle()`** создаёт объект-итератор с бесконечным перебором всех значений в итерируемом множестве (например, списке). Функция имеет простой синтаксис и принимает значение множества для перебора — `itertools.cycle(<коллекция>)`:

```
# Создадим список:
trees = ['Ольха','Берёза','Ясень','Кипарис']

# Создадим цикл, использующий бесконечный итератор cycle:
count = 0
for i in itertools.cycle(trees):
    if count > 6:
        break
    else:
        print(f'Итерация цикла: {count}, Элемент списка: {i}')
    count += 1 
```

```
Итерация цикла: 0, Элемент списка: Ольха
Итерация цикла: 1, Элемент списка: Берёза
Итерация цикла: 2, Элемент списка: Ясень
Итерация цикла: 3, Элемент списка: Кипарис
Итерация цикла: 4, Элемент списка: Ольха
Итерация цикла: 5, Элемент списка: Берёза
Итерация цикла: 6, Элемент списка: Ясень
 
```

В нашем примере итератор на каждом шаге цикла последовательно выдавал значения из списка `['Ольха','Берёза','Ясень','Кипарис']`. Когда значения закончились, итератор начал выдавать элементы списка с начала.

Такой подход к итерации объектов списка можно использовать не только для распределения заказов между курьерами, но и при распределении клиентов на несколько групп в последовательности их регистрации, например, подобное распределение пользователей между группами можно использовать при проведении А/В-теста:

```
# Создадим список групп:
groups = ['A','B','C']

# Создадим итератор, имитирующий генерацию id пользователей:
user_id = itertools.count(100000, 1)

# Создадим пустой список, в который будем добавлять информацию:
user_grop_list = []

# Создадим цикл, использующий бесконечный итератор cycle:
count = 0
for group in itertools.cycle(groups):
    # Для примера проведём 10 итераций:
    if count > 10:
        break
    else:
        user_grop_list.append(f'{next(user_id)}_{group}')
    count += 1
    
# Проверим результат:
print(user_grop_list) 
```

```
['100000_A', '100001_B', '100002_C', '100003_A', '100004_B', '100005_C', '100006_A', '100007_B', '100008_C', '100009_A', '100010_B'] 
```

В примере мы использовали итератор `itertools.count()` для имитации генерации бесконечного списка `user_id` с id пользователя. Затем в цикле использовали итератор `itertools.cycle()`, бесконечно перебирающий названия групп в списке `groups`. А в самом цикле к каждому новому `user_id` добавляли суффикс с названием группы из списка `groups`. Обратите внимание, что для итерации по `user_id` использовали функцию `next()`.

**Функция `repeat()`** создаёт объект-итератор, возвращающий один-единственный элемент бесконечное количество раз, если не было указано точное количество итераций. Синтаксис функции следующий `itertools.repeat(object, times)`, где `object` возвращаемое значение, а `times` — количество итераций, по умолчанию — бесконечное значение.

Например, компания решила дать дополнительную скидку в 15% первым 7 покупателям. Id покупателей поступают по мере покупки. Тогда общий цикл будет таким:

```
# Создадим итератор, имитирующий генерацию id пользователей:
user_id = itertools.count(100000, 1)

# Создадим пустой список, в который будем добавлять информацию:
user_action_list = []

# Создадим цикл, использующий бесконечный итератор repeat с количеством итераций 7:
for action in itertools.repeat('15%_Sale', 7):
    user_action_list.append(f'{next(user_id)}_{action}')
    
# Проверим результат:
print(user_action_list) 
```

```
['100000_15%_Sale', '100001_15%_Sale', '100002_15%_Sale', '100003_15%_Sale', '100004_15%_Sale', '100005_15%_Sale', '100006_15%_Sale'] 
```

В результате первым семи клиентам был добавлен префикс с названием скидки.

### Комбинаторные итераторы

Комбинаторные итераторы создают все возможные комбинации элементов из одного или нескольких итерируемых объектов. По своему функционалу они считаются конечными. Они полезны при решении задач, в которых нужно изучить все возможные комбинации данных, найти декартово произведение множеств или изучить возможные последовательности элементов в одном множестве значений.

Декартово произведение множеств — это множество, содержащее все возможные упорядоченные значения элементов исходных множеств. Например, есть два множества значений: множество `A`, содержащее значения `('X', 'Y')`, и второе множество `B`, содержащее значения `(0, 1)`, тогда декартово произведение множеств будет содержать такие варианты пересечения всех значений, как `('X0', 'X1', 'Y0', 'Y1')`.

Один из самых распространённых примеров декартова множества — все возможные результаты двух последовательных бросков игрального кубика, когда важна последовательность значений в получаемых вариантах множеств.
![[Pasted image 20260304175256.png]]

Декартово произведение множеств можно использовать в аналитике для поиска всех возможных сочетаний двух или нескольких множеств параметров. Например, можно перебрать все возможные сочетания товаров из двух категорий, чтобы оценить их потенциал совместных продаж, с учётом последовательности предложения.

В **Python** для создания объекта-итератора, содержащего все возможные варианты декартова произведения нескольких множеств, можно использовать функцию `product()` из модуля `itertools`.

**Функция `product()`** создаёт объект-итератор, определяющий все возможные последовательности элементов двух множеств, реализуя декартово произведение множеств. Функция имеет следующий синтаксис: `itertools.product(*iterables, repeat=1)`, где на месте `*iterables` через запятую перечислены все коллекции (списки, кортежи, множества или другие итерируемые объекты), а параметр `repeat` указывает, какое количество раз возможно задействовать исходные множества. По умолчанию это значение равно 1.

Например, найдём декартово произведение множеств `A` и `B`:

```
# Найдём декартово произведение множеств `A` и `B`:
A = ['A', 'B']
B = [0, 1]

for i in itertools.product(A, B):
    print(i)   
```

```
('A', 0)
('A', 1)
('B', 0)
('B', 1) 
```

Или посмотрим на возможные результаты по последовательному броску двух игровых кубиков:

```
# Найдём возможные результаты по последовательному броску двух игровых кубиков:
A = [1, 2, 3, 4, 5, 6]

for i, j in itertools.product(A, repeat=2):
    print(f'{i} : {j}')  
```

```
1 : 1
1 : 2
1 : 3

...

6 : 4
6 : 5
6 : 6 
```

В примере выше, поскольку два кубика идентичны, то множество значений от 1 до 6 мы использовали один раз, но передали функции `product()` параметр `repeat=2`.

Помимо декартова произведения множеств, существуют другие виды перестановок и перебора элементов во множествах. Рассмотрим наиболее распространённые из них и которые можно решить при помощи модуля `itertools` и функций:

- `permutations()`
- `combinations()`
- `combinations_with_replacement()`

Разберём по порядку каждую функцию. Обратите внимание, что в отличие от `product()`, который может работать с несколькими множествами, эти функции всегда работают с элементами одного множества.

**Функция `permutations()`** создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов одного множества с учётом их последовательности и уникальности. В одной последовательности не может быть несколько одних и тех же элементов множества. Функция имеет следующий синтаксис: `itertools.permutations(iterable, r=None)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств. По умолчанию этот параметр = `None` и соответствует количеству элементов во множестве.

Допустим, есть множество значений, например `['A', 'B', 'C']`, функция `permutations()` найдёт все возможные варианты сочетания трёх элементов между собой, то есть `['A', 'B', 'C']`, `['A', 'С', 'В']`, `['В', 'А', 'С']` и так далее. При этом порядок элементов в образованных последовательностях имеет значение: `['A', 'B', 'C']` и `['A', 'С', 'В']` — это разные последовательности.

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C']

for i in itertools.permutations(data):
    print(i)   
```

```
('A', 'B', 'C')
('A', 'C', 'B')
('B', 'A', 'C')
('B', 'C', 'A')
('C', 'A', 'B')
('C', 'B', 'A') 
```

В результате создано шесть уникальных вариантов сочетания элементов исходного множества `['A', 'B', 'C']`, каждый из которых содержит по три элемента. Количество элементов в итоговых последовательностях было задано по умолчанию, исходя из количества элементов в исходном множестве. Мы можем поменять это значение, например, уменьшив размерность, и указать два значения в каждом множестве, передав переменную `r=2`:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C']

for i in itertools.permutations(data, r=2):
    print(i)   
```

```
('A', 'B')
('A', 'C')
('B', 'A')
('B', 'C')
('C', 'A')
('C', 'B') 
```

Обратите внимание, что `r` не может быть больше, чем общее количество элементов в исходном множестве, поскольку значения в получаемых вариантах не должны дублироваться.

Такой перебор возможных последовательностей значений множества полезен для решения задач по выбору последовательности действий, продажи продуктов, запуска маркетинговых событий и т. д. Например, можно подобрать оптимальный маршрут для такси или курьерской доставки. Предположим, что есть три адреса доставки — с помощью `permutations()` можно получить все последовательности адресов. Используя эти последовательности и `api`геосервисов, можно рассчитать длину маршрута и время и выбрать самый оптимальный вариант.

---

**Функция `combinations()`** создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов множества заданной длины без дублирования значений и учёта их последовательности, в порядке их расположения в исходном множестве. Функция имеет следующий синтаксис: `itertools.combinations(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан. Это значение не может быть больше, чем общее количество элементов в исходном множестве.

Функция `combinations()` отличается от `permutations()` тем, что не учитывает последовательность элементов: ей важно их значение. Множества `['A', 'B', 'C']` и `['A', 'С', 'В']` будут рассмотрены как одно множество `['A', 'B', 'C']`.

Рассмотрим на примере, выберем все уникальные последовательности элементов множества `['A', 'B', 'C', 'D', 'E']`:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C', 'D', 'E']

for i in itertools.combinations(data, r=3):
    print(i)   
```

```
('A', 'B', 'C')
('A', 'B', 'D')
('A', 'B', 'E')
('A', 'C', 'D')
('A', 'C', 'E')
('A', 'D', 'E')
('B', 'C', 'D')
('B', 'C', 'E')
('B', 'D', 'E')
('C', 'D', 'E') 
```

В результате найдены все уникальные сочетания исходного множества, содержащие три различных элемента. При этом элементы в последовательностях указаны в порядке их сортировки в исходном множестве.

Такой перебор элементов множества и поиск уникальных сочетаний полезен при поиске всех возможных сочетаний элементов. Представим, что есть несколько продуктов: `Молоко`, `Сметана`, `Сгущёнка` и `Хрустящий хлеб` и поставлена задача собрать все возможные варианты из трёх продуктов.

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['Сгущёнка', 'Сметана', 'Молоко', 'Хрустящий хлеб']

for i in itertools.combinations(data, r=3):
    print(i)   
```

```
('Сгущёнка', 'Сметана', 'Молоко')
('Сгущёнка', 'Сметана', 'Хрустящий хлеб')
('Сгущёнка', 'Молоко', 'Хрустящий хлеб')
('Сметана', 'Молоко', 'Хрустящий хлеб') 
```

Возможные варианты на экране. Далее можно использовать эти сочетания для тестирования системы рекомендаций. Это поможет просмотреть, какие сочетания товаров могут быть предложены при покупке какого-то товара, и отобрать самые удачные сочетания. Например, если клиент заказал `Хрустящий хлеб`, то ему можно посоветовать продукты `Молоко` и `Сметана`.

---

В примерах выше функции из модуля `itertools` создавали последовательности значений множества значений, исходя из их уникальности, — в одном варианте не могли встретиться одни и те же элементы множества. Это полезная опция, когда необходимо учитывать уникальность товаров. Например, курьеру нужно посетить адрес доставки только один раз.

Но бывают случаи, когда необходимо учитывать возможное дублирование значений во множестве. Например, при прогнозе продаж товаров или наступлении невзаимосвязанных событий. Чтобы не учитывать уникальность значений в получаемых последовательностях, в модуле `itertools` существует функция `combinations_with_replacement()`.

---

**Функция `combinations_with_replacement()`** (комбинации с замещением) создаёт объект-итератор, содержащий все возможные варианты сочетания элементов множества заданной длины с возможностью дублирования друг друга. Синтаксис функции `combinations_with_replacement()` такой же, как функции `combinations()`.

Рассмотрим работу функции на таком примере: соберём все возможные уникальные варианты значений на двух игральных кубиках, при этом нам не так важно, какое было значение на первом кубике, а какое на втором. Пары значений `1`-`6` и `6`-`1` будут рассмотрены как один вариант — `1`-`6`. Важно итоговое уникальное количество комбинаций:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['1', '2', '3', '4', '5', '6']

for i in itertools.combinations_with_replacement(data, 2):
    print(i)   
```

```
('2', '5')
('2', '6')
('3', '3')
('3', '4')
('3', '5')
('3', '6')
('4', '4')
('4', '5')
('4', '6')
('5', '5')
('5', '6')
('6', '6')
```

В результате составлена последовательность всех возможных уникальных значений на двух кубиках без учёта, какое значение было на первом кубике и на втором, как будто мы бросали их вместе.

Использование функции `combinations_with_replacement()` может пригодиться при необходимости проанализировать сочетание товаров или событий с учётом их повторения.

Например, маркетологи розничной системы торговых точек по продаже продуктов питания подготовили скидочные купоны для покупателей, выдаваемые после покупки случайным образом. Всего существует три типа купонов, со скидками на 2, 5 и 7%. Если покупатель собрал три купона, то он может их применить при следующей покупке, при этом скидки трёх купонов суммируются. Например, если у покупателя талоны 2, 2 и 7, то итоговая скидка будет 11%.

Маркетологам стало интересно, какие могут быть получены итоговые скидки при различном сочетании купонов. Функция `combinations_with_replacement()` поможет рассмотреть все возможные варианты суммарного значения скидки:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = [2, 5, 7]

for i in itertools.combinations_with_replacement(data, r=3):
    print(f'Комбинация скидок: {i}, суммарная скидка: {sum(i)}')   
```

## Заключение

В этом уроке вы детально рассмотрели, что из себя представляют объекты-итераторы и какие функции могут их образовывать. Объекты-итераторы позволяют перебирать объекты в циклах и определяют правила, по которым этот перебор происходит.

Помимо конечных итераторов, которые завершают свой перебор при окончании всех возможных вариантов, когда количество итераций заранее определено, существуют бесконечные и комбинаторные итераторы. Они создаются с помощью библиотеки `itertools`. Это такие функции, как:

**Бесконечные итераторы `itertools`**

- `count()` — создаёт объект-итератор с бесконечной последовательностью чисел, с определённой последовательностью и началом. Синтаксис: `itertools.count(start, step)`, где `start` начальное значение, а `step` — интервал до следующего значения.
- `cycle()` — создаёт объект-итератор с бесконечным перебором всех значений в итерируемом множестве (например, в списке). Синтаксис: `itertools.cycle(iterables)`, где `iterables` — объект для итерации.
- `repeat()` — создаёт объект-итератор, возвращающий один-единственный элемент бесконечное количество раз. Синтаксис: `itertools.repeat(object, times)`, где `object` возвращаемое значение, а `times` — количество итераций, по умолчанию — бесконечное значение.

**Комбинаторные итераторы `itertools`**

- `product()` — создаёт объект-итератор, определяющий все возможные последовательности элементов множеств, реализуя декартово произведение множеств. Синтаксис: `itertools.product(*iterables, repeat=1)`, где на месте `*iterables` через запятую перечислены все коллекции (списки, кортежи, множества или другие итерируемые объекты), а параметр `repeat` указывает, какое количество раз возможно задействовать исходные множества. По умолчанию это значение равно `1`.
- `permutations()` — создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов одного множества с учётом их последовательности и уникальности. То есть в одной последовательности не может быть несколько одних и тех же элементов множества. Синтаксис: `itertools.permutations(iterable, r=None)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств. По умолчанию этот параметр `= None` и соответствует количеству элементов во множестве.
- `combinations()` — создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов множества заданной длины без дублирования значений и учёта их последовательности, в порядке их расположения в исходном множестве. Синтаксис: `itertools.combinations(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан.
- `combinations_with_replacement()` — создаёт объект-итератор, содержащий все возможные варианты сочетания элементов множества заданной длины с возможностью дублирования друг друга. Синтаксис: `itertools.combinations_with_replacement(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан.
