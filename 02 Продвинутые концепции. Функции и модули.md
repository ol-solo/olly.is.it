# Организация циклов
## Обработка списков в цикле

Циклы можно использовать не только для вывода значений элементов списка или их преобразования, но и для обработки значений при группировке данных. Например, нужно подсчитать сумму или произведение элементов списка, найти минимальное или максимальное значение или выполнить другое математическое действие. Часто при выполнении подобных вычислений на основе исходных списков создаются новые.

В этом уроке рассмотрим, как выполнять вычисления при обработке списков и создавать новые списки по результатам обработки исходных.

### Математические вычисления

Предположим, есть список с длительностью фильмов. Разберём на примере, как в цикле рассчитать сумму всех значений в списке:

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
total_duration = 0 # создадим переменную-счётчик

for duration in movies_duration:
    total_duration += duration

print(total_duration) 
```

Получаем:

```
1768 
```

Решение состоит из нескольких частей:

1. **Счётчик.** Мы создали переменную-счётчик `total_duration`, которая хранит результат вычислений. Присвоили ей начальное значение `0`.
2. **Цикл (**`for duration in movies_duration`**).** В этом цикле перебираем каждый элемент из списка `movies_duration`. На каждой итерации цикла переменной `duration` присваивается значение очередного элемента списка. _а_. **Вычисления (**`total_duration += duration`**).** На каждой итерации цикла текущее значение `duration` добавляется к значению счётчика. Таким образом, на выходе из цикла в `total_duration` будет сумма всех значений списка.
3. **Вывод на экран.** Здесь функции `print()` мы передаём переменную `total_duration`, которую создали с самого начала и потом меняли в цикле.

Этот пример показывает, как в цикле можно проводить математические операции над элементами цикла. Здесь мы использовали сложение значений с их накоплением, но в циклах можно также выполнять и другие математические действия и реализовывать более сложные вычисления.

Произведение элементов вычисляют тем же образом, но есть несколько важных особенностей:

- **Изменение значения счётчика:** при создании новой переменной счётчика начальное значение `0` необходимо изменить на `1`, чтобы на первом шаге цикла в счётчик сохранился первый элемент списка, умноженный на единицу, а не ноль. В противном случае общий результат будет равен нулю.
- **Изменение оператора счётчика:** значение счётчика будем менять оператором `*=`, а не `+=`.

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]

mult_duration = 1
for duration in movies_duration:
    mult_duration *= duration

print(mult_duration) 
```

В результате видим:

```
1663161131763038875968000 
```

Python легко справляется с такими огромными числами. В этом одно из преимуществ этого языка.

Другое преимущество Python — встроенные методы. Расчёты суммы и произведения встречаются в работе аналитика так часто, что для них в языке есть готовые функции. На практике в таких задачах вам не придётся каждый раз писать цикл:

- **Использование встроенной функции** `sum()` **для суммирования значений:** функция `sum()` — «сумма» принимает в качестве аргумента список и возвращает сумму его элементов:

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
total_duration = sum(movies_duration)
print(total_duration) 
```

Итог:

```
1768 
```

- **Использование функции** `prod()` **для произведения значений:** аналогично работает функция для расчёта произведения — `prod()` — от англ. product, «произведение». Она встроена в модуль `math` — придётся его импортировать:

```
import math 

movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
mult_duration = math.prod(movies_duration)
print(mult_duration) 
```

В результате:

```
1663161131763038875968000 
```

Обратите внимание, что выше мы использовали функцию `prod()`, которая не является встроенной функцией Python, а входит в состав модуля `math`. Поэтому в начале кода мы дополнительно импортировали модуль `math`, чтобы воспользоваться его функционалом. В дальнейших уроках мы подробнее разберём, как корректно импортировать дополнительные модули, какие модули существуют и в чём их преимущества.

### Максимум и минимум

Чтобы найти максимальное значение, мало цикла — ещё нужна условная конструкция с `if` — «если». С ней вы уже встречались в предыдущих темах.

Сам алгоритм остаётся схожим с предыдущим примером:

- Объявим переменную `max_value`, которая будет фиксировать максимальное значение, и установим в качестве начального значения первое значение списка `movies_duration[0]`:

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
max_value = movies_duration[0] 
```

- Запустим цикл `for duration in movies_duration`, который перебирает элементы списка `movies_duration`, присваивая значения переменной `duration`.
- В теле цикла сравним очередной элемент с результатом: `if duration > max_value:` если текущий элемент `duration` больше, чем текущее максимальное значение `max_value`, мы обновляем `max_value`.

```
for duration in movies_duration:
    if duration > max_value:
        max_value = duration 
```

- Когда цикл завершил работу, выведем результат на экран.

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
max_value = movies_duration[0]

for duration in movies_duration:
    if duration > max_value:
        max_value = duration

print(max_value) 
```

Итог:

```
201 
```

Алгоритм выберет максимальное значение, даже если список состоит только из отрицательных значений:

```
values = [-142, -175, -152, -195]
max_value = values[0]

for value in values:
    if value > max_value:
        max_value = value 

print(max_value) 
```

Результат:

```
-142 
```

Аналогичным образом можно найти минимальное значение. Для этого нужно изменить условие — `if value < min_value:`, в котором будем сравнивать каждое значение с минимальным и таким образом искать наименьшие:

```
values = [142, 175, 152, 195, 201, -142, -175, -152, -195]
min_value = values[0]

for value in values:
    if value < min_value:
        min_value = value 

print(min_value) 
```

Результат выполнения кода:

```
-195 
```

В Python также есть встроенные функции, которые находят минимальное — `min()` — «минимум» и максимальное `max()` — «максимум» значения из списка значений. Чтобы воспользоваться этими функциями, передадим им список:

```
movies_duration = [142, 175, 152, 195, 201, 154, 178, 139, 133, 163, 136]
max_duration = max(movies_duration)
min_duration = min(movies_duration)
print(f'Максимальное значение - {max_duration} \nМинимальное значение - {min_duration}') 
```

Получаем:

```
Максимальное значение - 201 
Минимальное значение - 133 
```

Для поиска минимального или максимального значения в списке лучше использовать функции `min()` и `max()` вместо перебора значений в цикле. Это позволит сократить время выполнения кода и сделать его более читаемым. Обычно циклы используются для реализации более сложных алгоритмов.

Представьте, что нам нужно найти самую низкую цену товара среди дорогих товаров. У нас есть несколько условий для цикла. Найдём самую низкую стоимость товара среди значений, состоящих из четырёх и более знаков:

```
values = [750, 341, 950, 899, 159, 3550, 1500, 1105, 1999]
min_value = max(values)

for value in values:
    if value > 1000:
        if value < min_value:
            min_value = value

print(min_value) 
```

В примере выше в качестве `min_value` назначено максимальное значение из списка `values`. Для этого мы использовали функцию `max(values)`. Затем в цикле реализовали два условия:

1. Первое условие `if value > 1000:` сравниваем значение списка `value` с `1000`, поскольку мы должны учитывать значения с четырьмя и более знаками.
2. Дальше, если условие истинное, идёт реализация второго условия `if value < min_value:`, которое будет искать минимальное значение из списка.

Результат будет следующим:

```
1105 
```

### Расчёт квадрата

На вход программе поступает список чисел. Каждый элемент — сторона квадрата. Нужно создать новый список, где каждому исходному элементу соответствует площадь квадрата:

```
[2, 10, 20, 3] -> [4, 100, 400, 9] 
```

Площадь квадрата находят по формуле: `a * a`, где `a` — сторона квадрата. В Python вместо `a * a` можно ещё написать `a ** 2`, где `**` — оператор возведения в степень.

Для создания нового списка с изменёнными значениями в цикле используем метод `append()`. Чтобы метод `append()` добавлял в новый список изменённые значения исходного списка, перед циклом создадим пустой список `squares`. В теле цикла будем добавлять изменённые значения исходного списка в новый список — `squares.append(side ** 2)`:

```
sides = [2, 10, 20, 3]

# создаём пустой список
squares = [] 
# объявляем цикл for, 
# который перебирает элементы списка sides
for side in sides: 
    # добавляем к squares 
    # квадрат очередного элемента
    squares.append(side ** 2) 
    
# после цикла «печатаем» результат
print('Список длин сторон квадратов:', sides)
print('Список площадей квадратов:', squares) 
```

Получаем:

```
Список длин сторон квадратов: [2, 10, 20, 3]
Список площадей квадратов: [4, 100, 400, 9] 
```

Если бы задача состояла в изменении исходного списка, помогла бы функция `range()`:

```
sides = [2, 10, 20, 3]

# выведем значения списка до цикла
print('Список длин сторон квадратов:', sides) 

# пройдёмся по индексам списка sides: 0, 1, 2, 3
for i in range(len(sides)):  
    # на каждом шаге обращаемся 
    # к очередному элементу списка и заменяем его 
    # на квадрат этого же элемента
    sides[i] = sides[i] ** 2 

# после цикла «печатаем» результат
print('Список превратился в список площадей квадратов:', sides)  
```

```
Список длин сторон квадратов: [2, 10, 20, 3]
Список превратился в список площадей квадратов: [4, 100, 400, 9] 
```

### Расчёт факториала

Факториал числа `n` — произведение всех целых чисел от 1 до `n`. Факториал означают так: `!`. Например, факториал числа `5`:

```
5! = 1 * 2 * 3 * 4 * 5 = 120  
```

Факториалы важны в комбинаторике — разделе математики, который занимается вопросами вроде:

- Сколько существует различных способов разложить объекты заданным образом?
- Сколько комбинаций можно получить из заданных объектов?

Посчитаем факториал числа `8`. Для этого используем диапазон чисел, который сгенерирует функция `range()`. Поскольку мы работаем с заданным диапазоном значений, то передадим функции `range()` два аргумента — начало и окончание необходимого диапазона — `range(1, 9)`:

```
number = 8 
fact = 1 
for i in range(1, number + 1): 
    fact *= i 
print(fact) 
```

Результат:

```
40320 
```

Объясним алгоритм использования цикла для расчёта факториала:

1. В начале задали необходимое число для нахождения факториала в переменной `number`. Оно равно `8`.
2. Задали переменную-счётчик `fact`, поскольку расчёт факториала начинается с 1, то и значение `fact` зададим равное `1`.
3. Следующим шагом объявляем цикл `for i in range(1, number + 1):`, чтобы перебирать значения от 1 до `number + 1`. Добавление 1 к `number` обеспечивает включение `number` в диапазон.
4. В теле цикла рассчитывается факториал — `fact *= i` , где значение переменной-счётчика домножается на текущее значение `i`.

Как часто бывает, вместо своего алгоритма можно использовать встроенную функцию `factorial()` — «факториал» из модуля `math`:

```
import math

print(math.factorial(8)) 
```

Итог:

```
40320 
```

### Подсчёт количества значений в строке

Счётчик, который мы использовали в примерах выше, можно также применять для подсчёта значений в строке. Например, подсчитаем, сколько знаков в строке и сколько из них — буква `о`:

```
text = 'Коровье молоко'
# счётчик, подсчитывающий общее количество символов
total_letters = 0 

# счётчик, подсчитывающий количество букв о
letter_o_cnt = 0 

for letter in text:
    # на каждом шаге увеличиваем счётчик символов
    total_letters += 1 
    # проверяем в условном выражении, является ли итератор буквой о
    if letter.lower() == 'о': 
        letter_o_cnt += 1       
    
print(f'Всего в строке {total_letters} знаков, из них {letter_o_cnt} букв "о"') 
```

В результате выполнения:

```
Всего в строке 14 знаков, из них 5 букв "о" 
```

В примере выше было задано два счётчика — один (`total_letters`) подсчитывал общее количество знаков в строке, второй (`letter_o_cnt`) увеличивал своё значение, только если переменная-итератор`letter`— принимала значение `о`.

Разберём пример выше:

1. В начале мы создали две переменных-счётчика `total_letters` и `letter_o_cnt` и установили начальные значения для них равные `0`. Счётчик `total_letters` будет увеличиваться на каждом шаге цикла для подсчёта общего числа символов в строке. Счётчик `letter_o_cnt` будет увеличиваться, если перебираемый символ исходной строки `text` будет буквой `о`.
2. Далее объявляем цикл `for letter in text:` для перебора символов в исходной строке, в которой хотим подсчитать количество символов.
3. В теле цикла `total_letters += 1` на каждом шаге увеличиваем счётчик `total_letters`, чтобы подсчитать общее количество символов в строке.
4. Затем следует условие `if letter.lower() == 'о':`, которое проверяет, является ли текущий символ буквой `о`. Метод `.lower()` используется для того, чтобы сделать сравнение регистронезависимым.
5. Если условие истинное, увеличиваем счётчик `letter_o_cnt` на 1 — `letter_o_cnt += 1`.
6. После завершения цикла распечатаем результаты.

Кстати, если бы мы считали общее количество символов в строке, то в теле цикла можно было бы не использовать переменную-итератор, а только увеличивать значение счётчика. Чтобы подчеркнуть, что значение переменной-итератора нам неважно, в качестве имени переменной-итератора обычно используется символ `_`, а сам счётчик называют латинской буквой `n`:

```
text = 'Коровье молоко'
n = 0 # счётчик, подсчитывающий общее количество символов
for _ in text:
    n += 1 # на каждом шаге увеличиваем счётчик знаков
    
print(f'Всего в строке {n} знаков') 
```

Итог:

```
Всего в строке 14 знаков 
```

В примере выше можно также использовать встроенную функцию Python `len()` — «длина», которая подсчитывает длину строки или другого множества:

```
text = 'Коровье молоко'
print(f'Всего в строке {len(text)} знаков') 
```

Результат:

```
Всего в строке 14 знаков 
```
## Использование различных функций в заголовке цикла

В прошлых уроках мы рассмотрели, как при помощи функции `range()` в заголовке цикла обратиться к индексу списка. Однако в заголовках циклов можно использовать и другие полезные функции. В этом уроке разберём, как:

- при помощи функции `enumerate()` использовать одновременно индекс и значение элементов списка;
- при помощи функции `zip()` в одном цикле использовать значения из нескольких списков.

### Индекс и значение элемента из списка. Функция `enumerate()`

Иногда, чтобы решить аналитическую задачу, нужно извлечь из списка и индекс элемента, и его значение. Например, для изменения исходных значений в списке, создания отдельных списков или словарей на основе существующих, для ранжирования данных или проведения сложных расчётов. В этом поможет функция `enumerate()` — «перечислять». Она применяется для списков и других итерируемых объектов.

Результат работы этой функции — объект-перечисление `enumerate object`, содержащий множество кортежей из двух элементов — индекса элемента исходного списка и значения этого элемента. Однако если обратиться к этому объекту напрямую, то мы получим только общую информацию о нём:

```
trees = ['Дуб','Берёза','Сосна','Ель']
enumerate_result = enumerate(trees) # сохраним результат работы функции enumerate()

print(enumerate_result) 
```

Результат:

```
<enumerate object at 0x00000177BB5CE110> 
```

Это происходит потому, что `enumerate object` — это объект-генератор. В Python так называется специальный тип объекта, который можно итерировать (перебирать, запрашивая у него следующие значения). При этом на каждой итерации перебираемые значения генерируются из списка, на основе которого был создан объект. Это позволяет сохранять вычислительные ресурсы при работе с большими объёмами данных. При этом в памяти хранится только последнее значение.

Разберём особенности работы объекта-генератора на жизненных примерах. Представьте, вы собрались на обед в классное кафе. Сегодня в меню вкусный витаминный салат, первое блюдо, второе блюдо, десерт и яблочный компот. Все блюда вместе можно представить как набор элементов. Теперь у нас есть два сценария:

- В первом случае все блюда приносят сразу вместе, и пустые тарелки не убирают. В таком случае стол должен быть большой, и при этом пустые тарелки остаются, а второе блюдо и вовсе может остыть.
- Во втором случае каждое блюдо приносят по мере употребления предыдущего. Это экономит место на обеденном столе, процесс обеда оптимизируется — пустые тарелки убираются со стола, снижается одновременно задействованное количество тарелок. С таким подходом проще придерживаться определённой заранее последовательности событий. При этом, если кто-то не захочет яблочный компот, то его не нужно будет специально подавать к столу.

Первый случай больше соответствует простому перебору значений из определённого множества, когда все элементы из этого множества зафиксированы и хранятся в памяти, даже если не используются. А второй пример больше характеризует работу объекта генератора в цикле: каждый новый элемент множества извлекается по мере необходимости его использования.

Мы уже сталкивались с такими объектами ранее — функция `range` генерирует последовательность чисел, которая воспроизводится по мере обращения.

Посмотрим, как выглядит содержимое объекта `enumerate`, перебрав его значения в цикле:

```
trees = ['Дуб','Береза','Сосна','Ель']

for i in enumerate(trees):
      print(i) 
```

Результат выполнения кода:

```
(0, 'Дуб')
(1, 'Берёза')
(2, 'Сосна')
(3, 'Ель') 
```

Здесь каждый элемент исходного списка `trees` представлен кортежем с индексом этого элемента и его значением, например, `(0, 'Дуб')`.

Поскольку в каждом кортеже два значения, то для их использования в цикле следует вместо одной переменной-итератора задать две:

- первая будет отображать индекс элемента (`i`);
- а вторая — его значение (`tree`).

Например:

```
trees = ['Яблоня','Ясень','Черёмуха','Ирга']

for i, tree in enumerate(trees): # зададим две переменные-итератора
    print(f"Дерево с индексом {i} в списке - {tree}") 
```

Итог:

```
Дерево с индексом 0 в списке - Яблоня
Дерево с индексом 1 в списке - Ясень
Дерево с индексом 2 в списке - Черёмуха
Дерево с индексом 3 в списке - Ирга 
```

Используя функцию `enumerate()`, можно изменять значения исходного списка по индексу по аналогии с тем, как делали в предыдущих уроках с помощью функции `range()`. Например, изменим стоимость товара, снизив её на 100 единиц:

```
prices = [1500, 3100, 7500, 4200]
print('Значения списка prices до изменения:', prices)

# уменьшим каждое значение списка prices на 100 единиц:
for i, price in enumerate(prices):
     prices[i] = price - 100
print('Результат изменения значений списка prices:', prices) 
```

Результат выполнения:

```
Значения списка prices до изменения: [1500, 3100, 7500, 4200]
Результат изменения значений списка prices: [1400, 3000, 7400, 4100] 
```

Разберём работу скрипта:

1. **Определение исходного списка:** в этом блоке был определён список `prices` с исходными значениями — `prices = [1500, 3100, 7500, 4200]`. Далее использована функция `print()` для вывода на экран исходных значений списка.
2. **Объявление цикла `for`:** далее был объявлен цикл `for i, price in enumerate(prices):`, в котором в заголовке используется функция `enumerate()`. На каждой итерации цикла функция `enumerate()` генерирует кортеж из двух значений — индекс элемента и его значение. В заголовке цикла определены две переменных-итератора: первая переменная-итератор `i` будет принимать значение индекса элемента списка, а вторая переменная-итератор `price` — значение элемента списка.
3. **Тело цикла:** в теле цикла прописана инструкция, которая, используя индекс элемента списка (`prices[i]`), изменяет его значение за счёт математической операции с исходным значением элемента списка — `price - 100`.
4. **Вывод на экран изменённых значений списка:** в завершении использована функция `print()` для вывода на экран значений списка после изменения.

Обратите внимание, что функция `enumerate()` возвращает индекс и значение всех элементов списка или другого итерируемого объекта. Это позволяет уже в самом начале цикла, в его заголовке определить как индекс элемента, так и его значение. Поэтому `enumerate()` предпочтительнее использовать при переборе и изменении элементов по их индексу вместо конструкции `range(len(some_list))`, которую мы использовали для примеров в предыдущих уроках. В примерах ниже результат будет одинаковый, но для изменения исходного списка используются различные подходы:

```
prices_a = [1500, 3100, 7500, 4200]
prices_b = [1500, 3100, 7500, 4200]

# уменьшим каждое значение списка prices на 100 единиц, используя enumerate:
for i, price in enumerate(prices_a):
     prices_a[i] = price - 100

# уменьшим каждое значение списка prices на 100 единиц, используя range(len(prices)):
for i in range(len(prices_b)):
     prices_b[i] = prices_b[i] - 100
        
print(prices_a)
print(prices_b) 
```

```
[1400, 3000, 7400, 4100]
[1400, 3000, 7400, 4100] 
```

Функция `enumerate()`имеет дополнительный параметр `start`, который позволяет задать начало нумерации значений. По умолчанию этот параметр равен `0`. Такой подход может быть полезным для составления нумерованных списков, где необходимо задать отличное от 0 начальное значение.

Например, пронумеруем последовательность действий пользователей на сайте:

```
funnel = ['registration','login','main_page','product_page','purchase_page']

for i, stage in enumerate(funnel, start=1):
    print(f'Этап воронки {i}: {stage}') 
```

```
Этап воронки 1: registration
Этап воронки 2: login
Этап воронки 3: main_page
Этап воронки 4: product_page
Этап воронки 5: purchase_page 
```

В примере выше в заголовке цикла использована функция `enumerate()` с параметром `start=1`, что позволило пронумеровать события пользователей в порядке их совершения (от первого до пятого события).

Однако в этом случае мы не сможем использовать переменную итератор `i` для перебора элементов списка по их индексу, так как индексы, возвращаемые `enumerate()` с `start=1`, не будут соответствовать индексам списка, и они не начнутся с `0`.

### Использование в цикле нескольких списков. Функция `zip()`

В цикле можно использовать несколько списков, если их предварительно объединить. Например, в одном списке содержится информация о товаре, а во втором — его стоимость. Чтобы вывести для каждого товара его стоимость в цикле, можно использовать в заголовке цикла функцию `zip()` — «упаковывать».

Функция `zip()` принимает в качестве аргумента список итерируемых объектов, элементы которых с одинаковым индексом будут объединены в кортеж. В результате будет создано множество кортежей, объединённых в `zip`-объект:

```
trees = ['Яблоня','Ясень','Черёмуха','Ирга']
prices = [10000, 15000, 13000, 14000]

# выведем значения zip-объекта, объединяющие два списка:
for i in zip(trees, prices):
     print(i) 
```

```
('Яблоня', 10000)
('Ясень', 15000)
('Черёмуха', 13000)
('Ирга', 14000) 
```

Как и в случае с функцией `enumerate()`, возвращаемый zip-объект будет объектом- генератором.

Для перебора кортежей, сформированных функцией `zip()` при объединении нескольких списков, можно использовать цикл `for` с несколькими переменными итераторами в заголовке:

```
trees = ['Яблоня','Ясень','Черёмуха','Ирга']
prices = [10000, 15000, 13000, 14000]

for tree, price in zip(trees, prices):
     print(f'{tree}, стоимость {price} руб.') 
```

```
Яблоня, стоимость 10000 руб.
Ясень, стоимость 15000 руб.
Черёмуха, стоимость 13000 руб.
Ирга, стоимость 14000 руб. 
```

В примере выше переменная-итератор `tree` на каждом шаге цикла принимает значение первого списка, который передан функции `zip` — в нашем случае это список `trees`. Вторая переменная-итератор `price` принимает значение следующего переданного функции `zip` списка — `prices`.

С помощью функции `zip()` можно объединять более двух списков. Например, объединим название товара, его стоимость и место реализации, если оно есть:

```
trees = ['Яблоня','Ясень','Черёмуха','Ирга']
prices = [10000, 15000, 13000, 14000]
markets = ['Алексеевка','Заречное']

# объединим списки:
for tree, price, market in zip(trees, prices, markets):
    print(f'{tree}, стоимость {price} руб., магазин: {market}') 
```

```
Яблоня, стоимость 10000 руб., магазин: Алексеевка
Ясень, стоимость 15000 руб., магазин: Заречное 
```

У функции `zip()` есть важная особенность: если количество элементов в циклах будет разным, то итоговое количество кортежей будет соответствовать количеству элементов в наикратчайшем из списков. В примере выше объединены только первые два элемента каждого списка, поскольку в списке `markets` было только два элемента. Таким образом, функция **`zip()`** объединяет элементы списков до тех пор, пока не достигнет конца самого короткого списка.

Если необходимо сложить все списки, когда они не равны по количеству элементов, можно использовать функцию `zip_longest()` из модуля `itertools`. Действие этой функции аналогично функции `zip()`, только в случае недостающих значений будет использовано значение `None`.

Такой подход полезен, когда необходимо сохранить все элементы из списка с самым большим количеством значений. Это позволит заполнить недостающие значения в одном или нескольких списках.

Вернёмся к примеру выше: в одном из рядов товарного ассортимента нет информации о месте реализации, и поэтому не попал в совмещённый список значений, которые использовались в цикле. Использование `zip_longest()` позволит сохранить все наименования товаров:

```
import itertools # импортируем модуль itertools 

trees = ['Яблоня','Ясень','Черёмуха','Ирга']
prices = [10000, 15000, 13000, 14000]
markets = ['Алексеевка','Заречное']

for tree, price, market in itertools.zip_longest(trees, prices, markets):
    print(f'{tree}, стоимость {price} руб., магазин: {market}') 
```

```
Яблоня, стоимость 10000 руб., магазин: Алексеевка
Ясень, стоимость 15000 руб., магазин: Заречное
Черёмуха, стоимость 13000 руб., магазин: None
Ирга, стоимость 14000 руб., магазин: None 
```

В первой строчке import itertools мы говорим Python, что хотим использовать функции из модуля itertools. Начинать обращение к таким функциям будем названием модуля с точкой. В уроках следующих тем мы рассмотрим этот механизм подробнее.

В функцию `zip_longest()` можно передать аргумент, который будет заполнять пропущенные значения: `fillvalue`:

```
import itertools

trees = ['Яблоня','Ясень','Черёмуха','Ирга']
prices = [10000, 15000, 13000, 14000]
markets = ['Алексеевка','Заречное']

# Выведем совместно значения списков:
for tree, price, market in itertools.zip_longest(trees, prices, markets, fillvalue='Нет данных'):
    print(f'{tree}, стоимость {price} руб., магазин: {market}') 
```

```
Яблоня, стоимость 10000 руб., магазин: Алексеевка
Ясень, стоимость 15000 руб., магазин: Заречное
Черёмуха, стоимость 13000 руб., магазин: Нет данных
Ирга, стоимость 14000 руб., магазин: Нет данных 
```

А теперь вперёд, к практике!

## Циклы с неизвестным количеством итераций

В предыдущих уроках мы разбирали примеры использования цикла `for`, когда общее число итераций заранее известно — оно или равно количеству элементов в списке, или задаётся с помощью последовательности чисел.

Однако бывают ситуации, когда количество итераций заранее неизвестно и зависит от некоторого условия — логического выражения. Такие ситуации возникают при разработке приложений на Python или создании сложных алгоритмов обработки и анализа данных:

- При проверке корректности ввода email в форме заказа, где количество итераций будет зависеть от правильности ввода информации — когда пользователь введёт верное или ожидаемое значение, цикл прекратит работу;
- При подборе наиболее оптимальных параметров для достижения необходимых условий, например, расчёты для оптимального распределения бюджета между рекламными каналами в зависимости от стоимости привлечения пользователя и конверсии в целевое действие;
- При обработке данных с учётом времени, например, при сборе данных для расчёта среднего чека покупок в интернет-магазине, совершённых в утренние часы;
- Когда необходимо обработать данные по заданным условиям, например, получить информацию по первым 30 покупателям, которые совершили покупки свыше 5000 рублей.

Для этого используют цикл Python `while` — «до тех пор, пока». Он будет повторять последовательность действий, пока условие будет истинным. Остановка цикла `while` произойдёт, когда условие станет ложным. Для этого в теле цикла изменяют используемые в условии переменные.

![](https://pictures.s3.yandex.net/resources/image_1704915576.png)

Разберём структуру цикла `while` на примере: создадим переменную `n`, которую будем увеличивать на каждой итерации цикла на 1, а сам цикл остановим, когда условие `n < 5` станет ложным, то есть, когда переменная `n` будет равна 5 или больше.

```
n = 1

# объявим цикл while
while n < 5: # заголовок цикла
    print(f'Значение переменной n = {n}') # тело цикла
    n = n+1

print('Инструкция после цикла: цикл окончен') # инструкция после окончания цикла 
```

```
Значение переменной n = 1
Значение переменной n = 2
Значение переменной n = 3
Значение переменной n = 4
Инструкция после цикла: цикл окончен 
```

Цикл `while` так же, как и цикл `for` состоит из заголовка и тела, но есть и другие элементы:

- Ключевое слово `while`;
- Условие, по которому цикл выбирает продолжить итерации или остановиться; в примере выше проверяется условие `n < 5` и пока это условие истинное, цикл будет совершать итерации;
- Знак `:`. Без него будет синтаксическая ошибка, а сам цикл не будет объявлен.

Важно, что при этом тело цикла содержит действие, которое будет изменять переменную, используемую в условии. В примере выше, при каждой итерации цикла происходит увеличение значений переменной `n` на 1.

Если переменная, которая используется в условии цикла не будет изменяться в теле цикла, то цикл будет продолжаться бесконечно, что может вызвать ошибку выполнения программы.

Рассмотрим работу цикла на примере с новогодними подарками: чтобы завернуть каждый подарок в красивую упаковку нужно 32 сантиметра упаковочной бумаги. Однако наши запасы бумаги не бесконечны и в распоряжении есть рулон длиной всего один метр. Посмотрим, сколько итераций цикла по упаковке подарка мы можем провести:

```
length = 100 # переменная, которая задаёт общую длину бумаги

# объявим цикл while
while length > 32: # условие, которое проверяет хватит ли бумаги на подарок
    length -= 32 # изменение переменной на каждой итерации
    print(f'Осталось {length} см упаковочной бумаги') 
```

```
Осталось 68 см упаковочной бумаги
Осталось 36 см упаковочной бумаги
Осталось 4 см упаковочной бумаги 
```

Разберём работу цикла из примера:

1. Перед циклом мы задали переменную `length`, в которую передали исходное значение — 100 см упаковочной бумаги: `length = 100`.
2. Затем в заголовке цикла задали условие `while length > 32`, которое проверяет длину оставшейся упаковочной бумаги. Если её достаточно для следующего подарка, то есть осталось больше 32 сантиметров и условие является истинным, то цикл проводит итерацию. А если условие будет ложным, цикл должен остановиться.
3. В теле цикла прописали инструкции, которые изменяют значение переменной `length`, уменьшая ее на 32: `length -= 32`. После этого на каждой итерации выводим длину оставшейся упаковочной бумаги — именно это значение будет сравниваться в условии заголовка цикла в следующей итерации цикла.
4. Когда оставшееся значение `length` будет меньше 32, цикл остановится.

Используя цикл `while` мы узнали, сколько подарков мы сможем упаковать, пока у нас не закончится бумага. В результате цикл выше совершил только 3 итерации. Значит бумаги хватит только на три подарка. Таким образом, цикл `while` будет выполняться, пока условие истинно, и остановится, когда условие станет ложным.

На практике аналитики часто используют цикл `while` вместе с переменной, которая подсчитывает количество итераций, а в условии прописывают необходимое их количество.

В примере выше мы выяснили, что подарочной упаковки хватит только на три подарка, поэтому нужно купить новую, которую хватит на пять подарков. В этом случае можем использовать цикл `while`, чтобы подсчитать необходимую длину упаковочной бумаги:

```
length = 0 # зададим переменную, которая будет учитывать затраченную длину бумаги
n = 1 # зададим начальное количество итераций - количество подарков

# объявим цикл
while n <= 5: # условие, которое проверяет значение перменной 'n'
    length += 32 # на каждой итерации к значению length будет добавлено 32
    print(f'Подарок {n}, израсходовано {length} см упаковочной бумаги')
    n += 1 # увеличиваем счётчик итераций на 1

print(f'Результат: для {n-1} подарков нужно {length} см упаковочной бумаги') 
```

Разберём работу цикла:

### Объект итератор и цикл `for`

Вы уже знакомы с циклом `for` и его основными компонентами. В примерах при создании цикла мы использовали переменные итераторы, поочерёдно перебирающие данные в списке, строке или в других коллекциях, в том числе созданных различными функциями, например `range()`, `zip()` или `enumerate()`. Немного подробнее остановимся на том, как **Python** перебирает объекты в цикле.

В **Python** существуют такие объекты-**итераторы**, помогающие циклу последовательно перебирать все элементы. Именно из итератора берётся необходимое значение из перебираемого списка:

```
# Создадим цикл, перебирающий все значения в списке trees:
trees = ['Ольха','Берёза','Ясень','Кипарис']

for tree in trees:
    print(tree) 
```

```
Ольха
Берёза
Ясень
Кипарис 
```

Результат работы цикла простой: на каждой новой строке отображён элемент списка `trees`. В этом случае цикл `for` взаимодействует не напрямую со списком `trees`, а с объектом-итератором, созданным на основании списка. Именно объект-итератор на каждом шаге цикла выбирает необходимое для перебора значение из списка `trees` и передаёт его в переменную-итератор `tree`.

Так происходит, пока все элементы не будут перебраны.

В коде выше мы четыре раза подряд обратились к объекту-итератору, используя функцию `next()`, и вывели на экран каждое последующее значение списка `trees`. Особенность работы объекта-итератора в том, что он не загружает все объекты исходной коллекции в память, а выводит их последовательно по одному, по мере обращения. Объект-итератор всегда знает, какое значение должно быть выведено при следующем обращении к нему. Эту информацию он хранит в памяти.

```
# Создадим список элементов:
trees = ['Ольха','Берёза','Ясень','Кипарис']

# Создадим объект-итератор:
trees_iter = iter(trees)

# Последовательно выведем на экран первые четыре элемента:
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter))
```

### Какие ещё функции возвращают итератор?

Объект-итератор возвращает не только функция `iter()`. Таких функций в среде программирования **Python** много. Они поддерживают использование функций `iter()` и `next()`, в них прописаны условия итерации объектов. Однако эти вопросы затрагивают особенности программирования на **Python** и выходят за рамки тех основных знаний, которые необходимы аналитику для работы. Поэтому разберём пример работы функций, возвращающих объект-итератор на примере уже знакомых функций.

В прошлых уроках вы работали в цикле с функциями `enumerate()` и `zip()`. Они также возвращают объекты, которые могут быть итерированы. Например, используем `enumerate()` для создания объекта-итератора и применим функцию `next()`:

```
# Создадим объект-итератор с помощью `enumerate()`:
trees_iter = enumerate(trees)

# Последовательно выведем на экран 4 первых элемента:
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter))
print(next(trees_iter)) 
```

```
(0, 'Ольха')
(1, 'Берёза')
(2, 'Ясень')
(3, 'Кипарис') 
```

Функция `next()` при каждом новом обращении к объекту-итератору выводила значения сформированного `enumerate()` кортежа. Если продолжить дальше использование функции `next(trees_iter)`, то итератор вернёт ошибку `StopIteration`.

Помимо функций `iter()`, `enumerate()` и `zip()`, в **Python** есть другие функции и модули, создающие объект-итератор из исходного множества значений, одну из которых мы рассмотрим далее — модуль `itertools`. Ещё язык **Python** позволяет создавать собственные объекты-итераторы с необходимыми правилами итерации объектов, но изучение этой темы выходит за рамки курса, поскольку аналитик данных работает уже с готовыми решениями встроенных функций **Python**.

### Модуль `itertools`

Модуль `itertools` (от английского «инструменты для создания итератора») представляет собой встроенный модуль **Python**, содержащий функции для создания различных объектов итераторов. Используя `itertools`, можно создавать такие типы итераторов:

- конечные;
- бесконечные;
- комбинаторные.

Конечные итераторы перебирают все объекты исходного множества и завершают свою работу, когда количество элементов множества закончилось. Например, это итераторы списка, или функций `enumerate()` и `zip()`. В модуле `itertools` конечные итераторы позволяют по-разному объединять несколько множеств для итерации или использовать условия для выбора итерируемых объектов из списка.

В прошлом уроке мы рассмотрели один из таких итераторов — `zip_longest`, на остальных конечных итераторах в рамках урока мы не будем останавливаться, так как по своей логике они дублируют действия со списками и другими коллекциями. Остановимся детальнее на бесконечных и комбинаторных итераторах.

### Бесконечные итераторы

В предыдущих примерах все итераторы, используемые в цикле `for`, имели определённое количество элементов и были конечными. Выше мы рассматривали, что если итерируемых значений больше нет, то итератор выведет ошибку исключение `StopIteration`.

Бесконечный итератор будет генерировать элементы для итерации до тех пор, пока цикл не остановится в явном виде, например, при помощи условного выражения и команды `break`, пока не будет достигнуто определённое условие итерации.

![](https://pictures.s3.yandex.net/resources/image_1729599793.png)

Бесконечный итератор может быть полезным в работе аналитика при создании цикла, когда нет чёткого представления об итоговом количестве элементов в перебираемом множестве значений или существует необходимость обработки поступающих новых данных в режиме реального времени. В этом бесконечные итераторы по своему смыслу похожи на цикл `while`.

Задачей, которая может требовать такой бесконечной генерации, может быть генерация уникальных идентификаторов, присваиваемых новым пользователям или заказам. Также бесконечный итератор может пригодиться при распределении задач между группами или сегментами — например, распределение нового заказа в службе доставки между активными курьерами.

Предположим, в службе доставки есть три курьера и нужно каждый новый заказ последовательно назначать каждому из этих заранее известных курьеров по очереди. В этом случае новые объекты — заказы — поступают из коллекции неизвестного размера, а категории — курьеры — присваиваются им циклично, по заранее заданному правилу, реализованному в виде итератора.

Ещё пример — перебор отображаемых рекламных баннеров в приложении, когда они последовательно сменяются через определённый промежуток времени. Такую задачу также можно осуществить при помощи бесконечных итераторов. Бесконечные итераторы также используются для непрерывной генерации данных при тестировании моделей машинного обучения или проверке работы баз данных, программного обеспечения.

В основном бесконечные итераторы используются больше в программировании, чем в анализе данных. Однако аналитику данных на **Python** полезно понимать общий принцип работы таких итераторов и быть знакомым с инструментами, помогающими их создать.

В модуле `itertools` существуют следующие функции, генерирующие бесконечные итераторы:

- `count()`
- `cycle()`
- `repeat()`

**Функция `count()`** создаёт объект-итератор с бесконечной последовательностью чисел, с определённой последовательностью и началом. Синтаксис функции следующий `itertools.count(start, step)`, где `start` начальное значение, а `step` — интервал до следующего значения.

```
# Создадим итератор, генерирующий бесконечную 
# последовательность чисел от 0 с шагом 5:

# Импорт библиотеки itertools:
import itertools

# Создадим цикл, использующий бесконечный итератор count:
for i in itertools.count(0, 5):
    if i > 50:
        break
    else:
        print(i) 
```

```
0
5
10
15
20
25
30
35
40
45
50 
```

В примере выше функция `itertools.count()` создала бесконечный итератор, начинающийся со значения `0` и шагом `5`. В результате цикл `for` получился бесконечным и, чтобы его остановить, было использовано условие, останавливающее цикл через команду `break`, если перебираемое значение выше `50`. В дальнейших примерах будет использован аналогичный подход для выхода из цикла.

В качестве шага можно использовать не только целые значения, но и вещественные числа, например, сделаем шаг не `5`, а `0.5`:

```
# Создадим цикл, использующий бесконечный итератор count:
for i in itertools.count(0, .5):
    if i > 2:
        break
    else:
        print(i) 
```

```

1.0
1.5
2.0 
```

**Функция `cycle()`** создаёт объект-итератор с бесконечным перебором всех значений в итерируемом множестве (например, списке). Функция имеет простой синтаксис и принимает значение множества для перебора — `itertools.cycle(<коллекция>)`:

```
# Создадим список:
trees = ['Ольха','Берёза','Ясень','Кипарис']

# Создадим цикл, использующий бесконечный итератор cycle:
count = 0
for i in itertools.cycle(trees):
    if count > 6:
        break
    else:
        print(f'Итерация цикла: {count}, Элемент списка: {i}')
    count += 1 
```

```
Итерация цикла: 0, Элемент списка: Ольха
Итерация цикла: 1, Элемент списка: Берёза
Итерация цикла: 2, Элемент списка: Ясень
Итерация цикла: 3, Элемент списка: Кипарис
Итерация цикла: 4, Элемент списка: Ольха
Итерация цикла: 5, Элемент списка: Берёза
Итерация цикла: 6, Элемент списка: Ясень
 
```

В нашем примере итератор на каждом шаге цикла последовательно выдавал значения из списка `['Ольха','Берёза','Ясень','Кипарис']`. Когда значения закончились, итератор начал выдавать элементы списка с начала.

Такой подход к итерации объектов списка можно использовать не только для распределения заказов между курьерами, но и при распределении клиентов на несколько групп в последовательности их регистрации, например, подобное распределение пользователей между группами можно использовать при проведении А/В-теста:

```
# Создадим список групп:
groups = ['A','B','C']

# Создадим итератор, имитирующий генерацию id пользователей:
user_id = itertools.count(100000, 1)

# Создадим пустой список, в который будем добавлять информацию:
user_grop_list = []

# Создадим цикл, использующий бесконечный итератор cycle:
count = 0
for group in itertools.cycle(groups):
    # Для примера проведём 10 итераций:
    if count > 10:
        break
    else:
        user_grop_list.append(f'{next(user_id)}_{group}')
    count += 1
    
# Проверим результат:
print(user_grop_list) 
```

```
['100000_A', '100001_B', '100002_C', '100003_A', '100004_B', '100005_C', '100006_A', '100007_B', '100008_C', '100009_A', '100010_B'] 
```

В примере мы использовали итератор `itertools.count()` для имитации генерации бесконечного списка `user_id` с id пользователя. Затем в цикле использовали итератор `itertools.cycle()`, бесконечно перебирающий названия групп в списке `groups`. А в самом цикле к каждому новому `user_id` добавляли суффикс с названием группы из списка `groups`. Обратите внимание, что для итерации по `user_id` использовали функцию `next()`.

**Функция `repeat()`** создаёт объект-итератор, возвращающий один-единственный элемент бесконечное количество раз, если не было указано точное количество итераций. Синтаксис функции следующий `itertools.repeat(object, times)`, где `object` возвращаемое значение, а `times` — количество итераций, по умолчанию — бесконечное значение.

Например, компания решила дать дополнительную скидку в 15% первым 7 покупателям. Id покупателей поступают по мере покупки. Тогда общий цикл будет таким:

```
# Создадим итератор, имитирующий генерацию id пользователей:
user_id = itertools.count(100000, 1)

# Создадим пустой список, в который будем добавлять информацию:
user_action_list = []

# Создадим цикл, использующий бесконечный итератор repeat с количеством итераций 7:
for action in itertools.repeat('15%_Sale', 7):
    user_action_list.append(f'{next(user_id)}_{action}')
    
# Проверим результат:
print(user_action_list) 
```

```
['100000_15%_Sale', '100001_15%_Sale', '100002_15%_Sale', '100003_15%_Sale', '100004_15%_Sale', '100005_15%_Sale', '100006_15%_Sale'] 
```

В результате первым семи клиентам был добавлен префикс с названием скидки.

### Комбинаторные итераторы

Комбинаторные итераторы создают все возможные комбинации элементов из одного или нескольких итерируемых объектов. По своему функционалу они считаются конечными. Они полезны при решении задач, в которых нужно изучить все возможные комбинации данных, найти декартово произведение множеств или изучить возможные последовательности элементов в одном множестве значений.

Декартово произведение множеств — это множество, содержащее все возможные упорядоченные значения элементов исходных множеств. Например, есть два множества значений: множество `A`, содержащее значения `('X', 'Y')`, и второе множество `B`, содержащее значения `(0, 1)`, тогда декартово произведение множеств будет содержать такие варианты пересечения всех значений, как `('X0', 'X1', 'Y0', 'Y1')`.

Один из самых распространённых примеров декартова множества — все возможные результаты двух последовательных бросков игрального кубика, когда важна последовательность значений в получаемых вариантах множеств.

⚀⚁⚂⚃⚄⚅⚀⚀⚀⚀⚁⚀⚂⚀⚃⚀⚄⚀⚅⚁⚁⚀⚁⚁⚁⚂⚁⚃⚁⚄⚁⚅⚂⚂⚀⚂⚁⚂⚂⚂⚃⚂⚄⚂⚅⚃⚃⚀⚃⚁⚃⚂⚃⚃⚃⚄⚃⚅⚄⚄⚀⚄⚁⚄⚂⚄⚃⚄⚄⚄⚅⚅⚅⚀⚅⚁⚅⚂⚅⚃⚅⚄⚅⚅⚀⚁⚂⚃⚄⚅​⚀⚀⚀⚁⚀⚂⚀⚃⚀⚄⚀⚅⚀​⚁⚀⚁⚁⚁⚂⚁⚃⚁⚄⚁⚅⚁​⚂⚀⚂⚁⚂⚂⚂⚃⚂⚄⚂⚅⚂​⚃⚀⚃⚁⚃⚂⚃⚃⚃⚄⚃⚅⚃​⚄⚀⚄⚁⚄⚂⚄⚃⚄⚄⚄⚅⚄​⚅⚀⚅⚁⚅⚂⚅⚃⚅⚄⚅⚅⚅​​

Декартово произведение множеств можно использовать в аналитике для поиска всех возможных сочетаний двух или нескольких множеств параметров. Например, можно перебрать все возможные сочетания товаров из двух категорий, чтобы оценить их потенциал совместных продаж, с учётом последовательности предложения.

В **Python** для создания объекта-итератора, содержащего все возможные варианты декартова произведения нескольких множеств, можно использовать функцию `product()` из модуля `itertools`.

**Функция `product()`** создаёт объект-итератор, определяющий все возможные последовательности элементов двух множеств, реализуя декартово произведение множеств. Функция имеет следующий синтаксис: `itertools.product(*iterables, repeat=1)`, где на месте `*iterables` через запятую перечислены все коллекции (списки, кортежи, множества или другие итерируемые объекты), а параметр `repeat` указывает, какое количество раз возможно задействовать исходные множества. По умолчанию это значение равно 1.

Например, найдём декартово произведение множеств `A` и `B`:

```
# Найдём декартово произведение множеств `A` и `B`:
A = ['A', 'B']
B = [0, 1]

for i in itertools.product(A, B):
    print(i)   
```

```
('A', 0)
('A', 1)
('B', 0)
('B', 1) 
```

Или посмотрим на возможные результаты по последовательному броску двух игровых кубиков:

```
# Найдём возможные результаты по последовательному броску двух игровых кубиков:
A = [1, 2, 3, 4, 5, 6]

for i, j in itertools.product(A, repeat=2):
    print(f'{i} : {j}')  
```

```
1 : 1
1 : 2
1 : 3

...

6 : 4
6 : 5
6 : 6 
```

В примере выше, поскольку два кубика идентичны, то множество значений от 1 до 6 мы использовали один раз, но передали функции `product()` параметр `repeat=2`.

Помимо декартова произведения множеств, существуют другие виды перестановок и перебора элементов во множествах. Рассмотрим наиболее распространённые из них и которые можно решить при помощи модуля `itertools` и функций:

- `permutations()`
- `combinations()`
- `combinations_with_replacement()`

Разберём по порядку каждую функцию. Обратите внимание, что в отличие от `product()`, который может работать с несколькими множествами, эти функции всегда работают с элементами одного множества.

**Функция `permutations()`** создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов одного множества с учётом их последовательности и уникальности. В одной последовательности не может быть несколько одних и тех же элементов множества. Функция имеет следующий синтаксис: `itertools.permutations(iterable, r=None)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств. По умолчанию этот параметр = `None` и соответствует количеству элементов во множестве.

Допустим, есть множество значений, например `['A', 'B', 'C']`, функция `permutations()` найдёт все возможные варианты сочетания трёх элементов между собой, то есть `['A', 'B', 'C']`, `['A', 'С', 'В']`, `['В', 'А', 'С']` и так далее. При этом порядок элементов в образованных последовательностях имеет значение: `['A', 'B', 'C']` и `['A', 'С', 'В']` — это разные последовательности.

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C']

for i in itertools.permutations(data):
    print(i)   
```

```
('A', 'B', 'C')
('A', 'C', 'B')
('B', 'A', 'C')
('B', 'C', 'A')
('C', 'A', 'B')
('C', 'B', 'A') 
```

В результате создано шесть уникальных вариантов сочетания элементов исходного множества `['A', 'B', 'C']`, каждый из которых содержит по три элемента. Количество элементов в итоговых последовательностях было задано по умолчанию, исходя из количества элементов в исходном множестве. Мы можем поменять это значение, например, уменьшив размерность, и указать два значения в каждом множестве, передав переменную `r=2`:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C']

for i in itertools.permutations(data, r=2):
    print(i)   
```

```
('A', 'B')
('A', 'C')
('B', 'A')
('B', 'C')
('C', 'A')
('C', 'B') 
```

Обратите внимание, что `r` не может быть больше, чем общее количество элементов в исходном множестве, поскольку значения в получаемых вариантах не должны дублироваться.

Такой перебор возможных последовательностей значений множества полезен для решения задач по выбору последовательности действий, продажи продуктов, запуска маркетинговых событий и т. д. Например, можно подобрать оптимальный маршрут для такси или курьерской доставки. Предположим, что есть три адреса доставки — с помощью `permutations()` можно получить все последовательности адресов. Используя эти последовательности и `api`геосервисов, можно рассчитать длину маршрута и время и выбрать самый оптимальный вариант.

---

**Функция `combinations()`** создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов множества заданной длины без дублирования значений и учёта их последовательности, в порядке их расположения в исходном множестве. Функция имеет следующий синтаксис: `itertools.combinations(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан. Это значение не может быть больше, чем общее количество элементов в исходном множестве.

Функция `combinations()` отличается от `permutations()` тем, что не учитывает последовательность элементов: ей важно их значение. Множества `['A', 'B', 'C']` и `['A', 'С', 'В']` будут рассмотрены как одно множество `['A', 'B', 'C']`.

Рассмотрим на примере, выберем все уникальные последовательности элементов множества `['A', 'B', 'C', 'D', 'E']`:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['A', 'B', 'C', 'D', 'E']

for i in itertools.combinations(data, r=3):
    print(i)   
```

```
('A', 'B', 'C')
('A', 'B', 'D')
('A', 'B', 'E')
('A', 'C', 'D')
('A', 'C', 'E')
('A', 'D', 'E')
('B', 'C', 'D')
('B', 'C', 'E')
('B', 'D', 'E')
('C', 'D', 'E') 
```

В результате найдены все уникальные сочетания исходного множества, содержащие три различных элемента. При этом элементы в последовательностях указаны в порядке их сортировки в исходном множестве.

Такой перебор элементов множества и поиск уникальных сочетаний полезен при поиске всех возможных сочетаний элементов. Представим, что есть несколько продуктов: `Молоко`, `Сметана`, `Сгущёнка` и `Хрустящий хлеб` и поставлена задача собрать все возможные варианты из трёх продуктов.

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['Сгущёнка', 'Сметана', 'Молоко', 'Хрустящий хлеб']

for i in itertools.combinations(data, r=3):
    print(i)   
```

```
('Сгущёнка', 'Сметана', 'Молоко')
('Сгущёнка', 'Сметана', 'Хрустящий хлеб')
('Сгущёнка', 'Молоко', 'Хрустящий хлеб')
('Сметана', 'Молоко', 'Хрустящий хлеб') 
```

Возможные варианты на экране. Далее можно использовать эти сочетания для тестирования системы рекомендаций. Это поможет просмотреть, какие сочетания товаров могут быть предложены при покупке какого-то товара, и отобрать самые удачные сочетания. Например, если клиент заказал `Хрустящий хлеб`, то ему можно посоветовать продукты `Молоко` и `Сметана`.

---

В примерах выше функции из модуля `itertools` создавали последовательности значений множества значений, исходя из их уникальности, — в одном варианте не могли встретиться одни и те же элементы множества. Это полезная опция, когда необходимо учитывать уникальность товаров. Например, курьеру нужно посетить адрес доставки только один раз.

Но бывают случаи, когда необходимо учитывать возможное дублирование значений во множестве. Например, при прогнозе продаж товаров или наступлении невзаимосвязанных событий. Чтобы не учитывать уникальность значений в получаемых последовательностях, в модуле `itertools` существует функция `combinations_with_replacement()`.

---

**Функция `combinations_with_replacement()`** (комбинации с замещением) создаёт объект-итератор, содержащий все возможные варианты сочетания элементов множества заданной длины с возможностью дублирования друг друга. Синтаксис функции `combinations_with_replacement()` такой же, как функции `combinations()`.

Рассмотрим работу функции на таком примере: соберём все возможные уникальные варианты значений на двух игральных кубиках, при этом нам не так важно, какое было значение на первом кубике, а какое на втором. Пары значений `1`-`6` и `6`-`1` будут рассмотрены как один вариант — `1`-`6`. Важно итоговое уникальное количество комбинаций:

```
# Найдём все возможные уникальные варианты сочетания элементов множества:
data = ['1', '2', '3', '4', '5', '6']

for i in itertools.combinations_with_replacement(data, 2):
    print(i)   
```

Использование функции `combinations_with_replacement()` может пригодиться при необходимости проанализировать сочетание товаров или событий с учётом их повторения.

Например, маркетологи розничной системы торговых точек по продаже продуктов питания подготовили скидочные купоны для покупателей, выдаваемые после покупки случайным образом. Всего существует три типа купонов, со скидками на 2, 5 и 7%. Если покупатель собрал три купона, то он может их применить при следующей покупке, при этом скидки трёх купонов суммируются. Например, если у покупателя талоны 2, 2 и 7, то итоговая скидка будет 11%.

Маркетологам стало интересно, какие могут быть получены итоговые скидки при различном сочетании купонов. Функция `combinations_with_replacement()` поможет рассмотреть все возможные варианты суммарного значения скидки:

```
data = [2, 5, 7]

for i in itertools.combinations_with_replacement(data, r=3):
    print(f'Комбинация скидок: {i}, суммарная скидка: {sum(i)}')   
```

### Заключение

В этом уроке вы детально рассмотрели, что из себя представляют объекты-итераторы и какие функции могут их образовывать. Объекты-итераторы позволяют перебирать объекты в циклах и определяют правила, по которым этот перебор происходит.

Помимо конечных итераторов, которые завершают свой перебор при окончании всех возможных вариантов, когда количество итераций заранее определено, существуют бесконечные и комбинаторные итераторы. Они создаются с помощью библиотеки `itertools`. Это такие функции, как:

**Бесконечные итераторы `itertools`**

- `count()` — создаёт объект-итератор с бесконечной последовательностью чисел, с определённой последовательностью и началом. Синтаксис: `itertools.count(start, step)`, где `start` начальное значение, а `step` — интервал до следующего значения.
- `cycle()` — создаёт объект-итератор с бесконечным перебором всех значений в итерируемом множестве (например, в списке). Синтаксис: `itertools.cycle(iterables)`, где `iterables` — объект для итерации.
- `repeat()` — создаёт объект-итератор, возвращающий один-единственный элемент бесконечное количество раз. Синтаксис: `itertools.repeat(object, times)`, где `object` возвращаемое значение, а `times` — количество итераций, по умолчанию — бесконечное значение.

**Комбинаторные итераторы `itertools`**

- `product()` — создаёт объект-итератор, определяющий все возможные последовательности элементов множеств, реализуя декартово произведение множеств. Синтаксис: `itertools.product(*iterables, repeat=1)`, где на месте `*iterables` через запятую перечислены все коллекции (списки, кортежи, множества или другие итерируемые объекты), а параметр `repeat` указывает, какое количество раз возможно задействовать исходные множества. По умолчанию это значение равно `1`.
- `permutations()` — создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов одного множества с учётом их последовательности и уникальности. То есть в одной последовательности не может быть несколько одних и тех же элементов множества. Синтаксис: `itertools.permutations(iterable, r=None)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств. По умолчанию этот параметр `= None` и соответствует количеству элементов во множестве.
- `combinations()` — создаёт объект-итератор, содержащий все возможные уникальные варианты сочетания элементов множества заданной длины без дублирования значений и учёта их последовательности, в порядке их расположения в исходном множестве. Синтаксис: `itertools.combinations(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан.
- `combinations_with_replacement()` — создаёт объект-итератор, содержащий все возможные варианты сочетания элементов множества заданной длины с возможностью дублирования друг друга. Синтаксис: `itertools.combinations_with_replacement(iterable, r)`, где `iterable` — множество значений, а параметр `r` определяет длину будущих сочетаний множеств и должен быть обязательно указан.

# Знакомство с пользовательскими функциями и встроенными методами
## Функции в Python

В этом спринте мы сфокусируемся на изучении функций, познакомимся со встроенными функциями, научимся писать функции самостоятельно и познакомимся с принципами оформления наборов функций в библиотеки. Для начала давайте разберёмся, что такое функция и как она помогает аналитику.

### Типичные задачи

Работа над аналитической задачей обычно выглядит так:

1. **Собрать и обработать данные.** На этом этапе нужно выгрузить данные из доступных источников, отфильтровать эти данные, очистить от некачественных или просто лишних результатов. Способы сбора и обработки данных бывают разные и зависят от специфики данных и задачи.
2. **Рассчитать необходимые показатели.** Когда данные подготовлены, нужно рассчитать необходимые показатели и проанализировать их в соответствии с запросом заказчика. Возможных показателей много, чаще всего это сумма, среднее, средневзвешенное и т. д. Бывают и специфические показатели. Если аналитик работает с заказчиком долго, он хорошо знаком с частыми запросами и регулярно их рассчитывает.
3. **Оформить отчёт.** В задаче важно не только получить числа и статистику, но и познакомить заказчика с результатами, то есть оформить отчёт в информативном и доступном формате, и презентовать его. Вид отчёта может быть разным: таблица, презентация, различные визуализации — обычно это обговаривают заранее, когда формируют задачу.

![](https://pictures.s3.yandex.net/resources/image_1706817715.png)

Почти в любой работе можно найти повторяющиеся действия. На бытовом уровне хорошими примерами будут:

- оплата квитанций — каждый месяц необходимо собрать показания, внести их в платёжную систему и провести платежи,
- кормление кота — ежедневно, минимум два раза в сутки,
- даже установка будильника — регулярное вечернее дело.

Рутинные действия обычно не требуют сильной вовлечённости, но отнимают время и часто уменьшают интерес к делу.

### Что такое функция

Хорошая новость — от рутины можно избавиться:

- для оплаты квитанций — подключить автоплатежи,
- для будильника — создать шаблон либо установить время пробуждения с помощью голосового помощника,
- коту приобрести умную автоматическую кормушку.

В работе над аналитическими задачами часто тоже можно столкнуться с тем, что часть кода повторяется сначала для одних данных, потом с незначительными изменениями для других. Например:

- очистка данных от аномальных значений,
- подсчёт и заполнение пропусков в данных,
- вычисление среднемесячных и среднегодовых показателей,
- построение распределений.

В таких случаях порой приходится дублировать фрагменты кода из уже завершённых проектов. И здесь точно так же повторяющиеся действия можно выделить из процесса в отдельный блок — использовать функции.

**Функция** — маленькая программа, созданная для решения некоторой задачи. Её пишут один раз и затем при необходимости применяют одной командой.

Сравните оформление списка дел на вечер в галерее: на первой иллюстрации он выглядит длинным «полотном», а на второй — структурированным, удобным и понятным.

**Основные преимущества применения функций:**

- **Повышается скорость работы.** Аналитик применяет свои наработки для решения похожих задач, а это быстрее, чем писать код заново.
- **Удобнее поддерживать код.** С функциями код становится структурированным, детали выносятся из основного алгоритма. Код проще анализировать, дорабатывать и переиспользовать.
- **Код проще отлаживать.** Опять же потому, что код более чистый, и всё логично разделено на блоки.

Если перед вами возникают задачи, в которых можно заменить повторяющиеся действия на общее решение, — это однозначно стоит сделать.

Более того, практика показывает, что полезно сохранять решения в виде функции — как будто это инструменты в тулбоксе, которые можно использовать в любой момент.

Неформально говоря, чем выпиливать вручную двадцать деталек, лучше сделать один раз форму для штамповки таких деталек и потом использовать их хоть тысячами.

В предыдущих уроках уже работали с функциями, которые встроены в Python, например, `print()` — для вывода текста или `type()` — чтобы узнать тип переменной. Они работают так же: создатели Python определили, какие операции разработчикам пригождаются часто, и оформили фрагменты кода отдельной программой, которую применяют одной командой. Можно создавать и собственные функции — они называются **пользовательскими**. О том, как это делать, мы поговорим в следующей теме.

Во всех предложенных ситуациях стоит вынести повторяющиеся блоки кода в функцию и оптимизировать процесс.

Каждую функцию можно представить как специалиста, которому вы даёте задание, а он своими специфическими знаниями и инструментами (блоками кода, который вы поместили в функцию) его выполняет.

Подведём итоги:

- В коде приходится сталкиваться с повтором написанного кода. Повторяющиеся блоки кода стоит выносить в функции. Функции упрощают работу и делают программу понятнее.
- Функция — маленькая программа, созданная для решения задачи. Её создают один раз и применяют при необходимости.
- Оформление части кода в функцию — первый шаг к автоматизации процесса работы.

### Методы

Некоторые функции работают с разными объектами. Например, в рассмотренную выше функцию `len()` можно подать и строку, и список, и таблицу. Возьмём список и строку и посчитаем их длину :

```
mylist = ['террариум']
print("Длина начального списка:", len(mylist))

mystr = mylist[0]
print("Длина строки «", mystr, "»:", len(mystr))

mylist = list(mystr)
print("Длина преобразованного списка:", len(mylist)) 
```

```
Длина начального списка: 1
Длина строки « террариум »: 9
Длина преобразованного списка: 9 
```

Функция прекрасно обработала все объекты.

Другие же функции привязаны к конкретному объекту и выполняют специфические операции. Например, бывает нужно преобразовать строку к нижнему регистру. Такая операция актуальна только для одного объекта — строки.

**Метод** — это функция, которая привязана к определённому объекту и выполняет операции, нужные в работе именно с этим объектом.

Метод вызывается в формате `объект.метод()`, то есть метод указывается через точку после имени объекта.

Примеры часто встречаемых операций в работе со строками:

- Разделить строку. Используем метод `.split()`, по умолчанию он делит строку по пробелам, но можно указать разделитель самостоятельно:

```
print("Есть три вида лжи: ложь, наглая ложь и статистика".split())
print("кто, где, когда".split(", ")) 
```

```
['Есть', 'три', 'вида', 'лжи:', 'ложь,', 'наглая', 'ложь', 'и', 'статистика']
```

## Знакомство с библиотеками

В предыдущем уроке вы познакомились с пользовательскими функциями и встроенными методами. Они позволяют ускорять работу над часто встречающимися задачами при помощи встроенных в Python решений. Эти функции доступны по умолчанию.

Но это не всё. Если для выполнения задачи требуется какой-либо специализированный набор функций, скорее всего, можно подобрать библиотеку, в которой найдутся подходящие.

**Библиотека** — коллекция готовых функций и методов, которые помогают в решении специализированных задач.

Функции в библиотеках недоступны по умолчанию — сначала нужно подключить подходящую библиотеку и уже потом использовать хранящиеся в ней функции.

**Примеры библиотек:**

- Библиотека Math хранит математические функции и константы: синус, косинус, число π и другие. Если в задаче от заказчика придётся делать много сложных математических вычислений, то разумно подключить эту библиотеку и использовать готовые решения. Подробнее с этой библиотекой можно ознакомиться [в официальной документации](https://docs.python.org/3/library/math.html).
- Библиотека TextBlob содержит функции для обработки текстовых данных и полезна в задачах по анализу настроений текста, исправлению орфографии и перевода, поиску синонимов и многих других. Ознакомиться подробнее можно [на странице с документацией](https://textblob.readthedocs.io/en/dev/).

Обратите внимание: функции в библиотеке — на одну тему. Например, в библиотеке Math много математического: логарифмы, интегралы, производные и т. д. Но искать в ней функции для обработки звуковых дорожек — бессмысленно.

Библиотек действительно много, найти подходящую можно поиском в интернете.

Библиотеки функций, как и встроенные функции по умолчанию, упрощают работу разработчика, экономят время и усилия на написание программ.

### Установка библиотек

Библиотека — это дополнение Python. Чтобы функции из неё стали доступны, её нужно установить. Иногда для функции библиотеки написаны с помощью функций другой библиотеки — тогда, чтобы всё корректно работало, нужно установить эту библиотеку тоже.

Загрузить библиотеку поможет система `pip`. Можно представлять `pip` как огромный магазин, в котором вы можете «приобрести» нужные библиотеки. При этом `pip` — магазин премиум-класса: если вы берёте шкаф, для сборки которого нужна отвёртка, а её у вас нет, то отвёртка будет идти в комплекте. Это значит, что `pip` самостоятельно проверяет, какие библиотеки нужны, чтобы загружаемая библиотека работала правильно.

Если вы работаете в Jupyter Notebook, достаточно использовать команду, вызывающую установку, в очередной ячейке. В другом случае нужно вызвать терминал или командную строку и ввести команду.

Давайте!

```
pip install numpy 
# numpy - библиотека, c которой вы познакомитесь 
```

Посмотреть все установленные библиотеки поможет команда:

```
pip list 
```

```
Package                       Version
----------------------------- -----------
numpy                          1.19.5
matplotlib                     3.5.2
seaborn                        0.11.2 
```

В примере вывода мы видим не только наименования библиотек, с которыми будем работать в этом уроке, но и цифры рядом. Это **версия библиотеки**.

Версия библиотеки — как версия приложения. Библиотека создаётся разработчиками в конкретный момент и с определёнными функциями. Со временем в Python появляются новые возможности, пользователи предлагают новые задачи — это побуждает разработчиков дорабатывать библиотеки.

Библиотеки поздних версий содержат новые инструменты и решения, а также доработанные старые — так же, как и более поздняя версия приложения.

Чтобы понять, какая версия новее, последовательно сравнивают числа — как и при сравнении обычных чисел: слева направо. Например, сравним версии 11.12.0 и 11.15.1:

1. Первое число у обеих версий — 11. Одинаково, значит, сравниваем следующие числа.
2. Вторые числа: 11.12.0 и 11.15.1 — сравним теперь их. 12<15, значит, версия 11.12.0 более ранняя, а версия 11.15.1 — более поздняя.

Неподходящая версия библиотеки может сломать код. Python отслеживает такие нюансы и при необходимости предлагает актуализировать версию.

Просматривая список из примера с командой `pip list`, обратим внимание на seaborn. Текущая версия библиотеки — 0.11.2. На момент подготовки материала есть более новая версия библиотеки — 0.13.0. Обновим её!

Сделать это можно разными способами:

```
# после названия через == указываем версию вручную
pip install seaborn==0.13.0  
```

```
# перед названием прописываем --upgrade,
# версию можно не указывать - установится последняя доступная 
pip install --upgrade seaborn  
```

|**Команда**|**Описание её работы**|
|---|---|
|`pip install numpy`|установить библиотеку numpy|
|`pip install --upgrade numpy`|получить последнюю доступную версию библиотеки numpy|
|`pip list`|получить список всех установленных библиотек|
|`pip install numpy==1.19.5`|установить библиотеку numpy версии 1.19.5|

- Важно отслеживать версию библиотеки, так как от этого зависят её функциональные возможности и совместимость с кодом, который планируется использовать.

## Часто используемые библиотеки

В прошлом уроке вы научились устанавливать библиотеки, в этом научитесь их использовать.

### Как использовать библиотеки

Сначала установленную библиотеку нужно импортировать, то есть подключить к программе. Без этого функции, которые содержит выбранная библиотека, работать не будут.

Импортировать библиотеку стоит в самом начале. Так код будет более читаемым, а работать с окружением, которое используется для выполнения программы, станет удобнее.

Как правило, для подключения библиотек используется следующий синтаксис:

`import название_библиотеки`.

После этого обратиться к функции можно так: `название_библиотеки.функция`. Например:

```
import math
num = 100
# из math вызываем функцию log10(), 
# возвращающую логарифм num по основанию 10
print(math.log10(num)) 
```

```
# результат
2.0
# 10 ^ 2 действительно 100 
```

Если известно, что потребуется постоянно использовать функции какой-то библиотеки, можно использовать синтаксис:

`from *имя_библиотеки* import *`.

Например:

```
from math import *
# pi - константа из библиотеки math
num = pi
# из math вызываем функцию cos(), 
# возвращающую cos переданного значения num
print(cos(num)) 
```

```
# результат - cos от числа pi
-1 
```

Такой подход возможен, но требует осторожности:

- Бывает, совпадают названия функций библиотеки и обычных встроенных функций. Из-за этого работа с функцией может выдать неожиданный результат.
- Программа постоянно будет «держать» в памяти все функции библиотеки, и код будет работать медленнее.

Дальше в уроке вы обзорно познакомитесь с библиотеками, которые часто используют аналитики данных. Более подробно функции и их работу вы разберёте дальше в курсе.

### NumPy

Эта библиотека вам встречалась в предыдущем уроке. Её часто используют для вычислений, которые сложнее арифметических. Она подходит для работы со сложным математическим аппаратом и большими объёмами данных. Аналитикам данных NumPy помогает, например, обрабатывать и агрегировать наборы данных.

Чтобы разобраться, как работает определённая функция NumPy, можно обратиться к [документации библиотеки](https://numpy.org/doc/stable/reference/index.html#reference).

В сравнении с библиотекой Math у NumPy больше готовых решений и возможностей, но иногда в них сложнее разобраться. В локальных вычислениях чаще используют Math, а вот эффективнее обработать огромный набор данных смогут функции NumPy.

Библиотеку NumPy аналитики и разработчики используют часто, в ней много полезных функций — в таких случаях указывают сокращённое название:

`import название_библиотеки as сокращённое_название`.

Сокращённые названия помогают писать код быстрее. Казалось бы, убираем всего несколько символов, но вероятность опечаток уменьшается, и в больших программах разница в скорости заметна.

Сокращённое название можно придумать самостоятельно, главное — оно должно быть уникальным в коде.

Для некоторых особенно популярных библиотек есть устойчивые сокращённые названия. Для NumPy это `np`. Мы рекомендуем использовать эти устойчивые сокращения — так проще обмениваться кодом, искать рекомендации, использовать код повторно. При этом не нужно задумываться о том, как вы сократили название в прошлый раз.

Посмотрите, как использовать сокращённое название на примере программы, вычисляющей квадратный корень числа:

```
import numpy as np
# из numpy вызываем функцию sqrt()
# возвращающую квадратный корень из переданного числа
print(np.sqrt(16))  
```

```
# результат - квадратный корень из 16
4.0 
```

### Matplotlib

Matplotlib — библиотека для визуализации. В ней очень много функций и методов, она сложно организована, поэтому обычно импортируют только нужную её часть. Мы будем использовать pyplot: [`matplotlib.pyplot`](https://matplotlib.org/2.0.2/api/pyplot_api.html#module-matplotlib.pyplot) — это раздел библиотеки с функциями, которые позволяют строить графики в разных стилях: для научных статей, отчётов, презентаций результатов маркетинговых исследований и т. д.

Для сравнения: в Matplotlib больше видов графиков и больше возможностей настроить их вид, чем в Excel.

Нам пригодится функция `.plot()`. Она принимает 2 массива:

- массив x — значения точек по горизонтальной оси,
- массив y — соответствующие им значения по вертикальной оси,
- стиль точек, которые покажутся на графике. Например, `'bo'` — синие круглые точки: `b` — blue — отвечает за синий цвет, `o` — обозначение круглой формы точки.

```
from matplotlib import pyplot as plt
x = [1,2,3,4,5,6,7] # задаём значения по горизотальной оси Х
y = [21,15,10,6,10,15,21] # задаём значения по вертикальной оси Y
plt.plot(x, y, 'bo') # строим график по заданным значениям, используем стиль ‘bo’ 
```

Вывод:

![](https://pictures.s3.yandex.net/resources/image_1707163175.png)

Подробнее о библиотеке и стилях можно почитать на [официальном сайте](https://matplotlib.org/stable/api/pyplot_summary.html), посмотреть готовые решения в [этом разделе сайта](https://matplotlib.org/stable/tutorials/pyplot.html#sphx-glr-tutorials-pyplot-py).

При использовании такого подхода можно не ссылаться на библиотеку, так как в этом случае мы подтягиваем сразу нужный раздел функций. Если импортировать всю библиотеку `import matplotlib`, то нужно обращаться к библиотеке, её разделу и уже затем указывать функцию: `matplotlib.pyplot.plot()`:

```
import matplotlib 
x = [1,2,3,4,5,6,7]
y = [21,15,10,6,10,15,21]
matplotlib.pyplot.plot(x, y, 'bo')
matplotlib.pyplot.axis([0, 8, 0, 25])
matplotlib.pyplot.show()
 
```

Функции выглядят громоздкими, и код читать сложнее. Поэтому используют первый вариант — он визуально облегчает код и упрощает его чтение.

В Matplotlib много функций и возможностей, она помогает создавать эффектные графики. Минус библиотеки в том, что для создания сложных графиков может потребоваться больше времени и кода, чем с другими библиотеками для визуализации.

### Seaborn

Ещё одна библиотека для визуализации — [Seaborn](https://seaborn.pydata.org/#). Эта аналог Matplotlib, который проще использовать — здесь много готовых решений. Например, она позволяет одной строчкой кода построить популярные графики: столбчатую диаграмму, парные графики для поиска взаимосвязи между данными, ящик с усами и т. д.

С помощью Seaborn часто визуализируют данные и связанную с ними статистику, чтобы их проанализировать и сделать выводы.

Для примера построим график распределения данных — гистограмму. В нём:

- откладывают значения по горизонтальной оси,
- значения на вертикальной оси показывают, как часто горизонтальные значения встречаются в данных.

Сгенерируем значения для задачи с помощью библиотеки Numpy:

`x = np.random.normal(0, 1, size = 1000)`.

Эта строчка кода создаст массив значений, распределённых по нормальному закону. С таким распределением аналитики часто встречаются на практике, поэтому полезно посмотреть, как оно выглядит.

Чтобы построить гистограмму, используем функцию `displot(x = x, kde = True)`. Здесь:

- `х` — сгенерированный массив данных,
- параметр `kde = True` построит кривую — сглаженную версию гистограммы.

```
import numpy as np
import seaborn as sns
# формируем данные
x = np.random.normal(0, 1, size = 1000)

# строим гистограмму с помощью функции из библиотеки seaborn
sns.displot(x = x, kde = True)  
```

К функции `displot()` мы обращаемся с указанием sns. Это сокращённое название Seaborn — библиотеки, в которой функция лежит.

## Математические функции

Работа аналитика связана с расчётами и, пожалуй, это одни из самых часто повторяемых действий. Обработать данные, вычислить показатели, получить статистику и результаты внедрения процесса — для всего этого нужно последовательно выполнить математические операции.

Чем больше вопросов нужно решить в задаче и чем больше данных нужно для решения, тем сложнее формулы и вычисления. В Python много встроенных математических функций и есть библиотеки с математическим аппаратом — они автоматизируют, облегчают и ускоряют вычисления.

Начнём с понятия «математической функции». Договоримся на берегу, что под этим понимаем.

**Математическая функция** — это правило, по которому передаваемым параметрам ставится в соответствие выходной результат.

Это было формальное определение, теперь представим приземлённое.

- У нас есть аппарат, на вход которому передаётся число.
- Внутри аппарата происходит какое-то преобразование входного числа. Например, оно умножается на 2.
- Преобразованное число возвращается как результат.

→xf=2⋅x→y = 2⋅xx​f=2⋅x​y=2⋅x​

### Как использовать библиотеки с математическими функциями

В прошлом уроке вы познакомились с двумя библиотеками, которые хранят математические функции: Math и Numpy.

Решим ещё одну задачу.

**Задача.** Компания «Феерические принтеры» производит детали для офисной техники. Одну из деталей создают так: берут шар, затем его пилят пополам. Площадь среза должна быть больше 7 см27 см2 и не превышать 10 см210 см2. Станки могут произвести шары таких диаметров: 1 см, 1,15 см, 2 см, 3,5 см, 5 см, 7 см. Какие шары подойдут?

**Решение.** Нам предстоит вычислить площади срезов доступных шаров и сравнить результаты с заданными ограничениями. Вариантов шаров много — есть смысл перейти к циклу. Освежить материал по циклам можно в теме «Циклы и их организация» в уроке «Понятие цикла. Элементы цикла».

**Вспомним формулы из геометрии.** Срез шара посередине — это круг, радиус которого равен радиусу самого шара. Площадь круга вычисляется по формуле: S=π⋅R2,S=π⋅R2, где S — площадь, RR — радиус, ππ — константа. Радиус равен половине диаметра.

![](https://pictures.s3.yandex.net/resources/image_1706889862.png)

Получается, у нас есть 6 вариантов шаров, для каждого надо сделать три действия: по диаметру посчитать радиус, затем возвести его в степень и домножить на π.π. Итого 18 операций!

**Составим короткий алгоритм решения:**

- 1) Для доступного диаметра вычисляем радиус.
- 2) По формуле вычисляем площадь.

Здесь нам пригодится библиотека Math:

- В ней есть константа ππ: `math.pi`.
- Воспользуемся функцией возведения в степень: `math.pow(radius ,2)`, где `radius` — посчитанный радиус, `2` — степень, в которую возводим.

Тогда всю формулу S=π⋅R2S=π⋅R2 запишем так:

`math.pi * math.pow(radius ,2)`.

- 3) Повторяем действия для остальных радиусов — в цикле.

Напишем цикл:

```
import math

# создаём список с возможными диаметрами шара
values = [1, 1.15, 2, 3.5, 5, 7] 

# проходимся по списку диаметров
for value in values: 
    radius = value/2 # радиус - это половина диаметра
  
    # находим площадь круга по формуле
    s = math.pi * math.pow(radius ,2)
  
    # выводим результат
    print('диаметр:',value,', площадь:',s)  
```

```

диаметр: 1.15 , площадь: 1.0386890710931251

диаметр: 5 , площадь: 19.634954084936208
диаметр: 7 , площадь: 38.48451000647496 
```

Осталось учесть ограничения и выбрать подходящий шар. По условию площадь среза должна быть от 7 см27 см2 до 10 см210 см2. Выведенные результаты показывают, что в этом случае подойдёт только диаметр 3,5 см.

Задать условие поможет оператор `if`:

```

# создаём список с возможными диаметрами шара

  radius = value/2 # радиус - это половина диаметра
```

Особенность библиотеки NumPy в том, что она может работать сразу с массивом чисел. Например, чтобы возвести в степень массив чисел, не нужно писать цикл — можно применить функцию `.power(массив, степень)`. Она пройдёт по элементам массива и возведёт каждый в нужную степень.

Сначала создадим массив значений, с которым будем работать. Нам поможет `numpy.arange()`. Эта функция:

- принимает три параметра: начальное и крайнее значения, а также шаг,
- создаёт массив, в котором находятся элементы от начального до крайнего значений с заданным шагом.

```
import numpy as np

# определяем начало и конец диапазона, шаг
start, stop, step = 1, 5, 1

# создаём массив значений, с которым будем работать
array = np.arange(start, stop+step, step)

print('Исходный массив:', array)
 
```

```
Исходный массив: [1 2 3 4 5] 
```

Если указать последнее число, которое мы хотим видеть в массиве, как 3, а шаг 0,5, то получится массив `[1. , 1.5, 2. , 2.5, 3. ]`.

Теперь возведём все элементы массива в 3 степень:

```
# возводим каждый элемент в 3 степень
print('Возведение в степень 3:', np.power(array, 3)) 
```

```

```

Давайте. Например, преобразуем каждый элемент массива по намеренно усложнённой формуле x3−x4⋅ln⁡xx3−x​4⋅lnx.

## Агрегирующие функции

Результаты работы агрегирующих функций необходимы для отчётов, принятия решений, просто ознакомления с теми данными, которые мы получили, особенно, если речь идет о большом количестве данных.

## Как найти сумму значений

С операцией сложения мы давно знакомы. Однако в работе аналитики зачастую сталкиваются с ситуациями, когда нужно сложить далеко не два числа: требуется узнать сумму десятков, сотен, тысяч значений. Если решать эту задачу циклом при помощи простого сложения, то мы будем вычислять массу ненужных нам промежуточных значений — это будет медленно и неэффективно. Решить подобные задачи можно быстрее и проще при помощи агрегирующей функции, сразу получив конечное значение.

Чтобы разобраться с суммированием — одной из самых простых агрегирующих функций, вспомните чек из магазина. Данные здесь — это цены отдельных товаров в списке, кассовый аппарат складывает все эти значения, и в графе «итог» вы видите сумму к оплате.

Реализуем упрощенный кассовый аппарат при помощи уже знакомой вам библиотеки NumPy.

Освежим в памяти список ингредиентов любимого многими салата «Оливье»:

|**Товар**|**Цена**|
|---|---|
|Картофель, 2 кг|150|
|Морковь мытая, 2 шт|35|
|Яйца куриные, 10 шт|80|
|Колбаса вареная «Докторская», 0.5 кг|300|
|Горошек зелёный консервированный, 1 уп.|89|
|Майонез, 100 г|92|
|Соль, 1 кг|34|
|Огурцы маринованные, 1 уп.|120|

На основе этих расценок, посчитаем итоговую сумму для такого чека.

Сначала приведём данные к виду, с которым можем работать: создадим массив, в который положим все цены из таблицы. Чтобы найти сумму набора данных, воспользуемся функцией `sum()`, внутрь которой мы включим набор данных.

```
import numpy as np # импортируем NumPy

#задаём список цен
price_list = [150, 35, 80, 300, 89, 92, 34, 120] 

np.sum(price_list) #находим сумму заданного массива
>900 
```

Значит, итоговая стоимость такого набора продуктов — 900 денег. Мы получили результат благодаря паре строчек кода: функция `np.sum()` сагрегировала данные — вычислила сумму.

## Как найти среднее и медианное значения

Вспомним теорию статистики:

Среднее значение (среднее) — сумма всех чисел, поделенная на общее количество чисел. Используется для измерения усредненного значения ряда данных.

Медианное значение (медиана) — значение, которое разделит отсортированные по возрастанию данные на две равные части. В выстроенном от наименьшего к наибольшему значению элементов ряду данных, медиана находится ровно посередине: значений меньше медианного ровно столько же, сколько значений большего.

И среднее, и медиана позволяют оценить «середину данных», но при этом далеко не всегда эта «середина» совпадает. Среднее зависит от самих значений — если в наборе данных найдётся один элемент значительно больше других, то он заметно повлияет на сумму чисел в наборе, и, как следствие, на среднее значение. А медиана больше зависит от соотношения количества больших и маленьких чисел. Насколько экстремальны сами значения — не так существенно.

Если оцениваемые значения отличаются друг от друга несущественно, например 10, 11, 11, 12, 10 — можно использовать среднее. Но если одно или несколько значений выбиваются из общего ряда, например: 10, 11, 36, 12, 11 — необходимо использовать медиану, так как аномально большое (или маленькое) значение сильно повлияет на результат. Вывод будет опираться на одну аномалию, а не на весь ряд данных в равной мере.

### Как объявить и назвать функцию

В Python функцию определяют с помощью ключевого слова `def`.

Создадим пустую функцию (да, в Python можно и такое). Когда нужно пропустить блок кода и не выполнять его, можно использовать ключевое слово `pass`. Это бывает полезно, например, при написании циклов.

**Создаём пустую функцию**

1. Сначала укажем `def`, затем через пробел название функции: `my_first_function`.
2. После названия ставим `()`. О том, что указывают в скобках, поговорим позднее.
3. После круглых скобок следует поставить `:` и перейти на новую строку. На новой строке образовался отступ — это часть синтаксиса.  
    
4. Теперь прописываем `pass`.

Функция готова:

```
def my_first_function():
    #наполнение функции допишем позднее
    #пока используем пустую инструкцию
    pass 
```

Обратите внимание на имя функции — оно создаётся по тем же правилам, что и название переменной. Напомним их. Название функции должно быть:

- **Синтаксически корректным.** Имя функции начинается с буквы или знака нижнего подчёркивания `_` и содержит только буквы, цифры и знак нижнего подчёркивания. Нельзя добавлять другие символы, например, `!`, и разделять слова пробелом.
- **Уникальным.** В программе не должно быть двух функций с одинаковыми наименованиями. Если нарушать это правило, код может работать совсем не так, как предполагалось: та функция, которая была определена первой, станет недоступна, и всё будет обрабатывать вторая.
- **Понятным.** Здесь несколько моментов:
    - Обычно используют англоязычные названия функций. Это помогает понимать код сотрудникам из разных стран и просто хороший тон в разработке.
    - Название функции должно описывать то, что она делает — это облегчит чтение кода. Например, если функция находит количество пользователей сайта за день, то хорошим названием будет `get_users_count_day()`.
    - В питоне принята «змеиная» запись, когда слова разделены нижним подчёркиванием и состоят из строчных букв. Без разделителей понять название функции может быть трудно.
- **В меру кратким.** Длинные названия выглядят громоздко, код становится плохо читаемым. Короткие названия облегчают код и упрощают его восприятие.

Напишем функцию, которая будет выводить на экран текст о завершении работы программы.

Создавать её будем аналогично пустой функции, изменим только тело функции — ту часть, где содержится основной код, то есть то, что будет делать функция.

1. Используем ключевое слово `def`.
2. Выбираем название функции. По правилам подойдёт `print_tech_info`.
3. После названия укажем `():`

Получаем:

```
def print_tech_info(): 
```

4. Перейдём к телу функции. Её цель — вывести информацию на экран. Здесь нам поможет наша старая знакомая — встроенная функция `print()`.

```

```

5. Для соблюдения синтаксиса функции, при переходе со строки с названием, необходимо соблюдать полученный отступ, как мы это делали, когда писали самую первую функцию.

Объединяем фрагменты кода вместе и получаем пользовательскую функцию `print_tech_info()`:


Вызовем функцию и посмотрим, как она работает. Для этого пропишем название функции и скобки. Скобки будут пустыми, так как у нашей функции нет аргументов:

### Аргументы

**Аргументы** функции — изменяемые вводные данные, которые функция принимает и использует в своей работе.

Например, функция считает сумму двух чисел — эти числа подаются ей как аргументы.

Аргументами функции могут быть число, строка, список, даже другая функция. Существуют функции, которые не принимают аргументы вообще, — именно про такой пример мы писали в прошлом уроке. Бывают функции, принимающие один или любое другое число аргументов.

Рассмотрим примеры.

Функция `triple_num()` принимает один аргумент и выводит его утроенное значение:

```
def triple_num(num) : 
    print(3*num) 
```

Вызовем функцию с аргументом `2024`:

```
triple_num(num=2024) 
```

Получим:

```
6072 
```

Можно использовать короткую запись:

```
triple_num(2024) 
```

Получим тот же результат:

```
6072 
```

Функция сама не может придумать значение аргумента — его нужно ей передать. Если не указать значение аргумента, то получим ошибку:

```
triple_num()# аргумент не задан
> TypeError: triple_num() missing 1 required positional argument: 'num' 
```

Функция `get_sum()` принимает два аргумента и выводит на экран их сумму:

```
def get_sum(first_num, second_num=12):
    print(first_num + second_num) 
```

Внутри скобок указано `second_num=12`. Здесь `12` — это **значение аргумента по умолчанию**. Это значит, что при вызове функции аргументу `second_num` необязательно присваивать значение. Если присвоим, то функция возьмёт его, а если нет — будет использовать `12`.

Вот как это работает:

```
get_sum(first_num=1) 
```

Вывод:

```
# в этом случае first_num = 1 (мы задали), second_num = 12 (значение по умолчанию)
13 
```

Вот так тоже можно:

```
get_sum(first_num=1, second_num=15) 
```

Вывод:

```
# в этом случае first_num = 1 (мы задали), second_num = 15 (мы задали)
16 
```

Для `first_num` значение по умолчанию не указано, значит, его нужно передать функции. Если не передать этой функции ни одного обязательного параметра, получим ошибку:

```

get_sum() 
```

Ошибка:

```

```

### Тело функции

Тело функции — часть, содержащая блок кода с основным алгоритмом.

После круглых скобок в объявлении функции следует поставить **`:`** и перейти на новую строку — здесь начинается тело функции. На новой строке появляется отступ — это важная часть синтаксиса функции в Python, его не нужно удалять.

Вы уже многое знаете о составляющих функции — давайте решим задачу с помощью собственной функции.

**Задача.** Создать подпись в электронном письме для сотрудников компании ООО «Баг и Фича». Подпись должна выводиться в формате:

С уважением, `Имя и Фамилия сотрудника` — — — — — — — — — `должность`, ООО «Баг и Фича»

**Решение.**

1. Определим функцию:

```
def get_signature(name, position): 
```

Здесь:

`def` — ключевое слово для объявления функции;

`get_signature` _—_ название функции (англ. signature — подпись_);_

`(name, position):` — аргументы функции. Параметры, которые изменяются в задаче, — это имя и должность. Именно эту информацию мы и будем передавать в скобках.

2. Результат работы функции — вывод подписи. В теле функции предстоит использовать встроенную функцию `print()`. Оформим вывод в нужном формате. В первой строке — фиксированная часть: `С уважением,`. После неё идёт изменяемая составляющая — имя и фамилия сотрудника, их нужно взять из аргумента `name`, который передаётся функции. Помним про отступ и прописываем этот шаг в тело функции:

```
print('С уважением,', name) 
```

Вторая строка — набор символов `— — — — — — — — —`. Здесь нет изменяемых параметров, так что следующий шаг в теле функции прост:

```
print('— — — — — — — — —') 
```

Следующая строка, как и первая, содержит фиксированную и изменяемую части. Оформим её по тому же принципу, что и первую:

```
print(position,', ООО «Баг и Фича»', sep = '') 
```

Фрагмент `sep = ''` помогает избежать лишнего пробела.

3. Соберём всё вместе:

```

def get_signature(name, position): 
# вывод по образцу
# name - параметр из аргументов функции
        print('С уважением,', name) 
        print('— — — — — — — — —')
# position- параметр из аргументов функции
        print(position,', ООО «Баг и Фича»', sep = '') 
```

Готово! Теперь можно вызвать `get_signature()` и передать ей значения для аргументов `name` и `position`.

6. Александра Павлова, должность — инженер данных.

Передаём аргументам соответствующие значения:

```

```

## Вывод результатов. return

Итак, в предыдущих ~~сериях~~ уроках вы разобрались с синтаксисом объявления функции, правилами её названия, аргументами и телом функции. В этом уроке познакомитесь с последней составляющей — оператором `return`.

После оператора `return` указывают то, что функция выдаст после окончания работы. Это позволяет использовать результат функции в других частях программы.

Оператор `return` используют, когда нужно сохранить результат работы функции — в виде переменной, списка или др. Например:

```
def multiply(num1, num2):
    # перемножаем два числа
    result = num1 * num2 
    # return возвращает значение result
    return result   
    
# res_1 присваиваем результат работы multiply()
res_1 = multiply(2,5) 
# то же для res_2
res_2 = multiply(4,2) 

# выведем res_1, чтобы посмотреть, что хранится в переменной
print(res_1) 
# то же для res_2
print(res_2) 
```

Получим:

```
# res_1
10 
# res_2
8  
```

Можно провести аналогию работы функции и блендера. На вход функции подают аргументы — как в блендер складывают ингредиенты: бананы, молоко, ягоды. Функция-блендер обрабатывает всё внутри. В `return` определим результат работы блендера — коктейль.

![](https://pictures.s3.yandex.net/resources/image_1707291725.png)

Оператор`return` завершает работу алгоритма, и всё, что написано после, Python не отрабатывает. Если коктейль из блендера перелили в стакан, то у блендера задач не осталось — логично!

Зависит от того, что делает функция и нужно ли сохранить результаты её работы. Возможные варианты:

- У функции вообще нет `return`. Например, такова функция `print()` — она просто выводит на экран информацию, не надо сохранять или использовать в коде результаты её работы.
- Один `return` — частый случай. Функция что-то посчитала и вернула результат. Например, функция `sqrt()` из библиотеки NumPy извлекает квадратный корень из числа и возвращает, что получилось — результат можно использовать дальше в программе.
- Несколько return — тоже «штатная» ситуация. Рассмотрим её на примере задачи.

**Задача.** В сервисе доставки еды запускают акцию. Условия такие: если сумма в чеке с учётом доставки превышает 1000 денег, то клиенту дают скидку 10%. Создадим функцию, которая:

- принимает сумму из ресторана и стоимость доставки;
- возвращает итоговую стоимость заказа с учётом условий акции.

1. **Разберём условие.** Аргументами функций будут параметры, которые меняются от заказа к заказу — это сумма чека из ресторана и стоимость доставки. По условию акции нужно посчитать сумму этих аргументов и сравнить её с 1000. Дальше возможны два варианта:
    - Если полная стоимость больше 1000, нужно применить скидку 10% к полной стоимости — то есть вернуть 90% от неё.  
        Для этого нужно сначала разделить сумму на 100 — так мы узнаем, сколько составляет 1 процент. А затем умножить полученное значение на 90.  
        Запишем формулу и упростим её:  
          
        стоимость100⋅90=стоимость⋅0,9.100стоимость​⋅90=стоимость⋅0,9.
    - Если полная стоимость меньше или равна 1000, скидка не начисляется. В этом случае нужно вернуть просто общую стоимость.
        
        Работать с несколькими вариантами поможет условная конструкция `if`-`else`, вернуть значение — оператор `return`.
        
2. Определим `amount_estimator`:

```
def amount_estimator(sum_cafe, sum_delivery): 
```

Здесь:

- `def` — ключевое слово для объявления функции;
- `amount_estimator` — название функции;
- `(sum_cafe, sum_delivery):` — аргументы функции: стоимость заказа в ресторане и стоимость доставки соответственно.

3. Напишем тело функции. Будем действовать так, как определили в шаге 1. Сначала найдём полную стоимость:

```
total_sum = sum_cafe + sum_delivery 
```

Теперь полученное значение нужно сравнить с 1000 и реализовать два исхода.

Используем оператор `if` для первого случая: если `total_sum` больше 1000. `if total_sum>1000:` Пропишем, что делать, если условие выполняется. В этом случае надо применить скидку. Формулу мы получили выше, переведём её в код: `total_sum*0.9`. Это значение вернём как результат функции, для этого прописываем оператор `return`: `return total_sum*0.9`

```
total_sum = sum_cafe + sum_delivery
if total_sum>1000:
    return total_sum*0.9 
```

Для работы со вторым случаем (`total_sum <= 1000`) достаточно использовать оператор `else`, так как других возможных случаев не может быть. Здесь просто вернём это значение: `return total_sum`.

Получаем такой фрагмент кода с условной конструкцией:

```
if total_sum>1000:
  return total_sum*0.9
else:
  return total_sum
 
```

4. Посмотрим на весь код целиком:

```
# объявление функции amount_estimator 
# с аргументами sum1, sum2
def amount_estimator(sum1, sum2):
    # вычисление полной стоимости заказа
    total_sum = sum1 + sum2
    # сравнение с 1000
    # выбор случая, когда сумма больше 1000 
    if total_sum > 1000:        
        # возвращение значения согласно условию 
        # акции: с начисленной скидкой
        return total_sum*0.9       
        # сравнение с 1000, выбор
        # всех оставшихся случаев
    else:       
        # возвращение значения согласно условию акции
        return total_sum            
```

# Создание функций для решения задач
## Создание функции с несколькими аргументами и встроенными методами

Как вы уже знаете из предыдущих уроков, в Python функции — это способ организации и повторного использования кода. Напомним, встроенные методы — это функции, уже включенные в Python, например, `len()` или `print()`. Пользовательские функции — это функции, которые создаются самим пользователем для выполнения разных задач.

Функции в программировании и аналитике можно сравнить с рецептом приготовления блюда. Если вам нужно несколько раз приготовить одно и то же блюдо, вы не изучаете рецепт каждый раз заново. Вместо этого вы создаёте рецепт, и следуете ему каждый раз, когда готовите. Функции работают так же: они помогают нам избежать повторения кода и делают его более читабельным и организованным.

А чтобы улучшить код, можно комбинировать встроенные методы с пользовательскими функциями. Такое комбинирование в Python может значительно повысить гибкость и эффективность вашего кода подобно тому, как в повседневной жизни мы используем сочетание инструментов и индивидуальных навыков для решения задач. Как например, если вы используете готовое тесто (встроенный метод) и добавляете свои ингредиенты (пользовательская функция) для создания уникальной пиццы.

В этом уроке вы узнайте, как комбинировать встроенные методы и пользовательские функции.

Для этого представим, что вы планируете организовать мероприятие, будь то праздник, вечеринка или корпоратив.

Для начала напишем простую функцию `create_invitation` с одним аргументом, которую мы используем для создания индивидуальных приглашений. Эта функция принимает имя гостя и возвращает строку с приглашением на мероприятие, содержащую это имя.

Для создания такой функции можно использовать f-строку, которая позволяет вставлять переменные и выражения непосредственно в текст. Чтобы её применить, нужно поставить букву f перед началом кавычек, а название аргумента обрамить фигурными скобками с каждой стороны.

```
def create_invitation(name):
    return f"Уважаемый(ая) {name}, приглашаем Вас на наше мероприятие!"

# Примеры вызова функции
print(create_invitation("Анна"))
print(create_invitation("Иван"))
print(create_invitation("Ольга")) 
```
Важные составляющие любой пользовательской функции — аргументы и возвращаемые значения. Напомним, аргумент — это значение, которое передаётся функции при её вызове. Эти значения перечисляются в круглых скобках и разделяются запятыми. Сравнивая пользовательскую функцию с калькулятором можно сказать что, аргументы — это числа, которые вы вводите (например, 2 и 3), а возвращаемое значение — это результат операции (например, сумма этих чисел).

Пользовательские функции в Python могут принимать разное количество аргументов и возвращать множество значений, перечисленных через запятую. Вместе они формируются в кортеж.

Для успешного проведения мероприятия нам необходимо подобрать разные продукты. Вдруг у кого-то аллергия? Начнём с выбора фруктов. Вам понадобятся яблоки, бананы и апельсины. Напишем такую функцию `fruit_count`, которая принимает количество яблок, бананов и апельсинов, а затем возвращает общее количество фруктов.

```
def fruit_count(apples, bananas, oranges):
    return apples + bananas + oranges 
```

Используем нашу функцию. Каждое число — это количество соответствующих фруктов. Возьмем 12 яблок, 13 бананов и 9 апельсинов.

```
print(fruit_count(12, 13, 9)) 
```

Фрукты могут различаться по количеству и виду (а не только яблоки, бананы и апельсины). В связи с этим правильнее записать их в виде списка list.

```
fruits = [12, 13, 9] 
```

Напомним, что если вы хотите просуммировать элементы внутри списка, вы можете использовать метод `sum(fruits)`. В свою очередь, если вы захотите узнать, каких фруктов у вас больше всего, вы можете воспользоваться методом `max(fruits)`. В вашем случае такая функция выдаст 13. И наоборот, метод `min(fruits)` покажет, каких фруктов у вас меньше всего. Тогда функция выдаст 9.

Вам нужно учесть количество продуктов, чтобы обеспечить достаточное и разнообразное угощение для гостей. После покупки фруктов, вычислим, сколько каждого вида останется, если каждый участник съест по одному фрукту. Предположим, что у нас будет восемь гостей:

```
def remaining_fruits(participants):
    # Количество каждого вида фруктов на начало мероприятия
    apples = 12
    bananas = 13
    oranges = 9

    # Вычисляем количество оставшихся фруктов
    remaining_apples = apples - participants
    remaining_bananas = bananas - participants
    remaining_oranges = oranges - participants

    # Печатаем результаты с использованием f-строк
    print(f"Оставшиеся яблоки: {remaining_apples}")
    print(f"Оставшиеся бананы: {remaining_bananas}")
    print(f"Оставшиеся апельсины: {remaining_oranges}")

# Вывод функции
remaining_fruits(8) 
```
Оставшиеся яблоки: 4 Оставшиеся бананы: 5 Оставшиеся апельсины: 1

Если 8 гостей съедят по 1 фрукту, то останется 4 яблока, 5 бананов и 1 апельсин. Важно помнить, что для определения оставшегося количества каждого вида фруктов необходимо вычесть количество гостей из первоначального количества фруктов, так как предполагается, что каждый гость съест по одному фрукту.

Мы рассмотрели несколько примеров использования функции в Python на основе задач по организации мероприятия. В следующем уроке вы примените эти знания на практике, разрабатывая функции для проведения лучшего мероприятия.

## Создание функций с циклами и условиями

Часто возникает ситуация, когда в рамках одной пользовательской функции нужно многократно выполнять одно или несколько действий. Например:

- помесячно рассчитать финансовый показатель, чтобы построить график его динамики;
- распределить клиентов по сегментам — для каждого клиента определить, в какой сегмент он попадает по заранее определённым условиям;
- перебрать все маркетинговые кампании, чтобы определить более и менее эффективные;
- проверить каждый склад и определить, достаточно ли на в нём запасов для территории, которую он обслуживает.

Такая обработка данных нетипична, и конкретной функции, которая сразу решит такую задачу для всего массива данных, часто нет. В таких случаях помогают циклы. Напомним: циклы — это конструкции, которые используются для повторения одного или нескольких действий множество раз. Ранее в курсе вы познакомились с двумя основными типами циклов: `for` и `while`. Освежить в памяти их синтасис можно в теме «Циклы и их организация».

Ещё в пользовательских функциях можно использовать условия — когда нужно выполнить разные блоки кода в зависимости от того, истинно ли заданное условие.

Более того, циклы и условия можно комбинировать! Например, функция для подсчёта среднего значения списка чисел может:

- с помощью условия проверять перед расчётом, не пуст ли список;
- а затем с помощью цикла перебирать все числа.

Обратимся к списку стоимостей покупок из магазина электронной техники.

```
purchases = [7000, 14000, 9000, 70000, 3000] 
```

Рассмотрим функцию `price_range()` , которая принимает список цен, минимальную и максимальную границы и возвращает список цен, которые попадают в заданный диапазон.

```
def price_range(prices, min_price, max_price):
    # Создаём пустой список для хранения результатов
    result = []  
    # Итерируем по каждому элементу в списке prices
    for price in prices:  
        # Проверяем, попадает ли цена в заданный диапазон
        if min_price <= price <= max_price:  
            # Если да, добавляем цену в список результатов
            result.append(price)  
    return result 
```

Используем данную функцию на нашем списке `purchases`. Зададим нижнюю границу равной 5 000, а верхнюю — 15 000.

```
purchases = [7000, 14000, 9000, 70000, 3000]

def price_range(prices, min_price, max_price):
    # Создаём пустой список для хранения результатов
    result = []  
    # Итерируем по каждому элементу в списке prices
    for price in prices:  
        # Проверяем, попадает ли цена в заданный диапазон
        if min_price <= price <= max_price:  
            # Если да, добавляем цену в список результатов
            result.append(price)  
    return result

print(price_range(purchases, 5000, 15000)) 
```

### Списковые включения

Более элегантно решать подобные задачи и создавать новые списки на основе существующих помогают **списковые включения** (list comprehensions). Они позволяют комбинировать цикл и условие — генерировать новый список, применяя выражение к каждому элементу итерируемого объекта (например, списка), и при необходимости фильтровать элементы с помощью условного выражения.

**Списковое включение** — цикл, который включён внутрь определения списка.

Сначала возьмём обычный цикл и создадим с его помощью список квадратов чисел:

```
numbers = [1, 2, 3, 4, 5]
squared_numbers = []

for x in numbers:
    squared_numbers.append(x**2)
print(squared_numbers) 
```

Вывод:

```
[1, 4, 9, 16, 25] 
```

Задача простая, а код немного громоздкий. Списковое включение поможет с этим справиться.

Простой пример спискового включения выглядит так:

```
numbers = [2, 3, 4, 5, 6]
squared_numbers = [x**2 for x in numbers]
print(squared_numbers) 
```

Вывод:

```
[4, 9, 16, 25, 36] 
```

В этом примере `squared_numbers` содержит квадраты чисел из списка `numbers`. Здесь `[x**2 for x in numbers]` — это списковое включение. Оно итерирует по каждому элементу `x` из списка `numbers` и генерирует новый список, содержащий квадраты этих чисел. Списковое включение может итерироваться также по генераторам `range` и другим итерируемым объектам.

Чтобы дополнительно фильтровать результат, нужно добавить в конце спискового включения `if` и после него логическое выражение, которое будет проверяться для каждого `х`.

Например, получим квадраты только чётных чисел от 1 до 10:

```
print([x**2 for x in range(1, 11) if x % 2 == 0]) 
```

```
[4, 16, 36, 64, 100] 
```

Коротко и элегантно!

Рассмотрим пример посложнее и запишем функцию `price_range()`, о которой мы говорили выше, с помощью списковых включений.

С помощью списка включений создадим новый список `prices`, который содержит только те цены, которые попадают в диапазон между `min_price` и `max_price`.

```
purchases = [7000, 14000, 9000, 70000, 3000]

def price_range(prices, min_price, max_price):
    return [price for price in prices if min_price <= price <= max_price]
print(price_range(purchases, 5000, 15000)) 
```

Вывод:

```
[7000, 14000, 9000] 
```

Что происходит в коде:

1. Сначала функция перебирает все элементы списка `prices`.
2. Затем для каждой цены (`price`) из списка `prices` проверяется, удовлетворяет ли она условию `min_price <= price <= max_price`. Это условие гарантирует, что в результат попадут только те цены, которые находятся в заданном диапазоне.
3. Если цена удовлетворяет условию, она добавляется в новый список.

Таким образом, результатом работы функции является список, содержащий только те цены, которые находятся в заданном диапазоне.

Результатом будет тот же список `[7000, 14000, 9000]`.
