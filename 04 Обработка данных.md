# Первые расчёты
## Что можно подсчитать, а что нет? Первые расчёты

В прошлых уроках вы узнали, в каком формате аналитик обычно изучает данные, что такое **DataFrame** и как в нём хранится информация, а также как быстро и эффективно фильтровать данные, чтобы получить выборку с необходимыми значениями. После получения необходимых данных аналитик переходит к знакомству с ними. На этом этапе важно понимать, в каком виде и качестве поступили данные для анализа, какой их объём, сколько уникальных значений в категориальных данных и какие статистические показатели имеют численные данные. А ещё — есть ли в них пропуски, дубликаты, ошибки, аномалии и выбросы.

Например, опечатки в названии категорий или сегментов могут привести к неточным подсчётам метрик для этих сегментов, дублирующие значения в данных могут завысить финансовые показатели, а аномально высокие или низкие значения (выбросы) могут повлиять на среднее арифметическое значение.

Когда данные полностью изучены и не содержат ошибок, то и полученные на их основе результаты будут более достоверными и надёжными. Следить за качеством полученных данных, проводить корректную их подготовку перед самим анализом данных — это зона ответственности аналитика.

Для получения необходимой информации о данных аналитики проводят различные расчёты. Часть из них может быть простым подсчётом количества различных значений, а другая часть будет базироваться на расчёте значений при агрегации данных.

В первом случае считать можно количество строк в датафрейме, общее количество данных в столбце, уникальные значения категориальных данных.

Подсчёт общего количества строк в датафрейме и уникальных значений помогает получить представление об объёме данных и их структуре. Например, подсчёт количества строк в пользовательских логах даст представление о количестве событий, а подсчёт количества уникальных id пользователей — о том, сколько пользователей совершили эти события.

Подсчёт количества уникальных значений по различным категориям помогает лучше понять содержимое данных: какая категория встречается чаще всего, а какая реже. Например, сколько уникальных категорий товаров представлены в данных или какие страницы приложения и сколько раз посещали пользователи.

Второй подход к знакомству с данными и их изучению — расчёт значений при агрегации данных. Он часто используется, когда нужно оценить разброс значений в выборке — найти минимальное и максимальное значения или среднее арифметическое. Изучение статистических показателей по столбцам позволяет оценить минимальные и максимальные значения численных данных или изучить их среднее значение. Например, подсчитать минимальную и максимальную сумму заказа, найти средний чек покупателя. Агрегировать данные можно по столбцам в целом, а можно по определённым категориям или группам. Например, рассчитать среднюю длительность просмотра контента онлайн-кинотеатра можно по всем просмотрам, а можно отдельно для каждой категории контента.

Агрегацию данных мы рассмотрим в последующих уроках, а в этом уроке остановимся на том, как проводить расчёты количества данных:

- как подсчитать количество строк в датафрейме с помощью свойства **DataFrame** `shape` и функции **pandas** `len()`, а также получить информацию о датафрейме с помощью метода `info()`;
- как найти количество значений в столбце с помощью метода `.count()`;
- что такое уникальные значения и как их подсчитать, используя методы `.unique()`, `.nunique()`, `.value_counts()`.

Изучать значения мы будем на примере базы данных Яндекс Музыки.

### Подсчёт числа строк в таблице

Подсчёт строк в датафрейме — одно из распространённых действий при знакомстве с данными. Он помогает понять объёмы данных, поступивших на анализ, или оценить размер выборки, которая была сформирована после фильтрации данных. Например, подсчёт количества строк в данных о продажах интернет-магазина за последний месяц, где каждая строка — это уникальная продажа, позволит оценить общее количество заказов.

### **Атрибут DataFrame `.shape`**

Перед анализом данных полезно изучить их объём, например, количество строк в датасете. Подсчитать количество строк **DataFrame** можно несколькими способами. Один из них — свойство (атрибут) **DataFrame** `shape` (от англ. «форма, облик»). Это свойство **DataFrame** возвращает кортеж, который содержит информацию о количестве строк и столбцов датафрейма:

```
# Импортируем библиотеку pandas:
import pandas as pd

# Загрузим датасет Яндекс Музыки и скорректируем названия столбцов:
music_df = pd.read_csv('music_log.csv').rename(columns={'  user_id':'user_id', 
                                'total play':'total_play', 'Artist':'artist'})

# Выведем информацию о количестве строк и столбцов с помощью свойства shape:
print(music_df.shape) 
```

```
(67963, 5) 
```

Первое значение (`67963`) — это количество строк, а второе (`5`) — количество столбцов. Чтобы вывести информацию только о количестве строк, можно использовать индексацию:

```
# Выведем информацию о количестве строк с помощью свойства shape:
print(f'Количество строк в датафрейме music_df: {music_df.shape[0]}') 
```

```
Количество строк в датафрейме music_df: 67963 
```

В некоторых предыдущих уроках вы уже использовали это свойство датафрейма для определения количества строк в датафрейме после фильтрации данных.

### **Функция Python `len()`**

Подсчитать количество строк в датафрейме можно с помощью функции **Python** `len()`. Эту функцию вы уже использовали для подсчёта количества значений в списке или для подсчёта значений в строке. С объектом **DataFrame** функция `len()` будет работать аналогичным способом: достаточно передать в неё датафрейм и в качестве результата появится количество строк:

```
# Подсчитаем количество строк в датафрейме music_df с помощью функции len():
print(f'Количество строк в датафрейме music_df: {len(music_df)}') 
```

```
Количество строк в датафрейме music_df: 67963 
```

Результат аналогичен результату `shape`. Преимущество функции `len()` в том, что она возвращает только количество строк, что упрощает вывод результата, в отличие от `shape`, когда нужно подсчитать только это значение. Также на большом объёме данных `len()` будет работать быстрее, чем `shape`.

Для изучения количества строк можно использовать метод pandas `.info()` (от англ. «информация»). Этот метод позволяет получить основную информацию о датафрейме, включая информацию об индексе и типе столбцов, количестве заполненных строк в каждом столбце, а также объёме использованной памяти.

### Подсчёт количества значений в столбце

Помимо подсчёта количества строк в датафрейме, часто бывает необходимость подсчитать общее количество значений в столбце, то есть строк, которые содержат данные. На стадии знакомства с данными это позволяет выявить возможные ошибки в данных — пропуски, а также предварительно оценить какие-то параметры или показатели. Например, в датасете с информацией о пользовательской активности количество строк в данных о заказах будет давать представление об их количестве. В последнем примере мы получали эту информацию с помощью метода `.info()` — столбец `Non-Null Count`:

![](https://pictures.s3.yandex.net/resources/4.4.1_2_1711222287.png)

_Метод .info() выводит в столбце Non-Null Count количество заполненных строк._

Однако в **pandas** существует ещё один способ получить информацию о количестве строк с данными — это метод `.count()` (от англ. «считать», «подсчитывать»). Этот метод можно применить как к самому **DataFrame**, так и к **Series**, то есть к столбцу датафрейма. Рассмотрим пример применения метода к **DataFrame**:

```
# Применим метод .count() к датафрейму music_df
# и подсчитаем количество заполненных строк:
print('Количество заполненных строк в датафрейме music_df:')
print(music_df.count()) 
```

```
Количество заполненных строк в датафрейме music_df:
user_id       67963
total_play    67399
artist        59647
genre         64661
track         64804
dtype: int64 
```

Результатом будет информация о заполненных строках в датафрейме в виде **Series**, где индекс — это название столбца, а значение **Series** — количество заполненных строк. Полученную информацию можно использовать для анализа полноты данных и для общей оценки наличия пропущенных значений в данных.

Метод `.count()` в случае применения к **DataFrame** имеет параметр `axis`, который можно использовать для подсчёта количества значений не в столбце, а в строках. По умолчанию значение параметра равно `0`, что означает подсчёт значений по столбцам. Если передать параметру `axis=1`, то подсчёт будет вестись по строкам:

```
# Применим метод .count() к датафрейму music_df
# И подсчитаем количество заполненных значений по строкам:
print('Количество заполненных строк в датафрейме music_df:')
print(music_df.count(axis=1)) 
```

```
Количество заполненных строк в датафрейме music_df:
0        5
1        5
2        5
3        4
4        5
        ..
67958    5
67959    5
67960    5
67961    5
67962    2
Length: 67963, dtype: int64 
```

Результатом также будет **Series**, где индекс — это индекс строки, а значение — количество столбцов, которые содержат значения. Такой подход можно использовать для проверки полноты данных по строкам. Например, создадим срез данных, который будет содержать неполные данные:

```
# Создадим логическую маску, где значение True будет соответствовать строкам,
# которые содержат неполные данные хотя бы в одном из столбцов:
mask = music_df.count(axis=1) < music_df.shape[1]

# Подсчитаем количество строк с неполными данными:
print(len(music_df[mask])) 
```

```
8767 
```

8767 строк содержат неполные данные хотя бы в одном из столбцов датафрейма.

В примере выше мы использовали логическую маску для подсчёта строк, которые содержат хотя бы один незаполненный столбец: `music_df.count(axis=1) < music_df.shape[1]`. В левой части логического условия для каждой строки датафрейма `music_df` было подсчитано число заполненных столбцов `.count(axis=1)`. Для этого мы использовали метод `.count()` с параметром `axis=1`. Затем значение было сравнено с общим количеством столбцов датафрейма `music_df`. Для этого мы использовали атрибут **DataFrame** `.shape`, второе значение, которое содержит количество столбцов `.shape[1]`.

Метод `.count()` можно также применить к определённому столбцу:

```
# Применим метод .count() к столбцу artist датафрейма music_df:
print('Количество заполненных строк в столбце artist:', music_df['artist'].count()) 
```

```
Количество заполненных строк в столбце artist: 59647 
```

Результатом будет целое число **int** — количество заполненных строк в столбце датафрейма. В отличие от применения к **DataFrame**, метод `.count()`, применяемый к столбцу, не имеет дополнительных параметров.

Обратите внимание, что вышеуказанные способы подсчитывают количество значений в столбце или строке, игнорируя незаполненные ячейки **DataFrame**. Поэтому для подсчёта общего количества строк в столбце следует использовать функцию **Python** `len()`. Как и в случае её применения к **DataFrame**, функция `len()` вернёт длину **Series**, то есть количество строк в столбце:

```
# Применим метод .count() к столбцу artist датафрейма music_df:
print('Общее количество строк в столбце artist:', len(music_df['artist'])) 
```

```
Общее количество строк в столбце artist: 67963 
```

### Подсчёт уникальных значений в столбце

В своей работе аналитик часто сталкивается с необходимостью подсчитать не только общее количество строк в столбце, но и количество уникальных значений в столбце. Например, количество уникальных пользователей, товаров, категорий. Для этого в **pandas** существует несколько методов, которые позволяют вывести уникальные значения, подсчитать их количество, а также изучить распределение данных по уникальным значениям.

Анализ уникальных значений также помогает получить общее представление о данных, например, узнать количество уникальных клиентов или товаров в магазине или найти уникальное количество заказов. Обычно уникальные значения подсчитываются для категориальных данных и данных, которые подразумевают уникальные номера — id. Например, id клиентов или транзакций.

### **Поиск уникальных значений в столбце, методы `.unique()` и `.nunique()`**

Для поиска уникальных значений в столбце используют метод `.unique()` (от англ. «уникальный»). Метод применяется к данным типа **Series**, то есть столбцам датафрейма. Например, найдём все уникальные значения в столбце с жанром музыкальной композиции:

```
# Выведем на экран уникальные значения музыкальных жанров:
print(music_df['genre'].unique()) 
```

Результатом будет массив данных. В примере с данными Яндекс Музыки массив получился большой, поскольку уникальных жанров много. Отобразим его небольшую часть:

```

 'hiphop' nan 'spoken' 'new' 'latin' 'extrememetal' 'instrumental'
 'classical' 'alternative' 'rock' 'german' 'french' 'metal' 'dubstep'
 'house' 'miscellaneous' 'rap' 'world' 'country' 'punk' 'rusrap' 'rnb'
 'beats' 'ukrrock' 'inspiritual' 'ruspop' 'caucasian' 'rusrock' 'dub'
 'soundtrack' 'folk' 'shanson' 'fairytail' 'hard-n-heavy' 'romance' ... ] 
```

При выводе подобных данных для их анализа полезно провести сортировку значений по возрастанию с использованием метода `.sort_values()` в цепочке методов:

```
# Выведем на экран уникальные значения мызыкальных жанров:
print(music_df['genre'].sort_values().unique()) 
```

```

 'bigroom' 'black' 'blip' 'bluegrass' 'blues' 'bolero' 'boleros'
 'bollywood' 'bossa' 'brazilian' 'breakbeat' 'breaks' 'broadway' 'broken' ... ] 
```

В этом случае метод `.sort_values()` применяется перед методом `.unique()`. Такой подход можно использовать для анализа уникальных значений категорий и поиска среди них значений, дублирующих друг друга, то есть неявных дубликатов. Например, выше можно заметить, что жанры `'bolero'` и `'boleros'` близки по смыслу. Эти два значения категорий дублируют друг друга, поэтому `'boleros'` можно заменить на `'bolero'`. Похожие кейсы мы рассматривали в предыдущих уроках с заменой данных.

Для подсчёта количества уникальных значений в **pandas** существует специальный метод — `.nunique()` (от англ. n — number — «количество уникальных»). Этот метод используется так же, как и `.unique()`:

```
# Выведем на экран количество уникальных значений музыкальных жанров:
print('Количество уникальных жанров:', music_df['genre'].nunique()) 
```

```
Количество уникальных жанров: 304 
```

Метод `.nunique()` возвращает целое число — количество уникальных значений. Данные Яндекс Музыки содержат информацию о 304 уникальных жанрах!

Метод `.nunique()` можно также применить к датафрейму:

```
# Подсчитаем количество уникальных значений во всех столбцах music_df:
print(music_df.nunique()) 
```

```
user_id       43240
total_play    37003
artist        55726
genre           304
track         51443
dtype: int64 
```

Результатом действия метода будет серия значений, в которой содержится информация о количестве уникальных значений в каждом столбце датафрейма.

### **Частота встречаемости уникальных значений, метод `.value_counts()`**

При анализе данных часто возникает необходимость не только найти и проверить корректность уникальных категориальных данных и подсчитать их общее количество, но и проанализировать распределение данных по этим категориям. Это значит — подсчитать частоту встречаемости уникальных значений в строках датафрейма. Например, это может быть полезным при подсчёте количества продаж товаров различной категории или определении количества просмотров фильмов различных категорий.

Для этого в **pandas** существует метод `.value_counts()` (от англ. «количество значений»). Этот метод можно применить как к отдельным столбцам **Series**, так и ко всему **DataFrame**. Подсчитаем частоту встречаемости уникальных значений жанров в данных Яндекс Музыки, применив метод `.value_counts()` к столбцу `genre`:

```
# Подсчитаем частоту встречаемости уникальных жанров:
print(music_df['genre'].value_counts())

При этом данные будут отсортированы в порядке убывания значений, то есть от популярных значений к редко встречающимся. ## Агрегирующие функции: sum, min, max

В прошлом уроке вы узнали, как подсчитать строки, столбцы и уникальные значения в данных. Это помогает оценивать объём данных и выделять в них уникальные фрагменты. Однако часто требуется не только посчитать число самих данных, но и поработать с их значениями.

Чаще всего данные содержат какие-то количественные значения, состоящие из целых или дробных чисел. Это значения разного содержания: количество просмотров и покупок, стоимость товара, длительность музыкального трека, скорость загрузки страниц приложения, время доставки продуктов и многое другое.

При анализе аналитик часто проводит какие-то операции с такими данными: суммирует, находит минимальные и максимальные значения, изучает статистические показатели, обобщает результаты по группам или категориям, сравнивает значения для различных сегментов между собой. Например, рассчитывает минимальное и максимальное время загрузки страницы приложения, средний чек покупки или среднее время доставки блюда из ресторана.

Функции, которые рассчитывают значения по набору данных, называют **агрегирующими функциями**. Эти функции обрабатывают множество значений, а в качестве результата возвращают одно значение — сводную информацию, которая характеризует данные. Например, минимальное или максимальное значение, среднее или медиану.

В **pandas** существуют различные методы, которые позволяют выполнять необходимые действия с объектами **DataFrame** и **Series**. В этом уроке вы узнаете:

- как подсчитать сумму значений в столбце с помощью метода `.sum()`;
- как найти минимальное и максимальное значения данных с помощью методов `.min()` и `.max()`.

А в следующих уроках познакомитесь с другими методами.

### Расчёт суммы значений столбца, метод `.sum()`

Подсчёт суммы значений по столбцу или определённому срезу данных используется для получения общей информации о данных. Например, он помогает проанализировать общую сумму, затраченную на рекламные каналы привлечения пользователей, или узнать суммарное количество купленных товаров в заказах.

Для этого в **pandas** существует метод `.sum()` — в переводе с английского «суммировать». Метод суммирует все значения в столбце датафрейма. Например, у вас есть данные о покупателях интернет-магазина, с которым вы познакомились в тренажёре одного из уроков. Вы можете найти суммарную стоимость всех покупок:

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
buyers = pd.read_csv('/datasets/orders_shop_buyers.csv')

# Подсчитаем сумму значений в столбце total_price:
print('Общая сумма покупок составляет:', buyers['total_price'].sum()) 
```

```
Общая сумма покупок составляет: 1200317.63 
```

При применении метода к столбцу результатом будет число. Метод `.sum()` можно применить как к серии данных, **Series**, так и ко всему датафрейму, то есть к объекту **DataFrame**. В последнем случае результатом будут сводные данные по всем столбцам, а результат работы метода будет представлен в виде серии, где индекс — название столбца, а значение — сумма:

```
# Подсчитаем сумму значений во всём датафрейме:
buyers.sum() 
```

```
first_name     AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA...
last_name      AcharyaAdamsAdebayoAgarwalAhmadAlamiAminAnandB...
email          AAcharya1987@outlook.infoAAdams1961@gmail.comA...
orders                                                      7123
items                                                      14325
total_price                                           1200317.63
dtype: object 
```

В случае количественных данных метод `.sum()` вернул сумму всех значений, а в случае текстовых — соединил все значения в одну строку. Иногда это плохо, потому что полезной информации в суммировании строковых данных часто нет, а подсчёт может занять много времени. Чтобы этого избежать, метод `.sum()` имеет параметр `numeric_only` со значением `True`, который будет подсчитывать сумму только в столбцах с количественными данными:

```
# Подсчитаем сумму значений в столбцах, которые содержат количественные данные:
buyers.sum(numeric_only=True) 
```

```
orders            7123.00
items            14325.00
total_price    1200317.63
dtype: float64 
```

Теперь суммы подсчитаны только для количественных значений. Можно провести первичный анализ данных и узнать, сколько было совершено заказов, сколько товаров куплено и на какую общую сумму.

При работе с несколькими столбцами можно подсчитывать сумму не только по столбцам, но и по строкам. За это отвечает параметр `axis`, значение которого по умолчание равно `0`, что подразумевает сложение значений в столбцах. Значение параметра `axis=1` позволит подсчитать сумму для каждой строки.

Рассмотрим на небольшом примере: создадим датафрейм `df`, который содержит сводную информацию о количестве регистраций пользователей на ресурсе по дням в разрезе мобильной и десктопной версии приложения:

```
# Импортируем библиотеку pandas:
import pandas as pd

# Создадим датафрейм с количеством новых регистраций по дням:
data = {'2024-01-17': [197, 117], '2024-01-18': [153, 71],
        '2024-01-19': [86, 120], '2024-01-20': [138, 108],
        '2024-01-21': [89, 138]}

df = pd.DataFrame(data, index=['mobile', 'desktop'])

df 
```

### Нахождение минимального и максимального значений, методы `.min()` и `.max()`

Аналитик в своей работе обычно изучает не одно значение, а целую выборку, которая даёт представление об исследуемом объекте. Например, стоимость различных продуктов в интернет-магазине, количество посетителей в день на сайте, длительность просмотров видеоконтента в онлайн-кинотеатре, время взаимодействия пользователя с ресурсом или количество скачиваний и установок приложения.

При этом значения в выборках могут широко варьироваться. Есть дорогие продукты или товары, есть более дешёвые. Есть режиссёрские версии фильмов на четыре часа, а есть короткометражные ленты на 20 минут. Особенно широкий разброс обычно приобретают те показатели, которые зависят от поведения пользователей. Существуют пользователи с высокой степенью вовлечения в продукт: они будут чаще других совершать заказы, дольше пользоваться приложением или проходить онлайн-игры несколько раз. А другие быстро забросят сайт или приложение.

![](https://pictures.s3.yandex.net/resources/4.4.1.2_1_1711222415.png)

Как правило, выборки различных значений и метрик имеют определённый разброс. Поэтому аналитику важно определить границы выборки и оценить её другие статистические параметры, которые мы ещё разберём отдельно. Минимальное значение в выборке — это самое наименьшее число или элемент. Например, самый низкий средний чек или самое короткое видео. Максимальное значение — самое высокое значение в выборке. Разница между минимумом и максимумом показывает интервал значений.

Изучение минимальных и максимальных значений также помогает обнаружить экстремальные значения или ошибки в данных. Например, отрицательную длительность фильма или стоимости заказа. Или нереалистично завышенные значения, такие как возраст клиента свыше 200 лет.

![](https://pictures.s3.yandex.net/resources/4.4.1.2_2_1711222420.png)

Чтобы найти минимальное или максимальное значение в столбце или в выборке данных, в **pandas** существуют специальные методы: `.min()` и `.max()`, в переводе с английского «минимум» и «максимум». Вернёмся к данным по продаже устройств и найдём минимальное и максимальное значения в столбце `total_price`:

```
# Найдём минимальное и максимальное значения в столбце total_price:
print('Минимальное значение в столбце total_price:', buyers['total_price'].min())
print('Максимальное значение в столбце total_price:', buyers['total_price'].max()) 
```

```
Минимальное значение в столбце total_price: 0.0
Максимальное значение в столбце total_price: 3174.0 
```

Результатом работы методов `.min()` и `.max()` будет числовое значение. Обратите внимание, что минимальное значение стоимости товара равно 0. Нужно проверить — это может быть как какая-то акция или бонус, так и ошибка в данных.

По аналогии с методом `.sum()` методы `.min()` и `.max()` можно применить для объектов типа **DataFrame**, тогда в результате будут минимальные значения по всем столбцам датафрейма:

```
# Найдём минимальные значения в каждом из столбцов:
buyers.min() 
```

```
first_name                             A
last_name                        Acharya
email          AAcharya1987@outlook.info
orders                                 0
items                                  0
total_price                          0.0
dtype: object 
```

Если данные в столбце представлены в виде строк, то методы `.min()` выведут на экран значения в соответствии с лексикографическим порядком, в котором строки сравниваются посимвольно.

В Python этот порядок основан на коде символов. Например, буква `F` имеет код `70`, а буква `ф` — `1092`. Соответственно, `.min()` покажет строку, которая начинается с символа, которому соответствует наименьшее значение кода символа, `.max()` — строку с максимальным значением:

```
# Найдём минимальное и максимальное значения в серии значений:
print('Минимальное значение в серии символов:', pd.Series(['F','ф']).min())
print('Максимальное значение в серии символов:', pd.Series(['F','ф']).max()) 
```

```
Минимальное значение в серии символов: F
Максимальное значение в серии символов: ф 
```

Узнать код символа можно, используя функцию **Python** `ord()`. Например, `ord('ф')` выдаст результат `1092`. Если несколько строк начинаются с одного символа, то далее сравниваются следующие символы.

Если серия будет содержать как строковые, так и числовые значения — найти минимальное или максимальное значения не получится. Работа методов `.min()` и `.max()` приведёт к ошибке.

Как и метод `.sum()`, методы `.min()` и `.max()` имеют параметр `axis`, который позволяет искать минимальные и максимальные значения по строкам. Вернёмся к сводным данным по регистрации пользователей в онлайн-приложении и найдём минимальное и максимальное количество зарегистрированных пользователей в разрезе каждого устройства:

```
# Импортируем библиотеку pandas:
import pandas as pd

# Создадим датафрейм с количеством новых регистраций по дням:
data = {'2024-01-17': [197, 117], '2024-01-18': [153, 71],
        '2024-01-19': [86, 120], '2024-01-20': [138, 108],
        '2024-01-21': [89, 138]}

df = pd.DataFrame(data, index=['mobile', 'desktop'])

# Найдём минимальное количество в разрезе строки:
print('Минимальное количество зарегистрированных пользователей по устройствам:')
print(df.min(axis=1))

# Найдём максимальное количество в разрезе строки:
print('Максимальное количество зарегистрированных пользователей по устройствам:')
print(df.max(axis=1)) 
```

```
Минимальное количество зарегистрированных пользователей по устройствам:
mobile     86
desktop    71
dtype: int64
Максимальное количество зарегистрированных пользователей по устройствам:
mobile     197
desktop    138
dtype: int64 
```

Для этого мы передали параметру `axis` методов `.min()` и `.max()` значение `1`.

По своему результату методы **pandas** `.sum()`, `.min()` и `.max()` очень похожи на одноимённые функции **Python** — `sum()`, `min()` и `max()`. В следующих уроках вы познакомитесь с другими, более сложными функциями, которые помогут получить ещё больше информации из имеющихся данных.

## Агрегирующие функции: mean, median

Из предыдущего урока вы знаете, что значения в выборке могут варьироваться в диапазоне от минимального к максимальному. Но иногда нужно знать не только диапазон, но и распределение значений внутри этого диапазона.

В этом уроке вы узнаете:

- Как подсчитать среднее арифметическое значение и медиану, используя методы **pandas** `.mean()` и `.median()`.
- Как с помощью метода `.describe()` получить основную статистическую информацию о данных.

### Расчёт среднего арифметического значения, метод `.mean()`

Перед расчётом среднего арифметического значения нужно разобраться, что это такое.

Представьте, что, находясь проездом в Новосибирске, вы решили встретиться с друзьями в одном из кафе и выпить чашечку кофе за приятным разговором. Перед встречей вам стало интересно, какие вообще цены на кофе в Новосибирске. Из открытых источников вы проанализировали 49 заведений и записали цену двойного эспрессо в каждом из них:

```
# Создадим серию, содержащую стоимость двойного эспрессо в кафе Новосибирска:
coffee = pd.Series([110, 110, 500, 85, 369.99, 110, 85, 85, 85, 85, 110, 93, 
                   95, 110, 85,  85, 110, 85, 85, 85, 85, 85, 110, 85, 85,
                   85, 85, 120, 75, 75, 85, 85, 90, 85, 85, 70, 85, 85, 85, 
                   85, 85, 100, 85, 85, 85, 85, 85, 100, 85])
                   
# Найдём минимальное и максимальное значения в серии значений:
print('Минимальное значение:', coffee.min())
print('Максимальное значение:', coffee.max()) 
```

```
Минимальное значение: 70.0
Максимальное значение: 500.0 
```

Получается, что стоимость двойного эспрессо варьируется от 70 до 500 рублей — в зависимости от заведения.

Минимум и максимум мы узнали, но не получили представления о том, сколько же обычно стоит чашка двойного эспрессо. 70 и 500 — большая разница, и хотелось бы получить какое-то типичное, среднее значение.

![](https://pictures.s3.yandex.net/resources/4-1-3_1_1711217631.png)

Часто для характеристики выборки одним значением используют среднее арифметическое. Вы наверняка сталкивались с ним в школе. Для его вычисления нужно сложить все значения выборки и поделить на их количество:

```
# Найдём сумму всех значений и подсчитаем их количество:
print('Сумма значений:', coffee.sum())
print('Количество значений:', coffee.count())

# Найдём среднее значение:
print(f'Средняя стоимость чашки кофе: {coffee.sum()/coffee.count():.2f} руб.') 
```

```
Сумма значений: 5092.99
Количество значений: 49
Средняя стоимость чашки кофе: 103.94 руб. 
```

По нашим расчётам, средняя стоимость чашки двойного эспрессо в Новосибирске — примерно `104 рубля`.

Конечно нет. В **pandas** существует специальный метод `.mean()`, с английского переводится как «средний». Он похож на другие методы, которые проводят агрегированные вычисления: `.sum()`, `.min()` или `.max()`. Например, вот как гораздо быстрее подсчитать среднее значение стоимости двойного эспрессо:

```
# Найдём среднее значение:
print(f'Средняя стоимость чашки кофе: {coffee.mean():.2f} руб.') 
```

```
Средняя стоимость чашки кофе: 103.94 руб. 
```

В примере выше метод `.mean()` был применён к серии значений. Его можно также использовать и для нескольких столбцов датафрейма:

```
# Найдём среднее значение по столбцам 'orders' и 'items':
print(buyers[['orders','items']].mean()) 
```

```
orders    0.875276
items     1.760261
dtype: float64 
```

И даже для всего датафрейма, чтобы оценить средние значения во всех численных столбцах:

```
# Найдём среднее значение по всем численным столбцам датафрейма:
print(buyers.mean(numeric_only=True)) 
```

```
orders           0.875276
items            1.760261
total_price    147.495408
dtype: float64 
```

При использовании расчёта средних значений для датафрейма, в котором есть не только числовые значения, лучше использовать параметр `numeric_only=True`. Так метод не попытается считать среднее значение столбцов, в которых нет чисел, — это поможет избежать ошибок и перегрузки памяти ненужными вычислениями.

Возвращаемся к кофе и переходим к новому методу.

### Расчёт медианы, метод `.median()`

Вы определили, что средняя стоимость двойного эспрессо составляет 103.94 руб. Но если отсортировать цены в порядке возрастания, получится интересная картина:

```
# Отсортируем значения в порядке возрастания:
print(list(coffee.sort_values())) 
```

```
[70.0, 75.0, 75.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 
85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 
85.0, 85.0, 85.0, 85.0, 85.0, 85.0, 90.0, 93.0, 95.0, 100.0, 100.0, 110.0, 110.0, 
110.0, 110.0, 110.0, 110.0, 110.0, 120.0, 369.99, 500.0] 
```

Невооружённым глазом видно, что большая часть значений не превышает 110 рублей, а значение в 85 рублей заметно преобладает. На этом фоне среднее значение в 104 рубля выглядит завышенным.

Два высоких значения, 500 и 369.99 рубля, слишком сильно завысили среднее значение. Если их отбросить, среднее значение будет заметно ниже:

```
# Найдём среднее значение без учёта дорогих кафе:
print('Средняя стоимость чашки кофе:', coffee.sort_values()[:-2].mean()) 
```

```
Средняя стоимость чашки кофе: 89.85106382978724 
```

Такие высокие или низкие значения, которые резко отличаются от преобладающих в выборке, в описательной статистике называют выбросами. Они могут сильно влиять на среднее значение. Как их находить и учитывать в процессе анализа данных, вы узнаете в будущих уроках.

Можно воспользоваться еще одним способом характеристики выборки — **медианой**.

**Медиана** — это значение, которое делит упорядоченный набор данных на две равные половины. Для нахождения медианы необходимо:

1. Упорядочить все значения в выборке в порядке возрастания или убывания.
2. Если количество значений в выборке нечётное, то значение ровно посередине и будет медианой.
3. Если количество значений в выборке чётное, то медиана — это среднее арифметическое двух средних значений в упорядоченной выборке.

Удобнее всего рассмотреть на примере. Вот медиана в наборе с нечётным числом элементов:

![В наборе данных из пяти отсортированных значений: 1, 3, 5, 7, 9, медианой будет значение, расположенное в середине, то есть третье по счёту из пяти — 5.](https://pictures.s3.yandex.net/resources/image_1710407776.png)

А вот — в наборе из чётного числа элементов:

![В наборе данных из шести отсортированных значений: 1, 3, 5, 7, 9, 11, медианой будет среднее арифметическое между 5 и 7, то есть 6.](https://pictures.s3.yandex.net/resources/image_1710407831.png)

Конечно, в библиотеке **pandas** есть специальный метод, который позволяет найти медиану, — `.median()`, с английского переводится как «медиана». Посчитаем медиану стоимости чашки двойного эспрессо:

```
# Найдём медиану стоимости двойного эспрессо в кафе:
print('Медианная стоимость чашки кофе:', coffee.median()) 
```

```
Медианная стоимость чашки кофе: 85.0 
```

Значение ещё ниже, чем среднее арифметическое, — даже с отброшенными выбросами. На практике это значит, что в более чем половине заведений из списка кофе будет стоить не дороже 85 рублей. Эта информация даже полезнее, чем знание о средней стоимости чашки кофе.

![](https://pictures.s3.yandex.net/resources/4-1-3_2_1711217713.png)

Метод `.median()` можно использовать так же, как и метод `.mean()`, — применять к отдельным столбцам датафрейма или ко всему датафрейму, чтобы подсчитать медиану для всех столбцов, содержащих числовые данные.

Среднее арифметическое и медиана — это две меры центральной тенденции. Их так называют, так как они показывают центр распределения значений в выборке. Эти меры можно использовать для описательной статистики.

**Медиана** менее чувствительна к выбросам, то есть устойчива к экстремально высоким и низким значениям. Однако она не даёт полного представления о выборке. Медиана сообщает только то, что половина значений в ней меньше, а половина — больше. А экстремальные значения вообще практически игнорируются, хотя они могут быть нужны.

**Среднее арифметическое** может нереалистично завышаться или занижаться из-за выбросов. Зато оно ничего не отбрасывает, поэтому, если требуется охарактеризовать выборку данных с учётом всех значений, лучше использовать именно его.

Среднее арифметическое и медиана — не взаимозаменяемые. Использовать медиану вместо среднего значения для расчёта каких-то метрик не рекомендуется. Называть медиану средним значением тоже не нужно.

У среднего арифметического есть интересная особенность: зная количество значений, можно рассчитать общую сумму значений в выборке. Например, если бы вы задались целью выпить чашку двойного эспрессо в каждом из 49 заведений Новосибирска, на это пришлось бы потратить следующую сумму:

```
# Найдём сумму всех значений в серии:
print('49 чашек кофе во всех кафе будет стоить:', coffee.mean()*coffee.count()) 
```

```
49 чашек кофе во всех кафе будет стоить: 5092.99 
```

Медиана не позволит получить такую информацию: умножение её на количество наблюдений не имеет никакого смысла. Поэтому для расчёта валовых показателей (например, валовой выручки) стоит использовать среднее значение — даже при аномальных выбросах.

### Получение статистической информации с помощью метода `.describe()`

Получить информацию о среднем арифметическом или медиане в **pandas** можно не только с помощью методов `.mean()` и `.median()`. Есть ещё метод `.describe()`, с английского переводится как «характеризовать». Метод позволяет получить статистическую информацию о данных в **DataFrame** или **Series**. Вот что он выведет в применение к списку со стоимостью кофе:

```
# Применим метод .describe() к серии значений:
print(coffee.describe()) 
```

```
count     49.000000
mean     103.938571
std       71.082917
min       70.000000
25%       85.000000
50%       85.000000
75%       95.000000
max      500.000000
dtype: float64 
```

Метод выводит следующую информацию:

- `count` — количество непустых значений, по которым были проведены расчёты статистических показателей. Если в каких-либо столбцах отсутствуют значения, **pandas** автоматически исключит эти значения при проведении расчётов.
- `mean` — среднее арифметическое значение, результат аналогичен методу `.mean()`.
- `std` — величина стандартного отклонения.
- `min` — минимальное значение в выборке данных, результат аналогичен методу `.min()`.
- `25%` — значение 25 перцентиль (первого квартиля Q1).
- `50%` — медиана, или значение 50 перцентиля (второго квартиля Q2), результат аналогичен методу `.median()`.
- `75%` — значение 75 перцентиля (третьего квартиля Q3).
- `max` — максимальное значение в выборке данных, результат аналогичен методу `.max()`.

Такой список значений вы получите, если примените метод `.describe()` к числовым данным.

Вывод будет следующий:

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
music_df = pd.read_csv('/datasets/music_log.csv').rename(columns={' user_id':'user_id', 'total play':'total_play', 'Artist':'artist'})

# Применим метод .describe() к категориальным данным столбца genre:
print(music_df['genre'].describe()) 
```

```
count     64661
unique      304
top         pop
freq       8599
Name: genre, dtype: object 
```

Здесь информация такая:

- `count` — то же самое количество непустых значений.
- `unique` — количество уникальных значений, результат аналогичен методу `.nunique()`.
- `top` — самое встречаемое значение — мода выборки, то значение, которое встречается чаще всего.
- `freq` — количество появлений самого встречаемого значения. Вместе с `top` результат будет аналогичен самому первому значению метода `.value_counts()`.

Если метод `.describe()` применить к датафрейму, то по умолчанию будут выведены только результаты для числовых значений. Например, применим метод `.describe()` к датафрейму Яндекс Музыки:

```
# Импортируем библиотеку pandas и загрузим данные:

## Расчёт агрегаций по условиям

В прошлых уроках вы узнали, как посчитать количество строк и уникальных значений, вычислить сумму значений и другие статические параметры по целому столбцу. Однако в своей работе аналитикам часто приходится проводить расчёты по выборке данных, то есть их части. Это нужно, чтобы сравнивать результаты между собой или анализировать данные определённой группы.

Например, типичная рабочая задача — подсчитать сумму затрат на закупку рекламного трафика в прошлом месяце и сравнить значения с текущими. Или сопоставить затраты по разным источникам привлечения трафика между собой. В этом случае данные агрегируют по какому-то конкретному условию — например, источнику трафика или месяцу. Это называется **агрегацией данных по условию**.

![](https://pictures.s3.yandex.net/resources/image_1710424123.png)

Такую агрегацию данных активно используют в продуктовой аналитике: там постоянно нужно сегментировать пользователей, сравнивать и анализировать их поведение. Например, подсчитать среднюю стоимость заказа клиентов из каждого региона и выявить регионы с максимальным и минимальным значениями.

В прошлых уроках вы узнали, что для создания выборки с помощью индексации данных можно использовать несколько операторов **DataFrame**: `[]`, `.loc[]` и `.iloc[]`. Ещё для этого подойдут запросы и метод `.query()`. Полученную выборку можно использовать для агрегации данных и подсчёта необходимых значений.

В этом уроке вы узнаете, как рассчитать агрегацию столбца по одному или нескольким условиям, используя логическую индексацию или запрос `.query()`. В этом вам помогут данные Яндекс Музыки.

### Агрегация данных и логическая индексация

Логическая индексация позволяет сделать срез необходимых для работы данных и агрегировать их. Это полезно, если нужно провести расчёты для определённой группы данных: категорий товаров, сегментов пользователей, временных отрезков. Например, рассчитать среднюю длительность просмотра видео в выходные или посчитать количество уникальных моделей в категории «Ноутбуки».

Пора потренироваться на практике — посчитать уникальное количество клиентов Яндекс Музыки, которые слушали композиции жанра фанк. Чтобы это сделать, нужно вспомнить, как создавать срезы.

Решить задачу с подсчётами поможет оператор `[]` с логическим условием: `music_df[music_df['genre'] == 'funk']`. Так вы получите выборку всех прослушанных людьми песен в жанре фанк.

Теперь вам нужно узнать, сколько уникальных пользователей слушало этот жанр. В этом поможет агрегация по столбцу `'user_id'`. Этот столбец содержит информацию об уникальных id пользователей, поэтому считать количество пользователей нужно именно по нему.

Каждому прослушиванию трека соответствует id пользователя, который его прослушал. Но если один пользователь слушал несколько композиций в стиле `'funk'`, то в данных будет несколько строк с разными треками, но одинаковым id этого пользователя. Поэтому для подсчётов понадобится метод `.nunique()`, который подсчитает количество уникальных значений в столбце:

```
# Подсчитаем уникальное количество пользователей, прослушавших композиции стиля funk:
print(music_df[music_df['genre'] == 'funk']['user_id'].nunique()) 
```

```
31 
```

В результате получили целое число — количество уникальных пользователей в срезе данных `music_df[music_df['genre'] == 'funk']['user_id']`.

Просто так не угадать. Нужно подсчитать долю пользователей, слушающих композиции в жанре `'funk'`, от общего количества пользователей.

```
# Подсчитаем долю пользователей, прослушавших композиции стиля funk:
funk_users = music_df[music_df['genre'] == 'funk']['user_id'].nunique()
total_users = music_df['user_id'].nunique()
funk_share = funk_users / total_users 

# Выведем на экран результат в процентах:
print(f'Процент пользователей, прослушавших композиции стиля funk, = {funk_share:.2%}') 
```

```
Процент пользователей, прослушавших композиции стиля funk, = 0.07% 
```

Логическая индексация позволяет задавать условие, относящееся к нескольким столбцам, и на основе этого фильтровать и агрегировать данные. Например, можно подсчитать количество уникальных исполнителей и песен стиля `'modern'`, длительность прослушивания которых `total_play` превысила 100 минут:

```
# Сохраним логическую маску в переменной mask:
mask = (music_df['genre'] == 'modern') & (music_df['total_play'] > 100)

# Создадим срез данных для агрегации, используя маску mask, 
# и подсчитаем уникальные значения:
print(music_df[mask][['artist','track']].nunique()) 
```

```
artist    12
track     13
dtype: int64 
```

Результат показывает, что в базе данных 13 песен стиля `'modern'` с длительностью прослушивания более 100 минут, а исполнили их 12 музыкантов.

Проводить агрегацию данных можно и с помощью оператора `.loc[]`. Здесь всё как в обычной индексации данных, которая позволяет создать необходимый срез данных и применить к нему агрегирующий метод. Например, представьте, что вам нужно оценить типичные значения времени прослушивания для композиций жанра `'alternative'`, то есть рассчитать среднее арифметическое значение.

Метод `.mean()` поможет найти среднее время прослушивания композиций жанра `alternative`:

```
# Используя оператор .loc[], найдём необходимые треки 
# и затем подсчитаем среднее значение в столбце total_play:
print(music_df.loc[music_df['genre'] == 'alternative', 'total_play'].mean()) 
```

```
103.20009635481279 
```

Среднее значение — 103 минуты. Из предыдущих уроков вы знаете, что среднее арифметическое — не единственная мера, которая позволяет получить оценку типичных значений в выборке.

**Да, ещё есть медиана**

Можно посчитать её и сравнить результаты:

```
# Используя оператор .loc[], найдём необходимые треки 
# и затем подсчитаем медиану значений в столбце total_play:
print(music_df.loc[music_df['genre'] == 'alternative', 'total_play'].median()) 
```

```
27.428287 
```

При агрегации значений с `.loc[]` можно комбинировать несколько условий, а также применять агрегацию к нескольким столбцам или всему срезу данных — всё как и с оператором `[]`.

### Агрегация данных с помощью метода `.query()`

Этот способ похож на агрегацию данных при логической индексации. Сначала нужно создать срез данных, затем применить к нему метод **pandas**, который проводит нужную агрегацию.

Например, можно сравнить количество уникальных треков `pop` и `rock`, длительность прослушивания которых больше 150 минут:

```
# Используя запрос query, создадим срез треков стиля pop и проведём агрегацию данных:
pop = music_df.query('genre == "pop" and total_play > 150')['track'].nunique()

# Используя запрос query, создадим срез треков стиля rock и проведём агрегацию данных:
rock = music_df.query('genre == "rock" and total_play > 150')['track'].nunique()

# Сравним результаты:
print(f'Количество уникальных треков стиля pop = {pop}, а стиля rock = {rock}') 
```

```
Количество уникальных треков стиля pop = 2661, а стиля rock = 2136 
```

Композиций стиля `pop` с длительностью прослушивания `> 150` больше.

Необязательно. Может быть, количество таких композиций выше просто потому, что поп-песен больше рока в абсолютном количестве. Чтобы это понять, можно подсчитать долю песен с длительностью прослушивания более `150` для каждого жанра.

Для решения этой задачи понадобится найти общее количество уникальных треков каждого стиля и рассчитать долю треков с временем прослушивания больше 150 минут. Это решение можно реализовать через цикл:

```
# Сохраним интересующие нас жанры в списке genres:
genres = ['pop','rock']

# Зададим цикл, который для каждого стиля будет проводить необходимые расчёты:
for genre in genres:
    
    # С помощью запроса и метода .nunique() подсчитаем общее количество треков:
    total_tracks = music_df.query('genre == @genre')['track'].nunique()
    
    # С помощью запроса и метода .nunique() подсчитаем количество популярных треков:
    popular_tracks = music_df.query('genre == @genre and total_play > 150')\
                     ['track'].nunique()
    
    # Выведем информацию на экран:
    print(f'Жанр {genre}: треков всего {total_tracks}',
          f'доля популярных {popular_tracks/total_tracks}') 
```

```
Жанр pop: треков всего 7389, доля популярных 0.36012992285830286
Жанр rock: треков всего 6036, доля популярных 0.3538767395626243 
```

```
# Сохраним интересующие нас жанры в списке genres:
genres = ['pop','rock']

# Зададим цикл, который для каждого стиля будет проводить необходимые расчёты:
for genre in genres:
    
    # С помощью запроса и метода .nunique() подсчитаем общее количество треков:
    total_tracks = music_df.query('genre == @genre')['track'].nunique()
    
    # С помощью запроса и метода .nunique() подсчитаем количество популярных треков:
    popular_tracks = music_df.query('genre == @genre and total_play > 150')\
                     ['track'].nunique()
    
    # Выведем информацию на экран:
    print(f'Жанр {genre}: треков всего {total_tracks:_.0f}',
          f'процент популярных {popular_tracks/total_tracks:.2%}') 
```

```
Жанр pop: треков всего 7_445, процент популярных 36.01%
Жанр rock: треков всего 6_052, процент популярных 35.36% 
```

Результаты показывают, что доля треков с длительностью прослушивания для стиля `pop` выше, чем доля для стиля `rock`, хотя значения близки друг к другу.

В примере выше мы применили форматирование чисел, указав после двоеточия в фигурных скобках формат для отображения числа. Знак нижнего подчёркивания `_` сразу после двоеточия означает, что мы хотим использовать его в качестве разделителя разрядов. `.0` или `.2` указывают на количество знаков после запятой, которое мы хотим видеть в выводимом числе. В конце мы указали `f` для числа с плавающей точкой и `%` для отображения процентов.

Можно сделать программу вариативной, чтобы она умела рассчитывать долю популярных треков разных жанров и с заданной длительностью прослушивания:

```
# Сохраним интересующие нас жанры в списке genres:
genres = ['pop','rock','dance']

# Зададим функцию, которая будет подсчитывать долю популярных треков:
def genre_info(df, genre, duration):
    '''
    Функция genre_info подсчитывает долю популярных треков для жанра.
    Принимает следующие аргументы:
    - df - название датафрейма с информацией о композициях
    - genre - название жанра, по которому будет проводиться фильтрация
    - duration - длительность прослушивания в минутах, свыше которой 
                 трек будет считаться популярным
    Функция в качестве результата выводит на экран необходимую информацию.
    '''

    # С помощью запроса и метода .nunique() подсчитаем общее количество треков:
    total_tracks = df.query('genre == @genre')['track'].nunique()
    
    # С помощью запроса и метода .nunique() подсчитаем количество популярных треков:
    pop_tracks = df.query('genre == @genre and total_play > @duration')['track'].nunique()
                     
    # Выведем информацию на экран:
    print(f'Жанр {genre}: треков всего {total_tracks:_.0f},',
          f'процент популярных (total_play > {duration}) = {pop_tracks/total_tracks:.2%}')
    
# Используем функцию для получения информации по трекам из списка genres:
for genre in genres:
    genre_info(music_df, genre, 150) 
```

```
Жанр pop: треков всего 7_445, процент популярных (total_play > 150) = 36.01%
Жанр rock: треков всего 6_052, процент популярных (total_play > 150) = 35.36%
Жанр dance: треков всего 5_613, процент популярных (total_play > 150) = 40.44% 
```

По выбранному критерию популярности — длительность прослушивания свыше 150 — популярнее треки жанра `dance`.

Агрегацию данных с помощью методов **pandas** можно использовать совместно с фильтрацией с помощью логической индексации, оператора `.loc[]` и запросов `.query()`. В этом случае алгоритм будет единым: сначала создаётся необходимый срез данных с учётом столбцов, по которым будут проводиться расчёты, а затем используется необходимый метод.

## Агрегация данных и метод .agg()

В прошлых уроках вы изучили, как проводить агрегацию данных с использованием различных методов **pandas**:

- `.sum()` — подсчитать сумму значений;
- `.min()` и `.max()` — найти минимальное и максимальное значения;
- `.mean()` и `.median()` — рассчитать среднее арифметическое значение и медиану.

Эти методы аналитики часто используют при анализе данных: находят среднюю стоимость покупки или количество просмотров видеоконтента, минимальное время регистрации или максимальную длительность пользовательской сессии. Результаты помогают получить представление о данных, оценить их вариативность или наиболее частые значения.

У этих методов есть особенность: чтобы применить их к одному набору данных, необходимо каждый раз прописывать отдельные инструкции. Например, чтобы найти минимальное и максимальное значения, следует написать две строки кода, поскольку применить эти методы в цепочке не получится:

```
# Найдём минимальное и максимальное значения в серии данных values:
min_value = values.min()
max_value = values.max() 
```

Однако в **pandas** существует метод `.agg()`, который помогает использовать несколько вычислений в одном действии с данными. В этом уроке мы разберём особенности этого метода.

### Метод `.agg()` и расчёт нескольких значений для одного столбца

Метод `.agg()` (от англ. «объединять, агрегировать») позволяет объединить несколько расчётов в одно действие и получить серию или датафрейм с агрегированными значениями. Для этого необходимо передать список функций или названия методов **pandas**, которые проводят расчёты.

Например, можно использовать следующие названия методов **pandas**, которые мы разбирали в предыдущих уроках:

- `'sum'` — для подсчёта суммы значений;
- `'min'` и `'max'` — для нахождения минимального и максимального значений;
- `'mean'` и `'median'` — для расчёта среднего арифметического значения и медианы;
- `'unique'` — для вывода всех уникальных значений;
- `'nunique'` — для подсчёта количества уникальных значений;
- `'count'` — для подсчёта общего количества значений.

Разберём действие метода на примере. По данным Яндекс Музыки, найдём среднее арифметическое значение и медиану для данных в столбце с временем прослушивания трека `total_play`:

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
music_df = pd.read_csv('/datasets/music_log.csv')

# Найдём среднее арифметическое значение и медиану в столбце total_play:
print(music_df['total_play'].agg(['mean','median'])) 
```

```
mean      98.899155
median    20.135056
Name: total_play, dtype: float64 
```

Результат представлен в виде серии значений, состоящей из названия метода и рассчитанного значения.

В примере выше мы использовали названия методов **pandas** в строковом виде. Однако в методе `.agg()` можно использовать другие агрегирующие функции для расчётов значений: функции **Python** или других библиотек (например, `numpy`):

```
# Импортируем библиотеку numpy:
import numpy as np

# Проведём агрегацию данных с использованием различных функций Python и numpy:
print(music_df['total_play'].agg([min, max, np.mean, np.median])) 
```

```

max       8638.736000
mean        98.899155
median      20.135056
Name: total_play, dtype: float64

> ```python
> df['column'].agg(['min','max','sum'])
> ```
>
>Можно использовать встроенные функции **Python** для расчёта необходимых значений:

### Агрегация различных срезов данных

Агрегация данных, особенно при анализе больших объёмов информации, позволяет получить общее представление о данных, выявить определённые закономерности или тенденции в изменении данных, а также создавать сводные таблицы и аналитические отчёты. Например, выделить популярные товары, рассчитать средний чек по группам товаров, изучить продажи по категориям товаров и так далее.

Использовать метод `.agg()` также можно для нескольких столбцов или всего датафрейма. Например, подсчитаем общее и уникальное количество значений в каждом столбце датафрейма Яндекс Музыки:

```
# Подсчитаем общее и уникальное количество значений:
print(music_df.agg(['count','nunique'])) 
```

```
         user_id  total_play  artist  genre  track
count      67963       67399   59647  64661  64804
nunique    43240       37003   55726    304  51443 
```

Результатом работы метода будет объект **DataFrame**, индексами которого будут названия значений — в нашем случае `count` и `nunique`. Столбцы же будут содержать рассчитанные значения для каждого столбца исходного датафрейма `music_df`.

Такую агрегацию удобно проводить для общего ознакомления с данными и расчётом простых метрик. Например, подсчитаем долю уникальных значений:

```
# Подсчитаем общее и уникальное количество значений и сохраним результат:
result = music_df.agg(['count','nunique']).T

# Подсчитаем долю уникальных значений в столбце unique_share:
result['unique_share'] = round(result['nunique'] / result['count'], 2)

# Выведем результат на экран:
print(result) 
```

```

В примере выше мы вначале подсчитали необходимые значения с помощью метода `.agg()`, а затем с помощью свойства датафрейма `.T` транспонировали датафрейм таким образом, чтобы столбцы стали индексом, а индекс — столбцами. Это позволило добавить новый столбец `'unique_share'` с долей уникальных пользователей.

Применить метод `.agg()` можно и к определённым столбцам. Для этого достаточно создать срез необходимых данных:

```
# Подсчитаем общее и уникальное количество значений в столбцах 'artist' и 'track':

### Дополнительные варианты агрегирования данных

В методе `.agg()` существуют другие варианты агрегирования данных датафрейма с использованием словаря и именованного агрегирования. Рассмотрим эти варианты детальнее.

### **Агрегирование данных датафрейма с использованием словаря**

Использование словаря позволяет подсчитать разные метрики для разных данных в одном действии. Например, при анализе продаж можно в одно действие подсчитать количество уникальных пользователей и количество покупок, которые они совершили. Для этого вместо списка функций передаётся словарь, который состоит из ключа — названия столбца датафрейма — и списка функций. Например, рассчитаем для столбцов `'artist'` и `'track'` общее и уникальное количество значений, а для столбца `'total_play'` изучим разброс значений:

```
# Для различных столбцов подсчитаем разные значения:
print(music_df.agg({'artist': ['count', 'nunique'],
                    'track': ['count', 'nunique'],
                    'total_play' : ['mean','median']})) 
```

```
          artist    track  total_play
count    59647.0  64804.0         NaN
nunique  55726.0  51443.0         NaN
mean         NaN      NaN   98.899155
median       NaN      NaN   20.135056 
```

В результате для каждого столбца были подсчитаны необходимые значения, а все результаты представлены в едином датафрейме. При этом для тех значений, которые не рассчитывались, стоят пропуски в виде `NaN`, что указывает на пропущенные значения в числовых данных. Такой подход агрегирования является полезным, когда для разных столбцов необходимо рассчитать различные значения.

### **Именное агрегирование**

Именное агрегирование в методе `.agg()` позволяет задавать название для каждого значения. Для этого используется следующий синтаксис:

```
# Используем именное агрегирование для подсчёта количества значений в столбце 'artist'
# и количества уникальных значений в столбце 'track':
print(music_df.agg(artist_сount=('artist', 'count'),
                   track_unique=('track', 'nunique'))) 
```

```
               artist    track
artist_сount  59647.0      NaN
track_unique      NaN  51443.0 
```

В примере выше методу `.agg()` было передано несколько аргументов `artist_сount` и `track_unique`, которые содержали кортежи с названием столбца и метода агрегации данных. В результате получился **DataFrame**, где индексом являются названия переданных аргументов. Такой подход агрегирования полезен, когда необходимо задать определённое название для рассчитанного значения.

### Использование пользовательской функции при агрегировании данных

Агрегацию данных можно проводить не только с использованием встроенных функций **Python**, **pandas** или других библиотек, но и с использованием собственных расчётов. Например, реализуем функцию, которая будет возвращать самое часто встречающееся значение на основе метода `value_counts`.

```
# Определим нашу функцию, которая будет принимать последовательности
def most_common (iterable):
    # чтобы value_counts точно сработал, объект должен быть Series
    series = pd.Series(iterable)

    # применяем метод value_counts и возвращаем верхнее значение индекса
    return series.value_counts().index[0]

# Применим функцию ко всем столбцам датасета
music_df.agg(most_common) 
```

```
user_id                    FE76AC63
total_play                      0.0
artist                     Two Feet
genre                           pop
track         How Deep Is Your Love
dtype: object 
```

Мы постарались сделать нашу функцию универсальной. Она будет работать со всем, что можно преобразовать в серию **pandas**: с массивом **numpy**, списком или генератором типа `range`. Так как нас интересует не сколько раз значение встречается, а только само значение, мы берём индекс серии, которую возвращает `value_counts`. А так как нам нужно лишь самое часто встречающееся значение, мы возвращаем именно его.

### **Использование неименованной `lambda`-функции в `.agg()`**

Если наша функция очень проста и мы используем её только один раз, у нас нет необходимости хранить её в переменной, соответствующей названию функции. В таких ситуациях мы можем использовать неименованную функцию. Неименованные функции определяются с помощью ключевого слова `lambda`. Сначала перепишем использованную ранее функцию так, чтобы весь код внутри функции стоял после слова return.

```
def most_common (iterable):
    return pd.Series(iterable).value_counts().index[0] 
```

Полным аналогом кода выше будет следующее:

```
most_common = lambda iterable: pd.Series(iterable).value_counts().index[0] 
```

После `lambda` и до двоеточия мы пишем все аргументы функции без скобок. Если нужно несколько аргументов, то используем между ними запятую. После двоеточия сразу пишем то, что было после return. Сохранять лямбда-функции в переменные, как это сделано выше, не принято. Если вы используете функцию в нескольких местах, определите её обычным образом.

Там, где мы раньше использовали ссылку на нашу функцию — её название, мы можем сразу написать лямбда-выражение. Давайте на примере рассмотрим, как методу `.agg()` передать инструкцию с использованием `lambda`-функции, — подсчитаем долю уникальных значений в столбцах датафрейма Яндекс Музыки:

```
# Подсчитаем долю уникальных значений в каждом столбце датафрейма:
print(music_df.agg([lambda x: round(x.nunique() / x.count(), 3)]).T) 
```

```
            <lambda>
user_id        0.636
total_play     0.549
artist         0.934
genre          0.005
track          0.794 
```

В примере выше `lambda`-функция для каждого столбца вначале подсчитала количество уникальных значений с помощью метода `nunique()` — `x.nunique()`, а затем поделила это значение на общее количество данных в столбце `x.count()`. Результаты мы округлили до трёх знаков с помощью функции `round()`, а затем транспонировали.

Использование `lambda`-функций полезно при агрегировании данных, поскольку позволяет сразу подсчитывать необходимые значения или метрики, особенно при выполнении небольшого количества операций. Это позволяет избежать определения отдельной функции — например, при расчёте конверсии, среднего времени пользовательских сессий или удержания пользователей.

---

Подведём итоги. Метод `.agg()` позволяет использовать несколько функций и расчётов при агрегации данных, что упрощает работу с данными и оптимизирует код. В методе `.agg()` можно использовать список агрегирующих функций **Python** или названия агрегирующих методов **pandas** и других библиотек, а также произвольные расчёты на базе пользовательских и `lambda`-функций.

Метод `.agg()` можно применять ко всему срезу данных или определённой выборке. А если вместо списка функций или их названий передать словарь, где в качестве ключа будет название столбца, а в качестве значения — список функций, то для различных данных можно подсчитать разные значения.

С этим багажом знаний мы переходим к следующему важному этапу в нашем пути изучения данных — первой аналитике. На следующем этапе наша цель — рассмотреть подходы к проведению исследовательского анализа данных, который позволит не только глубже понять структуру и основные характеристики наших данных, но и выявить потенциальные взаимосвязи и зависимости между различными переменными. Эти инструменты и методы помогут раскрыть скрытые инсайты, лежащие в основе наших данных, и сформулировать основу для дальнейшего анализа.

# Первая аналитика
## Расчёт по группам

Группировка данных позволяет объединять информацию по заданным признакам для более тщательного анализа данных. Этот процесс особенно важен при использовании метода `groupby` в библиотеке pandas — мощного механизма для извлечения значимых выводов из больших объёмов данных. Ранее вы уже использовали этот метод при работе с пропусками в разрезе категорий.

Основная цель группировки — упростить анализ данных, превратив обширные и сложные наборы информации в более мелкие и понятные сегменты. Это позволяет аналитикам легче обнаруживать тенденции, закономерности и аномалии, что упрощает процесс принятия обоснованных решений. Группировка данных решает широкий спектр бизнес-задач: от анализа продаж до измерения эффективности рекламных кампаний. Например, компании используют группировку, чтобы определить наиболее прибыльные категории товаров, сезонные колебания спроса или оценить эффективность маркетинговых стратегий по регионам. Также этот инструмент применяется для мониторинга времени реакции службы поддержки, позволяя выявлять проблемные зоны и оптимизировать процессы обслуживания клиентов.

Таким образом, группировка данных не только позволяет найти скрытые в данных закономерности, но и предоставляет ценные инсайты для улучшения бизнес-процессов, повышения эффективности работы и оптимизации стратегического планирования на основе данных.

### Метод `groupby()`

Для понимания расчёта агрегаций по группам с использованием метода `groupby()` в pandas представим, что у нас есть набор данных о продажах товаров. В нём есть информация о категории товара, названии товара и количестве продаж в регионах. Наша задача — проанализировать эти данные, сгруппировав их по категориям товаров, и затем рассчитать сумму, среднее значение и максимальное количество продаж в каждой категории. Для наглядности сформируем небольшой датафрейм `sales_df`.

||category|product|sales|region|
|---|---|---|---|---|
|0|Electronics|Smartphone|200|North|
|1|Clothing|Jeans|150|South|
|2|Electronics|Laptop|340|East|
|3|Furniture|Desk|450|West|
|4|Clothing|T-shirt|90|North|
|5|Electronics|Tablet|120|East|
|6|Furniture|Chair|230|South|
|7|Clothing|Hat|110|West|

Используем метод **`groupby()`** для группировки данных по категории товаров:

```
grouped_sales = sales_df.groupby('category') 
```

Конструкция `groupby('category')` говорит pandas сгруппировать эти данные по уникальным значениям в столбце `category`. В результате создаётся объект `DataFrameGroupBy`, который представляет собой коллекцию групп данных. Каждая группа соответствует уникальному значению в столбце `category`.

После применения `groupby()` pandas предоставляет обширный набор методов для агрегации данных, часть из них вам известны из предыдущей темы:

- `sum()` вычисляет сумму значений внутри каждой группы;
- `mean()` рассчитывает среднее значение для группы;
- `min()` и `max()` определяют минимальное и максимальное значения в группах;
- `count()` подсчитывает количество значений в каждой группе;
- `size()` возвращает размер каждой группы, включая NaN значения;
- `std()` и `var()` рассчитывают стандартное отклонение и дисперсию значений в группах, соответственно;
- `first()` и `last()` возвращают первое и последнее значение в каждой группе.

Эти методы могут быть применены цепочным образом напрямую к результату вызова `groupby()`, возвращая новый датафрейм с агрегированными значениями.

Применим некоторые агрегирующие функции, чтобы рассчитать общее количество продаж (**`sum`**), среднее количество продаж (**`mean`**) и максимальное количество продаж (**`max`**) для каждой категории товаров. Начнём с суммы.

total_sales = grouped_sales['sales'].sum()

```
sales_df.groupby('category')['sales'].max()

Он позволяет выполнить группировку и агрегацию данных в одной строке кода, делая код понятнее. Также с `groupby` вы сможете применять все те же агрегации, которые вы применяли к отфильтрованным данным.

Метод `groupby()` в pandas обладает несколькими аргументами, которые позволяют настраивать процесс группировки данных. Давайте подробно рассмотрим некоторые из основных аргументов.

### **Основные аргументы метода `groupby()`**

1. `by` — определяет критерии группировки. Это может быть имя столбца, список имен столбцов, отображение или функция.
2. `axis` — ось для группировки. **`0`** или **`'index'`** для группировки по строкам по умолчанию, **`1`** или **`'columns'`** для группировки по столбцам.
3. `as_index` — если **`True`** по умолчанию, ключи группировки будут использоваться в качестве индексов нового объекта. Если **`False`**, индексы будут как в исходном датафрейме, а ключи группировки добавлены как обычные столбцы.
4. `sort` — контролирует, следует ли сортировать результаты группировки по ключам. По умолчанию **`True`**.

**Использование `by`**

Группировка осуществляется по одному или нескольким ключам.

```
# Группировка по одной категории
grouped_by_category = sales_df.groupby(by='category')['sales'].sum()
print(grouped_by_category) 
```

```
category
Clothing       350
Electronics    660
Furniture      680 
```

Двойная группировка позволяет аналитикам одновременно рассматривать два уровня данных. Например, вы можете сначала группировать данные по региону, а затем внутри каждого региона группировать данные по категориям продуктов. Это предоставляет возможность увидеть не только картину по регионам, но и детализированный анализ по категориям внутри каждого региона.

```
# Группировка по двум категориям
grouped_by_region = sales_df.groupby(by=['region', 'category'])['sales'].sum()
print('Агрегированные данные:')
print(grouped_by_region)
print()
print('Тип объекта:',type(grouped_by_region))
print()
print('Индекс объекта:')
print(grouped_by_region.index) 
```

```

В этом примере данные сначала группируются по `region`, а затем внутри каждого региона — по `category`. Важный момент: несколько категорий подаются в качестве аргумента в виде списка `[ ]`. Можно не указывать имя параметра `by=` — в него передаётся первый неименованный аргумент.

После группировки мы применяем агрегирующую функцию для расчёта общих продаж по каждой категории в каждом регионе. Результатом будет датафрейм, в котором для каждой комбинации региона и категории указаны общие продажи. Обратите внимание — так как мы использовали синтаксис `['sales'].sum()`, у нас получилась серия — то есть всего одна колонка значений, а регион и категория попали в мультииндекс.

Этот вызов вернёт датафрейм, где `category` будет обычным столбцом, а не индексом.

# Группировка по двум категориям
grouped_by_region = sales_df.groupby(by=['region', 'category'], as_index=False)['sales'].sum()

### Агрегации по группам методом `agg()`

Вы уже знакомы с методом `agg()`, но теперь рассмотрим, как этот метод можно использовать совместно с методом `groupby()`.

Для расчёта агрегаций по группам методом `agg()` требуемые агрегации задаются с помощью словаря параметров. Такой подход позволяет применять конкретную функцию агрегации к определённой колонке в результате группировки. Это удобно, когда вам нужно выполнить различные агрегирующие операции для разных столбцов внутри одной группировки.

Предположим, мы хотим рассчитать суммарные продажи (`sum`) и среднее количество продаж (`mean`) для каждой категории товаров в каждом регионе. Для этого мы можем использовать метод `agg()`, передав ему словарь, где ключи — названия столбцов, а значениями — функции или их названия, которые мы хотим применить к соответствующим столбцам.

Общий вид конструкции:

```
df.groupby('колонка группировки').agg({'колонка агрегации': 'функция агрегации'}) 
```

Такая агрегация с группировкой состоит из трёх компонентов:

- `Колонка группировки` — каждое уникальное значение в этой колонке формирует свою группу.
- `Колонка агрегации` — колонка, по которой происходит агрегация до одного значения, обычно числовая.
- `Функция агрегации` — функция, применяемая к колонке агрегации (**`mean`**, **`min`**, **`max`** и другие).

```
# Группировка по категории товара с последующим расчётом агрегаций
grouped = sales_df.groupby('category').agg({
    'sales': ['sum', 'mean']
})

print(grouped) 
```

```
            sales            
              sum        mean
category                     
Clothing      350  116.666667
Electronics   660  220.000000
Furniture     680  340.000000 
```

По результатам формируются две колонки под `sales`, показывающие `sum` и `mean` продаж в каждой группе.

Аналогичным образом проходит агрегация по двойным группировкам:

```
# Группировка по региону и категории с последующим расчётом агрегаций
grouped = sales_df.groupby(['region', 'category']).agg({
    'sales': ['sum', 'mean']
})

print(grouped) 
```

В этом примере:

- Мы группируем `sales_df` по столбцам `region` и `category`, используя `.groupby(['region', 'category'])`.
- Затем применяем `.agg()` со словарём агрегаций, где указываем, что для столбца `sales` необходимо рассчитать `sum` и `mean`.

Результат будет таким:

```
                   sales       
                     sum   mean
region category                
East   Electronics   460  230.0
North  Clothing       90   90.0
       Electronics   200  200.0
South  Clothing      150  150.0
       Furniture     230  230.0
West   Clothing      110  110.0
       Furniture     450  450.0 
```

Здесь результаты агрегации представлены в мультииндексном датафрейме, где первый уровень индекса — это `region`, второй уровень — `category`, и две колонки под `sales`, показывающие `sum` и `mean` продаж в каждой группе.

Использование метода `agg()` со словарём параметров даёт возможность выполнять гибкие агрегации данных, применяя разные функции к разным столбцам в рамках одного вызова метода. Это особенно полезно, когда нужно выполнять сложный анализ данных.

Существуют и другие форматы записи группировки с агрегацией, через передачу ключевого аргумента в виде кортежа, вспомним их:

```
df.groupby('колонка группировки').agg(новая_колонка=('колонка агрегации', 'функция агрегации')) 
```

Таким образом можно задать имя новой колонки:

```
# Группировка по категории с заданием имени новой колонки
grouped = sales_df.groupby('category').agg(sales_sum=('sales', 'sum'))

print(grouped) 
```

```
             sales_sum
category              
Clothing           350
Electronics        660
Furniture          680 
```

В результате сформировалась новая колонка `sales_sum`.

Можно задать несколько ключевых аргументов для множественных агрегаций:

```
# Группировка по категории с заданием имени для новых колонок
grouped = sales_df.groupby(['region', 'category'])\
                        .agg(
                             avg_sales=('sales', 'mean'),
                             min_sales=('sales', 'min'), 
                             max_sales=('sales', 'max'))

print(grouped) 
```

```

North  Clothing          90.0         90         90
       Electronics      200.0        200        200
South  Clothing         150.0        150        150
       Furniture        230.0        230        230
West   Clothing         110.0        110        110
       Furniture        450.0        450        450 
```

В результате группировки по столбцам `region` и `category` и применения агрегирующих функций, мы получаем датафрейм, где для каждой уникальной комбинации региона и категории рассчитаны средние, минимальные и максимальные продажи. Таким образом, каждая комбинация региона и категории в исходных данных будет представлена в результате одной строкой, содержащей значения средних, минимальных и максимальных продаж.


##  Сводные таблицы
Сводная таблица упрощает анализ больших наборов данных, представляя информацию в удобной и понятной для восприятия форме. В бизнесе сводные таблицы применяются для быстрого и систематизированного анализа информации. Особенно они популярны в данных, где столбцы и строки представляют различные параметры или категории, а ячейки содержат агрегированные показатели, такие как сумма, среднее, максимум и минимум, по соответствующим группам.

Сводные таблицы широко используются для решения многих аналитических задач:

- анализ продаж по категориям товаров и регионам;
- суммирование ежемесячных или ежегодных доходов по отделам или проектам;
- сравнение эффективности сотрудников по различным показателям;
- оценка распределения бюджета на маркетинговые кампании по каналам и периодам.

Например, так может выглядеть сводная таблица среднего возраста пациентов клиники в разбивке по полу и диагнозу за один из дней.

### Разница между группировкой и сводными таблицами

Группировка данных часто используется для агрегирования информации по одному или нескольким критериям, но без сложной многомерной структуры. Она подходит для простых задач агрегации, таких как подсчёт, суммирование, или вычисление средних значений по определённым категориям.

Представьте, что у вас есть таблица с данными о продажах в различных магазинах, включая название магазина, категорию товара и сумму продаж. Вы хотите узнать общий объём продаж по каждому магазину.

||store_name|product_category|sales_amount|
|---|---|---|---|
|0|Store C|Home Decor|786|
|1|Store A|Furniture|355|
|2|Store C|Electronics|574|
|3|Store C|Electronics|907|
|4|Store A|Kitchenware|349|

В этом случае достаточно применить группировку по названию магазина и суммировать продажи. Здесь не требуется сложная многомерная агрегация, и `groupby()` отлично справляется с задачей.

```
# Пример кода на Python
sales.groupby('store_name')['sales_amount'].sum() 
```

```
store_name
Store A    2257
Store B    4660
Store C    4891
Name: sales_amount, dtype: int64 
```

Сводные таблицы используют, когда необходимо агрегировать данные по нескольким измерениям одновременно. Это позволяет создавать более сложные и многогранные представления данных, которые могут включать в себя суммы по нескольким категориям, средние значения, количество и другие параметры, распределённые по различным измерениям.

Используя ту же таблицу с данными о продажах, предположим, что теперь вы хотите анализировать продажи не только по магазинам, но и по категориям товаров внутри каждого магазина. А еще вам нужно видеть средние значения продаж по каждой категории для каждого магазина.

В этом случае `groupby()` уже не подойдёт из-за его ограниченности в многомерной агрегации, и лучше использовать сводные таблицы.

```
# Пример кода на Python для создания сводной таблицы
pd.pivot_table(
    data=sales, 
    values='sales_amount', 
    index='store_name', 
    columns='product_category', 
    aggfunc='mean') 
```

Этот код создаст сводную таблицу, где строки — это магазины, столбцы — категории товаров, а значения в ячейках — средние суммы продаж по каждой категории для каждого магазина. Таким образом получится более детальный и многогранный анализ продаж по сравнению с простой группировкой:

```
product_category  Electronics  Furniture  Home Appliances  Home Decor  \
store_name                                                              
Store A                 481.5      377.0              NaN         NaN   
Store B                 571.0      681.0       523.666667       578.0   
Store C                 740.5      589.0       376.000000       786.0   

product_category  Kitchenware  
store_name                     
Store A                 180.0  
Store B                   NaN  
Store C                 240.5   
```

Главное отличие между группировкой и сводными таблицами — в уровне сложности агрегации данных.

Хотя `groupby` и `pivot_table` могут использоваться для достижения аналогичных результатов, следует учитывать оптимальные области их применения: `groupby` для простых агрегаций, а `pivot_table` — для сложных многоуровневых агрегаций, так как `pivot_table` предназначены для обработки данных по нескольким измерениям одновременно.

От конкретной аналитической задачи и предпочтений пользователя в представлении результатов анализа зависит выбор между «длинным» и «широким» форматами данных. В длинном формате данных каждая строка представляет собой одно наблюдение. В широком формате данных присутствует несколько уровней индексов или столбцов, что позволяет организовать и представить данные в структурированной форме, например в сводной таблице.

### Построение сводной таблицы

Для построения сводной таблицы в pandas используется метод `pivot_table`. Рассмотрим его основные параметры на примере. Предположим, что у нас есть набор данных о пациентах клиники: пол, возрастная группа, диагноз и стоимость лечения:

||Gender|Age Group|Diagnosis|Treatment Cost|
|---|---|---|---|---|
|0|Female|46-55|Cold|363|
|1|Male|26-35|Cold|134|
|2|Female|26-35|Fever|305|
|3|Female|26-35|Cold|180|
|4|Female|46-55|Fever|149|

Мы хотим использовать сводные таблицы для анализа средней стоимости лечения по полу и возрастным группам, а также определить, какой диагноз наиболее распространён среди различных групп.

Каждая сводная таблица в pandas базируется на следующих ключевых компонентах:

1. `Индексные колонки (Index columns)` — одна или несколько колонок, значения которых используются для формирования строк результирующей таблицы. Каждая уникальная комбинация значений в этих колонках формирует отдельную строку.
2. `Колонки значений (Values columns)` — колонка или колонки, данные которых агрегируются. Чаще всего это числовые колонки, но в зависимости от функции агрегации, могут использоваться и нечисловые данные.
3. `Функция агрегации (Aggregation function)` — функция, применяемая к данным колонки значений. В случае сводных таблиц могут быть использованы различные функции (`'mean'`, `'min'`, `'max'`, `'sum'` и другие) для вычисления агрегированных значений по каждой группе.
4. `Колонки для группировки (Columns)` — опциональный параметр, который позволяет дополнительно разделить агрегированные данные по другому измерению, формируя столбцы результирующей таблицы.

```
df.pivot_table(
    values= 'колонка значений', 
    index=  'индексные колонки', 
    columns='колонки для группировки', 
    aggfunc='функция агрегации') 
```

Рассмотрим пример с медицинскими данными, чтобы продемонстрировать, как создаются сводные таблицы:

```
# Создаём сводную таблицу
pivot_treatment_cost = medical_df.pivot_table(
                                    values='Treatment Cost', 
                                    index=['Gender', 'Age Group'], 
                                    columns='Diagnosis', 
                                    aggfunc='sum')
print(pivot_treatment_cost)
 
```

```
Diagnosis          Cold     Fever     Flu
Gender Age Group
Female 18-25        NaN     391.0   846.0
       26-35      180.0     510.0     NaN
       36-45        NaN       NaN   143.0
       46-55      363.0     149.0  1718.0
Male   18-25      317.0       NaN     NaN
       26-35      503.0     153.0     NaN
       36-45      562.0     351.0     NaN
       46-55      717.0     459.0     NaN 
```

В этом примере:

- `values='Treatment Cost'` указывает, что мы агрегируем данные по стоимости лечения;
- `index=['Gender', 'Age Group']` задаёт пол и возрастную группу как индексы строк;
- `columns='Diagnosis'` используется для создания отдельных столбцов для каждого диагноза;
- `aggfunc='sum'` определяет, что мы рассчитываем суммарную стоимость лечения для каждой комбинации пола, возрастной группы и диагноза.

В итоге получим сводную таблицу, которая иллюстрирует распределение стоимости лечения по диагнозам в разрезе пола и возрастных групп пациентов.

Когда мы опускаем один из параметров, мы получаем разные виды агрегаций:

- без указания `aggfunc` применяется метод `mean`;
- без `values` pandas попытается агрегировать все числовые столбцы;

```
# Создаём сводную таблицу без values и aggfunc
pivot_treatment_cost = medical_df.pivot_table(index=['Gender', 'Age Group'], columns='Diagnosis')
print(pivot_treatment_cost)
 
```

```
                 Treatment Cost

### **Дополнительные аргументы метода `pivot_table`**

В дополнение к основным параметрам метода `pivot_table` существуют и другие параметры, которые могут пригодиться в анализе данных:

1. `fill_value`: этот параметр позволяет задать конкретное значение для замены пропущенных данных в результате агрегации. Он позволяет избежать отображения `NaN` в вашей сводной таблице.
2. `margins`: при установке в `True`, этот параметр вносит дополнительную строку или столбец в таблицу, предоставляя итоговые суммы или средние значения по всем группам. Это удобно для получения общего представления о данных.
3. `dropna`: если установлен в `True` (по умолчанию), этот параметр гарантирует, что в результате будут представлены только те столбцы, для которых есть хотя бы одно ненулевое значение. Это помогает сохранить чистоту и читаемость сводной таблицы, исключая столбцы, полностью состоящие из пропущенных значений.
4. `margins_name`: этот параметр позволяет задать настраиваемое имя для строки или столбца с итоговыми значениями, когда используется `margins=True`. По умолчанию эта строка или столбец называется **`'All'`**, но вы можете изменить его на более подходящее для вашего анализа название.

Эти параметры позволяют гибко настраивать процесс агрегации и представления результатов анализа в pandas. Рассмотрим несколько примеров ниже.

При работе со сводными таблицами важно переименовывать столбцы и сбрасывать мультииндекс. Это не просто технические приёмы обработки данных в pandas, но и важные шаги, которые могут повысить качество аналитической работы:

- Переименование столбцов позволяет заменить технические или слишком общие наименования на более информативные. Это особенно важно при подготовке отчётов или презентаций для аудитории, не знакомой с деталями вашего анализа. Чёткие и содержательные названия столбцов облегчают понимание ключевых выводов из данных.
- Сброс мультииндекса и преобразование его в столбцы делает структуру таблицы более простой, что упрощает выборку данных, фильтрацию и выполнение операций среза. Это также упрощает экспорт данных в другие форматы или инструменты анализа, где мультииндексы могут не поддерживаться или работать не так, как ожидалось.

Для статистического анализа или машинного обучения данные часто требуется представить в формате, где каждая строка представляет собой отдельный элемент (например, клиент, покупка, событие) и содержит всю доступную информацию об этом элементе в виде столбцов. С другой стороны, для целей визуализации или представления данных в отчётах может потребоваться другая структура данных, подходящая для интерпретации человеком. В этих случаях могут использоваться многоуровневые структуры или данные в широком формате, позволяющие более наглядно представить взаимосвязи и иерархии в данных. Исходя из этого, сброс индексов позволяет адаптировать формат данных под конкретные аналитические нужды.

## Сортировка данных
Сортировка данных — это фундаментальная операция, позволяющая упорядочивать данные в логической последовательности. Это может быть сортировка от меньшего к большему или от большего к меньшему. Сортировка играет ключевую роль в обработке и анализе данных, поскольку она помогает облегчить поиск закономерностей, тенденций и аномалий.

В бизнес-контексте сортировка используется для ранжирования продуктов по уровню продаж, определения самых ценных клиентов по объёму покупок, или упорядочивания задач по приоритету и срокам выполнения. Она помогает принимать обоснованные решения, оптимизировать процессы и улучшать взаимодействие с клиентами.

В профессиональной среде существуют стандарты представления информации. Несортированные данные воспринимаются как неаккуратные или неполные. В бизнесе и науке часто требуется быстро передать ключевую информацию. Сортированные данные позволяют быстрее обрабатывать информацию и принимать решения.

Сортировка позволяет нам организовать данные и облегчить их анализ. При этом сортировать датафрейм можно как по одному столбцу, так и по нескольким.

Метод `sort_values()` в pandas — это инструмент для сортировки данных в датафрейме или Series. Он позволяет упорядочивать данные по значениям одного или нескольких столбцов, как в порядке возрастания, так и убывания.

Синтаксис метода `sort_values()` выглядит так:

```
df.sort_values(by, axis=0, ascending=True, inplace=False, kind='quicksort', na_position='last') 
```

Разберём каждый параметр:

- `by` — имя или список имён столбцов, которые необходимо отсортировать.
- `axis` — 0 или `'index'`, 1 или `'columns'`, по умолчанию 0. Определяет, будет ли сортировка проводиться по строкам или столбцам.
- `ascending` — bool или list of bool, по умолчанию `True`. Указывает направление сортировки. `True` для сортировки по возрастанию, `False` для сортировки по убыванию. Если указан список, каждый элемент списка применяется к соответствующему столбцу в `by` соответственно порядку в списке.
- `na_position`: `'first'` или `'last'`, по умолчанию `'last'`. Определяет, куда помещать `NaN` значения: в начало или в конец отсортированного списка.
- `inplace` bool, по умолчанию `False`. Если установлен в `True`, изменения будут внесены в исходный датафрейм, без создания нового датафрейма. По умолчанию методы pandas возвращают измененную копию датафрейма, но с использованием `inplace=True` изменения будут применены к самому датафрейму.
- `key` — функция, которая будет применена ко всем значениям перед сортировкой. Может быть полезно указать, например `key=abs`, если вы хотите отсортировать числовые значения по их абсолютной величине. Это не приведёт к изменению значений в датафрейме.

Сортировка по одной колонке актуальна, когда требуется упорядочить данные по одному критерию:

- Для просмотра экстремальных значений: например, определения самых дорогих или самых дешёвых продуктов в списке.
- Для хронологической организации: когда данные необходимо расположить в порядке возрастания или убывания даты и времени, чтобы отслеживать тенденции или события во времени.
- Для подготовки к визуализации: для создания графиков, где важно представить данные в определённом порядке, например, столбчатая диаграмма с продажами по месяцам.
- Для упрощения поиска: когда нужно облегчить поиск данных, например, сортируя список клиентов по алфавиту.

Метод `sort_values()` позволяет проводить сортировку и по нескольким колонкам. Этот подход используется, когда анализ должен учитывать несколько уровней группировки или когда необходимо упорядочить данные по нескольким критериям одновременно:

- Многоуровневый анализ: например, сначала группировка данных по региону, а затем внутри каждого региона — сортировка по объёму продаж для выявления лидеров продаж в каждом регионе.
- Определение приоритетов: когда требуется организовать задачи или события с учётом нескольких факторов, например, сначала по срочности, а затем по важности.
- Детализированный анализ тенденций: для исследования, как изменения одной переменной влияют на другую в разных группах или категориях.
- Управление сложными данными: в случаях, когда одного критерия недостаточно для адекватной организации и анализа данных, например, при сортировке пациентов сначала по типу лечения, а затем по исходу лечения.

# Пропуски
## Что такое ничего?
Данные, с которыми работают аналитики, бывают неполными. Иногда они содержат пропущенные значения, которые могут повлиять на результат анализа, его точность и достоверность. Например, при оценке конверсии пользователя в покупку товара в приложении маркетплейса пропущенные события, связанные с добавлением товара в корзину и его оплатой, могут повлиять на итоговое значение конверсии — снизить её. Или если при оценке средней стоимости заказа пользователей в интернет-магазине будут отсутствовать данные для некоторых категорий товаров, то итоговые значения могут быть недостоверными.

Пропущенные значения могут возникать по разным причинам: из-за человеческого фактора при заполнении данных, технических сбоев, ошибок сбора и обработки данных. Например, при заполнении анкеты клиента банка сотрудник банка случайно пропустил поле с фамилией клиента.

Иногда пропуски — это не ошибка в данных, а их особенность. Например, пропуск в поле «Отчество» клиента банка может быть из-за того, что у клиента нет отчества.

При подготовке данных к исследованию аналитик изучает количество пропущенных значений и причины их появления. Это помогает определиться с действиями по их обработке: оставить пропуски, удалить данные, восстановить пропуски или заполнить их расчётными значениями. Вы узнаете:

- Какие виды пропусков существуют и почему они появляются.
- Какими основными подходами к обработке пропусков пользуются аналитики при подготовке данных к изучению.

### Виды и причины появления пропусков

Перед обработкой пропусков необходимо разобраться в причине их возникновения. Аналитик должен понять, существует ли какая-то закономерность в их появлении. Как правило, пропуски можно разделить на две основные категории:

1. Появление которых не зависит от самих данных, в их появлении нет закономерности.
2. Появление которых зависит от данных и позволяет установить закономерность.

![](https://pictures.s3.yandex.net/resources/4.4.2.1_1_1711204142.png)

_Разберём эти две категории по отдельности._

### Пропуски, не зависящие от данных

В этом случае значения пропущены полностью случайно, их появление не зависит от других известных или неизвестных признаков в данных.

Обычно пропуски этого типа связаны с технической ошибкой при сборе данных или человеческим фактором. Например, при передаче данных об активности пользователей в приложении маркетплейса возникали технические проблемы и для некоторых пользователей их действия в приложении просто не были зафиксированы. В этом случае пропуски будут распределены среди данных равномерно.

Пропуски такого типа называются **MСAR** — от английского Missing Completely At Random, «пропущенные значения полностью случайны».

### Пропуски, зависящие от данных

Бывает, что пропущенные значения характерны только для определённых категорий или признаков данных. Например, в активности пользователей в приложении маркетплейса пропуски возникают только у пользователей планшетной версии приложения. Это может быть связано с техническими проблемами при сборе данных для планшетной версии приложения. При этом внутри категории пропуски будут полностью случайны.

Такой вид пропусков называют **MAR** — от английского Missing At Random, «пропущенные значения случайны» (но уже не полностью!). То есть пропуски в данных распределены случайным образом, но только внутри части данных. Закономерность найти можно, но связана она будет только с категорией — внутри никакой логики уже не будет.

Кроме пропусков **MAR**, существуют пропуски **MNAR** — Missing Not At Random, «пропущенные значения не случайны». Они чётко зависят от каких-то признаков или самих данных, но мы эти признаки измерить не можем либо у нас для этого недостаточно данных.

Рассмотрим на примере информации о клиентах банка. При заполнении анкеты клиент может пропустить ответ на вопрос о своём среднем доходе, если он не хочет или не может по каким-то причинам его озвучивать. В этом случае мы не будем знать о такой ситуации — для нас такой пропуск в данных будет зависеть от неизвестных признаков.

Обратите внимание, что **MNAR** и **MAR** можно спутать, пока не будет полного понимания о зависимости пропусков от данных.

### Обработка пропущенных значений

Итак, объём пропусков и их причины установлены. После этого аналитикам нужно обработать эти пропущенные значения разными методами. Если стратегия обработки будет выбрана некорректно, результаты анализа будут искажены или недостаточно достоверны.

Важно знать, что не все библиотеки Python смогут корректно работать с данными, в которых есть пропуски. Например, это касается некоторых статистических моделей, с которыми вы познакомитесь в следующих уроках.

### Восстановление пропущенных значений

Самый лучший способ обработать пропущенные значения — это восстановить их. Можно запросить дополнительную информацию у коллег, собирающих данные, или найти их в открытом источнике. Например, если в базе данных книжного магазина есть пропуски в авторе изданий, то другая информация о книге поможет восстановить эти пропуски.

Также можно провести дополнительный сбор данных, если это возможно. Например, отправить клиентам рассылку с просьбой заполнить определённые поля в анкете.

Такой способ восстановления можно применять к пропускам любого типа, но он очень трудоёмкий. А иногда восстановить пропущенные значения вообще невозможно, например, если заполнившие анкеты люди не оставили контактных данных.

### Удаление данных с пропусками

Существует два подхода к удалению данных с пропусками:

1. Удаляются все строки с пропущенными значениями хотя бы в одном из столбцов. В этом случае анализ данных называют **анализом полных наблюдений** (от английского complete case analysis), поскольку данные после обработки не содержат пропусков.
2. Пропущенные значения удаляются только для тех данных, которые используются для анализа. Такой подход к удалению данных называют **анализом доступных данных** (от английского available case analysis), поскольку для расчётов будут использованы все возможные данные, даже с пропусками по другим признакам. Большинство методов pandas по умолчанию используют такой подход при расчёте различных значений, например, среднего или суммы.

**Плюсы метода.** Обычно удаление применяют для пропусков **MCAR**-типа. Поскольку пропуски случайны, их удаление не должно повлиять на основные характеристики выборки данных.

**Минусы метода.** К сожалению, удаление данных, особенно если применяется **анализ полных наблюдений**, может существенно уменьшить выборку и снизить репрезентативность результатов. Отбрасывать строки с пропусками следует с осторожностью, наблюдая, какой именно процент данных вы потеряете. Особенно это важно, если в датасете много столбцов с пропусками в разных местах.

С **MAR** и **MNAR** сложнее: удаление может вызвать смещение статистических свойств выборки. Например, исчезнут клиенты только из одного города, в котором была допущена ошибка при сборе данных.

С **анализом доступных данных** всё тоже непросто. При сравнении разных метрик размер выборки может получиться разный, и это плохо повлияет на результаты анализа.

### Индикаторный метод, или присвоение уникальной категории

В этом случае пропуски заменяют на выбранное заранее значение — число или категорию данных. При этом значения выбирают такие, которые в реальных данных не встречаются. Например, пропуски в информации о количестве детей можно заменить на отрицательное значение (-1), а пропуски в возрасте клиента — на заведомо нереальное значение (например, 999).

Такой подход можно использовать для замены пропусков разного типа, когда восстановить данные нельзя.

**Плюсы метода.** Он помогает сохранить структуру и первоначальный объём данных. Ещё индикаторный метод хорошо подходит для заполнения пропусков в столбцах с категориями — вы просто добавляете ещё одну категорию, которая теперь характеризует данные с пропусками.

**Минусы метода.** Этот метод требует помнить, какое именно значение использовалось в качестве индикаторного. Его придётся учитывать при анализе данных, чтобы оно случайно не попало в агрегацию и не испортило результаты. Кроме того, непосредственно для анализа это новое значение использовать будет нельзя.

### Замена пропусков с учётом других значений в столбце

В этом случае пропуски заменяются на агрегирующие значения. Например, среднее арифметическое или медиану по всему столбцу или по каждой из категорий отдельно, если это не противоречит данным. Использовать замену на одно значение следует осторожно, поскольку это может привести к неверным результатам, особенно если таких замен будет много.

Например, пропуски в данных о среднем доходе клиента можно заменить средним или медианным значением дохода в зависимости от других доступных характеристик (например, возраста, образования, занимаемой должности, региона проживания).

Ещё заменять пропуски в данных можно на основе предыдущих или следующих значений. Это подходит для данных, связанных с временными рядами (например, измерений температуры в течение дня). Если промежуток времени между изменениями маленький, то можно заполнить пропуск предыдущим или следующим значением.

**Плюсы метода.** Замена пропусков позволяет сохранить структуру данных и их объём. Обычно этот метод применяют для пропусков **MAR** или **MCAR**, когда можно проследить зависимость данных с пропусками от других известных признаков и данных.

**Минусы метода.** Такой подход может изменить изначальное распределение данных — сделать его более сконцентрированным вокруг средних значений и уменьшить разброс.

### Многомерные методы замены пропущенных значений

Все методы выше — одномерные (англ. single Imputation). В них для заполнения данных используют тот же самый признак. Однако при анализе данных, особенно при решении задач машинного обучения, используются **многомерные методы** (англ. multivariate Imputation). Такие методы позволяют заполнять пропущенные значения в одном столбце на основе данных других столбцов. Про такой процесс говорят: «импутировать данные на основании какой-то модели».

При реализации многомерных методов используют модели машинного обучения, которые позволяют смоделировать пропущенные значения. Это могут быть модели нейронных сетей или кластеризации данных либо регрессионные методы. Они позволяют предсказать недостающие значения, основываясь на данных из других столбцов.

**Плюсы методов.** Их можно использовать для создания сложных алгоритмов при замене пропусков типа **MNAR**. Часто такие методы дают достаточно реалистичные результаты, которые нельзя получить другими способами.

**Минусы методов.** Перед применением методов нужно убедиться, что данные действительно взаимосвязаны между собой, иначе предсказание будет некорректным. Также использование регрессионных моделей может снизить разнообразие значений в выборке.

- Лучше всего восстанавливать пропуски из доступных источников данных. Но часто это или слишком долго, или дорого, или вообще невозможно.
- Если пропусков немного и они полностью случайны, строки можно просто удалить.
- Если данные с пропусками как-то связаны с другими данными, их можно заменить на определённое значение с учётом других значений в столбце и категорий данных.
- Когда нельзя определить взаимосвязь пропусков с данными, но нужно сохранить структуру и первоначальный объём данных, пропуски можно заменить на индикаторное значение.

В следующих уроках вы на примерах разберёте основные подходы к обработке пропусков.

## Подсчёт и удаление пропусков
Перед тем как обработать пропуски и удалить строки с пропущенными значениями, нужно оценить их количество и долю от всего объёма данных. Так станет понятно, насколько данные полные и будут ли они репрезентативными для исследования после обработки.

В этом уроке на примерах мы расскажем:

- как подсчитать количество и долю данных с пропущенными значениями с помощью метода `.isna()`;
- как удалять строки с пропущенными значениями, используя метод `.dropna()`.

### Подсчёт количества пропусков в данных, метод `.isna()`

В предыдущих уроках вы изучили, что в **Pandas** существует несколько подходов для анализа общего количества строк в датафрейме и строк, содержащих данные. Вспомните, как использовались методы `.info()` и `.count()`, чтобы получить информацию о количестве строк с данными и атрибут датафрейма `shape`, или функция `len()` для анализа общего количества строк.

Представьте, что вам надо повторить исследование, которое делал коллега, но код в одной из ячеек, идущей сразу после загрузки датафрейма, удалён. Вы видите (9, 6) в результатах работы кода.
чем может быть это значение.
Это количество строк и столбцов в датафрейме — dataframe.shape

Свойство shape датафрейма содержит кортеж с количеством строк и столбцов. dataframe.count() — это не число, а отдельный столбец с количеством присутствующих значений, dataframe.info() выдаёт форматированную информацию, а функция len() принимает только один аргумент.

Обо всём по порядку. С помощью метода `.info()` можно вывести общую информацию о датафрейме, например, Яндекс Музыки:

```
# Получим общую информацию о датафрейме:
music_df.info() 
```

Информация выше показывает, что датафрейм состоит из 67963 строк. При этом не все столбцы полные: `total_play` содержит 67399 строк с данными, `artist` — 59647, `genre` — 64661, а столбец `track` — 64804 строки. Только один столбец `user_id` не содержит пропусков.

Аналогичную информацию можно получить с помощью метода `.count()`, применив его к датафрейму:

Полученная информация полезна для знакомства с данными: из неё можно узнать, что часть столбцов содержит пропущенные значения, при этом их количество разное.

Но для того, чтобы определить абсолютное количество пропущенных значений, нужно выполнить дополнительные действия (например, вычесть количество строк с данными из общего количества строк, содержащихся в датафрейме):

```
# Подсчитаем количество строк с пропусками:
print(music_df.shape[0] - music_df.count()) 
```

total_play     564
artist        8316
genre         3302
track         3159
dtype: int64 
```

Конечно, можно рассчитать количество пропущенных значений вручную, но в **Pandas** всё предусмотрели. Существует специальный метод, который позволяет работать с пропущенными значениями, — `.isna()`, от английского is not available, «недоступно». Так в **Pandas** обозначаются пропущенные значения.

Метод `.isna()` можно применить к датафрейму или серии данных. Он вернёт логическую или булевую маску пропущенных значений в объекте **DataFrame** или **Series**:

```
# Применим метод .isna() к датафрейму music_df:
print(music_df.isna()) 
```

```
       user_id  total_play  artist  genre  track
0        False       False   False  False  False
1        False       False   False  False  False
2        False       False   False  False  False
3        False       False    True  False  False
4        False       False   False  False  False
...        ...         ...     ...    ...    ... 
```

В результате применения метода к датафрейму `music_df` каждая ячейка данных проверяется на пропущенные значения. Если в них есть пропуски, то они получают логическое значение `True`, а если нет — значение `False`.

Такой подход позволяет точно понимать, какая строка в каком столбце имеет пропущенные значения. Эту информацию можно использовать для подсчёта количества пропущенных значений в данных, для фильтрации данных и составления сложных логических индексаций.

Результат метода `.isna()` — это логическая маска, состоящая из `True` и `False`. Подсчитав сумму значений по каждому столбцу, вы получите общее количество значений `True`, поскольку при проведении арифметических вычислений **Python** воспринимает логическое `True` как `1`, а `False` — как `0`.

Вот так можно посчитать количество пропущенных значений в каждом из столбцов датафрейма `music_df`, используя цепочку методов `.isna()` и `.sum()`:

```
# Подсчитаем количество строк с пропусками, используя .isna() и .sum():
print(music_df.isna().sum()) 
```

```
user_id          0
total_play     564
artist        8316
genre         3302
track         3159
dtype: int64 
```

В результате получилось количество данных с пропусками для каждого столбца датафрейма.

Цепочку методов `.isna()` и `.sum()` можно использовать для одного или нескольких столбцов:

```
# Подсчитаем количество строк с пропусками в определённых столбцах:
print(music_df[['artist','track']].isna().sum()) 
```

```
artist    8316
track     3159
dtype: int64 
```

В примерах выше использовалась цепочка методов, которая позволила подсчитать количество пропусков в каждом столбце. А какой код подсчитает общее количество пропусков во всех столбцах сразу?

`sum(df.isna().sum())`
Сработает! Вначале код подсчитает сумму пропусков по каждому столбцу `df.isna().sum()`, а затем с помощью функции **Python** `sum()` просуммирует все значения.

`df.isna().sum().sum()`
Такая цепочка методов позволит подсчитать вначале сумму в каждом столбце, а потом сумму по всем столбцам.

Методы `.isna()` и `.sum()` можно использовать не только для подсчёта количества пропусков в столбцах, но и для расчёта количества пропусков в строках. Для этого нужно передать параметр `axis=1` методу `.sum()`:

```
# Подсчитаем сумму пропусков по строкам:

```
# Найдём строки с тремя и более пропусками:
print(music_df.isna().sum(axis=1).loc[music_df.isna().sum(axis=1) >= 3])

### Подсчёт доли пропущенных значений

Теперь, когда вы обнаружили пропущенные значения и подсчитали их количество, пришло время оценить их объём — какую долю или процент от всех данных составляют пропуски. Для этого нужно поделить количество данных с пропусками на общее количество строк в столбце.

Чтобы получить общее количество строк в датафрейме, используйте функцию `len()` или атрибут датафрейма `shape`:

```
# Подсчитаем долю строк с пропусками:
print(music_df.isna().sum() / len(music_df)) 
```

```
user_id       0.000000
total_play    0.008299
artist        0.122361
genre         0.048585
track         0.046481
dtype: float64 
```

В примере выше цепочка методов `.isna()` и `.sum()` вернула серию значений с количеством пропусков в каждом столбце, а действие `/ len(music_df)` поделило каждое из значений на общее количество строк в датафрейме. Это позволило получить долю пропущенных значений.

Ещё для подсчёта долей пропущенных значений можно использовать свойство логических значений. Ведь метод `.isna()` возвращает логические значения `True` и `False`, то есть `1` и `0`, и доля значений `True` будет равна среднему значению по этому столбцу:

```
# Подсчитаем долю строк с пропусками:
print(music_df.isna().mean()) 
```

```
user_id       0.000000
total_play    0.008299
artist        0.122361
genre         0.048585
track         0.046481
dtype: float64 
```

Использование метода `.mean()` даёт те же результаты, что и действие `music_df.isna().sum() / len(music_df)`. Но в случае с `.mean()` код получился более компактным и не требующим дополнительных вычислений.

Чтобы перевести доли пропущенных значений в процентный вид, нужно отформатировать каждое значение — с этим отлично справится `lambda`-функция. Чтобы её применить, используйте метод `.apply()`:

```
# Подсчитаем процент пропущенных значений:
print(music_df.isna().mean().apply(lambda x: f'{x:.1%}')) 
```

```
user_id        0.0%
total_play     0.8%
artist        12.2%
genre          4.9%
track          4.6%
dtype: float64 
```

Зная долю или процент пропущенных значений, можно оценить влияние пропусков на предстоящий результат.

Например, пропуски в столбце `total_play` составляют менее 1%. Такой низкий процент не позволяет достоверно диагностировать тип пропусков, поскольку сложно изучить зависимость пропусков от других данных. Можно предположить, что это пропуски типа **MCAR** и, скорее всего, они не повлияют на результат.

А вот количество пропусков в столбце `artist` составляет около 12%, и если эти пропуски типа **MAR** или **MNAR**, то удаление такого объёма данных может привести к серьёзному искажению результатов.

Теперь вы знаете, как с помощью метода `.isna()` можно подсчитать количество и долю пропущенных значений. Но, кроме этого, метод полезен для анализа пропущенных значений — в качестве логической маски. В следующем уроке мы детально разберём, как использовать метод `.isna()` при логической индексации и замене пропущенных значений.

### Удаление строк с пропусками, метод `.dropna()`

В прошлом уроке мы говорили, что одним из подходов к обработке данных с пропусками является их удаление. Такой подход рационально использовать, когда пропуски носят абсолютно случайный характер, то есть представлены типом **MCAR**. Но сначала важно убедиться, что оставшихся после удаления данных достаточно для проведения анализа. В этом случае объём данных будет зависеть от цели анализа данных.

Чтобы удалить все строки, которые содержат хотя бы одно пропущенное значение, в **Pandas** можно использовать метод `.dropna()` — от английского drop и not available, «удалить» и «недоступный».

Если применить метод `.dropna()` к датафрейму `music_df`, удалятся все строки, где есть хотя бы один пропуск. Получится такой результат:

```
# Удалим все строки с пропусками и сохраним результат в новом датафрейме:
filtered_df = music_df.dropna()

# Проверим результат удаления пропущенных значений:
print(filtered_df.isna().sum()) 
```

```
user_id       0
total_play    0
artist        0
genre         0
track         0
dtype: int64 
```

В результате все строки с пропущенными значениями были удалены и при этом сократился общий объём данных:

```
# Сравним количество строк до и после удаления пропущенных значений:
print(music_df.shape[0], filtered_df.shape[0]) 
```

```
67963 59196 
```

Важно помнить, что при использовании метода `.dropna()` для всего датафрейма будут удалены все строки, содержащие хотя бы одно пропущенное значение. Этот метод может быть эффективным, если пропуски в данных несущественны или их удаление не повлияет на результат, иначе это может привести к потере большого объёма данных.

Количество строк уменьшилось. В этом случае полезно подсчитать процент отфильтрованных данных:

```
# Подсчитаем процент отфильтрованных данных:
print(round((1 - filtered_df.shape[0] / music_df.shape[0])*100, 2)) 
```

```

```

В результате удаления данных с пропущенными значениями было отфильтровано около 13% строк.

Метод `.dropna()`, который применили ко всему датафрейму, удаляет все строки с пропусками. Иногда это не то, что нам нужно.

Представьте, есть столбец с большим количеством пропущенных значений, который не используется в анализе данных. Если удалить все строки с пропусками в этом столбце, можно потерять большую часть данных и исказить результаты анализа данных. Иногда это делает анализ данных невозможным. В таком случае лучше удалить пропуски только в тех столбцах, которые участвуют в расчёте значений.

Предлагаем подсчитать, например, среднюю длительность прослушивания треков, по данным Яндекс Музыки, — столбец `total_play`. Вначале ещё раз оцените распределение пропущенных значений в данных:

music_df.info() 
```

Меньше 1% данных содержат пропуски в столбце `total_play`, при этом пропуски в названии трека составляют около 5%, а пропуски в названии исполнителя — около 12%.

Предыдущий пример показал, что если удалить все пропуски, то потеряется около 13% данных. А если удалить только в столбце `total_play` — около 1% данных.

Выход такой: если в расчётах важно, чтобы только столбец `total_play` не содержал пропущенных значений, то можно применить метод `.dropna()` только к столбцу `total_play`. Для этого метод `.dropna()` имеет параметр `subset`. В этом параметре указывают названия столбцов, где необходимо удалить строки с пропущенными значениями:

```
# Удалим все строки с пропусками в столбце total_play:
filtered_df = music_df.dropna(subset='total_play')

# Проверим результат удаления пропущенных значений:
print(filtered_df.isna().sum()) 
```

```
# Подсчитаем среднее значение по столбцу total_play:
mean_total_play = filtered_df['total_play'].mean()

print(f'Средняя длительность прослушивания трека = {mean_total_play:.2f} мин.') 
```

```

```

Именно такой подход используют по умолчанию большинство методов pandas, которые рассчитывают агрегирующие значения, например `mean()` и `median()`.

Сравните результаты:

## Замена пропусков
В предыдущих уроках мы рассмотрели основные причины появления пропусков и наиболее распространённые подходы к их обработке. В частности, в прошлом уроке мы изучили механизмы удаления пропусков.

Однако в некоторых случаях удаление пропусков может исказить результаты анализа данных. Это произойдёт, если их много (более 5%) и они не являются полностью случайными (пропуски типа MAR или MNAR). Например, если пропущенные значения взаимосвязаны с другими данными (сегментом пользователей или группой товаров), удаление пропусков может исказить результаты по этому сегменту или группе.

Поэтому вместо удаления пропусков аналитики часто заменяют их на какое-то определённое значение. Это может быть индикаторное значение — число или фраза, которая говорит о том, что в этом столбце были пропуски, или результат вычисления по имеющимся данным (например, среднее значение или медиана).

В этом уроке мы рассмотрим базовые подходы к замене пропущенных значений и разберём:

- что представляют собой пропуски в **pandas**;
- как работает метод `.fillna()` и как провести замену пропусков на индикаторное значение;
- как заменить пропуски с помощью `.loc[]` или индексации и метода `.isna()`.

### Pandas и пропущенные значения

Кроме объектов `None`, пропущенные значения в **pandas** могут быть представлены в виде:

- `NaN`-значений (Not a Number) — если пропуски в численных данных;
- `NaT`-значений (Not a Time) — если пропуски во временных данных типа `datetime`.

Значения `NaN` и `NaT` обозначают отсутствие данных или неизвестное значение. Сравнение этих значений с другими с помощью стандартных операторов сравнения не даст ожидаемого результата. Поскольку значения `NaN` и `NaT` для **pandas** как бы не существуют, то они не могут быть ничему равны, даже самим себе.

Для работы с пропущенными значениями в **pandas** существуют специальные методы. В прошлых уроках мы рассмотрели некоторые из них: `.isna()`, генерирующий булеву маску для отсутствующих значений, и `.dropna()`, удаляющий все строки или столбцы с пропущенными значениями.

В **pandas** существует ещё один метод для работы с пропущенными значениями — `.fillna()` (от англ. «заполнить»), который позволяет заполнить пропущенные значения.

### Замена пропусков с помощью метода `.fillna()`

Базовый синтаксис метода `.fillna()` простой: ему передаётся то значение, на которое необходимо заменить все пропуски в **Series** или **DataFrame**. Это может быть индикаторное значение, среднее, медиана или предыдущее значение в данных.

Далее мы на примерах разберём использование метода `.fillna()` в различных ситуациях.

#### **Замена пропусков на значение-индикатор**

Для числовых данных в качестве индикаторного значения используют такое число, которое не встречается в данных. Например, для возраста клиента можно использовать отрицательное значение `-1` или заведомо нереально высокое значение (например, `999`).

Иногда для замены пропущенных значений в числовых данных в качестве индикатора используют `0`. Однако в этом случае следует убедиться, что данные не могут содержать это значение. Например, `0` в данных о возрасте клиента может означать новорождённого, а `0` в стоимости товара — что это мог быть подарок или товар по акции. Поэтому перед заменой пропусков на индикаторное значение стоит убедиться, что этого значения нет в исходных данных.

Проверим количество пропусков в датафрейме **Яндекс Музыки** и заменим пропуски в столбце `'total_play'` индикаторным значением.

```
# Импортируем библиотеку pandas и загрузим данные:
import pandas as pd
music_df = pd.read_csv('/datasets/music_log.csv')

# Подсчитаем количество пропусков и их долю в данных:
print(music_df.isna().agg(['sum', 'mean']).T) 
```

```
               sum      mean
user_id        0.0  0.000000
total_play   564.0  0.008299
artist      8316.0  0.122361
genre       3302.0  0.048585
track       3159.0  0.046481 
```

Количество пропусков в столбце `'total_play'` — 564 записи, что составляет менее 1%. Заменим пропуски на индикаторное значение, в качестве которого возьмём `0`. Однако перед заменой данных проверим, действительно ли данные не содержат нулевое значение в столбце `'total_play'`:

```
# Найдём минимальное значение для столбца 'total_play':
print(music_df['total_play'].min()) 
```

```
0.0 
```

Нулевое значение есть! Значит, `0` нельзя использовать в качестве индикаторных значений. Но зато нет отрицательных значений, поэтому в качестве индикаторного значения можно выбрать `-1`. Используем метод `.fillna()` и применим его к столбцу `'total_play'`:

```
# Заменим все пропущенные значения в столбце 'total_play' на -1:
music_df['total_play'].fillna(-1, inplace=True)

# Проверим результат: подсчитаем количество пропущенных значений:
print(music_df['total_play'].isna().sum())

# Подсчитаем количество данных в столбце total_play со значением -1:
print(music_df.query('total_play == -1')['total_play'].count()) 
```

```
0
564 
```

В результате все пропуски в столбце `'total_play'` были заменены на `-1`. Обратите внимание, что, как и большинство методов **pandas**, метод `.fillna()` имеет параметр `inplace`, который позволяет вносить изменения в датафрейм без перезаписи.

Метод `.fillna()` можно применить к нескольким столбцам или ко всему датафрейму по аналогии с другими методами **pandas**, которые мы рассматривали в предыдущих уроках.

Помимо параметра `inplace`, метод `.fillna()` имеет ряд других полезных параметров, которые можно использовать при работе с пропусками;

- `value` — определяет значение, на которое будут заменены пропуски; это может быть не только единичное значение — число или строка, но также словарь значений, объект типа **Series** или **DataFrame**;
- параметры `limit` и `axis` — помогают заменить только определённое количество первых встреченных пропущенных значений в каждом столбце (значение `axis=0` — значение по умолчанию) или строке (`axis=1`). Например, при параметрах `limit=5, axis=1` будут заменены первые пять пропущенных значений в каждой строке датафрейма.

Остановимся более детально на параметре `value`. В примерах выше при использовании базового синтаксиса метода параметру `value` мы передавали число или строку. Однако бывают случаи, когда необходимо использовать разные индикаторные значения для разных данных. Например, в анкете клиента пропуск в графе «возраст» можно заменить на `-1`, а пропуск в графе «высшее образование» на «нет сведений».

В этом поможет словарь, где в качестве ключа будет название столбца, а в качестве значения — индикаторное значение. Например, заменим пропуски в столбце `'total_play'` на `-1`, а в столбце `'genre'` на `'unknown'`:

```
# Проведём замену пропусков, используя словарь:
music_df.fillna({'total_play':-1, 'genre':'unknown'}, inplace=True)

# Проверим результат: подсчитаем количество пропущенных значений:
print(music_df.isna().sum()) 
```

```
user_id          0
total_play       0
artist        8316
genre            0
track         3159
dtype: int64 
```

В столбцах `'total_play'` и `'genre'` больше нет пропусков! Однако стоит учитывать, что теперь в столбце `'total_play'` содержатся отрицательные значения, которые будут влиять на результат вычислений: поиска среднего значения или медианы, суммы или минимального значения. Поэтому подобные расчёты лучше проводить или до замены пропусков на индикаторное значение, или фильтровать строки с таким значением. Например, рассчитаем среднее значение в столбце `'total_play'` без индикаторного значения и с ним:

```
# Проведём замену пропусков, используя словарь:
music_df.fillna({'total_play':-1, 'genre':'unknown'}, inplace=True)

# Проверим результат:
print('Среднее значение с учётом индикаторного значения:')
print(music_df['total_play'].mean())

print('Среднее значение без учёта индикаторного значения (исходное):')
print(music_df.query('total_play != -1')['total_play'].mean()) 
```

```
Среднее значение с учётом индикаторного значения:
98.07012832781753
Среднее значение без учёта индикаторного значения (исходное):
98.89915475813383 
```

Среднее значение изменилось. Поэтому, если вы заменяете пропуски на индикаторное значение, не забывайте об этом при дальнейшем анализе данных.

### **Замена пропусков на среднее значение или медиану**

Один из распространённых подходов к замене пропусков в числовых данных — их замена на среднее значение или медиану по этому столбцу. Обычно это делают, когда пропуски не зависят от других данных, не показывают закономерностей с категориями и группами данных, то есть являются пропусками типа MCAR и их количество не превышает нескольких процентов. Например, при анализе среднего времени загрузки приложения вы обнаружили случайные пропуски в столбце с временем загрузки начального экрана. Доля пропусков составила 1%, а их анализ не показал зависимости от типа устройства и других характеристик приложения. В этом случае можно заменить значения на среднее время загрузки.

Для этого передадим методу `.fillna()` среднее значение по заменяемым данным. Например, обработаем таким образом пропуски в столбце `'total_play'`:

```
# Заменим все пропущенные значения в столбце 'total_play' на среднее значение:
music_df['total_play'].fillna(music_df['total_play'].mean(), inplace=True) 
```

Заменить можно и на медиану, если вместо `.mean()` использовать `.median()`.

Такой подход позволяет избежать удаления данных и в целом не будет влиять на среднее арифметическое значение в выборке, если заменить пропуски на среднее значение. Но это может повлиять на другие статистические показатели, если пропусков было много. В этом случае может измениться медиана, мода или сам тип распределения данных. Например, сравним распределение значений `'total_play'` в интервале от 50 до 300 минут до и после замены данных на среднее значение:

![Замена пропущенных значений на среднее или медиану может влиять на тип распределения данных.]

![](https://pictures.s3.yandex.net/resources/4.4.3.5_1_1711204757.png)

На графиках выше хорошо видно, как после замены значений на среднее появился пик в районе 90–100 минут — это результат того, что все пропуски в столбце `'total_play'` заменили на одно из значений данных. Это изменение нужно учитывать при анализе данных. Если далее будут использованы только средние значения, то замена не повлияет на результат анализа, а вот при более сложных расчётах или метриках результаты анализа могут исказиться в сторону более средних значений, особенно если количество пропусков в данных большое (более 5%).

Поэтому аналитики часто используют замену по средним значениям или медиане отдельно для различных категорий или сегментов данных. Более подробно мы разберём такой подход в следующем уроке.

### Замена пропусков с помощью `.loc[]` или индексации и метода `.isna()`

Заменить отсутствующие данные можно с помощью логической индексации и метода `.isna()`, поскольку метод `.isna()` возвращает булеву маску. В этом случае результат метода `.isna()` можно использовать в качестве логической маски `.loc[]`. Например, заменим все пропуски в столбце `'genre'` на `'unknown'`:

```
# Заменим пропуски в столбце genre на значение "unknown":
music_df.loc[music_df['genre'].isna(), 'genre'] = "unknown"

# Проверим результат замены:
music_df.query('genre == "unknown"')['genre'].count() 
```

```
3302 
```

В примере выше логическая индексация проводилась с помощью `.loc[]`: в качестве логической маски был использован результат работы метода `.isna()` — `music_df['genre'].isna()`. Это позволило отобрать все данные с пропущенными значениями. После этого в `.loc[]` был указан столбец `'genre'`, пропущенные значения в котором следует заменить на индикаторное значение. Их также можно заменить на среднее значение:

```
# Заменим пропуски в столбце genre на среднее значение:
music_df.loc[music_df['total_play'].isna(), 'total_play'] = music_df['total_play'].mean() 
```

Преимущество этого подхода в том, что можно создавать сложные логические условия и заменять пропущенные значения в различных категориях или сегментах данных по разному алгоритму. Такой подход широко используется при замене пропусков типа MAR. Более детально мы рассмотрим его в следующем уроке, а сейчас немного попрактикуемся в замене пропусков.

## Замена пропусков в зависимости от категории
В прошлых уроках вы разобрали, как удалить пропуски или заменить их на определённое заранее значение: индикатор, среднее арифметическое значение или медиану. Для этого вы использовали методы `.dropna()`, `.fillna()`, логическую индексацию и метод `.isna()`.

Несмотря на то что замена пропусков индикаторным значением позволяет подготовить датафрейм к анализу, не всегда получится использовать в полной мере значение-индикатор в расчётах.

Например, замена пропусков на среднее арифметическое или медиану позволяет подставить наиболее типичные для выборки значения, но может привести к искажённому результату. Особенно если пропущенные значения неслучайны. Мы разбирали этот момент при анализе гистограмм распределения значений до и после замены пропусков: на гистограмме распределения значений появлялся выраженный пик значений.

Замену пропущенных значений на среднее или медиану хорошо проводить, когда обрабатывается выборка достаточно однородных значений — без сильной вариации данных.

Например, надо проанализировать объявления о продаже квартир в новом жилом микрорайоне, расположенном в пригороде крупного города. В базе данных есть информация о транспортной доступности района — время до ближайшей железнодорожной станции. Однако в некоторых объявлениях эта информация отсутствует. Поскольку все дома в микрорайоне расположены рядом и примерно на одном расстоянии от станции, то можно использовать среднее значение или медиану для замены пропусков.

А если анализировать придётся информацию о нескольких микрорайонах, которые расположены на различном удалении от города, использовать среднее значение или медиану по всем данным будет уже некорректно. Тогда для одних объявлений время до ближайшей станции будет завышено, а для других, наоборот, занижено. В этом случае замену значений необходимо проводить с учётом расположения микрорайонов, в которых продаются квартиры. То есть по отдельным группам или категориям объявлений.

Проводить замену пропущенных значений по группам или категориям можно не только на средние значения или медиану, но и на конкретное значение. Например, если в микрорайоне, где расположены продаваемые квартиры, нет станции метро, то пропуски в столбце с этой информацией можно заменить на «отсутствует» или «нет».

Перед заменой пропущенных значений в определённых группах или категориях данных нужно убедиться, что значения с пропусками имеют зависимость от выделенных категорий или групп данных. В этом случае аналитик изучает данные и выдвигает различные гипотезы по их взаимосвязи между собой.

Например, средний доход клиента может зависеть от уровня образования, стажа работы, должности, сферы деятельности, региона и других признаков. Или значения пробега подержанного автомобиля могут зависеть от года выпуска, типа или модели автомобиля.

В этом уроке мы расскажем, как заменить пропуски в зависимости от категории на конкретное значение или на агрегированные значения по категориям.

### Выбор категорий для замены пропущенных значений

После того как удалось определить тип пропусков, можно подумать над стратегией их обработки:

- Поскольку пропуски в столбце `total_income` для категории «студент», скорее всего, говорят о том, что у этой категории клиентов нет стабильного дохода, их можно заменить на нулевое значение.

- Пропуски в столбце `age` для категорий «пенсионер» и «жен», вероятно, зависят от самих категорий. Значит, их тоже можно заменить на среднее значение или медиану, остальные пропуски в столбце `age`, скорее всего, носят случайный характер, и их можно отфильтровать.

Обратите внимание, что среднее арифметическое значение, как правило, выше, чем медиана. Возможно, в данных есть высокие значения по доходу, поэтому для замены пропусков в столбце `total_income` лучше выбрать медиану для каждой категории.

Так можно заменить пропущенные данные в столбце `total_income` по уровню образования или по типу занятости. Или совместить эти две категории.

Для выбора данных опирайтесь на их количество: чем их больше, тем показательнее средние значения и медиана. А если категории резко различаются по количеству данных, это может повлиять на результат. Как, например, в категории «учёная степень»: в ней небольшое количество строк, по которым сложно получить репрезентативные результаты.

Проверим количество данных для двух совмещённых категорий клиентов — по типу занятости и образованию. Такую группировку данных можно сделать с помощью цикла, как в примерах выше, но в Pandas существует специальные методы. С помощью методов `.groupby()` и `.pivot_table()` можно создавать сводные таблицы и группировать данные. О них детально расскажем в следующих уроках.

А сейчас используем `.groupby()` для группировки данных по двум группам категорий: `education` и `income_type`. Метод `.groupby()` можно совместить с методом `.agg()` и подсчитать медиану и количество данных:

mask = (clients_df['income_type'] == 'пенсионер') & (clients_df['gender'] == 'жен')

### Замена пропусков по категориям на агрегированные значения по данным категориям

Заменить пропуски в столбце `total_income` на медиану по категориям `education` и `income_type` можно так же, как в предыдущем примере. Но тогда понадобится использовать вложенные циклы и для каждой пары категорий из `education` и `income_type` подсчитывать медиану и проводить замену. Это слишком объёмный способ, если используется много категорий:

```
# Заменим пропуски для категории клиентов на медианное значение:

# Объявим цикл, который будет перебирать данные для каждой категории по типу занятости,
# кроме категории "студент":
for income_type in df.query('income_type != "студент"')['income_type'].unique():

    # Объявим вложенный цикл, который будет перебирать каждую категорию
    # клиентов по типу образования:
    for education in df['education'].unique():

        # Создадим логическую маску для создания необходимого среза данных:
        mask = (df['income_type'] == income_type) & (df['education'] == education)

        # Используя логическую маску, рассчитаем медиану total_income:
        median_value = df.loc[mask, 'total_income'].median()

        # Проведём замену пропущенных значений на медиану:
        df.loc[mask & df['total_income'].isna(), 'total_income'] = median_value 
```

Для выполнения аналогичных действий в **Pandas** существует специальный метод `.transform()`. Он в связке с методом группировки данных `.groupby()` и методом замены пропущенных значений `.fillna()` позволяет оперативно заполнить пропущенные значения по результатам группировки данных.

Разберём работу на примере. Заменим пропущенные значения в столбце `total_income` на медиану по категориям `education` и `income_type`:

```
# Заменим пропуски для категории клиентов на медианное значение:
df['total_income'] = df['total_income'].fillna(
    df.groupby(['education','income_type'])['total_income'].transform('median')) 
```

Этот метод используется для применения функций или методов к группам данных внутри **DataFrame**. Он возвращает результат такого же формата, как и исходные данные.

Вот пример работы этого метода:

```

print(df.groupby(['education','income_type'])['total_income'].transform('median')) 
```

Результатом работы `.transform()` является серия данных. В этой серии каждая строка равна медиане значений `total_income` по категориям `education` и `income_type`. Теперь эти значения можно использовать для замены пропущенных значений в столбце `total_income`.

Алгоритм всего кода будет следующий:

1. `df.groupby(['education', 'income_type'])`: происходит группировка данных по двум столбцам: `education` и `income_type`;
2. `['total_income'].transform('median')`: для каждой категории `education` и `income_type` происходит подсчёт медианного значения в столбце `total_income`;
3. `df['total_income'].fillna(…)` проводит замену каждого пропущенного значения по результатам расчёта медианного значения.

Теперь все пропуски заменены по категориям на агрегированные значения по данным категориям. С этими данными можно работать: в них нет широких обобщений, усреднённых значений, которые могли бы исказить результаты.

### Замена пропусков по категориям на конкретное значение

Приступим к обработке пропусков в столбце `total_income` и заменим пропуски для категории клиентов `студенты` на значение `0`. Для этого используем логическую индексацию:

```
# Заменим пропуски для категории клиентов студенты на значение 0
# Создадим логическую маску для создания необходимого среза данных:
mask = (clients_df['income_type'] == 'студент') & (clients_df['total_income'].isna())

# Проведём замену пропущенных значений на 0:
clients_df.loc[mask , 'total_income'] = 0 
```

В примере выше, в операторе `.loc[]`, мы передали логическую маску с двумя условиями, которые должны быть истинными:

- `clients_df['income_type'] == 'студент'` — категория клиента по типу занятости должна быть равной `'студент'`, так мы оставили только необходимую категорию клиентов;
- `clients_df['total_income'].isna())` — значение в столбце `total_income` должно быть пропущенным, то есть `True`.

В результате выполнения такого кода все пропуски для категории студентов в столбце `total_income` заменены на `0`.

Выше мы решили заменить пропуски в столбце `age` для категории клиентов женского пола и занятости «пенсионер» на медианное значение по этой категории. Проведём эту замену данных.

## Особенности работы с пропусками
В предыдущих уроках вы изучили основные типы пропусков и причины их возникновения, а также разобрали базовые подходы к работе с пропущенными значениями. Напомним: пропуски могут быть полностью случайными (MCAR) или зависеть от других признаков или значений в данных, которые известны аналитику (MAR) и неизвестны (MNAR).

Конечно, лучше всего восстановить пропуски из исходных данных или других источников информации, но это не всегда возможно. Поэтому существуют базовые подходы к обработке пропусков:

- полное или частичное удаление;
- замена на индикаторное значение или на заданное значение в зависимости от категории данных.

Более сложные подходы к замене пропусков базируются на многомерных методах и зависят от поставленных целей анализа данных.

В этом уроке мы рассмотрим дополнительные особенности работы с пропущенными значениями, которые пригодятся при подготовке данных к анализу.

### Изменение типа данных с пропусками

Наличие пропусков в данных влияет не только на их анализ и его результаты, но и на то, в каком виде данные с пропущенными значениями находятся в датафрейме. Когда в столбце датафрейма содержатся пропуски, то тип данных может измениться следующим образом: целые числа `int` меняются на `float` — числа с плавающий запятой.

Это происходит, потому что у целочисленных типов данных, таких как `int`, фиксированный формат хранения — один бит на знак (если он есть), оставшиеся биты однозначно превращаются в число. Поэтому `int`-числа занимают мало места, но не хранят значения «бесконечность» `inf` или «неопределённость» `NaN`, а тип `float` предоставляет такую возможность.

При этом если изначально тип данных был `float`, то он не изменится:

```
# Загрузим необходимые библиотеки:
import pandas as pd
import numpy as np

# Создадим небольшой датафрейм для примера:
df = pd.DataFrame({'column_A': [1, 2, np.nan], 
                   'column_B': [0.5, np.nan, 5.75], 
                   'column_С': [1, 2, 5]})

# Проверим типы данных:
print(df.dtypes) 
```

```
column_A    float64
column_B    float64
column_С      int64
dtype: object 
```

Как видно из результатов выше, тип данных в столбце `column_A`, где были только целочисленные значения и пропуски, — `float`, тогда как в столбце `column_С`, где только целочисленные значения, — тип данных `int` не изменился, поскольку в этом столбце не было пропусков.

Если исходно данные были категориальные или содержали строки, то их тип не поменяется. Например, данные со строками:

```
# Загрузим необходимые библиотеки:
import pandas as pd
import numpy as np

# Создадим небольшой датафрейм для примера:
df = pd.DataFrame({'column_A': ['string_a', 'string_b', np.nan], 
                   'column_B': ['string_a', 'string_b', 'string_c']})

# Проверим типы данных:
print(df.dtypes) 
```

```
column_A    object
column_B    object
dtype: object 
```

Если данные содержали информацию о времени, то их тип также не изменится:

```
import pandas as pd
import numpy as np

df = pd.DataFrame({'column_A': [1, 2, np.nan], 
                   'column_B': [pd.to_datetime('2020-08-12'), 
                                pd.to_datetime('2020-08-13'), np.nan]})

print(df.dtypes) 
```

```
column_A           float64
column_B    datetime64[ns]
dtype: object 
```

### Особенность пропущенных значений Pandas и запросы `.query()`

В Pandas в качестве пропусков используются пропуски Numpy. Одно из их свойств можно использовать при написании запросов в `.query()`. Особенность пропусков Numpy заключается в том, что они не соответствуют никакому значению и считаются несравнимым с любым другим значением, включая само себя. При сравнении как с каким-то значением, так и между собой результат всегда будет `False`. Посмотрите на результат следующего кода:

```
# Загрузим необходимые библиотеки:
import pandas as pd
import numpy as np

# Сравним пропуски друг с другом:
print(np.nan == np.nan) 
```

```
False 
```

Результат сравнения пропусков — `False`. Это свойство можно использовать при фильтрации данных в запросе `.query()`:

- Если нужно оставить данные в столбце без пропусков, то в запросе достаточно указать, что значение в столбце равно самому себе: `column == column`. Соответственно, все значения, кроме пропущенных, при таком сравнении вернут логическое значение `True`. Именно эти значения войдут в срез данных:

```
# Загрузим необходимые библиотеки:
import pandas as pd
import numpy as np

# Создадим небольшой датафрейм для примера:
df = pd.DataFrame({'column_A': [1, 2, np.nan, 4, 5, np.nan], 
                   'column_B': ['a', np.nan, 'с', 'd', np.nan, 'f']})

# Отфильтруем пропущенные значения в столбце column_A:
print(df.query('column_A == column_A')) 
```

```
   column_A column_B
0       1.0        a
1       2.0      NaN
3       4.0        d
4       5.0      NaN 
```

В результате в столбце `column_A` отфильтровались все пропуски. В этом случае запрос `column_A == column_A` будет соответствовать запросу `column_A.isna() == False`.

- Если необходимо, наоборот, оставить пропущенные значения в столбце, например, чтобы проследить их взаимосвязь с другими данными, то можно использовать оператор неравенства:

```
# Оставим только пропущенные значения в столбце column_A:
print(df.query('column_A != column_A')) 
```

```
   column_A column_B
2       NaN        с
5       NaN        f 
```

В результате в столбце `column_A` отфильтрованы все данные, кроме пропусков. В этом случае запрос `column_A != column_A` будет соответствовать запросу `column_A.isna() == True`.

### Когда можно оставить пропуски в исходном виде

Вы уже изучили различные подходы к восстановлению пропущенных значений в данных. Однако не всегда требуется полностью восстановить пропущенные значения в данных или удалить их. Иногда пропуски можно оставить как они есть. В первую очередь это зависит от целей анализа данных.

Если результат анализа данных — расчёт экономических показателей (например, выручки или затрат компании, метрик монетизации, среднего чека), а в результате важно показать истинное значение изменения изучаемого параметра, то лучше использовать только имеющиеся данные, без замены пропущенных значений. В этом случае восстановление пропущенных значений, например, среднего или медианы может изменить истинное значение показателя. Такой жест может привести к ошибочным результатам и принятию неверных решений. Да, возможно без восстановления пропущенных значений результаты будут базироваться на меньшем количестве данных, но они будут надёжнее.

Чаще замену данных используют при изучении общих тенденций изменения параметров или создании моделей поведения метрики или клиентов. Например, при создании модели оттока клиентов в банковской сфере или при изучении динамики среднего чека во времени, когда важно общее направление изменения данных. Обобщая, такой подход пригоден в тех случаях, когда важны не истинные значения исследуемого параметра или метрики, а общая информация о них или выявление общих закономерностей.

Поэтому важно помнить, что принятие решения по обработке пропусков следует обосновать, исследуя причины их возникновения и потенциальное влияние на результаты анализа.

Выберите ситуации, когда пропуски можно заменить на среднее или медиану, или восстановить иным способом, а когда лучше оставить как есть.

### Почему важно изучить причины возникновения пропусков

Пропущенные значения можно рассматривать как ошибки в данных. Для контроля качества данных и их анализа, а также оптимизации сбора данных пропущенные значения следует изучить и предположить причины их возникновения. Это важно в независимости от того, как вы будете обрабатывать пропуски — удалите их, оставите как есть или замените на какие-то значения.

Изучение пропусков на стадии подготовки данных к анализу поможет выявить недочёты в сборе данных или подсветить проблемные места в данных. Эти результаты можно оформить в отдельный отчёт, так называемый баг-репорт с информацией об основных ошибках в данных и возможных причинах их появления. Такая информация пригодится коллегам, собирающим данные.

Например, если вы зафиксировали случайные пропуски MCAR в дате и времени регистрации пользовательских событий, то это может указывать на проблемы с серверным оборудованием или с системой регистрации событий, а может быть и с работой самого приложения. Это будет важно для коллег, занимающихся технической составляющей продукта.

Или, например, обнаружились пропуски MAR в данных из нескольких региональных магазинов крупной сети. Это может говорить о том, что сбор информации в различных точках продаж не оптимизирован, на что следует обратить внимание менеджерам.

Обнаружение пропусков MNAR и изучение возможных факторов, которые их определяют, позволяют оптимизировать процесс сбора информации. Например, запросить дополнительную информацию, которая поможет найти закономерность в появлении этих пропусков. Например, если клиент банка скрывает свой доход, то в анкету можно добавить вопросы, которые могут помочь с выявлением закономерностей: например, если есть предположение, что клиенты с высоким доходом скрывают его, то общий уровень дохода можно определить, зная косвенную информацию, например, количество машин и их марки, количество туристических поездок и другую релевантную информацию.

Улучшение процесса сбора данных позволит получить достоверную информацию. А анализ таких данных поможет принять оптимальные решения для бизнеса.

## Что такое дубликаты и откуда они появляются
По мере углубления в мир анализа данных с помощью Python наш арсенал методов работы с данными неуклонно расширяется. Мы уже научились выделять нужные данные, применять к ним фильтрацию и анализировать первичные результаты. Теперь мы сталкиваемся с новым вызовом — обеспечением качества исходных данных.

Даже самые продуманные процедуры не гарантируют абсолютной чистоты наших данных. При этом мелкие на первый взгляд неточности — дубликаты записей и выбросы — могут серьёзно исказить наши аналитические выводы.

Дубликаты в данных — это не просто маленькие камешки в нашем аналитическом ботинке. Они могут быть как обычными повторениями записей, так и следствием сложных процессов сбора, обработки или объединения данных из различных источников. Понимание того, откуда берутся дубликаты, как они влияют на наши данные и как с ними бороться, — ключ к повышению качества и надёжности наших аналитических выводов. В следующих уроках мы подробно разберём, что такое дубликаты, откуда они появляются и какие методы существуют для их обнаружения и устранения, чтобы ваши аналитические модели были построены на точных данных.

### Что такое дубликаты?

Дубликаты в данных — это повторяющиеся записи, которые могут появляться в результате различных процессов: от ошибок ввода данных до технических проблем в ходе сбора и обработки информации.

Представьте себе, что вы составляете список покупок и случайно вписали молоко дважды. В мире анализа данных подобная ситуация не редкость: одна и та же информация часто появляется более одного раза.

![](https://pictures.s3.yandex.net/resources/4.4.5.1_1_1711222575.png)

Дубликаты искажают реальное представление о данных, что может привести к неправильным заключениям. Например, если данные о продажах одного продукта по ошибке дублируются, это может создать ложное впечатление о высоком спросе на него.

Повторяющиеся данные могут оказать негативное влияние на статистический анализ, снижая точность вычисления среднего арифметического, медианы, дисперсии и других метрик. Это, в свою очередь, ведёт к принятию неверных решений на основе данных.

Удаление дубликатов является критически важным этапом предобработки данных, поскольку чистые и точные данные — это основа надёжного анализа и правильных бизнес-решений.

Существуют две категории дубликатов: **явные** и **неявные**. Для упрощения поиска информации по этой теме полезно знать и английские термины: exact используется для обозначения явных дубликатов, а fuzzy — для неявных.

Поговорим о каждой из категорий более детально, так как подходы к работе с ними различаются.

### Явные дубликаты

Явные дубликаты появляются, когда один и тот же элемент данных повторяется в точности: все детали и значения полей совпадают.

Например, в базе данных клиентов интернет-магазина один и тот же клиент зарегистрирован дважды с абсолютно одинаковыми данными (имя, фамилия, электронная почта и так далее) из-за технической ошибки или повторного ввода данных.

Подобные ситуации часто возникают при объединении данных из разнообразных источников без должной проверки на уникальность или при переносе информации между системами. Иногда из-за недочётов в дизайне интерфейса или его неоднозначности покупатели могут несколько раз повторять одни и те же действия (например, отправлять форму заказа), что в итоге приводит к появлению копий записей.

### Неявные дубликаты

Неявные дубликаты появляются, когда данные фактически описывают один и тот же объект или событие, но представлены немного по-разному — из-за различий в форматировании, опечаток или использования синонимов.

В том же интернет-магазине клиент зарегистрировался дважды: один раз как Иван Иванов с электронной почтой ivanov@example.com, а второй раз — как И. Иванов с той же электронной почтой. Хотя физически это один и тот же человек, различия в написании имени создают неявные дубликаты в базе данных.

Неявные дубликаты сложнее обнаружить и устранить, поскольку для их идентификации требуется анализ контекста и использование более сложных алгоритмов сопоставления данных, о которых мы поговорим в следующих уроках.

## Работа с дубликатами
В этом уроке мы сосредоточимся на ключевых аспектах работы с дубликатами в данных. Умение обнаруживать и корректно обрабатывать дубликаты является важным навыком для любого специалиста, работающего с данными, так как наличие дубликатов может искажать результаты анализа и приводить к неверным выводам. Например, вы можете переоценить количество проданных товаров, среднюю сумму заказа и другие важные показатели, что может повлечь за собой неправильные решения по развитию бизнеса.

Как вы уже знаете, явные дубликаты — это записи в данных, которые полностью идентичны друг другу по всем признакам. Обнаружение таких дубликатов начинается с визуального осмотра данных (если объём данных это позволяет) или использования специализированных функций в таких инструментах, как **pandas** в Python.

### Функция `duplicated()` в pandas

Функция `duplicated()` в библиотеке **pandas** используется для обнаружения дубликатов в **DataFrame**. Она возвращает булево значение, показывающее для каждой строки, является ли она дубликатом предыдущих строк.

Эта функция просматривает **DataFrame** сверху вниз. Для каждой строки она проверяет, встречалась ли уже точно такая же комбинация значений в рассмотренных колонках. Если да, то для этой строки функция возвращает `True`, указывая на то, что это дубликат. Для первого вхождения уникальной комбинации значений возвращается `False`.

```

    |---:|:------------|:--------------|--------------------:|:---------------|:---------------------------|
    |  0 | B001        | T001          |                 100 | 2023-01-01     | False                      |
    |  1 | B002        | T002          |                 150 | 2023-01-02     | False                      |
    |  2 | B003        | X003          |                 120 | 2023-01-03     | False                      |
    |  3 | B001        | T001          |                 100 | 2023-01-01     | True                       |
    |  4 | B004        | X001          |                 100 | 2023-01-01     | еще не проставил           |
     
```

### Аргументы `duplicated()`

Функция `duplicated()` позволяет настроить свою работу под конкретные задачи с помощью аргумента `keep`.

`keep` (необязательный аргумент) определяет, какие из дубликатов следует считать таковыми. Возможные значения (если не указать явное значение, то по умолчанию будет использоваться `'first'` ):

- `'first'`: помечает как дубликаты всё, кроме первого вхождения.
- `'last'`: помечает как дубликаты всё, кроме последнего вхождения.
- `False`: помечает все вхождения как дубликаты.

Эти примеры иллюстрируют, как функция `duplicated()` может использоваться для идентификации дубликатов в данных. Важно определить в разрезе специфики данных и целей анализа, какие именно дубликаты нуждаются в удалении, имеет ли значение наличие дублированных строк и как это повлияет на интерпретацию результатов.

### Работа с неявными дубликатами

### Стандартизация данных

Приведение данных, которые подвержены вариациям, к единой форме позволит преобразовать неявные дубликаты в явные. Тем самым мы упростим процесс очистки данных и повысим точность последующего анализа.

Особое внимание стоит уделить процессу стандартизации текстовых данных. Приведение текста к единой форме является фундаментальным шагом, который значительно упрощает последующее выявление и удаление неявных дубликатов. Давайте подробнее рассмотрим, как данный подход используется на текстовых данных и почему это так важно.

**Приведение к нижнему регистру** — один из самых простых и эффективных способов уменьшить количество вариаций в текстовых данных. Оно помогает избежать распространённой проблемы, когда одно и то же слово или фраза встречается в разных регистрах.

Ещё один способ предобработки данных — **удаление лишних пробелов и знаков пунктуации**, которые могут привести к появлению неявных дубликатов. Использование методов обрезки пробелов и очистки от специальных символов значительно улучшает качество данных.

Используем собственную функцию `standardize_column`, которая принимает столбец `DataFrame` в качестве аргумента и выполняет серию операций для стандартизации его содержимого. В функции последовательно приводятся к нижнему регистру все символы, удаляются заданные специальные символы и обрезаются лишние пробелы в начале и конце строк.

```
# Создание функции стандартизации
def standardize_column(column):
    # приведение к нижнему регистру
    column = column.str.lower()

    # удаление специальных символов
    for char in ['"', '!', '?', '.', ',']:  
        column = column.str.replace(char, '')

    # удаление пробелов в начале и конце строки
    column = column.str.strip()  
    return column

# Копирование датасета
orders_df_new = orders_df.copy()

# Применение функции стандартизации к нужным колонкам
orders_df_new['Клиент'] = standardize_column(orders_df_new['Клиент'])
orders_df_new['Адрес доставки'] = standardize_column(orders_df_new['Адрес доставки'])

# Вывод изменённого DataFrame для проверки результатов стандартизации
display(orders_df_new) 
```

Для проверки эффективности стандартизации можно проверить количество дубликатов до и после стандартизации с помощью функции `duplicated()` или вручную, выбрав несколько случайных записей и проверив, были ли они правильно стандартизированы.

Для обнаружения неявных дубликатов в данных, особенно в больших и сложных наборах, можно использовать и более продвинутые методы и технологии (в рамках данного курса они рассматриваться не будут). Эти методы часто применяются в областях, где высока вероятность возникновения вариаций в данных, таких как естественный язык или клиентские базы данных. Вот некоторые из этих методов:

- **Расстояние Левенштейна (Levenshtein Distance):** мера различия между двумя строками, выраженная через минимальное количество односимвольных редакций (вставки, удаления, замены), необходимых для преобразования одной строки в другую.
- **Токенизация и сравнение множеств:** разбивка строк на множества слов или токенов и сравнение этих множеств с помощью метрик сходства.
- **Семантическое сопоставление:** использование методов обработки естественного языка (NLP) для анализа семантического содержания текста, что позволяет определять сходство на основе не только формы, но и значения слов и выражений.
- **Нахождение наибольшей общей подпоследовательности (LCS):** алгоритмическая задача, которая заключается в поиске самой длинной последовательности, являющейся общим подмножеством двух или более последовательностей.
В этом разделе мы рассмотрели различные аспекты работы с дубликатами в данных — явными и неявными. Мы научились подсчитывать их количество и использовать некоторые подходы к их обработке.

В обработке и анализе данных есть важный принцип, который звучит как "garbage in, garbage out" (GIGO). Этот принцип подчёркивает, что качество выходных данных напрямую зависит от качества входных данных. Если данные изначально некачественные, грязные или плохо подготовленные, то и результаты анализа на их основе будут недостоверными. Поэтому предварительная очистка и подготовка данных, включая удаление дубликатов, является критически важной задачей, чтобы обеспечить точность и надёжность аналитических выводов.

## Что такое выбросы
После того как вы разобрались, как обнаруживать и устранять явные и неявные дубликаты в данных, следует разобраться, как работать с выбросами.

**Выбросы** — это значения в данных, которые заметно отличаются от основной массы наблюдений, и их наличие может кардинально изменить выводы, сделанные на основе данных.

Например, большинство транзакций клиента банка могут представлять собой небольшие и регулярные покупки, но вдруг появляется несколько очень крупных транзакций. Эти большие транзакции могут быть выбросами и указывать на необычное поведение, например, покупку дорогого актива, мошенничество или ошибку ввода данных. Идентификация и анализ таких выбросов критически важны для финансовых институтов для обнаружения подозрительной активности и предотвращения мошенничества.

В здравоохранении выбросы могут проявляться в виде необычно высоких или низких значений в показателях пациентов. Например, если большинство пациентов в наборе данных имеют уровень сахара в крови в пределах нормы, но есть несколько случаев с экстремально высокими или низкими показателями, эти значения могут рассматриваться как выбросы. Они могут указывать на редкие медицинские состояния или острые заболевания, требующие немедленного внимания. Распознавание таких выбросов может способствовать ранней диагностике и своевременному лечению.

Важно понимать, что такие значения могут быть индикаторами как ошибок ввода или обработки данных, так и важных, редких явлений, которые необходимо учитывать. Вот несколько подходов, которые могут помочь в этом:

- **Проверка источника данных.** Ошибки при сборе данных или в процессе их передачи могут привести к аномальным значениям.
- **Графический анализ.** Визуализация данных с помощью гистограмм, ящиков с усами `boxplot` или точечной диаграммы `scatterplot` может помочь визуально определить выбросы и оценить их характер.
- **Расчёт статистических показателей.** Использование статистических мер, таких как стандартное отклонение, межквартильный размах поможет определить, насколько далеко значение лежит от среднего по набору данных.
- **Консультации с экспертами.** Обсуждение аномальных значений с экспертами в соответствующей области может помочь определить их возможное происхождение и значимость.
- **Исторические данные.** Сравнение с историческими данными поможет понять, являются ли подобные значения нормой для конкретного контекста.

С графическими и математическими подходами определения выбросов вы познакомитесь в следующих уроках.

Влияние выбросов на аналитические процессы и статистические модели может быть значительным, поскольку они способны исказить расчёты среднего, стандартного отклонения и других важных показателей. Поэтому точное определение и эффективная обработка выбросов являются критически важными для глубокого анализа данных.

### Классификация выбросов

Различают три класса выбросов в данных:

- глобальные выбросы (global оutliers);
- коллективные (collective оutliers);
- контекстные (сontextual оutliers).

Рассмотрим каждый из них более подробно.

#### Глобальные выбросы

Глобальные выбросы — это точки данных, которые значительно отличаются от общего распределения набора данных. Эти выбросы стоят особняком и не соответствуют общей тенденции или распределению данных.

Такие выбросы могут существенно исказить результаты анализа, например, влияя на расчёт средних значений, что ведёт к неправильным выводам и решениям. В контексте статистического анализа, глобальные выбросы могут ввести в заблуждение, предполагая несуществующие тенденции или отклонения.

Представьте, что в небольшой компании по продаже книг большинство сотрудников получают зарплату в пределах 30-40 тысяч рублей, но один из сотрудников ошибся при заполнении данных и написал вместо 30 тысяч рублей — 300 тысяч рублей. Если рассчитать среднюю зарплату без учёта этого выброса, получится одно значение, существенно отличное от реальной картинки при его учёте, что может ввести в заблуждение относительно стандартов оплаты труда в компании.

```
# Создание двух списков зарплат
salaries_without_outlier = [30, 32, 35, 31, 33, 34, 36, 37, 38, 39]  # Зарплаты без топ-менеджера
salaries_with_outlier = [30, 32, 35, 31, 33, 34, 36, 37, 38, 300]  # Зарплаты с ошибкой

# Рассчет средних зарплат
average_salary_without_outlier = sum(salaries_without_outlier) / len(salaries_without_outlier)
average_salary_with_outlier = sum(salaries_with_outlier) / len(salaries_with_outlier)

print(f'Средняя зарплата в компании без ошибки: {average_salary_without_outlier} тыс. руб')
print(f'Средняя зарплата в компании с ошибкой: {average_salary_with_outlier} тыс. руб') 
```

```
Средняя зарплата в компании без ошибки: 34.5 тыс. руб
Средняя зарплата в компании с ошибкой: 60.6 тыс. руб 
```

Средняя зарплата без учета выброса составляет 34.5 тысяч рублей, что отражает реальную картину оплаты труда для большинства сотрудников. Однако при включении в расчет зарплаты с ошибкой, средняя зарплата увеличивается до 60.6 тысяч рублей.

#### Коллективные выбросы

Коллективные выбросы — это группы точек данных, которые вместе значительно отличаются от общего распределения набора данных. Коллективные выбросы могут не считаться выбросами при индивидуальном рассмотрении, но как группа они демонстрируют необычное поведение.

![](https://pictures.s3.yandex.net/resources/4.4.4.4_2_1711299184.png)

_Визуальное представление коллективного выброса_

Влияние на данные

Такие выбросы важны для выявления структурных аномалий в данных, которые указывают на необычные явления, неочевидные при анализе отдельных точек.

Обработка коллективных выбросов зависит от конкретного случая использования и может включать дальнейший анализ поведения группы и выявление факторов, которые спровоцировали появление выбросов.

Например, аналитик изучает ежедневные продажи продукта в течение года и обнаруживает, что в один период продажи в декабре резко возрастают. Если рассматривать каждый месяц в отдельности, декабрьские данные могут не показаться выбросами, поскольку этот скачок повторяется каждый год и, таким образом, является ожидаемым для бизнеса. Однако, если смотреть на данные в контексте всего года, то группа точек данных, соответствующих декабрьским продажам, становится коллективным выбросом. Так происходит, потому что в совокупности они значительно отличаются от продаж в остальные месяцы и вносят существенное исключение в общий тренд.

Чтобы обнаружить коллективные выбросы, обычно применяют методы разделения данных на сегменты или группировки их по кластерам.

Например, в контексте медицинских данных коллективные выбросы могут сигнализировать о начале эпидемии или вспышке болезни. В таких случаях целесообразно использовать кластеризацию для объединения схожих инцидентов и методы выявления аномалий для идентификации и анализа необычных моделей распространения заболеваний.

#### Контекстные выбросы

Контекстные выбросы — это точки данных, которые значительно отличаются от ожидаемого поведения в конкретном контексте или подгруппе. В отличие от коллективных выбросов, которые выделяются как группа данных, отклоняющихся от общей тенденции, контекстные выбросы требуют понимания специфических обстоятельств или условий для их адекватной интерпретации.

Это особенно важно в таких областях, как временные ряды, географические данные и любые другие сценарии, где поведение данных интерпретируется в рамках конкретного контекста.

![](https://pictures.s3.yandex.net/resources/4.4.4.4_3_1711299202.png)

_Визуальное представление контекстного выброса_

Если в определённый месяц температура значительно отличается от обычной для этого времени года, это будет считаться контекстным выбросом, так как температура является аномальной в контексте ожидаемых погодных условий для заданного месяца.

Влияние на данные

В анализе временных рядов, например, в данных о продажах, контекстный выброс может проявляться в виде резкого увеличения продаж в определённый период, который соответствует праздничным дням или сезонным распродажам. Без учёта этого контекста анализ моделей потребления может привести к неправильным выводам об общих трендах потребительского спроса. Если рассматривать сырые, неагрегированные данные, такое увеличение продаж в конкретный день можно причислить к коллективному выбросу, поскольку он представляет собой группу значений, выделяющихся из общего распределения. Однако при анализе агрегированных данных по дням, где каждый день представлен одним значением, например, общим объёмом продаж за день, то же самое увеличение продаж уже будет восприниматься как контекстный выброс.

Важно учитывать контекст и общие условия, прежде чем делать выводы о том, является ли что-то действительно выбросом или нет. На основе неверно интерпретированных данных могут быть приняты неправильные стратегические решения.

К примеру, компания может ошибочно решить изменить стратегию ценообразования, считая, что спрос на продукт нестабилен, если не учесть сезонные колебания спроса как контекстный фактор.

Понимание природы и причин выбросов играет ключевую роль в их классификации и последующем анализе.

Предположим, мы анализируем данные по уровню холестерина у пациентов в течение года. В большинстве случаев значения распределены довольно равномерно, но в определённый период наблюдается резкий всплеск высоких значений. Если рассматривать эти данные без контекста, можно было бы классифицировать весь этот период как коллективный выброс, поскольку группа данных значительно отклоняется от остальной части распределения.

Однако, дальнейший анализ показывает, что этот всплеск совпал с вирусной эпидемией, в результате которой у многих пациентов возникли осложнения, влияющие на уровень холестерина. Таким образом, после установления косвенной связи между всплеском значений холестерина и эпидемией, этот выброс перестаёт быть глобальным или коллективным, поскольку теперь он имеет чёткий контекст — вирусное заболевание. В этом случае, выбросы становятся контекстными, указывая не на ошибку данных или случайное отклонение, а на специфическое медицинское явление, требующее дополнительного внимания.

Вы рассмотрели классификацию выбросов по окружению. Если вы работаете с датасетом с несколькими численными столбцами, то стоит обращать внимание на то, связан ли выброс в строке только с одним столбцом — тогда говорят про одномерный выброс, или сразу с несколькими — в этом случае речь про многомерный выброс.

Дополнительно выбросы часто разделяют по причинам возникновения:

- **Выбросы из-за ошибок при вводе данных.** Например, опечатки, ввод значения не в то поле и прочие.
- **Выбросы из-за ошибок измерения.** Это выбросы, вызванные ошибками при измерении данных. Это могут быть приборные ошибки, например рулетка, которая начинается не от нуля и отнимает пару сантиметров от объектов измерений.
- **Структурные выбросы.** Это выбросы, которые возникают из-за изменений в структуре данных или в системе, производящей данные. Например, введение нового продукта на рынке может вызвать изменение в паттерне продаж и порождать выбросы.
- **Выбросы из-за смещений в данных.** В городе установили датчики температуры, чтобы определять погоду на улице. Один из датчиков установили неверно — на трубу отопления. Данные с этого датчика будут смещёнными относительно данных с других датчиков. По окружению их стоит отнести к коллективным выбросам.
- **Выбросы из-за аномальных событий.** Это выбросы, вызванные реальными аномалиями или событиями, которые привели к значительным отклонениям от обычного хода событий. Например, финансовый кризис может вызвать выбросы в финансовых данных. Такие выбросы можно отнести к контекстным.

Это открытый список, могут быть другие, более подробные или более крупные типы, например выбросы из-за ошибок при вводе данных можно отнести к выбросам из-за человеческого фактора, можно выделять выбросы из-за изменений в условиях окружающей среды и тому подобные.
