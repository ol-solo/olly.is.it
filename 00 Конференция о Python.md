# Python и аналитика

Как говорит спикер нашей конференции, современная аналитика не может работать без программирования. А если и может, то это будет поверхностный анализ без возможности оптимизации и автоматизации.

В этом уроке вместе с Олегом выясним, почему аналитики всё чаще берутся за код и какие преимущества это даёт. Разберёмся, как программирование помогает решать аналитические задачи быстрее и точнее, а также почему именно Python стал фаворитом среди всех возможных инструментов.

# Как аналитика связана с программированием

Сегодня аналитика всё больше связана с программированием и конкретно языком Python по нескольким причинам:

1. **Мощные инструменты анализа данных** За последнее время появилось множество инструментов анализа данных, которые позволяют аналитикам обрабатывать и визуализировать огромные объёмы информации. Эти инструменты включают в себя как специализированные программы, так и библиотеки для различных языков программирования. Например, библиотека pandas для Python способна обрабатывать данные намного быстрее табличных программ.
2. **Развитие области машинного обучения и искусственного интеллекта** Современные аналитики не ограничиваются простыми методами анализа данных. С использованием программирования и машинного обучения они могут прогнозировать будущие события, классифицировать объекты и группировать их по признакам. Однако возможности машинных методов на этом не заканчиваются. Уже сейчас они могут работать с текстами и изображениями, автоматизировать ввод данных с отсканированных документов и определять подозрительные транзакции.
3. **Требования к автоматизации и оптимизации** Современные компании всё больше ориентированы на данные и стремятся извлекать максимальную пользу из доступной информации. При этом предприниматели не готовы ждать недели или месяцы, пока данные превратятся в отчёты и выводы. Поэтому бизнес нуждается в автоматизации и оптимизации анализа данных, что часто требует использования программирования. Каждый раз, когда мы запускаем любую программу, нам не нужно её создавать заново: она уже сохранена в скрипт. Для аналитики это тоже подходит: запустил программу — и данные обработались.
4. **Большие объёмы данных** В современном мире объёмы данных постоянно растут. Для эффективной обработки и анализа такого массива необходимо программирование. С его помощью можно создать оптимизированные и масштабируемые решения. То, что несколько лет назад казалось огромным объёмом (например, два-три миллиона строк), сейчас обрабатывается без проблем благодаря коду.

# Что такое Python в аналитике

У Python множество сфер применения, благодаря чему он стал одним из самых популярных языков программирования.

Вот основные преимущества Python:

1. **Простота синтаксиса** Синтаксис Python прост и легко читаем. Он напоминает английский язык, что делает его более доступным для новичков. Во многих языках программирования используются скобки или точки с запятой, чтобы отделять элементы кода. В Python такого нет, что делает его проще.
2. **Обширная стандартная библиотека** Python поставляется с обширной стандартной библиотекой, имеющей множество полезных модулей для решения различных задач. Благодаря этому можно написать достаточно мощную программу, не прибегая к сторонним библиотекам и не создавая огромное количество кода.
3. **Большое сообщество и документация** Существует масштабное сообщество пользователей Python, которые активно делятся [знаниями и опытом](https://habr.com/ru/hubs/python/articles/). Есть множество ресурсов: документация, форумы и учебные материалы, которые помогают новичкам быстро освоить язык. [Ответов и решений](https://stackoverflow.com/questions/tagged/python) в сети настолько много, что практически любую проблему можно решить запросом в поисковике.
4. **Многоцелевое использование** Python подходит для широкого спектра задач: веб-разработки, научных вычислений, анализа данных, автоматизации, игр и многого другого. Это делает его привлекательным для различных проектов и областей.

Теперь посмотрим, как выполнить такой расчёт с помощью Python:
```python
import pandas as pd
calls = pd.read_csv('/datasets/calls.csv')
calls['month'] = pd.to_datetime(calls['call_date']).dt.month
calls.groupby('month')['duration'].sum()
print(calls.groupby('month')['duration'].sum())
```




Результат получился таким же, но есть нюансы.

Код повторяет действия программы:

Такая программа выполняется за доли секунд и может быть универсальной: теперь схожую таблицу можно обработать этим же кодом, просто запустив его.

Вы можете возразить, что подсчёт сумм по месяцам — это простая задача, наверняка в сложных ситуациях кода намного больше. Но, написав код один раз, вы без затрат времени решаете задачу для любого аналогичного случая и не настраиваете вручную каждый новый отчёт. С кодом проще отслеживать и устранять ошибки: когда вся логика в одном месте, не нужно беспокоиться о том, что случайный клик в таблице приведёт к неправильному результату.
