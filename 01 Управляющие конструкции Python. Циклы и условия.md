# Основы синтаксиса Python
## Введение

Аналитики работают с данными, но иногда данных бывает слишком много, а исследования — усложняются.

Иногда данные разбросаны по разным таблицам: одни хранятся на вашем компьютере, другие в базах данных. Одни таблицы — среднего размера и обновляются ежедневно, а другие — обновляются ежеминутно и увеличиваются до десятка миллионов строк. Чем больше данных или их источников, тем труднее аналитику обрабатывать их «руками».

Представьте, что вы путешествуете за рубежом. Вы спрашиваете у прохожего, как добраться до нужного вам места. Чтобы человек точно вас понял, лучше говорить на известном ему языке. ### Что такое Python?

Python — это один из самых популярных языков программирования. На нём пишут как простые программы, так и сложные многосоставные системы. На Python можно сделать калькулятор, который складывает два числа, а можно реализовать сложный алгоритм для машинного обучения. Можно даже написать свою игру.

А ещё на этом языке работают с данными, его используют дата-аналитики, дата-сайентисты и дата-инженеры. Python (без использования оптимизированных библиотек) — это язык, на котором можно быстро писать, он хорошо подходит для хранения и обработки сложных структур данных.

Есть множество других языков, все они решают разные задачи, например:

- **JavaScript, TypeScript** — используют в веб-разработке. Благодаря программам на этих языках вы можете кликать на кнопки на сайте, чтобы выполнить определённые действия.
- **Swift, Kotlin** — эти языки используют при разработке iOS и Android-приложений.
- **Go, Java, C++** — используют для систем с высокой нагрузкой, где важна скорость работы и стабильность. Например, их используют в банковских системах. В частности на С++ написана операционная система Windows.
- **R** — используют, как и Python, для работы с данными. Чаще всего в научной среде, реже — в бизнесе.

Каждый язык хорош для своих задач. Для задач аналитика данных подходит Python.

Вот несколько примеров, где аналитик может использовать Python:

- Автоматизировать рутину — создание еженедельного отчёта и отправка его по почте руководству без ручного вмешательства.
- Провести анализ данных — сбор данных из разных источников и их исследование.

### Как это работает?

Кратко обратимся к основам. Компьютер понимает только язык нулей и единиц — это и есть машинный код. Чтобы перевести код вашей программы в машинный код, Python использует специальную программу — интерпретатор.

Интерпретатор — это программа, которая последовательно читает по строчкам ваш код на Python и переводит каждую в машинный код с набором инструкций.

![](https://pictures.s3.yandex.net/resources/image_1702641417.png)

Python называют **интерпретируемым** языком как раз из-за интерпретатора. Ещё есть **компилируемые** языки, в них переводом с языка программирования на машинный код занимается компилятор. Это тоже программа, но в отличие от интерпретатора, она читает код программы сразу, а не построчно. Чтобы компилятор вас правильно понял, вы должны писать программу в соответствии с правилами **синтаксиса** этого языка.

### Почему аналитики пишут на Python?

Аналитики работают с данными: собирают, обрабатывают, передают. И в большинстве случаев используют именно Python. Для начала выясним, зачем вообще аналитикам использовать язык программирования, ведь уже есть готовые приложения, например Excel.

1. **В работе с приложениями вы сталкиваетесь с ограничениями.** Например, в Excel нельзя работать с файлами больше 1 миллиона строк (если точно, то 1 048 576 строк). Это серьёзная проблема при работе с большими данными. В Python таких ограничений нет.
2. **Ручную работу нужно автоматизировать.** Ваша программа может запускаться и совершать те действия, которые вы делаете каждый день как по расписанию. Например, делать запросы в базу данных и создавать отчёт на основе полученных данных. Отвлекаясь от рутины, вы можете уделить внимание более сложным задачам.

Итак, аналитику нужно написать программу. Разберёмся, почему аналитики используют именно Python для своих задач:

1. В первую очередь **всё дело в простоте его синтаксиса**. Не нужно быть разработчиком или чётко представлять устройство компьютера, чтобы освоить базовые понятия и написать свою первую программу на Python. А ещё на Python легче читать и понимать чужой код.
2. На Python **реализовано огромное количество готовых модулей**, которые упрощают работу с данными. Это сокращает время написания кода — не приходится переписывать то, что уже написано другими людьми.
3. Используя только Python, **можно провести полноценное исследование**: загрузить данные из таблицы, исследовать данные, подготовить графики и сохранить результаты. Например, получить данные из нескольких маркетплейсов по интересующей категории товаров, исследовать и визуализировать изменения цены и на основе этого предсказать цены, представить результаты исследования конкурентов организации, занимающейся продажей товаров этой категории.

## Переменные. Вывод переменной на экран

Компьютер «запоминает» информацию, например, строки, числа или целые таблицы, пока выполняется программа.

В этом уроке вы:

- узнаете, как компьютер запоминает данные;
- узнаете, что такое переменные, зачем они нужны и как их правильно называть;
- научитесь выводить переменные на экран.

### Как компьютер хранит данные?

Компьютер хранит информацию в памяти. Память компьютера похожа на хранилище в банке — всё хранится по своим ячейкам. В таких ячейках мы можем хранить всё, что нужно программе: числа, строки или таблицы.

У каждой ячейки есть свой номер. Зная номер ячейки, мы можем получить значение, которое там хранится. Номер ячейки может выглядеть так: `139004536422736`.

В Python такая ячейка в памяти называется объектом, а номер ячейки называется ссылкой на объект.

![На изображении 9 ячеек: 3 ряда по вертикали и 3 ряда по горзонтали. Все они представляют собой память. В первой ячейке написано «Привет!». К ней ведёт стрелка с блоком, подписанным «Объект»](https://pictures.s3.yandex.net/resources/image_1702647740.png)

Использовать такие ссылки в явном виде неудобно: они длинные, ими тяжело пользоваться, а ещё они могут меняться в процессе выполнения программы. Было бы гораздо удобнее, если мы могли дать имя ячейке, которое не будет меняться. В таком случае мы будем использовать имя вместо номера. Именно для этого нужны **переменные**.

### Что такое переменная?

Переменная — это имя для объекта. Например, `a` или `x`, или `robot` — это всё может быть названием переменной. Это «цитаты» из кода, поэтому мы пишем их моноширинным текстом отличающегося формата.

Когда вы первый раз указываете название переменной, это называется _объявить переменную_. В Python переменную всегда объявляют, присваивая ей какое-нибудь значение. Присвоить переменной значение — значит сообщить компьютеру, что в переменной содержится это значение. Достаточно написать имя переменной, поставить `=` и по правую сторону от знака равенства ввести значение, которое вы хотите присвоить переменной. `=` — это оператор присваивания, он сообщает компьютеру, какое действие нужно выполнить — в нашем случае — присвоить значение переменной. Чуть позже мы подробнее расскажем о разных операторах.

![На изображении текст: robot = ‘Cat’. Robot это имя переменной. = это оператор присваивания. ‘Cat’ это значение, которое присваиваем переменной.](https://pictures.s3.yandex.net/resources/image_1702647769.png)

Переменные упрощают работу с данными. Любую строку или число можно сохранить в переменной с именем, которое отражает смысл этого значения. В обычной жизни мы пользуемся похожим трюком:

- Когда добавляем новый контакт в память телефона, мы указываем имя, чтобы не запоминать весь номер.
- Даём названия для файлов и папок на рабочем столе компьютера.

Создадим или, как вы уже узнали из этого урока, объявим переменную на языке Python:

```
robot = 'Cat' 
```

Выше объявлена переменная `robot` с содержимым `Cat`, кавычки вокруг `Cat` нужны были, чтобы Python «понял», что это строка, а не название другой переменной. О строках мы расскажем позже.

Вот ещё пример. Количество конфеток в вашем кармане:

```
candies = 24 
```

Символ `=` — оператор присваивания. Слева от него имя переменной, `candies`. Справа — значение, `24`.

Если вам подарили ещё одну конфетку и вы убрали её в карман, то можно заменить значение переменной `candies`, дописав ещё одну строчку:

```
candies = 24
candies = 25 
```

Мы снова присвоили значение переменной `candies`, и теперь она хранит число `25`.

### Как выбрать имя переменной

Уделите особое внимание названию переменной. Следуйте правилам синтаксиса и хорошему стилю.

**Правила синтаксиса:**

- Имя должно быть уникальным в рамках программы. Оно не должно совпадать с другими именами в выполняемой программе. Если у переменных полностью совпадают имена — значит, это одна переменная.
- Имя начинается с буквы или знака нижнего подчёркивания `_`.
- Имя содержит только буквы, цифры и знак нижнего подчёркивания. Добавлять в имя символы вроде `!` или разделять слова пробелом нельзя.
- Имена переменных — регистрозависимые. `Floors`, `floors` и `FLOORS` — три разные переменные, потому что их имена состоят из символов разного регистра: строчных и прописных.

Если нарушить правила, программа не заработает. Если забыть про стиль — разобраться в коде будет сложно.

**Правила хорошего тона:**

Старайтесь сделать ваш код понятным для себя и тех, кто, возможно, будет работать с ним в будущем. Не редки ситуации, когда вам придётся возвращаться к вашим старым «творениям» — и если не следовать некоторым правилам, то разобраться с кодом будет непросто.

- Имя выражает смысл переменной. Например, для цены товара подойдёт имя `price`, а для названия — `product`.

```
price = 399
product = 'pizza' 
```

- Лучше использовать английские слова, а не писать русские латиницей. Если не хватает владения английским — не страшно. Пока вы изучаете Python, ваш английский улучшается сам собой. А когда сомневаетесь в выборе слова, проверьте себя в онлайн-переводчике — аналитики тоже так делают.
- Когда имя состоит из нескольких слов, их пишут через знак_подчёркивания. Например, имя покупателя — `customer_name`, доход за предыдущий месяц — `previous_month_income`. Такую форму записи называют «змеиным регистром» (англ. Snake case) — в противоположность «верблюжьему» (англ. Camel case): `floorNumberTwo`. Легко запомнить Python — значит, змеиный регистр.
- В имени используют только строчные буквы. И вы знаете, почему так.

Для Python есть целый стандарт языка — [PEP-8](https://peps.python.org/pep-0008/). Это собрание правил хорошего стиля, которого стараются придерживаться все, кто пишут на Python.

### Вывод переменной на экран

Мы научились присваивать значение переменной и даже заменять его другим значением. Но что делать, если нужно узнать, какое значение хранится в переменной прямо сейчас? Обычно для этого выводят переменную на экран:

```
price = 399
product = 'pizza'

print(price) 
```

```
399 
```

Для вывода на экран мы использовали функцию `print()`. Это первая функция, с которой вы познакомитесь. То, что находится в круглых скобках `( )`, называют аргументами функции. В нашем случае `price` — аргумент функции `print()`.

Вызвав ещё раз функцию `print()`, мы можем вывести значение в переменной `product`, но уже на новой строке.

```
price = 399
product = 'pizza'

print(price)
print(product) 
```

```
399
pizza 
```

В конце каждой строки функция `print()` ставит символ `\n` — это символ перевода на следующую строку. Такой символ называют специальным.

Если выполнить код, в котором несколько раз подряд вызывается функция `print()`, то на самом деле результат будет следующий:
![[Pasted image 20260319111955.png]]

Но мы увидим:

```
399
pizza 
```

Компьютер видит специальный символ `\n` и переводит `pizza` на новую строку. Именно поэтому при каждом вызове `print()` значение выводится на новой строке.

Используя аргумент `end`, можно указать, какой символ нужно ставить вместо `\n`. Например, если подставить дефис `-`, то при каждом вызове функции значение будет печататься на той же строке.

```
print(price, end= ' - ')
print(product) 
```

```
399 - pizza 
```

Для красоты мы добавили пробелы справа и слева от дефиса.

Функция `print()` может вывести на экран сразу несколько значений:

```
price = 399
product = 'pizza'

print(price, product) 
```

```
399 pizza 
```

В таком случае аргументы перечисляются через запятую и выводятся на экран по порядку в одну строку.

Значения разделяются пробелом. Используя аргумент `sep` (от англ. separate — разделить), можно указать любой другой символ. Как и в примере выше, используем дефис:

```
price = 399
product = 'pizza'

print(price, product, sep = ' — ') 
```

```
399 — pizza 
```

## Арифметические операции

В этом уроке вы:

- познакомитесь с арифметическими действиями в Python;
- научитесь применять арифметические операторы на реальных задачах.

### Арифметические операции

В Python есть все те арифметические операции, к которым вы привыкли:

- `+` — сложение;
- `-` — вычитание;
- `*` — умножение;
- `/` — деление.

```
print(2 + 2)
print(27 / 9) 
```

```
4
3.0 
```

Это похоже на работу с калькулятором. Арифметические знаки в Python называют операторами, например `+` — оператор сложения. Операторы принято отделять пробелами — так легче читать код.

### Порядок действий

Порядок выполнения арифметических операций в Python соответствует привычному «школьному»:

1. Действия в скобках.
2. Умножение, деление.
3. Сложение, вычитание.

Решим такой пример: `2 + 5 * 4 / 2 - 8 + 3 * (3 + 2)`.

- Сначала скобки: `(3 + 2) = 5`. Получаем `2 + 5 * 4 / 2 - 8 + 3 * 5`.
- Потом идём слева направо. Умножение: `5 * 4 = 20`. Пример сокращается до `2 + 20 / 2 - 8 + 3 * 5`.
- Затем деление: `20 / 2 = 10`. Получаем `2 + 10 - 8 + 3 * 5`.
- Умножаем три на пять, остаётся `2 + 10 - 8 + 15`.
- Складываем два и десять, получаем `12 - 8 + 15`.
- После вычитания остаётся `4 + 15`.
- Ответ: `19`.
    

Вот как справится Python:

```
print(2 + 5 * 4 / 2 - 8 + 3 * (3 + 2)) 
```

```
19.0  
```

Ответ тот же.

Почему результат выглядит так?

Вместо `19` мы увидели `19.0`, откуда добавилось `.0`? Результат деления — это всегда число с плавающей точкой.

### Используем переменные

Результаты арифметических операций можно сохранять в переменных. Например, посчитаем, сколько всего было посетителей сайта и посчитаем среди них долю пользователей мобильной версии. Мы знаем только то, что с мобильной версии пришло 1103 человек, а с десктопной — 5402.

```
all_users = 1103 + 5402 #Всего пользователей с учётом мобильной и десктопной версии

print(all_users) 
```

```
6505 
```

Сначала выполняется всё, что в правой части, а затем результат присваивается переменной.

Обычно арифметические операции проводят не над значениями, а над переменными, в которых эти значения хранятся. Это даёт возможность переиспользовать код для решения похожих задачек и быстрее в нём ориентироваться. Посмотрите на примере: мы сохраним количество посетителей по версиям сайта в переменных и снова посчитаем общее количество:

```
desktop_users = 5402 #Пользователи десктоп-версии
mobile_users = 1103 #Пользователи мобильной версии
all_users = desktop_users + mobile_users #Всего пользователей

print(all_users) 
```

```
6505 
```

Получили тот же результат! При таком способе записи мы точно знаем, из чего складывается значение в переменной `all_users`, и даже комментарий не обязателен.

А теперь попробуем найти долю мобильной версии:

```
mobile_share = mobile_users / all_users

print(mobile_share) 
```

```
0.1695618754803997 
```

Оказалось, что в данных, которые нам передали, закралась ошибка, и количество трафика в мобильной версии должно быть в два раза больше. Нам необходимо увеличить значение в переменной `mobile_users` в два раза:

```
mobile_users = 1103
mobile_users = mobile_users * 2

print(mobile_users)

```


Мы изменили значение в переменной `mobile_users` и присвоили результат той же самой переменной. Это называется `переприсваиванием` Как и с обычным присваиванием — сначала вычисляется то, что справа, затем присваивается результат переменной слева.

Обычно в таком случае используют укороченную запись:

```
mobile_users = 1103

print(mobile_users)
```


Вместо того чтобы дублировать в коде переменную `mobile_users`, мы используем оператор `*=`, который доумножает и сохраняет результат в ту же переменную одновременно.

```
items = 10

print(items)

print(items)

print(items)
```

```
20
```

### Тоже операторы

На самом деле в Python есть много других операторов, мы же показали самые основные. Ещё вы можете встретить такие:

- `//` — деление нацело. Возвращается результат деления, округлённый в меньшую сторону, даже если результат положительный. Бывает полезным, когда можно пренебречь точностью.
- `%` — деление с остатком. Возвращает остаток от деления. Полезно для проверки того, что значения делятся друг на друга.
- `**` — возведение в степень. С помощью возведения в степень можно, например, найти корень числа (степень 1/2).

```
x = 100
y = 10
power = 0.5

print(x // y)
print(x % y)
print(x ** power) 
```

```
10
0
10.0 
```

```
var = 27 / 0

```

# Определение данных и их типов
## Типы данных

В работе с компьютером вы сталкиваетесь с разными типами файлов: PDF файлы для документов, Power Point файлы для презентаций, Excel файлы для таблиц и другие. Разные типы файлов помогают пользователю работать с содержимым документов, компьютер по-разному их хранит и читает. Например, удобно использовать Excel файлы для таблиц, но неудобно — для текстовых записей или презентаций. Другими словами, типы файлов служат разным целям.

В Python всё устроено похоже, только вместо файлов используют объекты. Объекты тоже бывают разных типов. Каждый тип хорошо подходит для своих задач.

В этом уроке вы узнаете:

- какие ещё типы данных бывают;
- для какой задачи какой тип подходит;
- как работать с каждым типом.

### Какие бывают типы данных

Итак, разные типы данных подходят для разных объектов. Чаще всего вы будете сталкиваться с такими объектами:

- числа — целые и дробные;
- логические переменные — правда или ложь (да или нет);
- строки — текст.

Основные типы данных, о которых пойдёт речь в этом уроке, представлены в таблице:

|**Тип**|**Примеры**|**Как обозначается**|
|---|---|---|
|Вещественный|`-10.0`, `.1`, `1.23`|float|
|Целочисленный|`-1`, `12`, `123`|int|
|Логический|`True` / `False`|bool|
|Строка|`'Я строка'`|str|

Со строками и числами вы уже знакомы.

Тип объекта, или тип данных, определяет, в каком виде значения хранятся и записываются. Для записи строки используют кавычки, а число записывают без них.

```
text = 'Я строка' # Это строка

```

Ещё тип данных определяет, какие операции можно проводить над значениями, а какие нельзя. Например, с помощью оператора `/` можно делить числа, но нельзя разделить одну строку на другую.

Рассмотрим типы данных подробнее.

Числа используются везде, где необходимы вычисления.

- Дробные числа — в типе `float`. Этот тип называют **вещественным**, или **числом с плавающей точкой**.

Известно, что всего на сайте было 6505 пользователей, и доля пользователей мобильной версии была 20%. Найдём количество пользователей мобильной версии.

В анализе используют не только числа — иногда нужно работать со строками. Внутри строки могут быть любые символы: буквы, цифры и прочие символы.

Строки всегда записывают в кавычках.

```
text = 'Я строка' # text содержит строку
```

Кавычки необязательно должны быть одинарными, двойные тоже подойдут:

```
text = "Я тоже строка"

print(type(text)) 
```

```

```

Разные виды кавычек полезны, когда внутри строки есть другие кавычки. Они подсказывают Python, что кавычки внутри — часть строки.

```
text1 = "Я тоже 'строка', но с 'кавычками' внутри"
text2 = 'Я другая "строка", тоже с "кавычками" внутри'
```

```
pushkin_text = """Я помню чудное мгновенье:
```

`SyntaxError` — ошибка синтаксиса. Здесь Python не понял, где заканчивается строка. Интерпретатор ожидал на новой строке новую команду, а не продолжение строки.

### Как работать со строками

Для работы со строками определены собственные операции, например, поиск подстроки, замена символов и так далее. Об этом подробно поговорим чуть позже. Но некоторые арифметические операторы всё же можно использовать и со строками, но работать они будут немного по-другому, не как с числами.

**Сложение строк**

Для строк задана операция сложения `+`, так мы объединим две или несколько строк вместе, как бы «склеим» их.

Объединение нескольких строк называют **конкатенацией**.

```
title = 'Название магазина'
shop_name = 'Моторизированный котокраб'

print(title + shop_name) 
```

```
Название магазинаМоторизированный котокраб 
```

Слова в середине склеились. Чтобы расклеить их, добавим промежуточное слагаемое — строку из двоеточия и пробела:

```
title = 'Название магазина'
shop_name = 'Моторизированный котокраб'

print(title + ': ' + shop_name) 
```

```
Название магазина: Моторизированный котокраб 
```

**Умножение на число**

В математике сложение одинаковых чисел можно заменить умножением — то же и со строками. Если нужно сложить строку с самой собой несколько раз, то можно вместо этого умножить строку на число.

Например, создадим разделитель для блоков текста:

```
print("=" * 20) # Повторим символ 20 раз 
```

```
==================== 
```

Украсим разделитель:

```
# Украсим разделитель
print("<" + "=" * 30 + ">") 
```

```
<==============================> 
```

Это тоже конкатенация, как и при сложении, строка «склеивается» сама с собой несколько раз подряд.

Как видите, можно складывать и числа, и строки — мы используем один оператор `+`, но операции работают по-разному. Если оператор `+` применяется к числам, то результатом будет математическое сложение. Если этот же оператор применяется к строкам, то результатом будет их конкатенация.

Свойство, при котором одни и те же операторы работают с разными типами объектов, но по-разному, называют **полиморфизмом**.

Полиморфизм — один из принципов объектно-ориентированного подхода в программировании. С этим термином вы встретитесь, если продолжите изучение Python и погрузитесь в более сложные темы.

**Невыполнимые операции**

Если попробовать сложить строку и число вместе, то получим ошибку:

```
counter = 0
without_days = 'Дней без кода на Python: '

print(without_days + counter) 
```

```
TypeError: can only concatenate str (not "int") to str 
```

`TypeError` — ошибка при работе с типами. В нашем случае она сообщает, что нельзя «склеить» строку и число вместе. К строке можно «приклеить» только другую строку. Подробнее о том, как решить эту проблему, поговорим в следующем уроке.

### Логический тип

Объекты логического типа хранят значения `True` или `False`. Ещё этот тип называют булевым (от англ. bool).

Обычно такие данные — флаг для какого-то признака. Например, включён ли свет дома:

```
light_on = True # Свет включён
light_on = False # Свет выключен 
```

Если в приложении есть премиум-подписка, то переменная логического типа подскажет, есть у пользователя подписка или нет.

```
is_premium = True # Есть подписка
is_premium = False # Нет подписки 
```

### Как работать логическими переменными

Для работы с таким типом данных используют специальные логические операторы. Основных логических операторов три:

|**Оператор**|**Что означает**|**Как работает**|
|---|---|---|
|and|И|даёт истину, только если и справа и слева от него истина|
|or|ИЛИ|даёт в результате истину, если справа или слева от него есть истина|
|not|НЕ|меняет истину на ложь, и наоборот|

Примеры работы:

```
print(True and True)  #Правда и Правда даёт Правду
print(True and False) #Правда и Ложь даёт Ложь
print(True or False)  #Правда или Ложь даёт Правду
print(not True)       #Не Правда даёт Ложь 
```

```
True
False
True
False 
```

С помощью логических операторов можно спроектировать логическую схему процессора или микросхемы. В дальнейшем мы будем использовать их для проверки сложных условий.

С переменными логического типа также можно совершать арифметические операции. В таком случае Python считает, что `True` это `1`, а `False` это `0`:

```
True + True # Складываем две правды 
```

```
2 
```

Например, таким способом можно посчитать количество пользователей с премиум-подпиской. Просто сложим значения всех булевых переменных в столбце — получим нужное число.

Если вам сложно запомнить, как работают and и or — and можно представить как знак умножения, а or — как знак сложения.

В некоторых языках программирования при создании переменной нужно явно указывать её тип. Это свойство называют **статической типизацией**. Python не из их числа, в нём не нужно специально указывать, какой тип данных ему использовать — он сам справляется с этой задачей, когда создаёт переменную. Это свойство называют **динамической типизацией**. Благодаря этой особенности писать код на Python удобнее.

Узнать, какой тип Python использует для хранения объекта, поможет функция `type()`:

```
number_int = 123 #Это целое число
number_float = 1.2345 #Это число с плавающей точкой

print(type(number_int))
print(type(number_float)) 
```

```
<class 'int'>
<class 'float'> 
```

## Преобразование типов

**В этом уроке вы узнаете:**

- как преобразовать тип данных;
- для каких задач пригодится преобразование;
- какие данные не получится преобразовать.

Тип данных определяет, какие операции можно проводить над значениями, а какие нельзя. Вы уже знаете, что складывать строки и числа не получится — будет ошибка. Чтобы решить эту проблему, необходимо преобразовать типы данных — превратить все строки в числа или наоборот, все числа в строки.

Например, чтобы объединить строку `'Количество месяцев в году:'` и число `12` необходимо преобразовать число в строку — `int` в `str`. Тогда, после сложения, получится строка: `'Количество месяцев в году: 12'`

Бывают и обратные ситуации. Например, число хранится в виде строки: `'24'`. Чтобы сложить его с другим числом, сперва нужно преобразовать строку в число — `str` в `int`.

### Целые числа

Вы уже вычисляли количество пользователей мобильной версии сайта, зная долю от общего числа.

```
all_users = 6505
mobile_share = 0.2
mobile_users = all_users * mobile_share

print(mobile_users)
print(type(mobile_users)) 
```

```
1301.0
<class 'float'> 
```

Напомним: результатом операции с `int` и `float` будет `float`, то есть вещественное число. Но фактически мы получаем целое число `1301` — у него нет дробной части, а если точнее, она равна нулю.

В таком случае полезно преобразовать тип данных: перевести значение из вещественного типа в целочисленный, из `float` в `int`. Тогда тип данных перейдёт в соответствие со значением в переменной.

Основные функции для преобразования типов в Python:

- `int()` — превращает объект в целое число;
- `float()` — превращает объект в вещественное число;
- `str()` — превращает объект в строку.

Применим функцию `int()`, чтобы привести значение к целочисленному типу:

```
all_users = 6505
mobile_share = 0.2
mobile_users = int(all_users * mobile_share)

print(mobile_users)
print(type(mobile_users)) 
```

```
1301
<class 'int'> 
```

Теперь в переменной `mobile_users` хранится целое число.

### Дробные числа

Когда переменная хранит вещественное число с ненулевой дробной частью, функция `int()` просто отбрасывает дробную часть. Положительные числа округляются в меньшую сторону, а отрицательные — в большую сторону.

```
temperature = 36.6 #Температура здорового человека

print(int(temperature)) 
```

```
36 
```

Но если нужно просто округлить дробное число, то как правило используют функцию математического округления `round()` — она округляет числа к ближайшему целому, в пограничных случаях (1.5, 2.5, 3.5 и т.п) работает в большую сторону.

Если бы мы хотели округлить число до значения 4, то выглядело бы это так:

![Две оси, на которых отложены числа 3 3 с половиной 4 4 с половиной и 5. На первой оси интервал от трёх с половиной до четырёх с половиной выделен красным. Число четыре с половиной не включено в интервал. Под этой осью подпись: round(x)→ 4. На второй оси интервал от четырёх до пяти. Под ним подпись int(x) → 4.](https://pictures.s3.yandex.net/resources/image_1703187548.png)

Для функции `round()` можно указать точность, с которой округляется число. По умолчанию — до целого числа, но можно указать, чтобы округление было, например, до двух знаков после запятой:

```

temperature = 36.6666 # Очень точная температура здорового человека

print(round(temperature), type(round(temperature))) # Округлим до ближайшего целого
print(round(temperature, 2), type(round(temperature, 2))) # Округлим до двух знаков после запятой 
```

```
37 <class 'int'>
36.67 <class 'float'> 
```

### Числа и строки

С числами вроде всё понятно. А что со строками?

Представим, что вы хотите сложить два числа, которые записаны в строках:

```
price_cart = '1399.9' #Стоимость продуктов в магазине
price_bag = '10' #Стоимость пакета

final_price = price_cart + price_bag

print(final_price)
print(type(final_price)) 
```

```
1399.910
<class 'str'> 
```

Получили совсем не то, что ожидали. Мы объединили строки, а хотели сложить числа. Python воспринимает значения `'1399.9'` и `'10'` как строки, будто там обычный текст, как например `'Я точно знаю, что это строка'`. Числа там видим только мы. Но выход есть — строку можно преобразовать в число.

```
price_cart = '1399.9' #Стоимость корзины с продуктами в магазине
price_bag = '10' #Стоимость пакета

final_price = float(price_cart) + int(price_bag)

print(final_price)
print(type(final_price)) 
```

```
1409.9
<class 'float'> 
```

А что, если мы хотим к строке прибавить число? Здесь всё то же самое — просто преобразуем строку к числу.

Например, стоимость пакета хранится в виде числа, а стоимость корзины как строка.

```
price_cart = '1399.9' #Стоимость корзины с продуктами в магазине
price_bag = 10 #Стоимость пакета

final_price = float(price_cart) + price_bag

print(final_price)
print(type(final_price)) 
```

```
1409.9
<class 'float'> 
```

Бывает и наоборот: необходимо прибавить к строке число. Такая задача встречается довольно часто. Вы встречались с похожей задачей и получили ошибку `TypeError`. Для решения этой проблемы преобразуем число к строке с помощью функции `str()`.

```
text = 'Месяцев в году: '
months_in_year = 12

final_text = text + str(months_in_year)

print(final_text)
print(type(final_text)) 
```

```
Месяцев в году: 12
<class 'str'> 
```

Работает и никаких ошибок!

Если в результате операций ожидается число, то приведите все переменные к числу, используя `float()` или `int()`. Если ожидается строка, то приведите всё к строке, используя `str()`.

Но что, если в строке записан текст, а не число? Или несколько чисел, разделённые пробелом? Или число, у которого есть ещё какие-нибудь символы. Во всех этих случаях функции `int()` и `float()` не сработают:

```
price = '149.9 руб.'

print(int(price)) 
```

```
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ValueError: invalid literal for int() with base 10: '149.9 руб.' 
```

Ошибка `ValueError` сообщает, что функция `int()` получила строку, которую не может преобразовать в число. Любое число можно преобразовать к строке, но строку можно преобразовать к числу только, если она не содержит никаких лишних символов, кроме цифр.

Исключения:

- знак `+` или `-` перед цифрами если число положительное или отрицательное соответственно.

```
print(int('+123'))
print(int('-123')) 
```

```
123
-123 
```

- Если число дробное, то можно использовать символ точки `.` для разделения целой и дробной части.

```
print(float('0.1')) 
```

```
0.1 
```

- Пробелы по краям (справа или слева)

```
print(int('  123  ')) 
```

```
123 
```

### Логические переменные

Логические данные — это те, которые принимают значения _True_ или _False_. Эти данные тоже можно преобразовывать:

```
is_true = True
is_false = False

print(int(is_true), int(is_false))
print(str(is_true), str(is_false)) 
```

```
1 0
'True' 'False' 
```

Если преобразовать логическую переменную к числу, то True — это всегда 1, а False — всегда 0. А если преобразовать в строку, то получим `'True'` и `'False'` соответственно.

А что если наоборот, преобразовать число к типу `bool`? В таком случае любое не нулевое число станет `True`. Это работает как с целыми, так и с вещественными числами.

Для преобразования воспользуемся функцией `bool()`:

```
temperature = 36.6
zero = 0

print(bool(temperature), bool(zero)) 
```

```
True False 
```

А что со строками? Любая не пустая строка тоже примет значение _True_

```
text = 'Это строка превратится в True'
empty_str = ''

print(bool(text), bool(empty_str)) 
```

```
True False 
```

А теперь пора практиковать полученные знания!

## Строки

Вы уже знакомы со строками. Как правило, текстовая информация состоит из строк. Для её хранения и используют переменные типа «строка».

Присвоим переменной `hi` строку `Привет!`:

```
hi = 'Привет!' 
```

Если посмотреть немного глубже, то строка — это последовательность символов. Например, строка `Привет!` это последовательность символов `П`, `р`, `и`, `в`, `е`, `т` и восклицательный знак `!` .

В этом уроке вы узнаете:

- как устроены строки;
- что такое индекс и срез строки.

### Длина строки

Длина строки — это количество символов в ней. Строки могут быть сколь угодно длинными — ограничением станет только объём памяти вашего компьютера.

Для подсчёта длины используют функцию `len()`. В качестве аргумента `len()` принимает строку, а возвращает её длину — целое число:

```
some_string = 'В этой строке 25 символов'

print(len(some_string)) 
```

Результат:

```

25 
```

Строка может быть и пустой. В таком случае в ней не будет никаких символов, а для создания такой строки просто ничего не запишем в кавычках — ставим две кавычки подряд:

```
lonely_empty_string = '' 
```

Длина пустой строки равна нулю:

```
print(len(lonely_empty_string)) 
```

Итог:

```
0 
```

### Индексация строк

Символы в строке идут друг за другом, при этом у каждого — свой порядковый номер. Этот номер называют индексом.

![](https://pictures.s3.yandex.net/resources/image_1705604816.png)

Нумерация в этой последовательности символов непривычная. Отсчёт начинается с нуля, а не с единицы:

- индекс первого символа — `0`;
- второго — `1`;
- третьего — `2` и так далее.

В программировании отсчёт с нуля — частое явление.

К каждому символу можно обращаться по его индексу, а для указания индекса используют квадратные скобки `[]`:

```
this_is_string = 'Это строка'

print(this_is_string[0])
print(this_is_string[4]) 
```

В результате получим:

```
Э
с 
```

Для получения символа можно использовать и обратный индекс — это когда мы считаем символы с конца строки. В таком случае мы указываем его отрицательным.

![](https://pictures.s3.yandex.net/resources/image_1705604837.png)

Понятнее станет на примере:

```
this_is_string = 'Это строка'

print(this_is_string[-1]) # -1 это первый символ с конца, то есть самый последний в строке
print(this_is_string[-2]) # -2 это второй символ с конца или предпоследний в строке 
```

```
а
к 
```

Если нужно обратиться к последнему символу строки, то используют либо длину строки `len() - 1`, либо обратный индекс `-1`.

```
this_is_string = 'Это строка'

print(this_is_string[-1])
print(this_is_string[len(this_is_string) - 1]) 
```

Получим:

```
a
a 
```

### **Ошибки при обращении по индексу**

Если обратиться к индексу, который больше длины строки, произойдёт ошибка:

```
word = 'слово'

print(word[123]) #Выведем 123-й символ строки word 
```

Получаем:

```
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: string index out of range 
```

Ошибка `IndexError` происходит при обращении к несуществующему индексу. Ещё эту ошибку называют выходом за границы _**_допустимого диапазона значений индекса.

Другая ошибка возникнет, если попытаться изменить символ строки, обратившись к нему по индексу:

```
word = 'bbc'
word[0] = 'a' # Заменим самый первый символ строки на 'a' 
```

```
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'str' object does not support item assignment 
```

Строки — неизменяемый тип данных. Можно только сохранить новую строку вместо старой:

```

word = 'bbc'
word = 'abc'
 
```

Но изменить существующую строку не получится.

### **Срезы строк**

Вы научились получать символ по его индексу. Теперь попробуйте получить целые фрагменты из строки. Эти фрагменты называют подстроками или срезами.

```
password = 'qwerty'
print(password[0:3]) 
```

```
qwe 
```

Обратите внимание, вывелось только три символа. Четвёртый по порядку и третий по индексу символ r не вывелся.

### **Что такое срезы**

Срезы — почти как индексы. Только они позволяют получить не один, а сразу несколько идущих подряд символов строки.

![](https://pictures.s3.yandex.net/resources/image_1705605205.png)

Чтобы получить срез, используют конструкцию: `[start:end:step]`.

`start` — индекс элемента, с которого начинается срез. В нашем примере `start = 0`.

`end` — индекс элемента, до которого продолжается срез. Важно, что элемент с этим индексом не включается в сам срез. В нашем примере последний элемент среза под индексом 2, а значит, `end = 3`.

`step` — шаг, с которым выбираются символы. По умолчанию `step=1`, и его можно не использовать.

```
text = 'Это строка'
substring = text[0:3]
print(substring) 
```

```
Это 
```

Теперь попробуем по-другому.

```
city = 'Paris'
substring = city[3:5] # Получаем срез элементов с 3 по 5 (не включая 5)
print(substring) 
```

```
is 
```

Здесь мы запросили два символа: начиная с индекса `3` и заканчивая `5`. Последний символ диапазона при этом не входит в результат. Результатом стала подстрока — часть исходной строки.

Если делать срез с самого начала строки, то необязательно указывать нулевой индекс. Можно просто пропустить эту часть:

```
# Если начало среза не указано, то срез начинается с нулевого символа.
print(city[:3]) # то же, что и city[0:3] 
```

Результат:

```
Par 
```

И наоборот, если конец подстроки совпадает с концом исходной строки, можно пропустить индекс конца среза:

```
# Если конец среза не указан, то срез продолжается до конца строки.
print(city[2:]) # тоже, что и city[2:5] 
```

В итоге:

```
ris 
```

А можно не указывать ни конец, ни начало. В таком случае вернётся точная копия строки:

```
print(city[:]) # то же, что и city[0:5] 
```

Получим:

```
Paris  
```

Так же, как и при обращении к одному элементу, в срезах можно использовать обратные индексы. Например, если мы захотим выбрать последние три элемента строки, то можем сделать вот так:

```
print(city[-3:]) # тоже, что и city[2:5] 
```

Результатом будет:

```
ris  
```

Используя `step`, мы можем указать шаг, с которым получаем символы. Например, сохраним все нечётные символы в строке:

```
city = 'Paris'
substring = city[::2] # Из всей строки выбираем каждый нечётный элемент
print(substring) 
```

```
Prs 
```

Если указать `step = -1`, то вернётся строка в обратном порядке:

```
city = 'Paris'
reverse_string = city[::-1]
print(reverse_string) 
```

```
siraP 
```

### Как работать со строками

В этом уроке вы узнаете:

- как работать со строками;
- что такое экранирование и форматирование.

Вы уже знакомы с основными операциями со строками: сложение `+` и умножение на число `*`. Повторим основные принципы. Оператор сложения `+` возвращает строку, склеенную из двух:

```
hi = 'Привет!'
question = 'Как дела?'

#Добавим пробел между привествием и вопросом
print(hi + ' ' + question)  
```

```
Привет! Как дела? 
```

А оператор умножения `*` возвращает строку, в которой исходная повторяется заданное количество раз:

```
s = 'Повтори эту строку 5 раз'
print(s * 5) 
```

```
Повтори эту строку 5 разПовтори эту строку 5 разПовтори эту строку 5 разПовтори эту строку 5 разПовтори эту строку 5 раз
 
```

В Python есть набор встроенных **методов**, которые предназначены для работы со строками. Методы — это функции, как, например, `print()` или `len()`, но которые могут работать только с определённым типом данных (в нашем случае — это строки `str`). Методы вызываются через точку после самой строки:

```
word = 'привет'

# метод upper сделает все буквы прописными
new_word = word.upper()
print(new_word) 
```

```
ПРИВЕТ 
```

Используя встроенные методы, вам не придётся самостоятельно создавать функции для решения тех же задач.

Несколько полезных методов:

- `replace()` — заменит часть строки;
- `find()` — ищет подстроку в строке;
- `upper()` — заменяет все буквы в строке на прописные;
- `lower()` — заменяет все буквы на строчные.

### **Замена части строки**

Метод `replace()` принимает два аргумента:

1. подстроку, которую нужно заменить;
2. подстроку, которая заменит исходную.

Посмотрим, как это работает:

```
some_string = 'leaving'
print(some_string)

# заменяем подстроку leav строкой stay
some_string = some_string.replace('leav', 'stay')
print(some_string) 
```

```
leaving
staying 
```

Метод вернёт новую строку с внесёнными изменениями. Её можно сохранить в той же переменной, из которой метод получил исходную строку.

### **Поиск подстроки**

Метод `find()` ищет подстроку в строке. Если найдёт — вернёт индекс символа, начиная с которого подстрока встречается в исходной строке.

Чаще всего методу передают один аргумент — подстроку, которую он должен найти:

```
word = 'hey, hello, hello, hello, how low?'
# подстрока hello входит в строку несколько раз,
# но метод вернёт место начала первого вхождения
ind = word.find('hello')

print(ind) 
```

```
5 
```

Для `find()` можно задать диапазон индексов, в котором он должен искать:

```
word = 'hey, hello, hello, hello, how low?'

# Три аргумента: подстрока, индекс начала поиска, индекса завершения, увеличенный на 1
ind = word.find('hello', 15, 25)

print(ind) 
```

```
19 
```

Инструкцию в этом коде можно прочитать так: «Найди первое вхождение `hello` в строке из переменной `word`, начиная с пятнадцатого символа и заканчивая двадцать четвёртым».

Если `find()` ничего не найдёт, он вернёт `-1`:

```
hi = 'Привет!'
bye_ind = hi.find('Пока!')

print(bye_ind)
 
```

```
-1 
```

У метода `find()` есть аналог — `rfind()`, тут поиск подстроки начинается с правой стороны. Если `find()` ищет первое вхождение подстроки, то `rfind()` ищет последнее вхождение.

```
word = 'hey, hello, hello, hello, how low?'
# подстрока hello входит в строку несколько раз,
# но метод вернёт номер последнего вхождения
ind = word.rfind('hello')

print(ind) 
```

```
19 
```

### **Изменение регистра**

Методы `upper()` и `lower()` возвращают строки, в которых исходная строка приведена к верхнему и нижнему регистру соответственно:

```
word = 'AbRacaDABRA'
lower_word = word.lower()
upper_word = word.upper()
print(lower_word, upper_word)
 
```

```
abracadabra ABRACADABRA
 
```

### Экранирование специальных символов

Кавычки сигнализируют Python о начале и конце строки. Но иногда кавычки должны стать частью самой строки:

```
rule = 'Строки в Python помещают в одинарные кавычки, вот так: 'текст'.'

print(rule) 
```

Такая запись вызовет ошибку.

```
rule = 'Строки в Python помещают в одинарные кавычки, вот так: 'текст'.'
                                                                ^
SyntaxError: invalid syntax 
```

С точки зрения Python, строка закончилась после второй одинарной кавычки после слов «вот так:». Что делать с оставшейся частью, он не догадается.

Чтобы Python воспринимал кавычки как часть строки, ему нужна специальная отметка:

```
rule = 'Строки в Python помещают в одинарные кавычки, вот так: \'текст\''

print(rule) 
```

```
Строки в Python помещают в одинарные кавычки, вот так: 'текст' 
```

К кавычкам мы добавили слэш `\`, и теперь всё заработало, как мы и хотели.

Эта техника называется экранированием. При экранировании Python считывает специальный символ и подставляет вместо него то, что считает нужным. Символы для экранирования начинаются со слэша `\`.

```
phrase = 'I don\'t want to stop'
print(phrase)
print(len(phrase)) 
```

```
I don't want to stop
20 
```

Для Python символ `\` перед кавычкой `'` означает, что кавычка это не конец строки, а её часть.

Есть и другие специальные символы, которые используют для экранирования:

- `\n` — перенос строки;
- `\t` — табуляция;
- `\\` — слэш.

Такие сочетания считаются за один символ.

```
s = 'Для пирога понадобятся:\nМолоко\nМасло растительное\nЯйца\nСахар\nМука'
print(s) 
```

```
Для пирога понадобятся:
Молоко
Масло растительное
Яйца
Сахар
Мука 
```

За счёт параметра end по умолчанию при нескольких вызовах функции print каждая строка будет выведена отдельно.

```
milk = 'Молоко\t120мл'
еggs = 'Яйца\t3 шт.'

print(milk)
print(еggs) 
```

```
Молоко    120мл
Яйца      3 шт. 
```

Также крайне полезен на практике метод `.strip`. Возможно у вас были случаи, когда при копировании текста вы захватывали лишние пробелы в начале или в конце строки? Заметить это сложно, а для компьютера такие строки будут разными, и в базе данных при таком заполнении появятся не равные друг другу строки, например: `Иван`, `Иван` и `Иван`. Метод `.strip` позволяет этого избежать, вот как он просто работает:

```
print('       Иван Иванович Иванов       '.strip()) 
```

```
Иван Иванович Иванов 
```

Пробелы в середине строки остались нетронутыми.

### Форматирование строк

Иногда встречаются задачи, где необходимо составить строку из готового шаблона. Например, из бизнес-показателей составить сообщение для отчёта.

![](https://pictures.s3.yandex.net/resources/image_1705606191.png)

Для решения такой задачи мы используем форматирование строк. Форматирование — это способ создания строки, в которой значения переменных подставляются в определённых местах, как по шаблону.

Для форматирования строки можно использовать метод `format()`. В строке в фигурных скобках `{ }` указывают название переменной для подстановки. Аргументами метода `format` будут эти самые переменные.

Разберём на примере:

```
s = 'Я {position}'.format(position='аналитик данных')

print(s) 
```

```
Я аналитик данных 
```

И ещё один пример:

```
cr_percent = 3
dau = 1000
dau_delta = 5

s = """
По итогам квартала конверсия составила {cr_percent}%.
В тоже время показатель DAU составил {dau} пользователей, 
что на {dau_delta}% выше прогноза.
"""
print(s.format(cr_percent = cr_percent, dau = dau, dau_delta = dau_delta)) 
```

```
По итогам квартала конверсия составила 3%.
В тоже время показатель DAU составил 1000 пользователей, 
что на 5% выше прогноза. 
```

Иногда удобнее использовать `f-строки`. Мы пишем символ `f` перед строкой и точно так же используем фигурные скобки `{ }` и именованные переменные, но использовать метод `format` уже необязательно.

```
cr_percent = 3
dau = 1000
dau_delta = 5

s = f"""
По итогам квартала конверсия составила {cr_percent}%.
В тоже время показатель DAU составил {dau} пользователей, 
что на {dau_delta}% выше прогноза.
"""
print(s) 
```

```
По итогам квартала конверсия составила 3%.
В тоже время показатель DAU составил 1000 пользователей, 
что на 5% выше прогноза. 
```

Получили то же самое!

## Списки

Вы познакомились с числами и строками — основными типами данных. Пришло время разобраться с более сложными объектами — со структурами данных.

Структуры позволяют хранить данные разных типов в удобном для работы виде. Ещё эти структуры называют коллекциями. Одна из таких структур в Python — списки.

**В этом уроке вы узнаете:**

- что такое список;
- как работать со списками;
- как связаны строки и списки.

### Что такое список?

Список — это упорядоченный набор элементов.

Представьте, что вы составляете распорядок дня на завтра. Для этого вы вспоминаете, какие дела хотели сделать, собираете их в одном месте, например, записываете на листочке, и нумеруете дела по важности. У вас получился **список** дел на день. Если вам нужно узнать, какое дело стоит первым на завтра, то вы просто открываете свой список и смотрите дело под номером `1`.

![Лист из ежедневника со списком дел. Список на утро: первое — пресс качат, второе — бегит, третье — турник, четвертое — анжумания. Список на вечер полностью повторяет пункты из второго списка.](https://pictures.s3.yandex.net/resources/image_1705582382.png)

В Python всё примерно так же — элементами списка были бы дела в нужном порядке. К каждому элементу мы могли бы обращаться по его номеру в списке. Этот номер называют индексом. Вспомните, как мы обращались к символам в строках — со списками всё очень похоже.

### Создание списка

Для создания списка используют квадратные скобки `[ ]`, а элементы списка перечисляют через запятую.

Список объявляют так:

```
years = [1994, 1972, 2008] #Список с годами
print(years) 
```

Результат:

```
[1994, 1972, 2008] 
```

Список может быть и пустым, в таком случае в квадратных скобках ничего не будет:

```
# пустой список
empty_list = [] 
```

Пустота — явление парадоксальное. Когда она есть, ничего нет.

Список может содержать элементы разных типов данных:

```
# список из строки str, целого int и вещественного числа float
items = ['какая-то строка', 1234, 3.1415] 
```

Список может содержать другие списки, тогда его называют вложенным:

```
items_list = [[1, 2, 3], [1, 2, 3, 4], [5, 6]] 
```

Список — это отдельный тип объекта `list`:

```
years = [1994, 1972, 2008] # Список с годами
print(type(years)) # Выведем тип объекта 
```

```
class <list> 
```

Каждый элемент имеет свой порядковый номер или индекс. Как и в строках, нумерация начинается с `0`. Используя индекс, можно получить элемент из списка, обычно это называют — «обратиться к элементу списка».

![Слева подпись список list. За ней даты 1994, 1972, 2008. Под этой строкой подпись индексы. Под 1994 — индекс 0. Под 1972 — индекс 1, под 2008 — индекс 2.](https://pictures.s3.yandex.net/resources/image_1705582413.png)

Количество элементов в списке называют его длиной. Как и в случае строк, длину списка вернёт функция `len()`:

```
first_film = ['Побег из Шоушенка', 'США', 1994, 'драма', 142, 9.111]
print(len(first_film)) 
```

Вывод:

```
6 
```

Проверим на пустом списке:

```
empty_list = []
print(len(empty_list)) 
```

Результат:

```
0 
```

### Индексы и срезы

Каждый элемент списка связан со своим индексом — порядковым номером. По индексу получают отдельные значения и их группы — срезы. Всё как со строками.

Покажем, как это можно сделать:

```
first_film = ['Побег из Шоушенка', 'США', 1994, 'драма', 142, 9.111]
print(first_film[0], first_film[1], first_film[-1]) 
```

```
Побег из Шоушенка США 9.111 
```

Отсчитывать индексы можно как с конца, так и с начала:

- индексация с начала начинается с `0`;
- индексация с конца начинается с `-1`.

Независимо от длины списка, обращение по индексу — быстрая операция. Неважно, обратитесь вы к индексу `1`, `100` или `100000` — скорость будет одинаковой.

Если обратиться к индексу, который больше чем длина списка, то вернётся ошибка:

```
numbers = [1, 2, 3, 4, 5]
print(numbers[123]) 
```

```
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list index out of range 
```

### **Срезы**

С помощью срезов обращаются к диапазону элементов с помощью конструкции `[start:end:step]`. Всё так же, как со строками:

```
first_film = ['Побег из Шоушенка', 'США', 1994, 'драма', 142, 9.111]
#срезаются элементы с первого по третий
print(first_film[1:4])
#срезаются элементы с нулевого по второй
print(first_film[:3])
#срезаются элементы со второго по последний
print(first_film[2:]) 
```

```
['США', 1994, 'драма']
['Побег из Шоушенка', 'США', 1994]
[1994, 'драма', 142, 9.111] 
```

### Операции над списками

В отличие от строк, списки можно изменять:

Например, мы можем изменить элемент в списке, обращаясь к нему по индексу:

```
numbers = [1, 2, 3, 4, 5] # Создадим список
print(numbers) # Выведем содержимое списка

numbers[0] = 0 # Присвоим элементу с индексом 0 новое значение
print(numbers) # Снова выведем содержимое списка 
```

Результат:

```
[1, 2, 3, 4, 5]
[0, 2, 3, 4, 5] 
```

Свойство изменяемости объекта не всегда заметно, но сказывается на производительности. В отличие от строк, списки предназначены для работы с большими данными. Представьте, если бы вы работали с огромными списками на несколько тысяч элементов — при каждом изменении приходилось бы пересоздавать заново огромный список и выделять в памяти много места.

Изменяемость списков экономит ресурсы компьютера, которому не приходится создавать новые объекты при каждом изменении.

|**Строки**|**Списки**|
|---|---|
|Нельзя изменить символ в строке, обращаясь по индексу.|Можно изменить элемент в списке, обращаясь по индексу.|
|Нельзя добавить к имеющейся строке новый символ. Операция сложения + создаст новую строку.|Можно добавить новый элемент к списку.|

### **Добавление элементов в список**

Самый простой способ добавить элемент в список — вызвать метод `append()`. На вход метод принимает элемент, который нужно добавить. Метод добавляет элемент в конец списка.

Вот так можно добавить имя режиссёра фильма «Побег из Шоушенка» (Фрэнк Дарабонт) в конец списка:

```
first_film = ['Побег из Шоушенка', 'США', 1994, 'драма', 142, 9.111]
first_film.append('Фрэнк Дарабонт')

print(first_film) 
```

```
['Побег из Шоушенка', 'США', 1994, 'драма', 142, 9.111, 'Фрэнк Дарабонт'] 
```

Обратите внимание, что метод `append()` изменяет изначальный список, а не создаёт новый. То есть результатом выполнения инструкции `first_film.append('Фрэнк Дарабонт')` уже будет добавление элемента списка. Таким образом, нам не нужно ничего переприсваивать переменной `first_film`. После выполнения этой инструкции список уже будет изменённым.

### **Добавление нескольких элементов в список**

Иногда нам нужно расширить список другим списком — прибавить к одному списку другой. В случае строк мы бы назвали это конкатенацией.

Один из способов расширить список — вызвать метод `extend()`. На вход он принимает другой список, которым нужно расширить изначальный. Посмотрите, как это работает:

```
second_film = ['Крёстный отец', 'США', 1972] # Создаём список с описанием фильма
directors = ['Фрэнсис Форд Коппола', 'Нино Рота'] # Создаём список с режиссёрами фильма
# Расширим список second_film элементами из списка directors
second_film.extend(directors)

print(second_film)
 
```

```
['Крёстный отец', 'США', 1972, 'Фрэнсис Форд Коппола', 'Нино Рота'] 
```

Как и с `append()`, тут мы изменили изначальный список, и результат не пришлось переприсваивать.

Но есть и другой способ расширить список другим списком, более привычный — использовать оператор сложения `+`:

```
second_film = ['Крёстный отец', 'США', 1972] # Фильм
directors = ['Фрэнсис Форд Коппола', 'Нино Рота'] # Режиссёр и композитор фильма
# в конец списка добавим имя режиссёра и имя композитора

print(second_film + directors) 
```

```
['Крёстный отец', 'США', 1972, 'Фрэнсис Форд Коппола', 'Нино Рота'] 
```

### **Удаление элемента из списка**

Элементы можно не только добавлять, но и удалять. Один из методов, которым это можно сделать — метод `pop()`.

Он принимает индекс элемента, который нужно удалить, и возвращает сам элемент, который был удалён. Как это работает:

```
numbers = [1, 2, 3, 4, 5]
droped_element = numbers.pop(0) # Удалим первый элемент с индексом 0

# В переменной droped_element хранится элемент, который был удалён
print(droped_element) 
print(numbers) 
```

```
1
[2, 3, 4, 5] 
```

### **Сортировка списка**

Если список состоит из чисел, то их можно отсортировать в порядке возрастания или убывания. Для этого можно вызвать метод `sort()`:

```
numbers = [3, 2, 5, 4, 1]
numbers.sort()

print(numbers) 
```

Получим:

```
[1, 2, 3, 4, 5] 
```

Этот метод изменяет исходный список, сортируя его элементы.

По умолчанию список будет отсортирован в порядке возрастания — от меньшего к большему. Чтобы отсортировать наоборот в порядке убывания, используют параметр `reverse=True`:

```
numbers = [3, 2, 5, 4, 1]
numbers.sort(reverse=True)

print(numbers) 
```

```
[5, 4, 3, 2, 1] 
```

### **Получение минимального и максимального элемента в списке**

Периодически аналитик сталкивается с задачей — найти самый большой или самый маленький элемент в списке. Для этого можно воспользоваться функциями `min()` и `max()`:

```
numbers = [3, 2, 5, 4, 1]

print(max(numbers)) #Выведем максимальный элемент в списке
print(min(numbers)) #Выведем минимальный элемент в списке 
```

```
5
1 
```

### Строки и списки

**Разделить строку на несколько частей**

Вы уже знаете, что каждая строка состоит из отдельных символов. Используя функцию `list()`, можно преобразовать строку в список, где элементами списка станут символы строки:

```
text = 'Привет!'
list_text = list(text)

print(list_text) 
```

Результат:

```
['П', 'р', 'и', 'в', 'е', 'т', '!'] 
```

Теперь вы умеете разбивать строку на список символов!

Однако чаще встречается немного другая задача — разделить строку не на отдельные символы, а на отдельные подстроки, то есть разделить строку на несколько частей по какому-нибудь разделителю. Например, разбить абзац на отдельные предложения.

Познакомившись со списками, мы уже можем решить эту задачку.

Для этого используем метод `split()`. На вход метод принимает символ-разделитель — это символ, которым мы «дробим» строку. Метод возвращает список из разделённых строк.

Если мы хотим поделить абзац на отдельные предложения, то использовали бы точку `.` как разделитель.

```
text = 'Предложение первое. Второе предложение. Предложение номер три.'
sent_text = text.split('.') #Разделим строку на отдельные части по точке

print(type(sent_text)) #Выведем тип полученного результата
print(sent_text) #Выведем содержимое списка 
```

Получим:

```
<class 'list'>
['Предложение первое', ' Второе предложение', ' Предложение номер три'] 
```

Мы разбили текст на отдельные предложения, ура!

Теперь если нам нужно выделить из всего текста только второе предложение, то мы можем просто обратиться к полученному списку:

```
print(sent_text[1]) 
```

```
 Второе предложение 
```

Метод `split()` возвращает список строк, разъединённых указанным разделителем. В нашем случае — это символ точки.

**Объединить в строку**

Мы можем сделать и наоборот — собрать целую строку из элементов списка. Для этого используем метод `join()`:

```
sent_text = ['Предложение первое', 'Второе предложение', 'Предложение номер три']
sep = '.'
text = sep.join(sent_text)

print(text) 
```

Получим:

```
Предложение первое.Второе предложение.Предложение номер три 
```

Метод `join()` объединяет элементы списка в одну строку, разделяя сами элементы строкой `sep`.

Обычно после точек ставят пробелы. Добавим их:

```
sent_text = ['Предложение первое', 'Второе предложение', 'Предложение номер три']
sep = '. ' # точка с пробелом
text = sep.join(sent_text)

print(text) 
```

Результат улучшений:

```

Предложение первое. Второе предложение. Предложение номер три 
```

## Логика. Логические выражения, операторы сравнения

В повседневной жизни мы часто соглашаемся или не соглашаемся с утверждениями — другими словами, оцениваем их истинность. Например, утверждение «отрезок в 1 метр длиннее отрезка в 60 см» верно, или истинно, поскольку 100 см объективно больше, чем 60 см. Однако с высказыванием «отрезок в 60 см длиннее отрезка в 100 см» мы не согласимся. Это утверждение неверное, ошибочное или ложное.

Так же и в программировании можно сравнивать данные и получать в качестве результата информацию о том, истинно или ложно выражение. Такие выражения в математике или программировании называют **логическими выражениями**.

Логическое выражение — выражение, в котором сравниваются данные. Может быть истинно или ложно.

Например, если сравнить среднее количество посетителей в день на прошлой неделе с показателями текущей недели, можно проверить, было ли увеличение метрики или нет. Выражение, сравнивающее текущее и среднее количества посетителей, — логическое.

На практике аналитик часто реализовывает сложные алгоритмы, включающие сравнение данных между собой или с заданными значениями, фильтрацию данных или расчёты и анализ данных в зависимости от полученного ранее результата. Например:

- отобрать пользователей, совершивших определённое количество покупок;
- проанализировать клиентов, имеющих средний чек выше среднего значения;
- сделать срез данных за определённый месяц или для конкретного сегмента;
- изучить окупаемость рекламных источников с высокими показателями конверсии.

В таких случаях без логических выражений не обойтись.

В этом уроке вы узнаете:

- как составляется простое логическое выражение,
- какие основные логические операторы используются,
- что представляет собой результат логического выражения.

### Логические выражения и их результат

В Python результатом логического выражения могут быть только два логических значения:

- `True` — указывает на истинность выражения,
- `False` — подразумевает, что выражение ложно.

Такой тип данных называется `bool` — булевый тип данных.

Например, сравним два числа:

```
20 > 2 
```

Результат будет такой:

```
True 
```

В примере логическое выражение состоит из сравнения 20 и 2. Известно, что 20 больше 2, значит, выражение истинно, и результат работы кода выше — булево значение `True`.

### Логические операторы сравнения

Для создания логических выражений используют операторы сравнения, которые умеют сравнивать числа. Они возвращают значение типа `bool`.

В качестве операторов сравнения используют основные математические операторы сравнения вроде `>` и `<` практически в неизменном виде. Отличий два — операторы равенства (`==`) и неравенства (`!=`):

- Поскольку знак `=` означает присваивание значения переменной, то равенство показывают с помощью двойного «равно»: `==`.
- Для отрицания равенства вместо знака ≠ используют `!=`.

В Python часто используются такие операторы сравнения:

|**Логический оператор**|**Значение**|**Действие логического оператора**|**Когда возвращает `True`**|**Пример выражения с результатом `True`**|
|---|---|---|---|---|
|`==`|равно|сравнивает два значения между собой|когда значения равны|`5 == 5`|
|`!=`|не равно|сравнивает два значения между собой|когда значения не равны|`4 != 5`|
|`>`|больше|сравнивает два значения между собой|если первое больше второго|`5 > 1`|
|`<`|меньше|сравнивает два значения между собой|если первое меньше второго|`1 < 5`|
|`>=`|больше или равно|сравнивает два значения между собой|если первое больше второго или значения равны|`5 >= 1`  <br>`5 >= 5`|
|`<=`|меньше или равно|сравнивает два значения между собой|если первое меньше второго или значения равны|`1 <= 1`  <br>`5 <= 5`|

Значения, сравнивающиеся в логическом выражении, называются **операнд**. Например, в выражении `1 < 5` операнды — это 1 и 5.

Посмотрим работу операторов сравнения на числовых примерах:

```
x = 5
y = 10

# Равно
print(x == y) # False

# Не равно
print(x != y) # True

# Больше
print(x > y) # False

# Меньше
print(x < y) # True

# Больше или равно
print(x >= y) # False

# Меньше или равно
print(x <= y) # True 
```

Если логическое выражение истинное, то результатом будет `True` — такие примеры отмечены комментарием `# True` в листинге выше. А если логическое выражение неверное, то результатом выполнения кода будет `False`.

### Простое логическое выражение

Логические выражения, в котором используется только два операнда и один логический оператор сравнения, называют **простыми** **логическими выражениями**. В них выполняется только одна логическая операция.

В Python можно сравнивать не только числа, но и строки, списки и другие типы данных. Например, используем в качестве операндов арифметическое выражение, списки и строки:

```
(16*5) > 99 # False
20 <= 20 # True
['A','B','C'] == ['A','B','C'] # True
'одно слово' != 'второе слово' # True 
```

Обратите внимание на первый пример — в нём один операнд представляет собой арифметическое выражение: `(16*5)`. Мы использовали скобки для лучшей читаемости, но можно их не ставить. Приоритет арифметических действий выше, чем у сравнений. Это значит, что Python сначала всё считает и только потом сравнивает результаты.

Сравнивать можно и переменные:

```
a = 25
b = 39

b <= a 
```

Результат будет такой:

```
False 
```

### Сравнение данных разных типов

Данные разных типов можно сравнивать с помощью операторов равенства (`==`) и неравенства (`!=`). Например:

```
'20' == 20 
```

Результат получим такой:

```
False 
```

Результат `False`, поскольку строка `'20'` не равна числу 20.

Если попытаться сравнить данные разных типов с помощью операторов `>`, `<`, `>=` или `<=`, то получим ошибку:

```
'20' >= 20 
```

```
TypeError: '>=' not supported between instances of 'str' and 'int' 
```

Исключение — сравнение `int` и `float`. Это разные типы данных, но это числа, так что их можно сравнивать с помощью любых операторов сравнения: `==`, `!=`, `>`, `<`, `>=` и `<=`.

## Сложные логические выражения

Вы познакомились с простыми логическими выражениями — они состоят из двух значений — операнд — и оператора сравнения. В работе аналитика часто необходимо оперировать и более сложными конструкциями, то есть сравнивать несколько параметров или выполнять сложные логические условия.

Например:

- Проанализировать окупаемость вложений в определённые рекламные источники за фиксированный период времени — здесь два условия: по рекламным источникам и по периоду времени.
- Изучить определённый сегмент пользователей — сегментирование бывает узким и накладывает несколько условий на аудиторию. Например, пользователи с ежемесячным доходом выше среднего, имеющие детей-школьников в возрасте от 14 до 17 лет. Здесь ограничения на доход, на наличие детей, на нижнюю границу возраста и на верхнюю — целых четыре! И это не предел.

- как сочетать разные операторы сравнения и условия;
- как создавать сложные логические выражения с помощью логических операторов `and`, `or`, `not`;
- в каком порядке выполняются действия в сложных логических условиях.

## Сложные логические выражения

Простое логическое выражение состоит из двух операнд, а сложное — из нескольких простых логических выражений, сочетание которых определяет результат всего выражения.

Например, необходимо проверить, что `5 больше 2 и меньше 10`. В этом примере есть два простых логических выражения: `5 > 2` и `5 < 10`. Если по условию оба эти выражения истинны, то исходное сложное выражение `5 больше 2 и меньше 10` тоже будет истинным. А если хотя бы одно из простых выражений ложно, то и общий результат будет ложным.

### Логические операторы

Для объединения нескольких простых логических выражений между собой в одно сложное в Python используются специальные логические операторы — `and`, `or`, `not`. Вы с ними уже встречались в уроке по типам переменных и сейчас познакомитесь подробнее.

### **Операторы `and` и `or`**

Оператор `and` называют ещё «логическое И» или «логическое умножение». Он возвращает `True`, только если оба логических значения — `True`.

В остальных случаях оператор `and` вернёт `False`. Например, `and` вернёт `False`, если объединит ложное и истинное выражение. Также `and` вернёт `False`, если объединит два ложных выражения:

```
print(True and True) # вернёт True, так как оба логические значения True
print(True and False) # вернёт False, так как одно из логических значений False
print(False and True) # вернёт False, так как одно из логических значений False
print(False and False) # вернёт False, так как оба логические значения False 
```

Результат будет следующий:

```
True
False
False
False 
```

На практике такое условие применяется, когда необходимо соответствие данных одновременно нескольким критериями. Например, нужно проанализировать покупателей, использующих мобильное приложение и зарегистрированных в определённом регионе, например, Красноярском крае. Оба условия — обязательны. Если клиент живёт в Красноярском крае, но не использует приложение, то он не подходит для анализа. Или если использует приложение, но живёт в Челябинской области — тоже не подходит. Выбрать нужных клиентов по двум обязательным критериям поможет оператор `and`.

Оператор `or` называют также «логическое ИЛИ», «логическое сложение». Он возвращает `True`, если **хотя бы** одно логическое значение тоже `True`. Оператор `or` вернёт `False` только если оба логические значения будут `False`.

```
print(True or True) # вернёт True, так как оба логические значения True
print(True or False) # вернёт True, так как одно из логических значений True
print(False or True) # вернёт True, так как одно из логических значений True
print(False or False) # вернёт False, так как оба логические значения False 
```

Результат будет следующий:

```
True
True
True
False 
```

На практике такой оператор применяется, когда анализируют данные, удовлетворяющие одному из условий. Например, аналитик онлайн-кинотеатра исследует данные по количеству просмотров фильмов двух жанров: семейный и нон-фикшн. Отобрать фильмы для анализа поможет оператор or. Если фильм имеет какой-то из этих жанров, то он попадёт в выборку. Не попадут в выборку фильмы других жанров — боевики, триллеры, комедии. Итоговая выборка данных будет содержат несколько значений категории.

Логические операторы `and` и `or` являются **бинарными** логическими операторами, поскольку применяются к двум операндам, стоящим слева и справа.

Приведём все возможные сочетания операторов `and` и `or` в таблице истинности:

|**Выражение А**|**Выражение В**|**A and B**|**A or B**|
|---|---|---|---|
|True|True|True|True|
|True|False|False|True|
|False|True|False|True|
|False|False|False|False|

### Оператор `not`

Логический оператор `not` устроен иначе. Он не объединяет логические значения, а меняет одно значение на противоположное:

- `True` на `False`,
- `False` на `True`.

Разберём его работу на примерах:

```
print(not True) # вернёт False, так исходное логическое значение True
print(not False) # вернёт True, так исходное логическое значение False 
```

Результат будет следующий:

```
False
True 
```

В первом случае исходное логическое значение было равно `True`, поэтому `not` вернул `False`. Во втором случае — наоборот, оператор вернул `True`, так как исходное логическое значение было равным `False`.

Пример применения `not` на практике

Логическое отрицание часто используется, когда необходимо исключить какие-то значения. Например, в интернет-магазине электроники есть категории товаров: «Телефоны», «Смарт-часы», «Компьютеры», «Ноутбуки», «Планшеты», «Телевизоры» и другие. Также есть пока нерассортированные товары, они хранятся в разделе «Без категории».

Представим, что нам нужно выбрать все товары, у которых категория задана. Перечислять все категории долго: их может быть много, к тому же есть вероятность что-то упустить. И тут как раз пригодится оператор `not.` Вместо того чтобы перечислять все категории, достаточно:

1. Выбрать товары без категории: они получат логический результат `True`, а остальные товары — `False`.
2. «Обратить» выборку с помощью оператора `not`. Теперь товары без категории станут `False`, а истинными станут как раз товары, у которые категория задана.

Добавим оператор `not` в таблицу истинности:

|**Выражение А**|**Выражение В**|**A and B**|**A or B**|**not A**|**not B**|
|---|---|---|---|---|---|
|True|True|True|True|False|False|
|True|False|False|True|False|True|
|False|True|False|True|True|False|
|False|False|False|False|True|True|

Оператор `not` является **унарным**, поскольку применяется к одному операнду, стоящему после него. Этим он отличается от бинарных `and` и `or`.

### Составляем сложное логическое выражение

Разберёмся, как составить сложное логическое выражение.

Вернёмся к примеру из начала урока и проверим, что `5 больше 2 и меньше 10`. Здесь нужно соединить два простых логических выражения: `5 > 2` и `5 < 10`. Какой логический оператор лучше выбрать для такого соединения?

С логическим оператором определились, теперь пропишем пример и посмотрим на результат:

```
print((5 > 2) and (5 < 10))
```

Результат будет следующий:


Результат `True`, то есть выражение истинно! Действительно, 5 больше 2 и меньше 10.

Для наглядности перепишем пример: зададим несколько переменных, которые сохранят в себе результат простых логических выражений, и отобразим их результат:

```
a = 5 > 2 # первое простое логическое выражение
```

Первое и второе простые логические выражения вернули `True`, это определило общий результат.

### Битовые операторы `&` и `|`

В Python, помимо стандартных числовых операций, есть операторы для побитовых операций над целыми числами в их двоичном представлении.

Битовые операторы соответствуют логическим операторам, которые вы уже знаете:

|битовый оператор|значение|
|---|---|
|`&`|и (`and`)|
|`\|`|или (`or`)|
|`~`|не (`not`)|

Важно запомнить соответствие операторов в таблице — это знание пригодится вам уже в следующем модуле. А сейчас рассмотрим несколько примеров.

Логические значения `True` и `False` можно рассматривать как 1 и 0, то есть можно применять логические операторы к числам. В двоичной системе счисления числа записывают с помощью 1 и 0, и операторы работают побитово. Битовый оператор берёт последовательно бит из одного числа и соответствующий бит из другого и получает снова 1 или 0. В результате получается последовательность единичек и нулей, которую можно обратно перевести в десятичную систему.

**Пример работы `&`**

```
a = 5 # в двоичной системе - это 101
b = 3 # в двоичной системе - это 011

result = a & b
print(result) # Вывод: 1 (в двоичной системе - это 001) 
```

Здесь a и b представлены в двоичной системе: 5 — это `101`, а 3 — это `011`. Для тройки мы дописали слева 0, чтобы получилось одинаковое количество символов у обоих операндов.

Оператор `&` выполняет логическое «и» для каждой пары битов.

![Поразрядно применяем оператор & к двоичным записям чисел a и b. Первый слева разряд: 1&0=0. Второй слева: 0&1=0. Третий слева: 1&1=1. Итого получаем: a&b=001](https://pictures.s3.yandex.net/resources/image_1705671256.png)

Результат содержит только тот бит, который присутствует в a и в b. В двоичной системе получается `001`, в десятичной это 1.

**Пример работы `|`**

Оператор `|` работает как побитовое логическое «или»:

```
a = 5 # 0b0101
b = 3 # 0b0011

result = a | b
print(result) # Вывод: 7 (0b0111) 
```

Здесь результат содержит те биты, которые были установлены в 1 хотя бы в одном из чисел a или b, это `111`, что в десятичной системе соответствует числу 7.

![Поразрядно применяем оператор | к двоичным записям чисел a и b. Первый слева разряд: 1|0=1. Второй слева: 0|1=1. Третий слева: 1|1=1. Итого получаем: a|b=111](https://pictures.s3.yandex.net/resources/image_1705671280.png)

### Последовательность выполнения условий

Итак, вы изучили много операторов: математические, операторы сравнения, битовые, логические. С помощью всего этого многообразия аналитики записывают разные логические условия: формируют простые, а потом объединяют их в сложные.

В примере выше мы использовали два оператора сравнения и один логический, но условия могут быть намного сложнее. В сложном выражении с несколькими операторами важен порядок выполнения. Как в арифметике — сначала умножение, потом сложение.

**Стандартный порядок в Python** такой:

1. операции в скобках;
2. возведение в степень `**`;
3. битовый оператор `~`;
4. математические операторы — умножение `*` и деление `/`, затем сложение `+` и вычитание `-`;
5. битовые операторы `&` и `|`;
6. операторы сравнения — `<`, `<=`, `>`, `>=`, `!=`, `==`;
7. логические операторы сравнения `not`, `and` и `or`.

Если встречаются операции с одним приоритетом, то они выполняются последовательно слева направо.

Посмотрим на примере, как обрабатываются сложные условия.

```
x = 2
y = 15
z = 15

print((x > 5) and (y < 10) or (z == y)) 
```

Результат будет следующий:

```
True 
```

В начале выполняются операции в скобках, слева направо:

```
x = 2
y = 15
z = 15

step_1 = x > 5  # действие 1
step_2 = y < 10 # действие 2
step_3 = z == y # действие 3

print(f'Действие 1: {step_1}')
print(f'Действие 2: {step_2}')
print(f'Действие 3: {step_3}') 
```

Результат будет следующий:

```
Действие 1: False
Действие 2: False
Действие 3: True 
```

Затем выполняются логические операторы сравнения `and` и `or`, тоже слева направо:

- вначале `and`: `(x > 5) and (y < 10)`,
- затем результат этого действия был использован с оператором `or` и логическим выражением `(z == y)`:

```
x = 2
y = 15
z = 15

step_1 = x > 5  # действие 1
step_2 = y < 10 # действие 2
step_3 = z == y # действие 3

print(f'Действие 1: {step_1}')
print(f'Действие 2: {step_2}')
print(f'Действие 3: {step_3}')

step_4 = step_1 and step_2 # действие 4
step_5 = step_4 or step_3 # действие 3

print(f'Действие 4: {step_4}')
print(f'Действие 5: {step_5}') 
```

Результат:

```
Действие 1: False
Действие 2: False
Действие 3: True
Действие 4: False
Действие 5: True 
```

Последнее пятое действие результирующее — оно определяет итоговый результат логического выражения.

Последовательность выполнения логического выражения можно представить такой схемой:

![](https://pictures.s3.yandex.net/resources/image_1705671331.png)

Теперь расставим дополнительные скобки и посмотрим, что получится.

x = 2 y = 15 z = 15 print((x > 5) and ((y < 10) or (z == y)))

Результат:

False

С помощью скобок мы поменяли порядок выполнения операторов `and` и `or`, в итоге получили другой результат. Пример показывает, как важно следить за приоритетом операций.

Чтобы избежать неоднозначностей и сделать код яснее, можно: - выносить отдельные логические операции в переменные; - использовать скобки и с их помощью явно указывать порядок выполнения.

## Подробнее о возвращаемых элементах True и False

В прошлом уроке вы познакомились с новым типом данных — `bool`, или булевый тип. Он имеет только два логических значения: `True` — истина и `False` — ложь.

В повседневной жизни мы постоянно сталкиваемся с аналогом этого типа данных:

- мобильный телефон может быть заряжен — это `True`, разряжен — это `False`;
- экзамен на получение прав на управление автотранспортом можно сдать — это `True`, а можно завалить — это `False`;
- ребёнок притащил из школы двойку — это `False` или положительную отметку — это `True`.

В программировании и анализе данных этот тип тоже часто встречается:

- пользователь совершил покупку — это `True`, ещё не купил продукт — `False`;
- страница сайта загрузилась — `True`, при загрузке произошла ошибка — `False`;
- средний чек с покупки вышел на необходимый уровень — `True`, не дотянул — `False`.

В этом уроке вы:

- детальнее разберётесь в особенностях булевого типа данных;
- узнаете, как перевести значения из других типов в булевый;
- выясните, какие функции Python возвращают `True` или `False`.

### Тип данных `bool`

В булевом типе данных хранят логические значения и представляют результаты логических выражений.

Логические значения часто представляют как числа: `True` — это 1, а `False` — 0. В Python логические значения можно явно преобразовать в целочисленные (`int`):

```
print(int(False), int(True)) 
```

Результат преобразования будет такой:

```
0 1 
```

Python автоматически преобразовывает значения `True` и `False` в `1` и `0`, когда это необходимо. Убедимся в этом — сложим `True` и `False`:

```
print(False + True) 
```

Получим целочисленный результат:

```
1 
```

Здесь арифметическое действие с `True` и `False` проведено, как с числами: `1 + 0 = 1`.

В других операциях логические значения тоже ведут себя как числа.

Это бывает полезно, когда необходимо подсчитать количество значений, отвечающих заданным условиям. Например, есть список с информацией о том, совершили пользователи покупки или нет. Каждое значение — информация об отдельном пользователе:

```
is_buyer = [True, True, False, True, False, False, False, False, True, True] 
```

Можно представить этот список в виде чисел `1` и `0`. Тогда, сложив все значения списка, фактически мы посчитаем количество единичек, то есть получим количество пользователей, совершивших покупку:

```
is_buyer = [True, True, False, True, False, False, False, False, True, True, False]
print(f'Всего посетителей: {len(is_buyer)}, из них совершили покупку {sum(is_buyer)}') 
```

Результат будет следующий:

```
Всего посетителей: 11, из них совершили покупку 5 
```

В примере выше функция `sum()` позволяет подсчитать сумму всех значений в списке.

### Преобразование из других типов данных

Ещё одно применение булевого типа данных — ситуации, когда необходимо проверить наличие информации в переменной. Например, все ли поля заполнены в форме регистрации или все ли товары имеют необходимую информацию.

В таких случаях данные других типов можно преобразовать в булевый тип. Для этого используют встроенную функцию Python `bool()`. Она возвращает значение `False`, если в исходных данных:

- число 0;
- что-то пустое: строка, список или пустой кортеж;
- значение None.

В остальных случаях функция `bool()` вернёт `True`.

Например:

|**Пример**|**Результат**|**Пояснение**|
|---|---|---|
|`bool('string')`|`True`|Строка с любыми символами преобразуется в `True`.|
|`bool('')`|`False`|Пустая строка преобразуется в `False`.|
|`bool(['A','B'])`|`True`|Список непустой — `True`.|
|`bool([])`|`False`|Пустой список — `False`.|
|`bool(-1995.5)`|`True`|Любое ненулевое число преобразуется в `True`.|
|`bool(0)`|`False`|0 преобразуется в `False`.|

### Функции-предикаты

Иногда нужно не просто проверить наличие значения в переменной, а проверить что-то другое — особенно часто проверяют корректность заполнения данных. Например, убедиться в валидности номера телефона поможет проверка того, что строка состоит только из цифр.

В подобных случаях помогают **функции-предикаты**. В программировании это функции, которые возвращают булево значение (`True` или `False`). Они помогают проверить условия или выполнить логические операции.

Примеры функций-предикатов:

- `islower()` — возвращает `True`, если в строке нет прописных букв;
- `isdigit()` — возвращает `True`, если строка состоит только из числовых знаков;
- `isalpha()` — возвращает `True`, если в строке только буквы, без пробелов и знаков препинания;
- `isspace()` — возвращает `True`, если в строке только пробелы.

Если условие не соблюдено, эти функции вернут `False`.

Примеры работы:

|**Метод**|**Пример**|**Результат**|**Пояснение**|
|---|---|---|---|
|`.islower()`|`'hello'.islower()`|`True`|В строке все символы представлены в нижнем регистре|
|`.isdigit()`|`'777'.isdigit()`|`True`|В строке все символы представлены в виде чисел|
|`.isalpha()`|`'здесь есть пробелы, знаки препинания'.isalpha()`|`False`|В строке есть пробелы, поэтому результат ложный|
|`.isspace()`|`' '.isspace()`|`True`|В строке только пробелы|

Ещё примеры функций-предикатов

Часто используют строковые методы, которые проверяют соответствие начала и окончания строки заданному набору символов:

- `startswith(prefix)` — возвращает `True`, если строка начинается с указанного набором символов `prefix`;
- `endswith(suffix)` — возвращает `True`, если строка заканчивается указанным набором символов `suffix`;

```
print('hello'.startswith('h')) # проверяем, начинается ли hello с h
print('hello'.endswith('O')) # проверяем, заканчивается ли hello символом O 
```

Результат будет следующий:

```
True
False 
```

Хотя `hello` заканчивается `o`, но в функции мы уточняем про другой символ — заглавную `O`. Эта функция и многие другие чувствительны к регистру.

Другой пример функций-предикатов — функции `all()` и `any()`. Они работают с пустыми значениями в списках и других множествах:

- `all()` — возвращает `True`, если все значения списка `True`. Например, функция поможет проверить, что заполнены все обязательные поля в форме регистрации. Она вернёт `True`, если всё заполнено, и `False`, если чего-то недостаёт.
- `any()` — возвращает `True`, если хотя бы одно значение списка `True`. Например, клиент может указать разные способы связи: номер телефона, имейл, ник в Телеграме. Эта функция поможет проверить, что указан хоть один способ связи. Она вернёт `True`, если клиент указал, как с ним можно связаться, и `False`, если клиент не указал ни одного способа.

```
print(all(['A','B',None,'D'])) 
print(any(['A','B',None,'D'])) 
```

Результат будет следующий:

```
False
True 
```

Итак, в этом и предыдущем уроке вы познакомились с ситуациями, когда булевый тип данных возвращает:

- логические выражения с операторами сравнения;
- функции-предикаты;
- преобразования данных других типов.

Ещё логические значения `True` или `False` получаются при использовании логических операторов `and`, `or`, `not`. Следующий урок — про них.

## Условная конструкция

В повседневной жизни мы часто действуем по ситуации. То есть мы выбираем свои действия в зависимости от того, какие условия наступили. Например, если на улице дождь, то мы возьмём зонт. Если небо ясное, то о зонте мы и не подумаем.

![](https://pictures.s3.yandex.net/resources/image_1705497588.png)

В анализе данных тоже, бывает, приходится действовать по ситуации. Например:

- Если пользователь совершил покупку, его относят к покупателям и включают в дальнейший анализ. Если ничего не купил, то не включают.
- Или если конверсия по какому-то источнику ниже ожидаемой, то стоит обратить внимание на этот источник и скорректировать закупку трафика. Если же с конверсией всё в порядке, то вложения в рекламную кампанию можно оставить на прежнем уровне.

В таких случаях программа выполняет один или другой блок кода — в зависимости от наступления определённых условий.

В предыдущих уроках вы разбирали примеры, в которых все строчки кода выполнялись одна за другой. А в этом уроке вы узнаете, как с помощью условных конструкций управлять выполнением различных частей кода.

### Условный оператор `if`

В Python существует специальный оператор `if` — он проверяет, истинно ли заданное условие. Если условие истинно, то выполняются определённые действия. Если условие ложно, то программа пропустит этот блок кода и перейдёт дальше.

Например, фрагмент инструкции по сборам может иметь вид:

![Список: сумка; если на улице дождь, то взять зонт; телефон; ключи.](https://pictures.s3.yandex.net/resources/image_1706090344.png)

Выбор действий зависит от условия: если на улице идёт дождь, то человек возьмёт зонт; дождя нет — пропустит пункт с дождём и перейдёт к следующему. В любом случае в конце концов мы соберёмся и выйдем на улицу, просто в одном случае с зонтом, а в другом — без.

Так и в программировании, при использовании условного оператора появляются различные пути выполнения кода.

Эти разные части кода, которые выполняются или нет в зависимости от определённого условия, обычно называют **«ветки»** кода, или ветвление кода.

Синтаксис простой условной конструкции с оператором `if` следующий:

![После условного оператора if нужно написать логическое выражение, после которого поставить разделить — двоеточие. На следующей строке — сделать отступ в четыре пробела и далее написать ветку кода на случай, если логическое выражение истинно. Для примера из текста урока получится так: 
if weather == 'дождь':
(отступ в четыре пробела) print('взять зонт')](https://pictures.s3.yandex.net/resources/1.3.6_skhema2_3_1707996029.png)

В структуре условной конструкции можно выделить два элемента — его заголовок и блок с инструкциями (тело условной конструкции).

1. **Заголовок условной конструкции `if`** — его первая строка, которая должна содержать:
    - Ключевое слово `if`: условный оператор, который предшествует самому условию.
    - Логическое выражение: то условие, которое необходимо проверить. Оно может быть простым или сложным, или включать функции-предикаты. Здесь часто используют операторы сравнения, вспомнить о них подробнее можно в уроке «Логика. Логические выражения, операторы сравнения».
    - Двоеточие: без него будет синтаксическая ошибка, а сам условный оператор не будет объявлен.
2. **Тело условной конструкции `if`** — набор инструкций, которые выполняются, если логическое выражение истинное. Тело может содержать как одну функцию, так и несколько функций — ограничений нет. Тело условной конструкции нужно отделить отступом от заголовка. Если этого не сделать, инструкции выполнятся независимо от условия. Отступ — это четыре пробела. Если количество пробелов будет другим — возникнет синтаксическая ошибка.

Напишем условную конструкцию, которая будет делать определённые действия, если логическое выражение `weather == 'дождь'` истинное:

```
# объявим переменную weather с информацией о погоде
weather = 'дождь'

# условная конструкция
if weather == 'дождь':  # заголовок условной конструкции
    print('Взять зонт') # тело условной конструкции 
```

Результат выполнения кода:

```
Взять зонт 
```
Обратите внимание, что код выше не пытается описать все возможные случаи. Если логическое выражение вернёт `False`, ничего не произойдёт:

```
# линейное выполнение алгоритма
weather = 'солнце'

# условная конструкция
if weather == 'дождь':  # заголовок условной конструкции
    print('Взять зонт') # тело условной конструкции 
```

Результат выполнения кода:

Пусто! Python просто пропустил код из условной конструкции, а поскольку дальше не было инструкций, то ничего не произошло. Такую условную конструкцию называют **неполной**.

### Сложные условные конструкции и оператор `if`

В предыдущих примерах мы рассматривали простое логическое выражение, но в условной конструкции можно использовать и сложные логические выражения или функции-предикаты — то есть то, что вернёт логическое значение `True` или `False`.

Например, выведем сообщение о возможных проблемах с работой десктопного приложения:

```
device = 'PC'
day = 'Friday'

if device == 'PC' and day == 'Friday':  
    print('По техническим причинам в ночь с пятницы на субботу могут быть сбои в работе приложения.') 
```

Здесь в `if` стоит сложное логическое выражение с `and`.

Результат выполнения кода:

```
По техническим причинам в ночь с пятницы на субботу могут быть сбои в работе приложения. 
```

Поскольку два условия были истинными, на экране появилось сообщение.

## Дополнительные ветки кода

В прошлом уроке мы рассматривали условную конструкцию, которая выполняла действия, если условие было истинным. Вернёмся к примеру про дождь — если на улице дождь, нужно взять зонт. Это пример условного оператора с одним блоком действий, которые случатся только тогда, когда проверяемое условие верно. В программировании такую ситуацию называют условной конструкцией с одной веткой кода или инструкций.

Однако часто нужно прорабатывать действия не только, когда условие истинное, но и когда условие ложное. Например, если оплата покупок в магазине прошла успешно, мы можем их забрать, а если оплата не прошла, то менеджер магазина попросит провести оплату ещё раз или откажет в покупке. В этом случае у нас есть несколько сценариев процесса покупки. Так же и в программировании при проверке условия можно прописать несколько путей развития кода или алгоритма анализа данных. В этом случае мы можем говорить о нескольких ветвях кода.

Например, если пользователь совершил покупку, то мы зададим ему новую роль — покупатель, а если не совершал, то оставим это значение ложным. Или если товар купили больше заданного количества раз, то присвоим ему категорию «популярный товар», а если нет, то другую, например, «спрос ниже среднего».

В этом уроке мы рассмотрим:

- ветвление кода в условной конструкции `if-else`;
- какие есть ключевые слова для создания сложных алгоритмов проверки различных условий.

### Ветка кода при ложном условии. Ключевое слово `else`

Если необходимо выполнить действия, при которых условная конструкция вернёт `False`, используют ключевое слово `else` — от английского «иначе». Для этого к условной конструкции добавляют ещё два блока:

![Три блока на одной строке: условный оператор if, логическое выражение weather == дождь, разделитель — двоеточие. Следующий блок с новой строки: print взять зонт. Блок подписан — ветка кода на случай дождя. Следующая строка: ключевое слово else и двоеточие. Новая строка, блок print помыть машину. Подпись к блоку — ветка кода на случай дождя](https://pictures.s3.yandex.net/resources/image_1706087729.png)

1. **Ключевое слово `else`:** предшествует блоку программы, который выполнится в ином случае — если условие ложное. После `else` тоже ставится двоеточие.
2. **Альтернативное действие:** новая ветка кода, которая содержит инструкции, выполняемые, если условное утверждение окажется ложным, то есть оператор сравнения вернёт `False`. Этот блок также начинают отступом в четыре пробела.

```
weather = 'солнце'

if weather == 'дождь':
# первая ветка кода
    print('Взять зонт')
else:
# вторая ветка кода
    print('Помыть машину') 
```

Результат выполнения кода:

```
Помыть машину 
```

Такая конструкция имеет две ветви кода, в которых прописаны инструкции на любой случай:

- `True` — если идёт дождь, нет смысла мыть машину, а вот зонт лучше захватить.
- `False` — если дождя нет, то и зонт ни к чему. Зато можно помыть машину.

### Дополнительные ветки кода. Ключевое слово `elif`

Ветвление программы, как и принятие решений в реальном мире, не всегда ограничивается двумя вариантами. Если на улице не дождь, а солнце, то вы можете не просто оставить зонт дома, но и надеть панаму. А если там не дождь, а снег — надеть шапку и шарф.

Для того чтобы реализовать условную конструкцию с различным количеством ветвления кода, используется конструкция `elif` — от английских else «еще, иначе» и if «если». Для этого к условной конструкции `if` перед конструкцией `else` добавляют ещё два блока:

![Три блока на одной строке: условный оператор if, логическое выражение weather == дождь, разделитель — двоеточие. Следующий блок с новой строки: print взять зонт. Блок подписан — ветка кода на случай дождя. Следующая строка: условный оператор elif, логическое выражение weather == солнце, разделитель — двоеточие. Следующий блок с новой строки: print взять панаму с подписью ветка кода на случай без дождя. Следующая строка: условный оператор elif, логическое выражение weather == снег, разделитель — двоеточие. Следующий блок с новой строки: print надеть шапку и шарф с подписью ветка кода на случай снега. Следующая строка ключевое слово else и двоеточие. Следующая строка print на улице отличная погода.](https://pictures.s3.yandex.net/resources/image_1706087881.png)

Блок условий `elif` по структуре аналогичен конструкции `if` и состоит из заголовка и тела конструкции:

1. **Заголовок блока `elif`**:
    
    - **Ключевое слово `elif`**, которое предшествует логическому условию.
    - **Логическое выражение:** то условие, которое необходимо проверить. Оно может быть простым или сложным, или состоять из функций-предикатов.
    - **Двоеточие:** без него будет синтаксическая ошибка.
    
2. **Тело условной конструкции `elif`** — набор инструкций, которые выполняются, если логическое выражение истинное. Как и в случае условной конструкции `if`, этот блок также начинают отступом в четыре пробела.
    

```
weather = 'снег'

if weather == 'дождь': # если на улице дождь
# первая ветка кода
    print('Взять зонт')
elif weather == 'солнце': # если на улице солнце
# вторая ветка кода
    print('Взять панаму')
elif weather == 'снег': # если на улице снег
# третья ветка кода
    print('Надеть шапку и шарф')
else: # если все три условных выражения ложные
# четвёртая ветка кода
    print('На улице отличная погода') 
```

Результат выполнения кода:

```
Надеть шапку и шарф 
```

В примере выше используются две конструкции с `elif`: если на улице солнце и если снег. Алгоритм выполнения кода и проверки условий следующий:

- Если первое условие `if` ложное — `if weather == 'дождь':`, то проверяется следующее условие `elif weather == 'солнце':`.
- Второе условие тоже было ложным, поэтому проверка условий продолжилась и перешла к условию `elif weather == 'снег':`.
- Третье условие оказалось истинным, и исполнилась ветка кода после этого условия — `print('Надеть шапку и шарф')`, а выполнение самой условной конструкции завершилось.

В случае если ни одно из условий не является истинным, то выполняется блок `else` (если он присутствует).

Такое ветвление часто используют для задач по сегментации или классификации. Разберём на примере. Необходимо провести сегментацию товаров детского магазина по возрастному признаку:

1. младенцы до 1 года;
2. дети от 1 года до 7 лет;
3. дети от 7 до 11 лет;
4. дети от 11 до 14 лет;
5. 14 и выше лет.

Эта задача как раз для условной конструкции с `if`, `elif` и `else`. Начнём с подсчёта необходимого количества ветвей кода: сегментация товаров предполагает пять независимых ветвей кода, в которых будет определён возрастной сегмент.

1. Зададим возраст ребенка, для которого предназначен детский товар, переменной `age` и пропишем первое условие — если возраст больше 14 лет, то инструкции выведут на экран название сегмента — `14 и выше лет`:

```
age = 0 # переменная, содержащая возраст
segment = '' # переменная, которая будет хранить информацию о возрастном сегменте

if age > 14: # первая ветка кода
    segment = '14 и выше лет' 
```

2. Далее, если первое логическое выражение `age > 14` будет ложным, то проверим следующий возрастной сегмент — `дети от 11 до 14 лет`. Для этого используем логическое выражение `age >= 11` и ключевое слово `elif`:

```
age = 0 # переменная, содержащая возраст
segment = '' # переменная, которая будет хранить информацию о возрастном сегменте

if age > 14: # первая ветка кода
    segment = '14 и выше лет'
elif age >= 11: # вторая ветка кода
    segment = 'дети от 11 до 14 лет' 
```

3. Далее по аналогии пропишем остальные условия для сегментов `дети от 7 до 11 лет`, `дети от 1 года до 7 лет` и `младенцы до 1 года`. Для этого также используем ключевое слово `elif`:

```
age = 0 # зададим переменную, которая будет содержать возраст
segment = '' # зададим переменную, которая будет хранить информацию о возрастном сегменте

if age > 14: # первая ветка кода
    segment = '14 и выше лет'
elif age >= 11: # вторая ветка кода
    segment = 'дети от 11 до 14 лет'
elif age >= 7: # третья ветка кода
    segment = 'дети от 7 до 11 лет'
elif age >= 1: # четвёртая ветка кода
    segment = 'дети от 1 года до 7 лет'
elif age >= 0: # пятая ветка кода
    segment = 'младенцы до 1 года'

print(segment) 
```

4. Мы получили основной алгоритм условной конструкции! Если сейчас мы запустим код, то результат будет следующим:

```
младенцы до 1 года 
```

Результат верный — исходно переменная `age = 0`, поэтому и сегмент — `младенцы до 1 года`. Однако если в переменной будет отрицательное значение, то результата сегментации мы не получим. В подобных случаях полезно дополнить условную конструкцию блоком `else`, который позволит выполнить инструкции, если все условия выше оказались ложными:

```
else:
    segment = 'без сегмента' 
```

5. В итоговом виде условная конструкция будет такой:

```
age = 0 # зададим переменную, которая будет содержать возраст
segment = '' # зададим переменную, которая будет хранить информацию о возрастном сегменте

if age > 14: # первая ветка кода
    segment = '14 и выше лет'
elif age >= 11: # вторая ветка кода
    segment = 'дети от 11 до 14 лет'
elif age >= 7: # третья ветка кода
    segment = 'дети от 7 до 11 лет'
elif age >= 1: # четвёртая ветка кода
    segment = 'дети от 1 года до семи лет'
elif age >= 0: # пятая ветка кода
    segment = 'младенцы до 1 года'
else:
    segment = 'без сегмента'

print(segment) 
```

А результат выполнения при значении `age = 8` будет:

```
дети от 7 до 11 лет 
```

# Циклы
## Понятие цикла. Элементы цикла

В этом уроке вы познакомитесь с циклами в Python. Они помогают аналитику избавиться от рутины и позволяют выполнять однотипные повторяющиеся команды множество раз. Разберём на примерах, зачем нужны циклы, как правильно их объявлять и из каких элементов они состоят.

### Цикл

В своей работе аналитик часто сталкивается с необходимостью выполнять одни и те же действия много раз, используя различные срезы данных. Например, для каждого клиента из списка вывести информацию для рассылки электронных писем или для каждой категории товаров подготовить сводный отчёт по заказам, покупателям, среднему чеку и другим показателям. Эти задачи могут различаться по сложности, но их объединяет то, что существует набор действий, которые нужно повторить для каждого элемента из списка.

![](https://pictures.s3.yandex.net/resources/image_1706292215.png)

В жизни такой инструмент был бы незаменим. Скажем, вы подготовили подарки друзьям на новогодние праздники и хотите обернуть их в красивую упаковочную бумагу. Вы составили список друзей и приступили к делу. Для каждого друга из списка:

1. подписали поздравительную открытку;
2. положили её к подарку;
3. отрезали упаковочную бумагу;
4. завернули в неё подарок.

Получается, вы выполнили последовательность из четырёх действий для подготовки подарка для каждого друга из списка.

С этим отлично справится цикл — сначала для первого элемента из списка (подарок для первого друга) цикл выполнит определённые действия (подготовит подарок). После этого начинается новый шаг действий со следующим элементом списка (подарок для второго, третьего и так далее до самого последнего друга). Каждый такой шаг называется **итерацией цикла**. При этом сами действия на каждом шаге остаются неизменными. Цикл остановится, когда все элементы списка закончатся (для каждого друга подготовим подарки). Таким образом, общее количество итераций цикла будет равно количеству элементов в списке.

Рассмотрим примеры, когда цикл помогает оптимально решить однотипные задачи.

Использование цикла делает работу проще! Но дело не только в упрощении. Скопировать несколько строчек кода с одинаковыми командами — несложно. Но «копипаста» (от англ. copy _— «_копировать», paste _— «_вставить») одинаковых строчек кода — плохой тон в программировании. Дублирование кода может привести к возможным ошибкам в работе скрипта, если понадобится изменить алгоритм вычислений. В случае с циклом — изменить нужно только один блок кода. В случае копирования кода — каждый скопированный фрагмент.

Также циклы не только упрощают работу. Они делают код понятнее, и благодаря им код зачастую выполняется быстрее.

### Элементы цикла

Часто применяемый цикл Python для анализа данных — это `for` — от английского «для». Он повторяет заданную последовательность инструкций для каждого итерируемого объекта, например, списка, кортежа, строки.

Например, выведем на экран цифры от 1 до 3. Для этого напишем такой код:

```
for i in 1,2,3:
    print(i) 
```

Результат будет следующим:

```
1
2
3 
```

Немного усложним и выведем на экран последовательно все элементы списка с именами друзей, для которых будем готовить новогодние подарки:

```
friends = ['Юля', 'Гена', 'Гоша', 'Алиса', 'Слава'] # список друзей

for friend in friends: # заголовок цикла
    print('Имя друга:', friend) # тело цикла 
```

```
Имя друга: Юля
Имя друга: Гена
Имя друга: Гоша
Имя друга: Алиса
Имя друга: Слава 
```

Каждый цикл из примеров выше состоит из двух элементов — заголовка и тела. **Заголовок цикла** — его первая строка, которая должна содержать:

- Ключевое слово `for`;
- Переменную-итератор, которая перемещается по множеству значений слева направо, принимая на каждом шаге значение очередного элемента списка; в первом примере — это переменная `i`, а во втором — `friend`;
- Ключевое слово `in`;
- Множество значений, по которому движется итератор; в первом примере — это множество цифр от 1 до 3, а во втором — список друзей `friends`;
- Знак `:`. Без него будет синтаксическая ошибка, а сам цикл не будет объявлен.

**Тело цикла** — набор инструкций, которые выполняются на каждом шаге цикла. Каждая строка тела цикла обязательно отделяется отступом в четыре пробела. Так Python сможет определить, где начинается блок инструкций и где он заканчивается. А вот если тело цикла не отделить четырьмя пробелами, то инструкции выполнятся вне цикла. Если же количество пробелов будет иным — возникнет синтаксическая ошибка.

```
for переменная_итератор in список_элементов: # заголовок цикла
    # тело цикла - код, который выполняется для каждого элемента списка
    # каждая строка тела цикла отделяется четырьмя пробелами
    # количество строк в теле цикла неограничено
    pass 
```

Тело цикла обязательно должно содержать хотя бы одну инструкцию. Если необходимо оставить цикл пустым, используют команду `pass`, которая не выполняет никаких действий, но и не позволяет всему коду остановиться из-за ошибки выполнения.

Имя переменной-итератора в цикле можно задать любое. Традиционно эти имена образуют от имени списка элементов в единственном числе: например, если имя списка `friends`, то имя итератора дают `friend`, если список называется `orders`, то переменную называют `order`. Такой подход позволяет по названию итератора определить его принадлежность к тем или иным данным и делает код понятнее.

Если переменная-итератор перебирает числовые последовательности, то в качестве имени итератора обычно используют латинские буквы `i`, `j` или `k`, поскольку ряд чисел не означает количество чего-либо и является служебными значениями, необходимыми для работы цикла.

Рассмотрим ещё один пример и потренируемся правильно объявлять цикл.

```
string = 'Пример строки'
```

## Перебор элементов цикла

В прошлом уроке вы познакомились с циклами. Узнали, из чего они состоят и как объявляются, рассмотрели пример цикла `for` с последовательным перебором элементов списка. А в этом уроке вы:

- познакомитесь с особенностями использования итератора и научитесь определять количество итераций в цикле `for`;
- разберётесь, как использовать последовательность чисел в цикле, и научитесь перебирать индексы списков с помощью функции `range()`.

### Итератор цикла и количество итераций

Вернёмся к нашему примеру со списком друзей:

```
friends = ['Юля', 'Гена', 'Гоша', 'Алиса', 'Слава'] 

for friend in friends: 
    print('Имя друга:', friend) 
```

На каждой итерации цикла переменная-итератор `friend` меняет своё значение. На первом шаге цикла итератор `friend` принимает значение первого элемента (`'Юля'`) исходного списка `friends`. На второй — итератор `friend` принимает значение второго (`'Гена'`) элемента списка `friends` и так далее, пока не будут перебраны все элементы:

```
Имя друга: Юля
Имя друга: Гена
Имя друга: Гоша
Имя друга: Алиса
Имя друга: Слава 
```

В примере выше итератор последовательно перемещается по элементам списка, перебирая их слева направо. Цикл автоматически прекратит работу, когда переберёт все элементы списка.

![](https://pictures.s3.yandex.net/resources/image_1706293499.png)

В этом случае количество итераций цикла равно количеству элементов в списке. Например, в списке из примера выше имена пяти друзей, поэтому в результате работы цикла будет всего пять строк.

Рассмотрим ещё один пример.

В цикле итератор можно использовать не только для вывода его на экран, но и для совершения различных действий с ним или с его помощью.

Каких, например?

Представьте: есть список температур по шкале Фаренгейта, который нужно перевести в значения температур по шкале Цельсия. Для этого нужно из каждого значения температуры по шкале Фаренгейта вычесть 32 (это температура замерзания воды по шкале Фаренгейта) и поделить на 1.8 (это отношение интервала температур от замерзания воды до её кипения по шкале Фаренгейта к аналогичному интервалу по шкале Цельсия — 180/100). В этой ситуации нам нужно совершить одни и те же действия с каждым элементом списка и показать результат преобразования. С этим отлично справится цикл!

При этом если в теле цикла происходит изменение значения итератора, то это не изменит сам элемент исходного списка.

Например, список `prices` хранит цены товаров. Увеличим каждое значение цены на `50` единиц:

```
prices = [1500, 3100, 7500, 4200]
for price in prices:
     price += 50
     print(f'Новая цена товара: {price}')
print('Значения списка prices:', prices) 
```

```
Новая цена товара: 1550
Новая цена товара: 3150
Новая цена товара: 7550
Новая цена товара: 4250
Значения списка prices: [1500, 3100, 7500, 4200] 
```

В теле цикла цены скорректировались, но исходные значения не изменились. В этом случае изменения остались внутри цикла и не повлияли на другие переменные основного кода.

Внутри цикла `for` лучше избегать изменения итератора, поскольку в сложных циклах это может привести к различным ошибкам в работе кода. В примере выше для новой цены лучше назначить новый оператор внутри цикла. Это позволит использовать значение итератора несколько раз за одну итерацию:

```
prices = [1500, 3100, 7500, 4200]
for price in prices:
     new_price = price + 50
     print(f'Старая цена товара: {price}, новая цена товара: {new_price}') 
```

```
Старая цена товара: 1500, новая цена товара: 1550
Старая цена товара: 3100, новая цена товара: 3150
Старая цена товара: 7500, новая цена товара: 7550
Старая цена товара: 4200, новая цена товара: 4250 
```

Обратите внимание, что в теле цикла мы выполнили последовательно несколько инструкций и назначили новую переменную `new_price`. При этом эта переменная будет доступна только внутри тела цикла и будет задаваться каждый раз заново на новой итерации цикла. В таком случае переменную `new_price` называют локальной переменной цикла.

По видимости переменные могут быть локальными и глобальными, а выбор видимости зависит от области видимости.

Глобальные переменные можно использовать везде внутри программы, а локальные переменные будут доступны только в пределах локальной области, в которой они были заданы.

### Фиксированное количество итераций цикла `for`. Функция `range()`

До этого мы рассматривали циклы, в которых количество итераций было равно количеству элементов, итерируемых в цикле.

Чтобы задать конкретное количество итераций цикла, вызывают функцию `range()` — «диапазон». Функция `range(n)` по мере обращения к ней возвращает последовательность целых чисел длиной `n` в диапазоне от `0` до `n-1`. Правая граница — значение `n` исключено из диапазона значений, возвращаемого функцией.

Используя функцию `range()` в качестве последовательности значений, по которому движется итератор, мы можем определить необходимое количество итераций цикла:

```
for i in range(5): # зададим последовательность целых чисел - 0, 1, 2, 3, 4
    print(f'Значение итератора равно {i}') 
```

```
Значение итератора равно 0
Значение итератора равно 1
Значение итератора равно 2
Значение итератора равно 3
Значение итератора равно 4 
```

В заголовке цикла итератор `i` перебирает результаты работы функции `range()`, которая возвращает последовательность целых чисел от `0` до `5`, не включая последнее целое значение. То есть на каждом шаге цикла итератор меняет свое значение — на первом шаге итератор будет равен `0`, на втором шаге — `1`, на третьем — `2`, а на пятом итератор будет равен `4`. В результате цикл совершил пять итераций.

Рассмотрим такой пример — рассчитаем увеличение стоимости товара за год, если в каждый квартал стоимость будет расти на 5%. В этом нам поможет цикл с определённым количеством итераций. Зафиксируем исходную стоимость товара в переменной `price`, коэффициент увеличения стоимости — в переменной `coeff`, а количество итераций зададим равное четырём — количеству кварталов в году. В теле цикла на каждой итерации будем увеличивать значение текущей стоимости на 5%:

```
price = 100 # исходная стоимость товара
coeff = 1.05 # коэффициент увеличения стоимости на 5%

for i in range(0,4): # в заголовке установим количество итераций, равное 4
    price *= coeff # увеличиваем исходное значение на 5% на каждом шаге цикла
    print(f'Стоимость товара в квартале {i+1} равна {round(price,2)} руб.') 
```

```
Стоимость товара в квартале 1 равна 105.0 руб.
Стоимость товара в квартале 2 равна 110.25 руб.
Стоимость товара в квартале 3 равна 115.76 руб.
Стоимость товара в квартале 4 равна 121.55 руб. 
```

Цикл `for` на каждом шаге берёт значение `price` и увеличивает его на заданный коэффициент. Поскольку переменная `price` глобальная, то есть была задана в основной части кода, то изменения переменной сохраняются: в последующей итерации цикл будет обрабатывать изменённое на предыдущей итерации значение `price`.

В результате мы получили поквартальную динамику изменения значений `price`. Обратите внимание, что в выводе результат округлился до двух знаков после запятой. Для этого мы использовали функцию `round()` — «округлять», в которой задали количество знаков после запятой.

### Перебор индексов элементов списка в цикле

Если в функцию `range()` передать длину списка — результат работы функции `len()`, то последовательность чисел будет соответствовать индексам списка. Например, выведем знакомые нам имена друзей с помощью индекса элемента списка:

```
friends = ['Юля', 'Гена', 'Гоша', 'Алиса', 'Слава'] 

for i in range(len(friends)): 
    print('Имя друга:', friends[i]) 
```

```
Имя друга: Юля
Имя друга: Гена
Имя друга: Гоша
Имя друга: Алиса
Имя друга: Слава 
```

Итератор `i` перемещается по последовательности целых чисел от `0` до `len(friends) - 1`. А в теле цикла по значению итератора выводится элемент с индексом итератора — `i`: `friends[i]`.

Использование индекса в качестве итератора позволяет изменять значения элементов самого списка. Например, уменьшим каждое значение цены из списка `prices` на `100` единиц:

```
prices = [1500, 3100, 7500, 4200]
print('Значения списка prices до изменения:', prices)

# уменьшим каждое значение списка prices на 100 единиц:
for i in range(len(prices)): 
     prices[i] -= 100
print('Результат изменения значений списка prices:', prices) 
```

Результат работы цикла будет следующий:

```
Значения списка prices до изменения: [1500, 3100, 7500, 4200]
Результат изменения значений списка prices: [1400, 3000, 7400, 4100] 
```

Здесь цикл `for` на каждом шаге хранит очередной индекс элемента в переменной-итераторе `i`. В теле цикла алгоритм берёт элемент списка `prices` с индексом `i` и уменьшает его значение с помощью оператора `-=`.

Выше на примере вывода элементов списка мы рассмотрели общий алгоритм использования индекса в качестве переменной-итератора. Однако для вывода элементов коллекции лучше использовать перебор самого списка, как это мы делали выше. Индексы элементов списка лучше использовать для решения более сложных задач, где необходимо вносить изменения в исходный список, примеры которых мы рассмотрим далее.

### Перебор определённого интервала индексов элементов списка в цикле

С помощью функции `range()` можно перебрать заданный интервал индексов элементов множества. Для этого нужно передать в функцию `range()` два аргумента — начальное значение и конечное. Например, отобразим только первые три элемента списка друзей:

```
friends = ['Юля', 'Гена', 'Гоша', 'Алиса', 'Слава'] 

for i in range(0,3): 
    print('Имя друга:', friends[i]) 
```

```
Имя друга: Юля
Имя друга: Гена
Имя друга: Гоша 
```

В этом случае были показаны элементы с индексом 0, 1, 2 — последнее значение `3` в `range(0,3)` не было включено в последовательность.

В этом уроке мы рассмотрели, как работает итератор цикла `for`, как определяется количество итераций, и разобрали, как можно использовать возможности функции `range()` в цикле. Также рассмотрели, как можно менять значения перебираемых списков.

## Циклы и строки. Вложенные циклы

В прошлом уроке вы узнали, как происходит перебор значений списка в цикле, а также научились использовать функцию `range()`, чтобы контролировать количество итераций и определять интервал перебора элементов в списке. В этом уроке вы выясните:

- как происходит перебор индексов элементов строки в цикле;
- как изменять строки, создавая новые строки в цикле;
- особенности вложенных циклов.

### Перебор индексов элементов строки в цикле

Ранее мы использовали `range()` для обращения к индексам элементов списка. Также можно обращаться к буквам или любым другим символам строки через их индексы. Например, выведем на экран первые шесть символов в строке — они будут соответствовать номеру заказа:

```

string = '001642 Кизильник цельнокрайний (обыкновенный), 50 кустов'
for i in range(6): # переберём первые 6 элементов строки
    print(string[i], end="") 
```

Результат:

```
001642 
```

Такой перебор значений строки можно использовать для создания новой строки: изменим код выше и выделим номер заказа в отдельную строку `order_id` :

```

string = '001642 Кизильник цельнокрайний (обыкновенный), 50 кустов'
order_id = '' # зададим пустую строку
for i in range(6): # переберём первые 6 элементов строки
    order_id += string[i]
print(f'Номер заказа: {order_id}') 
```

Результат:

```
Номер заказа: 001642 
```

В результате работы цикла сформировалась новая строка, которая содержит только номер заказа. Обратите внимание, что перед циклом мы создали пустую строку `order_id`, к которой на каждой итерации цикла добавляли значение строки по индексу — значению переменной-итератора от 0 до 5.

Также с помощью функции `range()` можно выбрать только часть строки по аналогии с тем, как мы выделяли интервал элементов списка в прошлом уроке.

### Изменение значений строки с помощью цикла

Строки — это уникальные неизменяемые объекты, и мы не можем менять символы в строке, используя индекс элемента, как это делали в случае со списками. Однако, чтобы изменить конкретные значения исходной строки, можно создать новую строку, которая будет содержать изменённые значения.

Например, приведём строку `АП ЧЕХОВ` к виду `Чехов А.П.`. Для этого будем придерживаться такого алгоритма:

1. Создадим новую пустую строку `new_string` и добавим к ней первый символ фамилии автора — в исходной строке этот элемент имеет индекс `3`. При добавлении первого символа фамилии переведём его в верхний регистр — для этого будем использовать метод `upper()` — «верхний»:

```
string = 'АП ЧЕХОВ' # исходная строка
new_string = '' # создадим пустую строку
new_string += string[3].upper() # добавим к новой строке первый символ фамилии 
```

2. Используя цикл `for`, подсоединим остальную часть фамилии, которая начинается с индекса `4`, и переведём её в нижний регистр, для чего будем использовать метод `lower()` — «нижний»:

```
string = 'АП ЧЕХОВ' # исходная строка
new_string = '' 
new_string += string[3].upper()

for i in range(4, len(string)): # переберём в цикле остальную часть фамилии
    new_string += string[i].lower() # переведём символы в нижний регистр 
```

3. Добавим разделитель `` (пробел) между фамилией и инициалами:

```
string = 'АП ЧЕХОВ' # исходная строка
new_string = '' 
new_string += string[3].upper()

for i in range(4, len(string)): 
    new_string += string[i].lower()

new_string += ' ' # добавим разделитель фамилии и инициалов 
```

4. Используя цикл `for`, присоединим инициалы в верхнем регистре — это элементы исходной строки с индексами 0 и 1, добавив к ним точку — `.`:

```
string = 'АП ЧЕХОВ' # исходная строка
new_string = '' 
new_string += string[3].upper()

for i in range(4, len(string)): 
    new_string += string[i].lower()

new_string += ' '

for i in range(0, 2): # добавим инициалы и после них точку
    new_string += string[i].upper() + '.'  
```

5. Теперь осталось проверить результат — добавим вывод исходной строки и новой:

```
string = 'АП ЧЕХОВ' # исходная строка
new_string = '' 
new_string += string[3].upper()

for i in range(4, len(string)): 
    new_string += string[i].lower()

new_string += ' '

for i in range(0, 2): # добавим инициалы и после них точку
    new_string += string[i].upper() + '.' 

print(f'Исходная строка: {string}')
print(f'Новая строка: {new_string}') 
```

В итоге получим:

```
Исходная строка: АП ЧЕХОВ
Новая строка: Чехов А.П.  
```

Отлично, то что нужно! Строка была изменена, как это и планировалось!

### Изменение значения исходной строки после определённого символа

Выше мы изменяли значения исходной строки, зная индексы символов, которые нужно было изменить. Однако бывают ситуации, когда строки имеют разное количество символов. Например, есть строки, где указаны только имена и фамилии авторов:

```
author = ['Джон Стейнбек',
          'Дэниел Киз'] 
```

В этом случае мы не знаем точное расположение фамилии автора в каждой строке. Однако, чтобы провести какие-то преобразования исходной строки, мы можем найти определённый символ в строке и использовать его индекс.

Например, чтобы выделить фамилию автора в отдельную строку, найдём положение разделяющего слова пробела. Зная его индекс, мы сможем выделить фамилию в отдельную строку.

Для поиска пробела будем использовать сравнение строк в цикле с использованием условного оператора `if`:

1. В начале зададим новую переменную `space_position`, которая впоследствии будет содержать индекс пробела, и объявим цикл, который будет перебирать значения исходной строки по индексу. Чтобы отличать случаи, когда пробел был на начальной позиции от случаев, когда пробела в строке не было — в качестве начального значения `space_position` зададим пустую строку:

```
author = 'Джон Стейнбек'

space_position = ''
for i in range(len(author)): # объявим цикл, перебирающий исходную строку по индексу 
```

2. Затем в теле цикла пропишем условие, проверяющее, является ли перебираемый элемент строки пробелом или нет. Если условие верное, то изменим значение переменной `space_position` на значение индекса, то есть значение переменной-итератора `i`:

```
author = 'Джон Стейнбек'

space_position = ''
for i in range(len(author)):
    if author[i] == ' ': # зададим условие, сравнивающее значение символа с пробелом
        space_position = i
        
print(space_position) 
```

Результат будет следующим:

```
4 
```

Цикл выше перебрал все элементы строки, сравнивая их с пробелом, и установил значение переменной `space_position`, равное индексу пробела. Обратите внимание, что если бы в исходной строке было бы несколько пробелов, то значение переменной `space_position` приобрело бы значение индекса последнего пробела в исходной строке. Но поскольку в примере выше количество пробелов единичное, то будем использовать `space_position` дальше для сохранения в новой строке фамилии автора.

3. Для этого создадим новую пустую строку — `last_name`, в которую сохраним фамилию и объявим новый цикл. Он будет перебирать элементы исходной строки после пробела, то есть прибавлять к индексу пробела единицу:

```
author = 'Джон Стейнбек'

space_position = ''
for i in range(len(author)):
    if author[i] == ' ': 
        space_position = i
        
last_name = '' # создадим новую пустую строку
for i in range(space_position+1, len(author)): # объявим новый цикл
    last_name += author[i] # сохраним значения исходной строки в новую строку

print(f'Фамилия автора: {last_name}') 
```

В теле второго цикла также прописана инструкция на сохранение символов исходной строки и в новую строку. Результат будет таким:

```
Фамилия автора: Стейнбек 
```

Обратите внимание: в примере выше мы разобрали подход к перебору части строки с помощью разделителя. Отобразить необходимую часть строки после определённого символа по индексу можно, используя срез строки по индексу:

```
author = 'Джон Стейнбек'

space_position = ''
for i in range(len(author)):
    if author[i] == ' ': 
        space_position = i
        
last_name = author[space_position+1:] # используем срез строки 
```

Результат будет аналогичным. Также в Python существуют другие методы и функции для работы с строками и извлечения из них необходимой части (подстроки).

Потренируемся определять индекс определённого символа и использовать его для формирования новой строки.

Необходимо выделить домен электронной почты в отдельную строку, преобразовав символы в нижний регистр. У нас есть исходная строка и создана новая переменная:

```
email = 'somename@DOMAIN.zone'
domain = '' # новая строка, которая будет содержать почтовый домен
index = '' # индекс символа разделителя @ 
```
Какой код позволит выделить доменную область в нижнем регистре в новую строку?

```python
for i in range(len(email)):
    if email[i] == '@': 
        index = i
        
for i in range(index+1, len(email)):
    domain += email[i].lower()
```
### Вложенные циклы

В результате формат строки был приведён к единому виду. А что делать, если таких строк будет много? Например, у нас есть список инициалов и фамилий, который нужно отформатировать в едином стиле.

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов'] # исходный список 
```

В таком случае можно использовать несколько циклов, которые будут вложены один в другой. Первый цикл будет перебирать строки в списке `names`, а вложенные — изменять написание имени, то есть работать со строками. За основу возьмём предыдущий пример:

- Создадим новый пустой список `new_names`, а также объявим цикл, который будет перебирать значения исходного списка `names`. Переменная `name` используется для хранения текущей строки:

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов'] # исходный список
new_names = [] # создадим пустой список, в который будут помещены новые элементы

for name in names: # объявим первый цикл, перебирающий строки исходного цикла 
```

- В цикле создадим новую строку `new_string`, как и в предыдущем примере — она будет содержать новое написание фамилии и инициалы авторов. Затем добавим к ней первый символ фамилии автора — в исходной строке этот элемент имеет индекс `3`. При добавлении первого символа фамилии переведём его в верхний регистр:

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов']
new_names = [] 

for name in names:
    new_string = '' # создадим пустую строку
    new_string += name[3].upper() # добавим к новой строке первый символ фамилии 
```

- Объявим первый вложенный цикл, который перебирает остаток символов в текущей строке, начиная с пятого символа (индекс 4) до конца строки. Он добавляет каждый символ, преобразованный в нижний регистр, к переменной `new_string`.

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов']
new_names = [] 

for name in names:
    new_string = ''
    new_string += name[3].upper()

        # объявим первый вложенный цикл:
    for i in range(4, len(name)): # переберём в цикле остальную часть фамилии
            new_string += name[i].lower() # переведём символы в нижний регистр 
```

- Обратите внимание, что заголовок вложенного цикла находится в теле самого первого — основного цикла, который перебирает элементы списка `names`, поэтому тело вложенного цикла мы отделили дополнительными четырьмя пробелами.
- Дальше в теле основного цикла добавим разделитель фамилии и инициал — пробел:

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов']
new_names = [] 

for name in names:
    new_string = ''
    new_string += name[3].upper()

    for i in range(4, len(name)): 
            new_string += name[i].lower()

    new_string += ' ' 
```

- Теперь объявим второй вложенный цикл, который будет присоединять инициалы в верхнем регистре — это элементы исходной строки с индексами 0 и 1, добавив к ним точку — `.`:

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов']
new_names = [] 

for name in names:
    new_string = ''
    new_string += name[3].upper()

    for i in range(4, len(name)): 
            new_string += name[i].lower()

    new_string += ' '

    for i in range(0, 2): # добавим инициалы и после них точку
        new_string += name[i].upper() + '.'  
```

- После этого добавим новую строку `new_string` в список `new_names`. Для этого будем использовать метод `append()` и выведем результат на экран:

```
names = ['АП ЧЕХОВ','ФМ ДосТОЕВский','ма шолохов']
new_names = [] 

for name in names:
    new_string = ''
    new_string += name[3].upper()

    for i in range(4, len(name)): 
        new_string += name[i].lower()

    new_string += ' '

    for i in range(0, 2): # добавим инициалы и после них точку
        new_string += name[i].upper() + '.' 
    
    new_names.append(new_string) # в теле основного цикла добавим строку в новый список

print(f'Исходный список: {names}')
print(f'Новый список: {new_names}') 
```

- Получаем:

```
Исходный список: ['АП ЧЕХОВ', 'ФМ ДосТОЕВский', 'ма шолохов']
Новый список: ['Чехов А.П.', 'Достоевский Ф.М.', 'Шолохов М.А.'] 
```

В результате строки в исходном списке были изменены, как это было запланировано.

Было объявлено несколько циклов, которые находились в теле другого цикла. Как правило, цикл, который содержит в себе другой цикл (или несколько других циклов), называется **внешним**, или циклом первого уровня. Вложенные циклы также могут называться вложенными или **внутренними** циклами, или циклами второго, третьего и т. д. уровней.

![](https://pictures.s3.yandex.net/resources/image_1704912669.png)

В Python нет ограничения на количество вложенных списков. Однако необходимо помнить, что много вложенных циклов делает код сложным для понимания. При этом за одну итерацию внешнего цикла, внутренние циклы совершают все свои итерации. Поэтому общее количество итераций при использовании вложенных циклов будет равно произведению числа самостоятельных итераций всех циклов. Например, приведённый выше на иллюстрации код для вывода таблицы умножения совершит ровно 81 умножение, так как на каждую из 9 итераций внешнего цикла придётся 9 итераций внутреннего. При большом количестве итераций это может привести к увеличению времени работы скрипта.

Обратите внимание на форматирование кода вложенных списков: заголовок вложенного списка имеет отступ из четырёх пробелов, поскольку он находится в теле внешнего цикла. При этом тело вложенного цикла имеет также отступ в четыре пробела от положения заголовка этого цикла. Таким образом, от начала строки количество отступов тела вложенного списка с уровнем вложенности 2 будет равно уже восьми (2×4). А если, например, вложенный цикл будет иметь уровень вложенности 4, то количество отступов тела такого цикла возрастёт и будет составлять 4×4 = 16 отступов.

Обратите также внимание, что для создания нового списка мы использовали метод `append()` — «добавить». Этот метод позволяет добавлять элемент в конец списка, к которому метод применяется. Поэтому в начале кода мы создали новый пустой список `new_names`, а в теле цикла последовательно добавляли новые значения в этот список.
