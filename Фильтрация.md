# Фильтрация по столбцам
Для создания выборки данных, включающей только часть столбцов исходного **DataFrame**, применяют фильтрацию по столбцам.

## Фильтрация по столбцам: оператор DataFrame `[]`
чтобы вывести на экран содержимое одного столбца, достаточно указать его название (метку) в специальном операторе **DataFrame** — квадратных скобках `[]`.
```python
# выборка одного столбца из DataFrame df:
df['column_А']
```
Для создания выборки, состоящей из нескольких столбцов, необходимо указать их метки через запятую и передать в оператор `[]` в виде списка. Количество столбцов может быть неограниченным, а их последовательность может не совпадать с последовательностью столбцов в исходном датафрейме
```python
# выборка нескольких столбцов из DataFrame df:
df[['column_А', 'column_B', ... , 'column_N']]
```
> [!attention]
> когда мы создаем срез данных (то есть выборку значений для анализа), состоящий из одного столбца, то результатом будут данные **Series**. Когда мы выбираем несколько столбцов, то результат будет представлен в виде **DataFrame**, поскольку больше одного **Series** — это **DataFrame**.
```python
# выборка одного столбца из DataFrame df в виде данных DataFrame:
df[['column_А']]
```

Примеры:
```python
# выборка нескольких столбцов из DataFrame df и вывод первых пяти строк:
print(music_df[['artist','genre']].head())
```
## Сохранение результата фильтрации данных
```python
# сохраним срез данных music_df в переменную artists:
artists = music_df[['artist','genre']]
```
> [!important]
> при фильтрации мы сохраняем только необходимые нам данные. Поэтому, если сохранить результат фильтрации данных в переменной датафрейма, как мы это делаем ниже, то часть исходных данных удалится
> ```python
music_df = music_df[['artist','genre']]
>```

## Удаление столбцов. Метод `.drop()`
Иногда легче отбросить один или несколько столбцов, чем перечислять все столбцы, которые стоит оставить. 
метод **pandas**, который позволяет удалить лишние столбцы — `.drop()`. Для удаления столбцов следует использовать параметр `columns` метода `.drop()`, в котором нужно передать список со столбцами для удаления.
> [!important]
> Тут мы один за другим применили к нашим данным два метода, при этом последующий метод применяется к результатам предыдущего. Такой вызов методов называется цепочным.
>```python
># удалим из датафрейма music_df столбцы user_id и total_play:
print(music_df.drop(columns=['user_id','total_play']).head())
>```
Если необходимо сохранить результат фильтрации в исходном датафрейме, то метод `.drop()` позволяет это сделать, если указать параметр `inplace=True`:
```python
# удалим из датафрейма music_df столбцы user_id и total_play:
music_df.drop(columns=['user_id','total_play'], inplace=True)
print(music_df.head())
```
> [!important]
> при использовании параметра `inplace=True` нельзя использовать цепочку методов: в этом случае метод `.drop()` не будет возвращать никакого результата, и дальнейший цепочный вызов `.head()` приведёт к ошибке

Шпаргалка

|**Выборка данных**|**Код**|
|---|---|
|Выбрать один столбец в виде Series|`df['column_А']`|
|Выбрать один столбец в виде DataFrame|`df[['column_А']]`|
|Выбрать несколько столбцов|`df[['column_А', 'column_B']]`|
|Удалить столбцы методом `drop()`|`df.drop(columns=['column_А','column_B'])`|
|Удалить и сохранить результат в исходном датафрейме|`df.drop(columns=['column_А','column_B'], inplace=True)`|

# Фильтрация по строкам при помощи индексации
## Индексация
Любая таблица обычно состоит из строк и столбцов. Для того чтобы обратиться к информации в определённой ячейке или группе ячеек, необходимо знать номер строк и название столбцов, в которых они расположены — то есть их координаты в таблице. Такой способ запрашивать данные называют **индексацией**.
![[Pasted image 20260310105614.png]]

Для индексации в **pandas** существует несколько операторов. Наиболее распространёнными из них являются:

- оператор `[]` — позволяет проводить простую индексацию по целочисленному значению индекса строк или по меткам столбцов;
- оператор `.loc[]` — позволяет проводить индексацию по меткам столбцов и индексов;
- оператор `.iloc[]` — позволяет проводить индексацию по целочисленному индексу столбцов и строк.

## Фильтрация по строкам: оператор DataFrame `[]`
Если в квадратных скобках указать не названия столбцов, как мы делали в предыдущих уроках, а интервал значений целочисленного индекса, то будут выведены только определённые интервалы строк.
```python
# синтаксис оператора [] при выборе среза строк DataFrame:
df[начальное_значение_интервала:конечное_значение_интервала]
```
Если необходимо вывести определённые строки, начиная с первой, то первое значение в интервале можно опустить.
```python
# выведем первые три строки датафрейма music_df:
print(music_df[:3])
```
Аналогично можно вывести последние пять строк
```python
# Выведем последние пять строк датафрейма music_df:
print(music_df[-5:])
```
С положительным первым индексом будет так же: если оставить пустым второе значение в срезе данных, то в итоговый срез попадут все строки, начиная с заданной.
```python
# Выведем все строки, начиная с 11 и заканчивая последней строкой датафрейма:
print(music_df[11:])
```
> [!hint]
> А для того чтобы получить полную копию всего датасета, мы можем выполнить `music_df[:]` — такое применение оператора `[]` включит в результат все строки.

Если мы попробуем обратиться к строке с индексом 9 при помощи кода `music_df[9]`, то получим `KeyError` — в таком формате **pandas** ожидает от нас ввод названия столбца, а такого столбца в таблице нет. Поэтому для того, чтобы вывести одну строку, необходим такой срез, который будет включать только её.
```python
# Выведем нужную строку, используя срез строк -
# так pandas поймёт, что нужна индексация строк:
print(music_df[9:10])
```
> [!important]
> важную особенность: этот оператор использует целочисленные значения индекса для вывода строк (то есть порядковый номер строки).

```python
# проведём сортировку данных по значению столбца user_id и выведем первые 5 строк:
sorted_df = music_df.sort_values(by='user_id')
print(sorted_df.head())
```
метод `.sort_values()` По умолчанию метод проводит сортировку от меньшего значения к большему, поэтому, если есть необходимость провести сортировку в обратном порядке, используется параметр `ascending=False`

## Индексация по строкам и столбцам
При фильтрации данных с использованием оператора `[]` можно совмещать индексацию по строкам и столбцам. Это позволит обратиться к определённому диапазону ячеек. Для этого используется несколько операторов `[]` подряд. При этом последовательность операторов `[]` может быть любой, хотя обычно в начале указывают срез по строкам, а затем срез по столбцам.
```python
# выведем значения столбца genre в интервале строк с индексами 9 и 10:
print(music_df[9:11]['genre'])
```
![[Pasted image 20260310112621.png]]

|**Выборка (срез) данных**|**Оператор `[]`**|
|---|---|
|Один столбец (Series)|`df['column']`|
|Один столбец (DataFrame)|`df[['column']]`|
|Несколько столбцов|`df[['column_A','column_B']]`|
|Все строки, начиная с заданной|`df[0:]`|
|Все строки до заданной|`df[:5]` (не включая индекс 5)|
|Срез строк|`df[2:5]` (не включая индекс 5)|
|Одна строка|`df[4:5]`|
|Одна ячейка|`df[4:5]['column']`|

# Индексация и условные выражения
## Логическая или булевая маска
Задаём условие, которое которое возвращает для каждой строки значение `True` или `False`.
```python
# Используем логическую маску для фильтрации данных по значению столбца genre:
print(music_df[music_df['genre'] == 'metal'].head())
```
такие массивы логических значений, используемые для фильтрации данных, называют логическими или булевыми масками, а саму индексацию данных в этом случае называют логической индексацией.
Для создания логических масок можно использовать логические выражения с любыми логическими операторами: $==$, $!=$, $>$, $<$, $>=$, $<=$, а также содержащие функции-предикаты.
можно применять не только логические операторы сравнения, но и логические операторы отрицания.
```python
# Использование логических операторов отрицания в логической маске:
print(music_df[music_df['genre'] != 'pop'])
```
Обращаться к операторам отрицания можно не только в самой логической маске, но и для всей логической маски в целом.
```python
# Использование битового оператора отрицания в логической маске:
print(music_df[~(music_df['genre'] == 'pop')])
```
сложные логические выражения, включающие несколько условий
```python
# Использование сложного логического выражения в качестве логической маски:
music_df[(music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')]
```
Если для фильтрации используется сложное логическое выражение, или фильтрацию предполагается использовать несколько раз в разных частях кода, то лучше инициализировать логическую маску в отдельной переменной
```python
# Сохраним сложное логическое выражение в переменной mask:
mask = (music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')

# Проведём логическую индексацию данных с помощью логической маски mask:
filtered_df = music_df[mask]
```
использование логической маски можно комбинировать с другими операторами `[]` для сложной фильтрации данных.
```python
# Сохраним сложное логическое выражение в переменной mask:
mask = (music_df['genre'] == 'metal') | (music_df['genre'] == 'rock')

# Выведем на экран первые 7 строк данных в столбцах artist, genre и track:
print(music_df[mask][:7][['artist','genre','track']])
```
При использовании сложных логических последовательностей не всегда удобно записывать их в одной переменной или логической маске — обычно это делает код длинным и сложным для чтения. Поэтому, если необходимо использовать несколько последовательных условий фильтрации данных, то оптимально в начале применить первое условие и сохранить результат в отдельной переменной. А затем провести логическую индексацию.
```python
# Первым действием создадим новый срез данных по первому условию:
filtered_df = music_df[music_df['total_play'] >= 80]
filtered_df = filtered_df[filtered_df['total_play'] <= 130]
filtered_df = filtered_df[filtered_df['genre'] == 'jazz']
```
> [!important]
> Обратите внимание, что для сохранения результата фильтрации мы задали новую переменную `filtered_df`. При этом первая фильтрация данных — по значению `['total_play'] >= 80` проведена по данным исходного датафрейма `music_df`, а последующие фильтрации — с использованием среза данных `filtered_df`.

|**Выборка (срез) данных**|**оператор `[]`**|
|---|---|
|Один столбец (Series)|`df['column']`|
|Один столбец (DataFrame)|`df[['column']]`|
|Несколько столбцов|`df[['column_A','column_B']]`|
|Все строки, начиная с заданной|`df[0:]`|
|Все строки до заданной|`df[:5]` (не включая индекс 5)|
|Срез строк|`df[2:5]` (не включая индекс 5)|
|Одна строка|`df[4:5]`|
|Одна ячейка|`df[4:5]['artist']`|
|Логическая индексация|`df[mask]`|

# Фильтрация при помощи `.loc[]` и `.iloc[]`
Для более сложных алгоритмов, когда необходимо провести фильтрацию по строкам и столбцам, применяя сложные условия в логической маске, часто используют операторы `.loc[]` и `.iloc[]`. Они позволяют индексировать датафрейм по строкам и столбцам в одно действие.

Синтаксис операторов `.loc[]` и `.iloc[]`: в начале задаётся индексация по строкам, затем, через запятую, индексация по столбцам
```python
df.loc[строки, столбцы]
df.iloc[индексы строк, индексы столбцов]
```

Основное различие операторов `.loc[]` и `.iloc[]` в том, что:
- `.loc[]` выбирает строки и столбцы на основе их метки (названия), списка или интервала меток, а также с использованием логической индексации.
- `.iloc[]` выбирает строки и столбцы на основе их целочисленного индекса (порядкового номера), списка и интервала этих значений.
![[Pasted image 20260310121004.png]]

```python
# Выведем первые пять строк датафрейма music_df используя метод .head():
print(music_df.head())

# знаком $<$ отмечены необходимые строки

3  82F52E69  193.776327          Rixton      pop  Me And My Broken Heart  <
4  4166D680    3.007000  Henry Hall ...     jazz                    Home
```
метки индексов совпадают с их целочисленным значением, поскольку отсчёт строк начинается с нуля — первая строка, имеет индекс 0 и метку 0, вторая — 1 и так далее.

> [!note]
> Оператор `.loc[]` при использовании диапазона индексов выводит все строки включая последнее значение
> ```python
># Вывод среза строк с метками индексов 1, 2, 3 с помощью оператора .loc[]:
>print(music_df.loc[1:3])
>```
>```
>    user_id  total_play          artist    genre                   track
>1  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
>2  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
>3  82F52E69  193.776327          Rixton      pop  Me And My Broken Heart  
>```

> [!important]
>  Оператор `.iloc[]` обращается к целочисленному индексу строк или столбцов, то есть к их порядковому номеру. Его использование будет аналогично использованию оператора `[]` — последнее значение в интервале строк не будет включено в итоговый срез.
>  ```python
># Вывод среза строк с индексами 1, 2, 3 с помощью оператора .iloc[]:
>print(music_df.iloc[1:4])
>```
>```
>    user_id  total_play          artist    genre                   track
>1  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
>2  FB1E568E  282.981000    Stive Morgan  ambient             Love Planet  
>3  82F52E69  193.776327          Rixton      pop  Me And My Broken Heart  
>```

Если данные уже подвергались фильтрации или сортировке, то последовательность меток индексов может нарушиться.
```python
# Сохраним срез данных, включающий композиции с жанром dance:
```

Если индексы изменились и необходимо снова использовать целочисленные индексы, в pandas можно использовать метод `.reset_index()`
```python
# Сохраним срез данных, включающий композиции с жанром dance:
filtered_df = music_df[music_df['genre'] == 'dance']

# Сбросим метки индексов строк с помощью метода .reset_index():
```

## Логическая индексация в `.loc[]`
Логическую индексацию можно использовать только с оператором `.loc[]`. В качестве логической маски можно использовать логические выражения с простым оператором индексации `[]` 
```python
# Создадим логическую маску, фильтрующую данные по значению столбца artist:
mask = music_df['artist'] == 'Aura'

# Проведём фильтрацию данных с использованием логической маски:
print(music_df.loc[mask, 'artist':'track'])
```
или с оператором `.loc[]`.
```python
# Создадим логическую маску, фильтрующую данные согласно нескольким условиям:
mask = (music_df.loc[:,'artist'] == 'Aura') & (music_df.loc[:,'genre'] == 'dance')
print(music_df.loc[mask, 'artist':'track'])
```

# Фильтрация при помощи query
Метод `.query()` позволяет проводить фильтрацию данных по заданным условиям в виде одного запроса. Благодаря повышенной производительности метод подходит для работы с объёмными данными, состоящими из более десятка тысяч строк.
## задаём условие
Синтаксис метода `.query()` выглядит так: `.query('запрос')`. Запрос может содержать одно или несколько логических условий, по которым проводится фильтрация данных. При написании запросов используются только метки столбцов или индексов строк.
```python
# Проведём фильтрацию данных с помощью метода query:
dance_df = music_df.query('genre == "dance"')

print(dance_df.head())

     user_id  total_play      artist  genre          track
12  4023830D   46.885000    ChinKong  dance      Live Life
14  285648EF    0.100000  Pugs Atomz  dance        No Love
15  34811AC9    7.000000   Zak Moore  dance      Feel Good
18  919EFA26  272.039167     Snowday  dance       Patterns
22  9C97ECE2  214.256327      A-Life  dance  Don't Hurt Me
```
Запрос `genre == "dance"` аналогичен по смыслу и результату условию `music_df['genre'] == 'dance'`. Только он выглядит компактнее, поскольку метод `.query()` уже применяется к определённому датафрейму и при указании названия меток нет необходимости указывать название этого датафрейма.

> [!note]
> Сравните написание двух инструкций, использующих оператор `[]` или `.loc[]` и метод `.query()`:
> ```python
># Логическая индексация с помощью []:
>dance_df = music_df[music_df['genre'] == 'dance']
>
># Логическая индексация с помощью .loc[]:
>dance_df = music_df.loc[music_df['genre'] == 'dance']
>
># Запрос .query():
>dance_df = music_df.query('genre == "dance"')
>```

при составлении запроса `query()` значения для фильтрации можно сохранить в переменной, а в запросе сослаться на неё с помощью символа `@`:
```python
# Проведём фильтрацию данных с помощью метода query:
total_play_limit = 120
filtered_df = music_df.query('total_play >= @total_play_limit')

print(filtered_df .head())
```

В запросе можно использовать различные операции сравнения: $!=$, $>$, $>=$, $<$, $<=$, функции-предикаты, а также математические вычисления
```python
# Проведём фильтрацию данных с помощью метода query:
total_play_limit = 2
filtered_df = music_df.query('total_play/60 >= @total_play_limit')
```

## Комбинирование нескольких условий в одном запросе
Метод `.query()` позволяет использовать запросы, содержащие два и более условных выражения.
```python
# Проведём фильтрацию данных с помощью метода query:
filtered_df = music_df.query('total_play > 200 and genre == "pop"')
```
> [!important]
> В запросе `.query()` логические условия можно соединять логическими операторами `or` — «или», `and` — «и», `not` — «не». Приводить отдельные условия в скобках в этом случае нет необходимости, поскольку у логических операторов сравнения `not`, `and` и `or` наименьший приоритет при выполнении. Это отличает запросы `.query()` от логической индексации с помощью `[]` или `.loc[]`, где для соединения нескольких условий можно использовать только битовые операторы сравнения.

## Особенности фильтрации данных с помощью `.query()`
#### **Использование списков в условиях. Операторы `in` и `not in`**
В запросе `query()` можно сравнивать значения с элементами списка. Для этого используются логические операторы $=$ и $!=$, или операторы сравнения `in` и `not in`, которые представляют собой краткую запись метода `isin`
```python
# Проведём фильтрацию данных с помощью метода query и значений списка:
genres = ['postrock','folkmetal','thrash']
filtered_df = music_df.query('genre == @genres')
```

Аналогичный результат будет при использовании оператора `in`:
```python
# Проведём фильтрацию данных с помощью метода query и значений списка:
genres = ['postrock','folkmetal','thrash']
filtered_df = music_df.query('genre in @genres')
```

#### **Использование метки индекса строк**
В запросе `query()` можно обращаться к индексу датафрейма, используя ключевое слово `index`.
```python
# Проведём фильтрацию данных с помощью метода query:
filtered_df = music_df.query('index < 3')

print(filtered_df.head())

    user_id  total_play        artist    genre        track
0  BF6EA5AF   92.851388    Marina Rei      pop       Musica
1  FB1E568E  282.981000  Stive Morgan  ambient  Love Planet
2  FB1E568E  282.981000  Stive Morgan  ambient  Love Planet
```
условие `'index < 3'`, которое отбирает все строки с индексом меньше `3`. По своей логике подобный запрос работает как индексация `.loc[]` — `.loc[:2]`. 

> [!important]
> Обратите внимание, что если датафрейм содержит столбец `index`, то приоритет при фильтрации будет отдан столбцу с таким названием, а не индексу датафрейма.

Такой запрос будет аналогом оператора `.loc[2:3]`.
```python
# Проведём фильтрацию данных с помощью метода query - 
# сделаем срез строк с индексами 2 и 3:
filtered_df = music_df.query('1 < index < 4')
```
```python
# Проведём фильтрацию данных с помощью метода query:
# вывести только строки с заданным индексом
filtered_df = music_df.query('index in [2,46,345]')
```
#### **Использование методов `pandas`**
сохраним в отдельном срезе данные, значение `total_play` которых выше, чем среднее значение по этому столбцу. Для нахождения среднего значения используем метод pandas `.mean()`:
```python
# Проведём фильтрацию данных с помощью метода query:
filtered_df = music_df.query('total_play > total_play.mean()')

print(filtered_df.head())
```
#### **Сочетание `query()` с оператором `[]` для фильтрации по столбцам и строкам**
Основное назначение метода `.query()` — фильтрация по строкам. Однако, если необходимо провести фильтрацию по строкам и по столбцам, то запрос `.query()` можно совместить с оператором `[]` для вывода нужных столбцов
```python
# Проведём фильтрацию данных с помощью метода query:
genres = ['postrock','folkmetal','thrash']
filtered_df = music_df.query('genre in @genres')[['artist','track']]

print(filtered_df.head())
```
## Использование нестандартных названий столбцов в запросе `.query()`
Использование меток столбцов упрощает работу с запросами в `.query()`, однако, не все названия можно использовать в запросах. Если название столбца содержит:

- пробелы ``;
- операторы сравнения: $<$, $<=$, $>$, $>=$, $!=$, $==$;
- математические операторы: `+`, `-`, `/`, `*`;
- символы: `.`, `,`, `$`, `&`, `!`, `^`;
- скобки: `()` и `[]`;
- ключевые слова Python (`False, None, True, and, as, assert, break, class, continue, def, del, elif, else, except, finally, for, from, global, if, import, in, is, lambda, nonlocal, not, or, pass, raise, return, try, while, with, yield`);

то для их корректного использования в запросе необходимо использовать обратные кавычки — ````:
```python
# Примеры запросов, где названия столбцов указаны в обратных кавычках:
df.query('`artist-name` == "Finntroll"')
df.query('`price$` > 150')
df.query('`user name` == "Ирина"')
```

## Сравнение метода `.query()` и оператора `.loc[]`
Методы `.query()` и `.loc` позволяют проводить основные действия по фильтрации данных. Однако `.query()` предоставляет удобный способ написания условий фильтрации в одну строку, что упрощает код, особенно при сложных и длинных условиях.

Когда количество строк в данных превышает несколько десятков тысяч, фильтрация методом `.query()` может оптимизировать процесс обработки, поскольку проводится быстрее, чем логическая индексация с помощью оператора `.loc[]`.

Однако метод `.query()` в большей степени ориентирован на логические условия при использовании значений столбцов. При необходимости проводить индексацию строк по их индексу предпочтительнее использовать операторы индексации `[]`, `.loc[]` или `.iloc[]`. Они позволяют более точно указать диапазон меток индексов строк или использовать их целочисленное значение.

Также `.query()` позволяет проводить фильтрацию данных и создавать новые выборки данных, а индексация при помощи метода `.loc[]` может использоваться для изменения данных в индексируемых ячейках или группах ячеек.
