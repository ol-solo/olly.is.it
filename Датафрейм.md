## Что такое датафрейм
Датафрейм состоит из колонок и строк — как в таблице. У колонок есть названия, а у строки — индекс (номер).

С датафреймом можно работать как с обычной таблицей. Его можно изменять, дополнять и удалять.

Датафрейм — это двухмерная структура, у неё есть колонки и строки. Каждая колонка или строка — это объект особого типа — серия. Если мы обратимся к конкретной колонке или строке датафрейма, то будем работать уже с объектом типа серия.
![[Pasted image 20260308201808.png]]

У серий есть несколько атрибутов, основные — два: название `name` и тип объектов в ячейках `dtype` (англ. data type).

Тип объекта расскажет о том, какие данные хранит серия. Основные типы:
- `int`, `float` — числа;
- `datetime` — даты;
- `object` — объекты разных типов. Как правило, это обычные строки. Для хранения строк нет специального типа данных.

`name` — это название колонки
![[Pasted image 20260308202017.png]]

`name` — это название индекса для строки
![[Pasted image 20260308202043.png]]
![[Pasted image 20260308202100.png]]

Датафрейм — это таблица со значениями, а серии — это её строки и колонки, в которых эти значения хранятся.

## Создаём DataFrame
```
import pandas as pd 

df = pd.DataFrame() # создадим пустой датафрейм и сохраним его в переменной `df`

print(type(df)) 

<class 'pandas.core.frame.DataFrame'> # так выглядит полное название нашего объекта
```

> [!tip]
> Вы будете часто встречать сочетание `df` в названии переменных, которые хранят датафреймы. Это опознавательный знак, который подскажет, что мы работаем с объектом типа DataFrame. Это вовсе не обязательное правило, но оно делает код удобным для чтения.

## Дополнительные параметры
При создании датафрейма, мы можем указать специальные параметры, определяющие, какие данные и каким именно образом будут храниться в датафрейме:
- `data` — содержимое DataFrame;
- `index` — значения индексов в DataFrame;
- `columns` — название колонок в DataFrame.

## Как наполнить датафрейм данными
добавить к нему готовые колонки, в которых эти данные уже хранятся. Помните, что Series — это и есть колонки в DataFrame.
![[Pasted image 20260308202948.png]]
![[Pasted image 20260308203004.png]]

```python
import pandas as pd

df = pd.DataFrame() # Создали датафрейм

# создадим Series для стран и для столиц
сountry_series = pd.Series(['Россия', 'США', 'Англия', 'Италия'])
capital_series = pd.Series(['Москва', 'Вашингтон', 'Лондон', 'Рим'])

df['страна'] = сountry_series # В квадратных скобках указываем название колонки
df['столица'] = capital_series 

print(df)

 страна    столица
0   Россия     Москва
1      США  Вашингтон
2   Англия     Лондон
3   Италия        Рим
```
> [!tip]
> `display(df)` - вывести данные в удобном для чтения виде

Всё то же самое можно сделать и без объектов Series, а используя лишь списки.
```python
import pandas as pd

df = pd.DataFrame() # Создали датафрейм

df['страна'] = ['Россия', 'США', 'Англия', 'Италия'] # Добавили колонку стран
df['столица'] = ['Москва', 'Вашингтон', 'Лондон', 'Рим'] # Добавили колонку столиц
display(df)
```

Один из способов представить данные в табличном виде — это использовать вложенные списки (их называют двумерными).
```python
a_list = [[1, 'a'], [2, 'b'], [3, 'c']] # двумерный список
```

```python
a_list = [[1, 'a'],
          [2, 'b'],
          [3, 'c']] # всё тот же двумерный список
```
Этот же список мы можем передать как параметр `data` при создании датафрейма
```python
import pandas as pd

data_countries = [['Россия', 'Москва'],
                  ['США', 'Вашингтон'],
                  ['Великобритания', 'Лондон'],
                  ['Италия', 'Рим']]

df = pd.DataFrame(data=data_countries, columns=['страна', 'столица'])
display(df)
```
Ещё один способ дать названия колонкам — обратиться к атрибуту `columns` в датафрейме. Этот атрибут хранит названия для колонок в виде списка.
```python
import pandas as pd

data_countries = [['Россия', 'Москва'],
                  ['США', 'Вашингтон'],
                  ['Великобритания', 'Лондон'],
                  ['Италия', 'Рим']]

df = pd.DataFrame(data=data_countries)
df.columns = ['страна', 'столица'] # изменили атрибут columns

print(df.columns)

Index(['страна', 'столица'], dtype='object')
```
> [!warning]
> Если мы пропустим элемент в списке, то в датафрейм запишется пустое значение или `None`, при этом ошибки не будет и программа не остановится
```python
import pandas as pd

data_countries = [['Россия', 'Москва'],
                  ['Вашингтон'], # Пропустили США
                  ['Великобритания', 'Лондон'],
                  ['Италия']] # Пропустили Рим

df = pd.DataFrame(data=data_countries, columns=['страна', 'столица'])
display(df)
```
||**страна**|**столица**|
|---|---|---|
|0|Россия|Москва|
|1|Вашингтон|None|
|2|Великобритания|Лондон|
|3|Италия|None|
Вашингтон оказался не в той колонке, а вместо Рима теперь `None`.

параметр `data` относительно универсальный: он может принимать данные разных типов и форматов. Это могут быть словари, списки или даже другие датафреймы. но всё же ограничения есть. Необходимо, чтобы данные в `data` соответствовали определённым правилам, иначе превращение данных в датафрейм не сработает.

## Как сохранить таблицу в словаре
#### **Ключ — колонка**
ключи словаря — это названия колонок, а значения — набор значений в колонке.

```python
data_countries = {
    # По ключу храним набор значений в колонке
        'cтрана': pd.Series(['Россия', 'США', 'Великобритания', 'Италия']),
        'столица': pd.Series(['Москва', 'Вашингтон', 'Лондон', 'Рим'])
    }
# Создали датафрейм на основе данных data_countries
pd.DataFrame(data=data_countries)
```
если мы передадим списки, то Pandas всё равно поймет нас
```python
data_countries = {
        'cтрана': ['Россия', 'США', 'Великобритания', 'Италия'],
        'столица': ['Москва', 'Вашингтон', 'Лондон', 'Рим'] # Вместо Series обычные списки
    }

pd.DataFrame(data=data_countries)
```
> [!important]
> Ошибка *ValueError: arrays must all be same length* говорит о том, что размеры наших списков должны совпадать.
> Чтобы исправить это, запишем`None` в словарь

#### **Записи**
Используем список, где каждый элемент — строка таблицы в виде словаря. Ключ словаря — название колонки, значение — значение колонки для этой строки. Получается набор записей.
```python
# каждый элемент — запись в таблице
data_countries = [{'страна': 'Россия', 'столица': 'Москва'}, # строка 1
                  {'страна': 'США', 'столица': 'Вашингтон'}, # строка 2
                  {'страна': 'Великобритания', 'столица': 'Лондон'}, # строка 3
                  {'страна': 'Италия', 'столица': 'Рим'}]    # строка 4
df = pd.DataFrame(data=data_countries)
display(df)
```
|**страна**|**столица**|
|---|---|
|Россия|Москва|
|США|Вашингтон|
|Великобритания|Лондон|
|Италия|Рим|

> [!danger]
> `NaN` расшифровывается как Not a Number. Это отсутствующее значение, как и `None`. По умолчанию в датафреймах пропуски обозначаются именно как `NaN`.
```python
# каждый элемент — запись в таблице
data_countries = [{'страна': 'Россия', 'столица': 'Москва'}, # строка 1
                  {'страна': 'США', 'столица': 'Вашингтон'}, # строка 2
                  {'страна': 'Великобритания', 'столица': 'Лондон'}, # строка 3
                  {'Италия': 'Рим'}]       # строка 4
df = pd.DataFrame(data=data_countries)
display(df)
```
|страна|столица|Италия|
|---|---|---|
|0|Россия|Москва|NaN|
|1|США|Вашингтон|NaN|
|2|Великобритания|Лондон|NaN|
|3|NaN|NaN|Рим|
мы ошиблись и объявили новую колонку `Италия` со значением `Рим`. Но в других строках этой колонки нет, поэтому, когда такая колонка добавилась, в них появились пропущенные значения.

## Выгрузка датафрейма из файла
CSV расшифровывается как Comma-Separated Values или значения, разделённые запятой. Данные записываются по строчкам, а все значения по колонкам разделены запятыми. Первой строкой принято указывать названия для колонок.

Довольно часто разделителем оказывается знак табуляции `\t` — такие файлы иногда сохраняют в специальном формате TSV (tab-seperated values), но не всегда, табуляция может встретиться в CSV-файле.

`read_csv()` - выгрузить данные из CSV-файла, используют функцию pandas
```python
import pandas as pd

df = pd.read_csv('arts.csv')
```
параметр `sep` укажет на то, какой символ является разделителем
```python
df = read_csv('arts_v2.csv', sep='\t')
display(df)
```
метод `head()` выводит первые пять строк по умолчанию, но можно указать в скобках точное кол-во
```python
df.head(1)
```
метод `tail()` выводит последние несколько строк (тоже пять по умолчанию)
```python
df.tail(1)
```
методом `sample()` выводит случайную строку из датафрейма, в скобках можем указать количество случайных строк, которые мы хотим получить
```python
df.sample(3)
```
атрибут `shape` покажет размеры датафрейма. Размеры датафрейма хранятся в виде кортежа из двух значений `(Количество строк, Количество столбцов)`
```python
df.shape
(6, 3)
```
метод `info()`позволяет получить сразу всю важную информацию о нашем датафрейме:
1. `RangeIndex` даёт информацию об индексах: количество строк с индексами и как они распределены. В нашем случае 6 строк, со значениями от 0 до 5.
2. `Data columns` указывает на количество колонок. Для каждой колонки мы видим тип данных, которые она хранит `Dtype` и количество непустых значений `Non-Null`.
3. `Memory usage` говорит о том, какое место занимает датафрейм. В нашем случае это всего 272 байта.
```python
df.info()

Data columns (total 3 columns):
 #   Column   Non-Null Count  Dtype 
---  ------   --------------  -----

 1   Автор    6 non-null      object
 2   Год      6 non-null      int64

memory usage: 272.0+ bytes
```
