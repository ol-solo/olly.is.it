# Расчёт по группам. Практика

Группировка данных позволяет сегментировать данные по одной или нескольким переменным и применять к ним различные агрегирующие функции. В pandas для группировки данных используется метод `groupby()`. Вот основные концепты и советы по использованию группировки данных:

1. Основной синтаксис:

```
grouped = df.groupby('колонка группировки') 
```

Результат — это объект `GroupBy`, к которому можно применить агрегирующие функции.

2. Агрегирующие функции:
    - `mean()`, `sum()`, `min()`, `max()` и другие — позволяют рассчитать среднее, сумму, минимум, максимум и другие агрегаты для каждой группы;
    - `agg()` — используется для выполнения нескольких агрегаций одновременно или при необходимости применить пользовательские функции.
3. Группировка по нескольким ключам: Можно группировать данные по нескольким столбцам, передавая список:

```
grouped = df.groupby(['колонка группировки1', 'колонка группировки2']) 
```

4. Параметры `groupby()`:
    - `as_index`: если `True` (по умолчанию), ключи группировки используются как индексы нового объекта. Если `False`, ключи группировки добавляются как обычные столбцы.
    - `sort`: контролирует, следует ли сортировать результаты группировки по ключам. По умолчанию `True`.
5. Сброс индексов: После группировки и агрегации результат часто возвращается с мультииндексом. Использование `reset_index()` позволяет преобразовать мультииндекс в столбцы и сделать данные более читаемыми:

```
    result = df.groupby('колонка группировки')\
               .agg({'колонка расчётов': 'метод агрегации'})\
               .reset_index()
     
```

6. Использование `agg()` с словарями: Метод `agg()` можно использовать со словарями для указания агрегаций для разных столбцов:

```
    result = df.groupby('колонка группировки')\
               .agg({'column1': 'sum', 'column2': 'mean'})
     
```

7. Использование `agg()` с кортежем: Метод `agg()` можно использовать с кортежами для указания агрегаций для разных столбцов:

```
    result = df.groupby('колонка группировки')\
               .agg(новая_колонка=('column1', 'mean'))
     
```

# Сводные таблицы. Практика

Каждая сводная таблица в pandas базируется на следующих ключевых компонентах:

1. `Индексные колонки (Index columns)` — одна или несколько колонок, значения которых используются для формирования строк результирующей таблицы. Каждая уникальная комбинация значений в этих колонках формирует отдельную строку.
2. `Колонки значений (Values columns)` — колонка или колонки, данные которых агрегируются. Чаще всего это числовые колонки, но в зависимости от функции агрегации, могут использоваться и нечисловые данные.
3. `Функция агрегации (Aggregation function)` — функция, применяемая к данным колонки значений. В случае сводных таблиц могут быть использованы различные функции (`'mean'`, `'min'`, `'max'`, `'sum'` и т.д.) для вычисления агрегированных значений по каждой группе.
4. `Колонки для группировки (Columns)` — опциональный параметр, который позволяет дополнительно разделить агрегированные данные по другому измерению, формируя столбцы результирующей таблицы.

```
df.pivot_table(
    values= 'колонка значений', 
    index=  'индексные колонки', 
    columns='колонки для группировки', 
    aggfunc='функция агрегации') 
```

Когда мы опускаем один из параметров, мы получаем разные виды агрегаций:

- без указания `aggfunc` применяется метод `mean`;
- без `values` pandas попытается агрегировать все числовые столбцы;
- без `columns` данные будут агрегированы только по индексам;
- без `index` данные будут агрегированы в общую сумму по столбцам.

**Переименование столбцов**

```
# Переименование столбцов
pivot_renamed = pivot_example\
                 .rename(columns={
                   'колонка1': 'новая_колонка1',
                   'колонка2': 'новая_колонка1',
                   'колонка3': 'новая_колонка1'
                   }) 
```

**Сброс мультииндекса**

```

multi_level_pivot.columns = ['_'.join(col).strip() 
                             for col in multi_level_pivot.columns.values]

# Избавляемся от мультииндекса
multi_level_df = multi_level_pivot.reset_index() 
```

# Дубликаты. Практика

**Шпаргалка**

- Метод `DataFrame.duplicated()` помогает найти дубликаты. Если вызвать его без аргументов, он вернёт серию булевых значений, где `True` обозначает дубликаты (кроме первого вхождения).
- Метод `DataFrame.drop_duplicates()` удаляет дубликаты. По умолчанию этот метод оставляет первое вхождение и удаляет остальные дубликаты.
- Аргумент `keep` в методах `duplicated()` и `drop_duplicates()` определяет, какое из вхождений дублируемых данных будет сохранено (`'first'`, `'last'` или `False` — для удаления всех дубликатов).
- Аргумент `subset` позволяет указать столбцы, на основе которых будут идентифицированы дубликаты. Это может быть один столбец или список столбцов. Если `subset` не указан, по умолчанию используются все столбцы.

1.

Дубликаты — довольно частое явление при работе с данными. Особенно когда информация собирается из разных источников или в результате разных операций. Дубликаты могут исказить результаты анализа, поэтому очень важно обнаружить их перед обработкой данных.

В этом уроке вы поработаете с датафреймом, содержащим повторяющиеся записи. Ваша первая задача — посчитать количество дубликатов.

        'Age': [25, 30, 35, 25, 30, 35, 40]}

df = pd.DataFrame(data)

  

# Рассчитаем количество дубликатов:

count_duplicates = df.duplicated().sum()

2.

Если данные собираются в течение какого-то времени и информация о сущности может обновляться, часто именно последнее вхождение дубликата может быть самым актуальным. В таком случае нужно сохранить последнее вхождение каждого дублирующегося элемента, удалив предыдущие.

В этой задаче вам предстоит обработать тестовый датафрейм, содержащий дубликаты, и оставить только последнее вхождение каждого из них, удалив все предыдущие.

# Создадим тестовый датафрейм:

        'Age': [25, 30, 35, 25, 30, 35, 40]}

# Графические методы определения выбросов. Практика

Boxplot или «ящик с усами» — это способ изображения распределения числовых данных на основе их квартилей. Он может показать центральное значение, разброс и наличие выбросов в данных. Вот ключевые элементы boxplot:

- **Медиана (Q2/50-й перцентиль)** — линия внутри «ящика», которая показывает медианное значение данных.
- **Нижний (Q1/25-й перцентиль) и верхний (Q3/75-й перцентиль) квартили.** Границы «ящика» указывают на местоположение нижнего и верхнего квартилей. Расстояние между Q1 и Q3 называется межквартильным размахом (IQR).
- **Усы** — линии, выходящие из «ящика» к наиболее удалённым точкам данных, которые находятся в пределах 1.5 IQR от нижнего и верхнего квартилей. Точки за пределами усов считаются выбросами.
- **Выбросы** — точки данных, расположенные за пределами усов, обозначаются отдельными точками на графике и могут указывать на особенности или аномалии в данных.

Функция `plt.boxplot` имеет множество аргументов, позволяющих настраивать внешний вид и поведение диаграммы «ящик с усами». Вот описание основных аргументов, которые могут быть полезны:

- `x` — данные для построения диаграммы.
- `vert` — логический (булев) аргумент, указывающий на ориентацию диаграммы. Если `True` (по умолчанию), диаграмма рисуется вертикально. Если `False`, диаграмма рисуется горизонтально.
- `patch_artist` — если `True`, ящики будут заполнены цветом. По умолчанию `False`, что означает, что ящики не будут заполнены.
- `meanline` — если `True`, среднее значение будет отображено в виде линии, а не в виде точки. По умолчанию `False`.
- `showmeans` — логический аргумент, указывающий на необходимость отображения среднего значения на диаграмме. По умолчанию `False`.
- `showfliers` — логический аргумент, определяющий, будут ли отображаться выбросы на диаграмме. По умолчанию `True`.
- `flierprops` — словарь с параметрами, определяющими внешний вид маркеров выбросов (`fliers`). В нашем случае мы задали красный цвет `'color': 'red'` для маркеров выбросов.

Точечная диаграмма — один из наиболее удобных и простых инструментов для визуализации данных. Она позволяет выявлять закономерности и выбросы в данных, а также проводить сравнение по двум переменным.

Функция `plt.scatter()` из библиотеки `matplotlib` используется для создания точечной диаграммы. Она предоставляет множество аргументов для настройки внешнего вида и поведения диаграммы. Вот некоторые из основных аргументов:

- `x, y` — позиционные аргументы, обязательные для задания. `x` и `y` – это последовательности значений, которые определяют положение точек на диаграмме по осям X и Y соответственно.
- `s` — размер точек на диаграмме. Можно задать один размер для всех точек или массивом — разный размер для каждой точки.
- `c` — цвет точек. Может быть одним цветом для всех точек или массивом цветов / чисел, которые отобразятся в соответствии с количественной переменной и указанной цветовой картой.
- `marker` — форма маркера, используемого для точек. Например, `'o'` для круглых точек, `'s'` для квадратных и так далее.
- `cmap` — цветовая карта.
- `alpha` — прозрачность точек, значение между 0 (полностью прозрачно) и 1 (полностью непрозрачно).
- `linewidths` — ширина границы точек.
- `edgecolors` — цвет границы точек. Может быть задан для всех точек в целом или индивидуально для каждой точки.

# Математические методы определения выбросов. Практика

Метод `describe()` в pandas может быть применён как к отдельной колонке датафрейма, так и ко всему датафрейму. В случае применения к отдельной колонке он предоставит статистическую информацию именно по этой колонке.

Если столбец содержит числовые данные, то в результате вы получите следующую информацию:

- `count` — количество непустых значений в колонке;
- `mean` — среднее значение;
- `std` — стандартное отклонение;
- `min` — минимальное значение;
- `25%` — первый квартиль (25-й процентный показатель);
- `50%` — второй квартиль (средний процентный показатель, также называется медианой);
- `75%` — третий квартиль (75-й процентный показатель);
- `max` — максимальное значение.

Метод `describe()` имеет несколько аргументов, которые можно использовать для настройки его работы:

- `percentiles` — список процентилей, которые необходимо включить в вывод метода. По умолчанию выводятся процентили 25, 50 (медиана) и 75.
- `include` — список типов данных, которые необходимо включить в вывод метода. Допустимые значения: `'all'` (все типы данных), `'numeric_only'` (только числовые типы данных), `'object_only'` (только объектные типы данных), `'datetime_only'` (только временные данные). По умолчанию включаются только числовые типы данных.
- `exclude` — список типов данных, которые необходимо исключить из вывода метода. Допустимые значения: `'all'` (все типы данных), `'numeric_only'` (только числовые типы данных), `'object_only'` (только объектные типы данных), `'datetime_only'` (только временные данные).
- `datetime_is_numeric` — логический флаг, указывающий, следует ли считать временные данные числовыми. По умолчанию значение равно False.

Метод `quantile()` принимает в качестве аргумента значение от 0 до 1, чтобы получить процентиль по колонке датафрейма.

`df_level['колонка'].quantile(0.95)` вернёт 95-й процентиль по колонке.

Алгоритм действий для выделения выбросов по IQR:

1. Рассчитываются квартили Q1 и Q3 с помощью метода `describe()` или `quantile()`.
2. Рассчитывается межквартильный размах (IQR) как разность между Q3 и Q1: IQR = Q3 – Q1. Так будет понятно, на отрезке какой длины расположена наиболее близкая к медиане половина данных.
3. Вычисляются нижний и верхний пороги для выбросов:
    - нижний порог = Q1 – 1.5 * IQR
    - верхний порог = Q3 + 1.5 * IQR
4. Выделяются выбросы — те значения, которые выходят за пределы между нижним и верхним порогом.
