Ранее мы ознакомились с возможностями Python для решения задач с использованием математических функций и вычислений. В анализе данных также полезны агрегирующие функции. Агрегирующие функции могут обработать ряд значений и выдать статистику по этому набору данных — посчитать среднее, медиану, максимальное или минимальное значения. Результаты работы агрегирующих функций необходимы для отчётов, принятия решений, просто ознакомления с теми данными, которые мы получили, особенно, если речь идет о большом количестве данных.

Агрегирующие функции — это функции, которые обрабатывают ряд значений и объединяют их в одно результирующее значение.

В этом уроке вы познакомитесь с несколькими агрегирующими функциями, которые часто применяются в анализе данных.

## Как найти сумму значений

С операцией сложения мы давно знакомы. Однако в работе аналитики зачастую сталкиваются с ситуациями, когда нужно сложить далеко не два числа: требуется узнать сумму десятков, сотен, тысяч значений. Если решать эту задачу циклом при помощи простого сложения, то мы будем вычислять массу ненужных нам промежуточных значений — это будет медленно и неэффективно. Решить подобные задачи можно быстрее и проще при помощи агрегирующей функции, сразу получив конечное значение.

Чтобы разобраться с суммированием — одной из самых простых агрегирующих функций, вспомните чек из магазина. Данные здесь — это цены отдельных товаров в списке, кассовый аппарат складывает все эти значения, и в графе «итог» вы видите сумму к оплате.

Реализуем упрощенный кассовый аппарат при помощи уже знакомой вам библиотеки NumPy.

Освежим в памяти список ингредиентов любимого многими салата «Оливье»:

|**Товар**|**Цена**|
|---|---|
|Картофель, 2 кг|150|
|Морковь мытая, 2 шт|35|
|Яйца куриные, 10 шт|80|
|Колбаса вареная «Докторская», 0.5 кг|300|
|Горошек зелёный консервированный, 1 уп.|89|
|Майонез, 100 г|92|
|Соль, 1 кг|34|
|Огурцы маринованные, 1 уп.|120|

На основе этих расценок, посчитаем итоговую сумму для такого чека.

Сначала приведём данные к виду, с которым можем работать: создадим массив, в который положим все цены из таблицы. Чтобы найти сумму набора данных, воспользуемся функцией `sum()`, внутрь которой мы включим набор данных.

```
import numpy as np # импортируем NumPy

#задаём список цен
price_list = [150, 35, 80, 300, 89, 92, 34, 120] 

np.sum(price_list) #находим сумму заданного массива
>900 
```

Значит, итоговая стоимость такого набора продуктов — 900 денег. Мы получили результат благодаря паре строчек кода: функция `np.sum()` сагрегировала данные — вычислила сумму.

## Как найти среднее и медианное значения

Вспомним теорию статистики:

Среднее значение (среднее) — сумма всех чисел, поделенная на общее количество чисел. Используется для измерения усредненного значения ряда данных.

Медианное значение (медиана) — значение, которое разделит отсортированные по возрастанию данные на две равные части. В выстроенном от наименьшего к наибольшему значению элементов ряду данных, медиана находится ровно посередине: значений меньше медианного ровно столько же, сколько значений большего.

И среднее, и медиана позволяют оценить «середину данных», но при этом далеко не всегда эта «середина» совпадает. Среднее зависит от самих значений — если в наборе данных найдётся один элемент значительно больше других, то он заметно повлияет на сумму чисел в наборе, и, как следствие, на среднее значение. А медиана больше зависит от соотношения количества больших и маленьких чисел. Насколько экстремальны сами значения — не так существенно.

Если оцениваемые значения отличаются друг от друга несущественно, например 10, 11, 11, 12, 10 — можно использовать среднее. Но если одно или несколько значений выбиваются из общего ряда, например: 10, 11, 36, 12, 11 — необходимо использовать медиану, так как аномально большое (или маленькое) значение сильно повлияет на результат. Вывод будет опираться на одну аномалию, а не на весь ряд данных в равной мере.

Удостоверимся в этом, решив задачу. В таблице ниже — информация о количестве продаж у 15 сотрудников. Аналитику отдела необходимо подготовить отчёт о том, сколько товара в среднем продал каждый продавец.

|**Сотрудник**|**Количество проданных товаров**|
|---|---|
|Петрова А.Р.|15|
|Смирнов А.В.|11|
|Тихонов В.И.|6|
|Глазков Е.А.|13|
|Вострякова А.В.|19|
|Топалева О.В.|12|
|Павлова П.С.|37|
|Синицин А.С.|22|
|Бабенко Е.А.|7|
|Савинов Я.В.|16|
|Моисеев А.А.|49|
|Сергеева К.К.|20|
|Романенко Д.Ю.|2|
|Колосов А.С.|22|
|Весенина С.И.|15|

Для начала создадим массив с информацией о продажах из таблицы.

```
import numpy as np
#список, содержащий информацию о продажах
count_list = [15, 11, 6, 13, 19, 12, 37, 22, 7, 16, 49, 20, 2, 22, 15] 
```

В NumPy для определения среднего стоит использовать функцию `mean()`:

```
# вычисление среднего арифметического значения массива
np.mean(count_list) 
> 17.733333333333334 
```

Для нахождения медианы — `median()`:

```
# вычисление медианного значения массива
np.median(count_list) 
>15.0 
```

Наблюдаем различие: медиана составила 15, среднее значение ~ 17.73, округлим до целого — 18.

Какой же результат использовать в отчёте? Для этого нужно дополнительно оценить сами данные: понять нет ли среди них аномально больших или неожиданно малых значений. Для этого можно использовать ещё одну пару агрегирующих функций.

## Максимальное и минимальное значение

Функции для расчёта максимального и минимального значений полезны, когда данных в ряде много: для набора из пяти чисел легко определить минимальное и максимальное «на глаз», а если таких чисел будет сто, тысяча, больше ста тысяч?

Но тратить время на подобные вычисления вручную точно не стоит, ведь существуют функции `max()` и `min()` из библиотеки NumPy.

Вернёмся к нашей задаче и посчитаем минимальное и максимальное количество проданных товаров:

```
import numpy as np

# массив, содержащий информацию о продажах
count_list = [15, 11, 6, 13, 19, 12, 37, 22, 7, 16, 49, 20, 2, 22, 15]
np.min(count_list)
>>2
np.max(count_list)
>>49 
```

Полученные значения для минимума — 2 и максимума — 49 существенно отличаются от медианы, которая равна 15 и среднего 18. Мы можем предположить, что эти значения аномальны для нашей выборки — они слишком отличаются от большинства значений. В таком случае корректнее направить в отчёт медиану — это число действительно отобразит усреднённые результаты работы каждого сотрудника отдела.

Проверим себя, «агрегируя» знания по агрегирующим функциям.

Подведём итоги:

- В анализе данных агрегирующие функции используются, чтобы обработать ряды значений и получить статистику по наборам данных.
- Готовые агрегирующие функции можно найти в библиотеке NumPy:

— `np.sum()` — сумма; — `np.mean()` — среднее;

— `np.median()` — медиана; — `np.max()` и `np.min()` — максимальное и минимальное значения.

- NumPy — не единственная библиотека, в которой реализованы агрегирующие функции, в будущем мы познакомимся и с другими.
